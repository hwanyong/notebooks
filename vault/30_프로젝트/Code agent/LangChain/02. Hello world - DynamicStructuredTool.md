# LangChain과 Ollama를 활용한 도구 기반 AI 에이전트 구현

## 사전 준비

1. **Ollama 설치**
   - [Ollama 공식 사이트](https://ollama.ai)에서 환경에 맞는 버전을 다운로드하여 설치
   - Mac의 경우 Homebrew를 통해 설치 가능: `brew install ollama`

2. **필요한 모델 다운로드**
   ```bash
   ollama pull deepseek-r1:14b
   ```

3. **프로젝트 설정**
   ```bash
   # 필요한 패키지 설치
   pnpm install @langchain/ollama @langchain/core @langchain/langgraph
   pnpm install zod # 스키마 검증을 위해 필요
   ```

## 주요 컴포넌트 설명

### 1. 기본 임포트

```javascript
import { Ollama } from "@langchain/ollama";
import { StringOutputParser } from "@langchain/core/output_parsers";
import { ChatPromptTemplate } from "@langchain/core/prompts";
import { DynamicStructuredTool } from "@langchain/core/tools";
import { z } from "zod";
```

- **DynamicStructuredTool**: 동적으로 구조화된 도구를 생성하기 위한 클래스
- **zod**: 타입 스키마 정의 및 검증을 위한 라이브러리

### 2. 모델 초기화

```javascript
const model = new Ollama({
  baseUrl: "http://localhost:11434",
  model: "deepseek-r1:14b",
});
```

### 3. 프롬프트 템플릿

```javascript
const prompt = ChatPromptTemplate.fromTemplate(`
  Answer the following question about a city:
  Question: {input}
  If you need information about a city, use the search tool.
  Think through this step by step:
  1. Determine what city is being asked about
  2. Use the search tool to get information about the city
  3. Formulate a response based on the search results
`);
```

- 도구 사용을 위한 단계별 지시사항이 포함된 프롬프트
- 도시 정보 검색을 위한 구조화된 접근 방식 제시

### 4. 도구 구현

```javascript
const search = new DynamicStructuredTool({
  name: "search",
  description: "Search for information about a city",
  schema: z.object({
    query: z.string().describe("The city to search for"),
  }),
  func: async ({ query }) => {
    if (query.toLowerCase().includes("sf") ||
        query.toLowerCase().includes("san francisco")) {
      return "San Francisco is a city in the United States.";
    }
    return "I don't know that city.";
  },
});
```

- **DynamicStructuredTool**: 구조화된 입력을 받아 특정 기능을 수행하는 도구
- **schema**: zod를 사용한 입력 검증 스키마 정의
- **func**: 실제 검색 기능을 수행하는 비동기 함수

### 5. 실행 체인 구성

```javascript
const chain = prompt
  .pipe(model)
  .pipe(new StringOutputParser());
```

## 기존 버전과의 주요 차이점

1. **도구 기반 접근**
   - 기존: 단순 질의응답 구조
   - 신규: DynamicStructuredTool을 통한 외부 기능 통합

2. **구조화된 입력 처리**
   - 기존: 단순 문자열 입력
   - 신규: zod를 통한 스키마 기반 입력 검증

3. **단계적 문제 해결**
   - 기존: 직접적인 응답 생성
   - 신규: 도구를 활용한 단계별 문제 해결 접근

## 실행 방법



## 주의사항 및 개선 가능한 부분

1. **도구 기능 확장**

2. **에러 처리**

3. **응답 최적화**

   - 응답 포맷팅 개선 필요

4. **설정 관리**
   - 도구 설정을 외부 파일로 분리
   - 환경 변수를 통한 설정 관리
