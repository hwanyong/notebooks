# LangChain과 Ollama를 활용한 간단한 AI 챗봇 구현

## 사전 준비

1. **Ollama 설치**
   - [Ollama 공식 사이트](https://ollama.ai)에서 환경에 맞는 버전을 다운로드하여 설치
   - Mac의 경우 Homebrew를 통해 설치 가능: `brew install ollama`

2. **필요한 모델 다운로드**
   ```bash
   ollama pull deepseek-r1:14b
   ```

3. **프로젝트 설정**
   ```bash
   # 필요한 패키지 설치
   pnpm install @langchain/ollama @langchain/core
   pnpm install @types/node # process.exit(1) 타입을 위해 필요
   pnpm install tsx # TypeScript 실행을 위해 필요
   ```

## 프로젝트 구조

```plain
your-project/
├─ package.json
├─ pnpm-lock.yaml
├─ docs/
│  └─ hello-ollama.md
└─ src/
    └─ helloGraphOllama.ts
```

## 코드 설명

### 1. 주요 컴포넌트

```typescript
import { Ollama } from "@langchain/ollama";
import { StringOutputParser } from "@langchain/core/output_parsers";
import { ChatPromptTemplate } from "@langchain/core/prompts";
```

- **Ollama**: 로컬에서 실행되는 LLM 서버와의 통신을 담당
- **StringOutputParser**: LLM의 출력을 문자열로 변환
- **ChatPromptTemplate**: 프롬프트 템플릿 생성 및 관리

### 2. 처리 과정

1. **Ollama 초기화**
   ```typescript
   const ollama = new Ollama({
     baseUrl: "http://localhost:11434",
     model: "deepseek-r1:14b",
   });
   ```
   - 로컬에서 실행 중인 Ollama 서버에 연결
   - deepseek-r1:14b 모델 사용 지정

2. **프롬프트 템플릿 설정**
   ```typescript
   const prompt = ChatPromptTemplate.fromTemplate(
     "다음 주제에 대해 한 문장으로 설명해주세요: {topic}"
   );
   ```
   - 사용자 입력을 받을 수 있는 템플릿 형식 정의
   - {topic} 부분이 실제 입력으로 대체됨

3. **실행 체인 구성**
   ```typescript
   const chain = prompt.pipe(ollama).pipe(new StringOutputParser());
   ```
   - prompt → ollama → StringOutputParser 순서로 데이터 처리
   - pipe() 메서드로 각 단계를 연결

4. **결과 처리**
   ```typescript
   const result = await chain.invoke({
     topic: "인공지능",
   });
   console.log("결과:", result);
   ```
   - chain.invoke()로 실행
   - 결과를 콘솔에 출력

## 실행 방법

1. **Ollama 서버 실행**
   ```bash
   ollama serve
   ```

2. **코드 실행**
   ```bash
   pnpm exec tsx src/helloGraphOllama.ts
   ```

## 주의사항 및 문제점

1. **서버 의존성**
   - Ollama 서버가 실행 중이어야 함
   - 기본 포트(11434)가 사용 가능해야 함

2. **모델 크기**
   - deepseek-r1:14b는 약 14B 파라미터 모델로, 상당한 시스템 리소스 필요
   - 필요에 따라 더 작은 모델로 대체 가능

3. **에러 처리**
   - 현재는 기본적인 에러 처리만 구현됨
   - 실제 사용시 더 상세한 에러 처리 필요

4. **응답 형식**
   - 모델의 생각 과정이 모두 출력되는 문제
   - 필요한 경우 응답 파싱 로직 추가 필요

## 개선 가능한 부분

1. **프롬프트 최적화**
   - 현재는 단순한 프롬프트 사용
   - 시스템 프롬프트 추가로 응답 품질 개선 가능

2. **스트리밍 지원**
   - 현재는 일괄 응답 방식
   - 스트리밍 응답 구현으로 사용자 경험 개선 가능

3. **타입 안정성**
   - 입출력 타입 정의 추가 필요
   - zod 등을 활용한 런타임 타입 검증 도입 가능

4. **설정 외부화**
   - 모델명, 서버 URL 등을 환경변수로 분리
   - 설정 파일 도입 검토
