# LangGraph: 낮은 수준 공급자 선택 - 로우 레벨 문서

## LangGraph 구성 요소 상세 설명

프로젝트에서는 LangGraph 라이브러리를 활용하여 대화 상태 관리를 개선하고, ReAct 패턴 기반 에이전트를 구성합니다. 아래는 `helloGraphOllama.mjs` 파일에서 LangGraph 관련 코드와 각 구성 요소에 대한 상세 설명입니다:

### 1. 상태 저장 및 복원: MemorySaver
- `MemorySaver`는 대화의 상태를 저장하고 필요할 때 복원하여, 이전 대화 내용과 컨텍스트를 유지할 수 있게 합니다.
- 이를 통해, 에이전트가 장기간의 대화 세션에서도 일관된 상태를 유지할 수 있습니다.

```javascript
const agentCheckpointer = new MemorySaver();
```

### 2. 상태 그래프 구성: StateGraph
- `StateGraph`는 대화 흐름과 작업 단계를 그래프로 구성하여, 복잡한 대화 로직과 도구 호출을 체계적으로 관리합니다.
- 각 노드는 모델 호출, 도구 실행 등 특정 기능을 담당하며, 에이전트의 전체 행동 흐름을 제어합니다.

```javascript
const workflow = new StateGraph(StateAnnotation)
  .addNode("agent", callModel)       // LLM 모델 호출 노드
  .addNode("tools", toolNode)         // 도구 실행 노드
  .addEdge("__start__", "agent")      // 시작점에서 에이전트 노드로의 연결
  .addConditionalEdges("agent", shouldContinue)  // 조건에 따른 흐름 분기
  .addEdge("tools", "agent");         // 도구 실행 후 에이전트 노드로 복귀
```

### 3. 상태 주석 및 메시지 관리: Annotation와 messagesStateReducer
- `Annotation`은 대화 메시지에 대한 상태 정보를 저장하고 관리하는데 사용됩니다.
- `messagesStateReducer`는 새로운 메시지가 추가될 때 기존 상태를 어떻게 업데이트할지 결정하는 로직을 포함합니다.

```javascript
const StateAnnotation = Annotation.Root({
  messages: Annotation({ reducer: messagesStateReducer }),
});
```

### 4. 도구 노드: ToolNode
- `ToolNode`는 여러 도구를 하나의 노드로 통합하여, 도구 호출 로직을 단순화합니다.
- 이를 통해, 특정 작업에 맞는 도구들을 에이전트의 실행 흐름에 쉽게 연결할 수 있습니다.

```javascript
const toolNode = new ToolNode({
  tools,
});
```

이러한 LangGraph 구성 요소들을 통해, 프로젝트는 복잡한 대화 흐름과 도구 연동을 효과적으로 관리하며, 높은 수준의 상태 보존과 동적 응답 생성이 가능합니다.

## 환경 설정
### .env 파일 구성
`.env` 파일은 AI 공급자 선택을 위한 구성 매개 변수를 설정하는 데 사용됩니다. 예제 구성은 아래와 같습니다:

```
DEBUG_OUTPUT=true
AI_PROVIDER=ollama
OPENAI_API_KEY=sk-... (API 키 자리 표시자)
```

- `DEBUG_OUTPUT`: 상세 디버그 출력을 활성화합니다.
- `AI_PROVIDER`: 사용할 AI 공급자를 결정합니다. 옵션에는 `openai`와 `ollama`가 있습니다.
- `OPENAI_API_KEY`: `AI_PROVIDER`가 `openai`로 설정된 경우 사용됩니다.

## 코드 수준 공급자 선택
`helloGraphOllama.mjs` 파일은 선택된 AI 공급자가 어떻게 구성되고 초기화되는지를 보여줍니다. 아래는 관련 코드 스니펫입니다:

```javascript
let baseModel;
if (process.env.AI_PROVIDER === "openai") {
  baseModel = new ChatOpenAI({
    model: "o1-mini",
    openAIApiKey: process.env.OPENAI_API_KEY,
  });
} else if (process.env.AI_PROVIDER === "ollama") {
  baseModel = new ChatOllama({
    baseUrl: "http://localhost:11434",
    model: "deepseek-r1:14b",
  });
} else {
  throw new Error(`Unsupported AI_PROVIDER: ${process.env.AI_PROVIDER}`);
}
const model = baseModel;
```

이 코드는 `.env` 파일의 `AI_PROVIDER` 변수를 확인하여 올바른 모델 인스턴스를 초기화하는 방법을 보여줍니다.

## 핵심 함수 설명

이 두 함수는 에이전트가 도구를 호출하고, 필요한 경우 모델의 재호출을 관리하는 데 핵심적인 역할을 수행합니다.

### 1. handleToolCall 함수
- 이 함수는 모델의 응답 내용이 JSON 형식의 도구 호출 요청인지 확인합니다.
- JSON 파싱에 실패하면 원래 응답을 반환합니다.
- 도구가 존재하지 않거나 호출에 실패할 경우에도 원래 응답을 반환하도록 합니다.

```javascript
async function handleToolCall(responseContent, tools, debugLog) {
  debugLog("Handling tool call for response content:", JSON.stringify(responseContent));
  let parsed;
  try {
    parsed = JSON.parse(responseContent);
  } catch (err) {
    debugLog("Response is not a JSON tool call, error:", err.message);
    return responseContent; // early return
  }
  if (!parsed.action) return responseContent;

  const toolObj = tools.find((t) => t.name === parsed.action);
  if (!toolObj) {
    debugLog(`No tool found for action: ${parsed.action}`);
    return responseContent; // early return
  }
  debugLog(`Invoking tool "${parsed.action}" with parameters:`, parsed.parameters);
  const toolResult = await toolObj.fn(parsed.parameters).catch((err) => {
    debugLog("Tool invocation failed:", err.message);
    return responseContent; // early return if tool call fails
  });
  debugLog(`Tool "${parsed.action}" returned:`, toolResult);
  return toolResult;
}
```

### 2. invokeModelWithToolHandling 함수
- 이 함수는 모델을 호출하고, 도구 호출이 필요한 경우 추가적으로 처리합니다.
- 빈 응답을 감지하면 마지막 유효 메시지를 반환합니다.
- 도구 호출 결과가 원래 응답과 다르면, 새로운 메시지 목록을 구성하여 재호출합니다.

```javascript
async function invokeModelWithToolHandling(messages, model, tools, debugLog) {
  debugLog("Invoking model with tool handling. Current messages:", messages);
  const responseMessage = await model.invoke(messages);
  debugLog("Raw model response message:", responseMessage);
  let content = responseMessage.content;
  debugLog("Model output content:", content);

  // If we get an empty response, log and return the previous valid message
  if (!content.trim()) {
    debugLog("Empty output detected. Returning the last valid message instead.");
    return messages[messages.length - 1].content;
  }

  const toolOutput = await handleToolCall(content, tools, debugLog);
  if (toolOutput !== content) {
    debugLog("Tool output detected. Tool output:", toolOutput);
    const toolOutputMessage = new AIMessage(toolOutput);
    const newMessages = messages.concat(toolOutputMessage);
    debugLog("New messages for re-invocation:", newMessages);
    return invokeModelWithToolHandling(newMessages, model, tools, debugLog);
  }
  return content;
}
```
