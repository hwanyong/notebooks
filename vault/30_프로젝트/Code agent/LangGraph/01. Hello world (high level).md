# LangChain과 LangGraph를 활용한 상태 유지 AI 에이전트 구현

## 사전 준비

1. **Ollama 설치**
   - [Ollama 공식 사이트](https://ollama.ai)에서 환경에 맞는 버전을 다운로드하여 설치
   - Mac의 경우 Homebrew를 통해 설치 가능: `brew install ollama`

2. **필요한 모델 다운로드**
   ```bash
   ollama pull deepseek-r1:14b
   ```

3. **프로젝트 설정**
   ```bash
   # 필요한 패키지 설치
   pnpm install @langchain/ollama @langchain/core @langchain/langgraph
   pnpm install zod # 스키마 검증을 위해 필요
   ```

## 프로젝트 구조

```plain
your-project/
├─ package.json
├─ pnpm-lock.yaml
├─ docs/
│  └─ LangGraph:hello-ollama.md
└─ src/
    └─ helloGraphOllama.mjs
```

## 주요 컴포넌트 설명

### 1. 기본 임포트

```javascript
import { Ollama, ChatOllama } from "@langchain/ollama";
import { HumanMessage, SystemMessage } from "@langchain/core/messages";
import { createReactAgent } from "@langchain/langgraph/prebuilt";
import { MemorySaver } from "@langchain/langgraph";
```

- **ChatOllama**: Ollama의 채팅 모델 인터페이스
- **HumanMessage, SystemMessage**: 메시지 타입 정의
- **createReactAgent**: ReAct 패턴 기반의 에이전트 생성
- **MemorySaver**: 대화 상태 유지를 위한 메모리 관리자

### 2. 모델 초기화

```javascript
const model = new ChatOllama({
  baseUrl: "http://localhost:11434",
  model: "deepseek-r1:14b"
});
```

### 3. 에이전트 설정

```javascript
const agentCheckpointer = new MemorySaver();
const agent = createReactAgent({
  llm: model,
  tools: [],
  checkpointSaver: agentCheckpointer,
});
```

- **MemorySaver**: 대화 상태를 저장하고 복원하는 기능 제공
- **createReactAgent**:
  - LLM 모델을 사용하는 ReAct 패턴 에이전트 생성
  - 도구 목록과 체크포인트 저장소 설정 가능

### 4. 에이전트 실행

```javascript
const result = await agent.invoke(
  { messages: [new HumanMessage("explain about the sf")] },
  { configurable: { thread_id: "42" } },
);
```

- **invoke**: 에이전트에 메시지를 전달하고 응답을 받는 메서드
- **thread_id**: 대화 스레드를 구분하는 식별자
- **messages**: HumanMessage 형태로 전달되는 사용자 입력

## 주요 특징

1. **상태 유지 (Stateful) 대화**
   - MemorySaver를 통한 대화 컨텍스트 유지
   - thread_id를 통한 대화 세션 관리
   - 이전 대화 내용을 기억하고 참조 가능

2. **ReAct 패턴**
   - Reasoning과 Acting을 결합한 문제 해결 방식
   - 체계적인 사고 과정과 도구 사용을 통한 해결책 도출

3. **모듈화된 구조**
   - 모델, 도구, 메모리 관리가 분리된 설계
   - 각 컴포넌트의 독립적인 확장 및 수정 가능

## 실행 방법

1. **Ollama 서버 실행**
   ```bash
   ollama serve
   ```

2. **코드 실행**
   ```bash
   node src/helloGraphOllama.mjs
   ```

## 주의사항 및 개선 가능한 부분

1. **도구 통합**
   - 현재는 빈 도구 목록으로 설정됨
   - DynamicStructuredTool 등을 추가하여 기능 확장 가능

2. **에러 처리**
   - 네트워크 오류, 모델 로딩 실패 등에 대한 처리 필요
   - 체크포인트 저장/복원 실패에 대한 처리 보완 필요

3. **메모리 관리**
   - 장기 실행 시 메모리 사용량 모니터링 필요
   - 오래된 대화 세션 정리 로직 추가 검토

4. **설정 관리**
   - 모델 설정, 서버 URL 등을 환경 변수로 분리
   - 체크포인트 저장 위치 설정 옵션 추가
