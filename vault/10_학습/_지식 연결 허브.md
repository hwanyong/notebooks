# 🌐 지식 연결 허브 (Cross-Domain)

> 서로 다른 지식 영역을 가로지르는 **구조적 유사점·연결점**을 모으는 곳. CS·공학·기초과학·수학·AI를 넘나드는 "**다리(bridge)**"의 중앙 색인.
> 목표: *근본 이해 + 융합 + 통찰* — 한 영역에서 배운 구조가 다른 영역에서 어떻게 반복되는지 잇는다.

## 운영 원칙
- **다리 = 노트 A ↔ 노트 B 양방향 링크 + 한 줄 설명.** 두 노트는 서로 다른 영역(폴더)에 있어야 진짜 cross-domain.
- **공통 구조를 적는다** — "무엇이 같은가"(추상 패턴)를 한 줄로. 표면이 아니라 구조의 유사성.
- **태그 `#연결/<영역>`** 로 도메인 색인 — 그래프뷰에서 영역별로 묶인다.
- **2층 구조:** 한 코스 *안*에서의 타 분야 연결은 그 코스의 연결 지도에, 코스·영역을 *넘는* 연결은 여기에. (아래 '영역별 연결 지도' 참조)

## 🌉 영역 간 다리 (Bridges)

| 다리 | A 영역 | B 영역 | 공통 구조 (무엇이 같은가) |
|---|---|---|---|
| **매직 메서드 = 인터페이스 계약** | 개발/파이썬 [[파이썬 OOP — self · super() · 매직 메서드\|던더 메서드]] | AI [[07 데이터 로더 (Dataset · DataLoader)\|Dataset]] | 약속된 메서드(`__len__`·`__getitem__`)만 구현하면, 호출측(`DataLoader`)이 내부를 몰라도 **표준 규약처럼** 사용 = 인터페이스·프로토콜·다형성 |
| **명시 vs 암묵 (`self` vs `this`)** | 개발/파이썬 [[파이썬 OOP — self · super() · 매직 메서드\|self]] | CS/언어설계 `#연결/언어설계` | 같은 '자기참조'를 파이썬은 매개변수로 *명시*, Java/JS는 예약어로 *암묵* — 언어 설계 철학의 트레이드오프 |
| **관심사 분리 (Protocol vs Orchestrator)** | CS/소프트웨어 설계 `#연결/CS` | AI [[07 데이터 로더 (Dataset · DataLoader)\|Dataset·DataLoader]] | '무엇(규격·Dataset)'과 '어떻게(실행·DataLoader)'를 분리 → 겉은 단순, 속은 OS레벨 최적화(공유메모리·DMA·collate_fn) 은닉 |
| **추상화가 숨기는 OS 비용** | CS [[메모리·캐시·멀티프로세스 (시스템 원리)\|캐시·메모리·멀티프로세스]] `#연결/CS` | AI [[07 데이터 로더 (Dataset · DataLoader)\|DataLoader]] | GIL 우회 fork·공유 메모리·`pin_memory`+DMA·캐시 히트 — 우아한 인터페이스 뒤의 하드웨어 제어 |
| **OS마다 다른 `num_workers`** | CS [[IPC — 프로세스 간 통신 (공유 메모리·소켓·OS별)\|IPC·OS별]] `#연결/CS` | AI [[T03 PyTorch 데이터 로딩 성능 최적화\|DataLoader 성능]] | 같은 `num_workers=4`가 Linux는 fork·`/dev/shm`, mac/Win은 spawn으로 다르게 동작 → OS 넘나들 때 성능·크래시 예측 |
| **로컬 통신은 네트워크 스택 우회** | CS [[IPC — 프로세스 간 통신 (공유 메모리·소켓·OS별)\|IPC]] `#연결/네트워크` | CS/네트워크 (TCP/IP) | 같은 머신이면 TCP/IP 루프백 대신 공유메모리·UDS로 네트워크 스택을 통째 바이패스 → 수배 빠름 |
| **국소 최적점은 하나의 원리** | 통찰 [[엔트로피와 질서 — 최소 에너지 법칙과 엔지니어링\|엔트로피vs질서]] `#연결/최적화` | AI [[T01 경사 하강법과 훈련 루프\|지역 최솟값]] | 딥러닝 손실 지형의 안주 = 조직·코드의 국소 최적점 = 진화의 땜질 — 같은 최적화 수학 |
| **엔트로피 거스르기 = 엔지니어링** | 통찰 [[엔트로피와 질서 — 최소 에너지 법칙과 엔지니어링\|엔트로피vs질서]] `#연결/열역학` | SW공학·인지·AI | 무질서로 흐르는 게 자연(최소 에너지), 질서 유지는 고에너지 역행 — 코드·뇌·LLM 공통 |
| **정보 엔트로피 = 열역학 엔트로피** | AI [[LLM은 왜 길어질수록 평이해지나 — 어텐션 희석과 엔트로피\|LLM 평이화]] `#연결/정보이론` | 정보이론·열역학·통계 | 섀넌 `H=-ΣP log P` ≡ 볼츠만 `S=-kΣp ln p`; 제약 풀리면 분포 평탄화→Prior 회귀 (어텐션·softmax·베이즈 동시 작동) |
| **정적/동적 = 컴파일타임/런타임** | 통찰 [[정적 vs 동적 — 컴파일타임과 런타임이 가르는 것들\|정적vs동적]] `#연결/CS` | AI 지도학습([[08 훈련 루프 (Training Loop)\|훈련루프]]) vs [[강화학습 패러다임 — 정적 데이터 vs 실시간 상호작용\|강화학습]] | 사전 고정(지도/LLM·컴파일타임) vs 런타임 결정(강화) — CS의 stack/heap·`.a`/`.so`와 같은 잣대; o1=둘의 링킹 |
| **I/O 병목은 이중 버퍼링으로 숨긴다** | CS [[메모리·캐시·멀티프로세스 (시스템 원리)\|생산자–소비자·DMA]] `#연결/CS` | AI 훈련 루프 [[08 훈련 루프 (Training Loop)\|호스트–디바이스 병목]] | 느린 전송(PCIe RAM→VRAM)을 빠른 연산 뒤에 *겹쳐 숨기는* 구조 = OS 더블버퍼링·프리페치·DMA = `num_workers`·`pin_memory` (I/O bound를 compute bound로) |
| **메모리 계층 = 속도/크기 트레이드오프 반복** | CS [[메모리·캐시·멀티프로세스 (시스템 원리)\|캐시 계층]] `#연결/CS` | AI [[08 훈련 루프 (Training Loop)\|RAM↔VRAM 분업]] | 레지스터→캐시→RAM→SSD나 SSD→RAM큐→VRAM이나 같은 원리: 빠르고 작은 곳(VRAM=상주 회로)엔 핵심만, 느리고 큰 곳(RAM=완충·SSD=원본)은 흘려보냄 |
| **메모리 토폴로지(분리 vs 통합)가 병목을 정의** | CS 컴퓨터 구조 `#연결/CS` | AI [[08 훈련 루프 (Training Loop)\|UMA zero-copy]] · [[01 PyTorch 설치 트러블슈팅 (Apple Silicon)\|MPS]] | 같은 훈련 코드도 하드웨어 토폴로지가 병목을 바꾼다: x86+NVIDIA=PCIe 분리(이주 병목, `pin_memory`로 완화) / Apple UMA=통합(zero-copy, 전송 병목 0, 단 전용 VRAM 대역폭과 맞바꿈) |
| **grad 동기화(all-reduce) = 분산 합의/집합 통신** | CS [[IPC — 프로세스 간 통신 (공유 메모리·소켓·OS별)\|IPC·집합통신]] `#연결/분산시스템` | AI [[10 GPU 가속과 분산 훈련 (DDP)\|DDP grad 동기화]] | DDP가 매 backward마다 GPU grad를 평균하는 all-reduce = HPC/MPI의 집합통신·분산 합의 그 자체(NCCL=GPU판); `init_process_group`/MASTER_ADDR = 노드 부트스트랩 |
| **하드웨어 제약이 알고리즘 구조를 강제** | CS [[GPU 하드웨어 구조 — SIMT·SM·텐서코어 (ARM 관점)\|GPU SIMT·분기없는 코드]] `#연결/AI` | AI 딥러닝 아키텍처(계층적 행렬곱) | GPU는 분기 발산 탓에 `if`를 싫어함 → 딥러닝을 *분기 없는 계층적 행렬곱(GEMM)* 으로 설계해 맞춤. 인과 역전: GPU가 빨라서 행렬곱이 아니라 **GPU가 잘하는 형태로 딥러닝을 맞춘 것** |
| **행렬 = 공간을 변형하는 선형 변환** | 수학 [[3D 변환 행렬 — 동차좌표·회전·투영 (선형대수 응용)\|3D 변환(물리 공간)]] `#연결/수학` | AI [[05 퍼셉트론·편향·은닉층\|Wx+b(특징 공간)]] | 그래픽스의 회전·투영 행렬과 신경망의 `Wx+b`는 *같은 선형 변환* — 한쪽은 물리 공간을, 한쪽은 특징 공간을 행렬로 비튼다. 로봇 역기구학·카메라 투영도 같은 도구 |
| **신경망의 몸 = 행렬 (형식·가중치·학습가능성)** | 수학 [[2.2 행렬 곱셈·행렬식\|행렬곱]] `#연결/수학` | AI [[신경망은 왜 행렬인가 — 형식·가중치·학습가능성\|신경망=행렬 더미]] `#연결/AI` | 신경망 그림 엣지=가중치=행렬 원소(형식↔행렬), 모델 지식=원소 숫자. 표현·합성·**미분**이 동시에 닫히는 유일 구조라 *학습가능성이 행렬을 강제*; 선형 합성 붕괴가 비선형을 부름. CS로는 Software 2.0(코드=숫자) |
| **피타고라스·삼각함수 = 노름·코사인 유사도** | 수학/물리 [[벡터 — 기하·물리·선형대수를 관통하는 한 뿌리 (피타고라스·삼각함수)\|기하·물리 벡터]] `#연결/수학` | AI [[02 벡터 크기와 단위벡터 (노름)\|노름]]·코사인 유사도(RAG) | 노름=거대해진 피타고라스(N차원), 코사인 유사도=확장된 삼각함수 cosθ=A·B/‖A‖‖B‖. 기하·물리 직관이 그대로 고차원 AI로 |
| **'데이터 형상·흐름' 직관 = 선형대수 언어** | CS/아키텍처 직관 `#연결/CS` | 수학/통찰 [[왜 선형대수인가 — 데이터·알고리즘의 표준 언어\|선형대수=표준 언어]] | 머릿속 데이터 구조·흐름 사고를 정확히 전달하는 표준 어휘가 선형대수(shape·부분공간·투영·선형변환); 알고리즘도 점화식→상태전이행렬, 그래프→인접행렬, 순위→고유값(PageRank)으로 *행렬화*. 짝꿍은 이산수학 |
| **NumPy `strides` = C 포인터 산술** | CS/메모리 [[메모리·캐시·멀티프로세스 (시스템 원리)\|메모리 레이아웃]] `#연결/CS` | AI/탐구 [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)\|NumPy ndarray 내부]] | NumPy ndarray=1D 연속 블록+shape/strides. | **그래프 위 메시지 전달 (역전파 ≠ 라우팅, 골격만 공유)** | AI [[03 계산 그래프\|역전파]] `#연결/AI` | CS/네트워크 라우팅 `#연결/네트워크` | 역전파(역방향 자동미분·메시지=그래디언트·DAG 1패스)와 거리벡터 라우팅(Bellman-Ford·메시지=거리·순환 반복)은 *같은 알고리즘이 아니라* 둘 다 "노드가 이웃 메시지로 자기 값 갱신"하는 **메시지 전달**. 반정환만 갈면 같은 뼈대(GNN이 일반화). → [[두 전파 — 역전파 vs 라우팅 (그래프는 같고 메시지는 다르다)]] |

## 🗂️ 영역별 연결 지도 (코스 내부의 타 분야 연결)
영역 *안*에서 모은 사이드/타분야 연결은 각 코스의 지도에 있고, 여기서 모아 본다.

- **AI 암반 정복:** [[_지식 연결 지도]] — 네트워크(`#연결/네트워크`)·분산시스템(`#연결/분산시스템`)·물리(`#연결/물리`) 등 AI 노트가 타 분야와 잇는 연결.
- _(향후 CS·수학·기초과학 코스가 생기면 각 연결 지도를 여기 등록)_
