# 벡터 내적 — 점곱·기하·직교·아다마르곱·외적

> 📐 **수학 개념원자(SSoT)** — 선형대수. 정의·성질·증명·기하 직관. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[04 벡터 내적 — 점곱·기하·직교 + 아다마르곱·외적|04(AI)]]

> **내적(dot product([dɒt ˈprɒdʌkt], 닷 프로덕트, 닷 프로덕트), 점곱/스칼라곱)** = 선형대수에서 *가장 중요한 연산*. 두 벡터를 **하나의 수**로 요약해 "얼마나 닮았나(방향·유사도)"를 말한다. + §1.5의 아다마르곱(원소별)·외적(rank-1 행렬)까지.

> 🎯 **핵심 (TL;DR)**
> - **내적 $\delta=\sum_{i=1}^{n} a_i b_i$** — 대응 원소 곱의 합. *같은 차원에서만*. 예: $[1,2,3,4]\cdot[5,6,7,8]=70$.
> - **표기:** $a^\mathsf{T}a$(이 책) · $a\cdot b$ · $\langle a,b\rangle$. ⚠️ `np.dot`은 실은 *행렬곱* 구현 — 1D면 내적, (1,N)·(N,1)이면 (1,1)([[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]).
> - **스칼라 선형:** σ([ˈsɪɡmə], 시그머, 시그마) 스칼라곱 — $(\sigma v)\cdot w=\sigma(v\cdot w)$. 음수 σ면 부호만 반전.
> - **분배 법칙:** $a^\mathsf{T}(b+c)=a^\mathsf{T}b+a^\mathsf{T}c$.
> - **기하:** θ([ˈθiːtə], 시터, 세타) → $\delta=\cos\theta\,\lVert v\rVert\lVert w\rVert$ → **부호=각도**(예각+·직각0·둔각−). **직교(orthogonal([ɔːrˈθɒɡənəl], 오쏘거널, 오쏘고널)) ⇔ 내적 0** (암기).
> - **아다마르곱(Hadamard([ˈhædəmɑːr], 해더마르, 아다마르))** $a\odot b$ = 원소별 곱(차원 유지). **외적(outer product([ˈaʊtər ˈprɒdʌkt], 아우터 프로덕트, 아우터 프로덕트))** $vw^\mathsf{T}$ = 열×행 → **rank-1([ræŋk wʌn], 랭크 원, 랭크 1) 행렬**.

> [!tip] 🗺️ "○○곱" 헷갈릴 때 → [[_용어집#🔢 곱(product) 연산 한눈 지도|곱 연산 한눈 지도(용어집)]]
> 내적(→스칼라)·아다마르(→벡터)·외적(→행렬)·교차곱(3D 벡터)·행렬곱(변환) — 입력·출력·정체가 전부 다름. 헷갈리면 용어집 한 표로.

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 두 벡터가 있을 때 "둘이 얼마나 닮았나(방향·유사도)"를 **한 숫자**로 답해야 한다.
> 1. 벡터 두 개는 있지만 "닮음"을 하나의 수로 못 요약함 → 두 벡터를 한 스칼라로 압축할 연산 필요 → **내적($\sum a_i b_i$)**
> 2. 그 한 숫자가 *왜* 닮음인지 기하 의미가 모호 → 사잇각과 잇는 정의 필요 → **기하 정의 $\cos\theta\,\lVert v\rVert\lVert w\rVert$ → 직교=내적0**
> 3. 크기가 큰 벡터가 닮음 점수를 부풀림 → 크기 빼고 방향만 비교 필요 → **코사인 유사도($\delta/\lVert v\rVert\lVert w\rVert$)**
> 4. 곱셈에도 다른 종류가 필요(원소별 가중·rank-1 표) → 합산 안 하는 곱·펼치는 곱 필요 → **아다마르곱·외적(rank-1)**
> **한 줄:** 닮음을 한 숫자로(내적) → 각도로 해석(기하·직교) → 크기 제거(코사인 유사도) → 곁가지 곱(아다마르·외적).

> [!abstract] 📘 책 핵심 — [실전선대] §1.4 내적 · §1.5 다른 곱셈
> **① §1.4 내적 정의 (식 1-9)**
> $$ \delta = \sum_{i=1}^{n} a_i b_i \qquad (\text{대응 원소 곱의 합, 같은 차원만}) $$
> - 예(식 1-10): $[1,2,3,4]\cdot[5,6,7,8]=1\cdot5+2\cdot6+3\cdot7+4\cdot8=70$.
> - 표기: $a^\mathsf{T}a$(이 책, 행렬곱 표기) / $a\cdot b$ / $\langle a,b\rangle$. 결과는 **스칼라 하나**.
> - ⚠️ **`np.dot` 주의:** 실제론 *행렬곱*을 구현. 두 입력에 방향(행 vs 열)을 주면 *행벡터·열벡터일 때* 내적이 나온다(→ [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|(N,1) 규격]]).
> - **인덱싱:** 수학은 1-base, 파이썬은 0-base → 공식→코드 변환 시 주의.
>
> **② 스칼라 선형 + §1.4.1 분배 법칙**
> - $(\sigma v)\cdot w=\sigma\,(v\cdot w)$. `np.dot(10*v,w)=700`. 음수 σ는 부호만 반전(크기 동일) — [[1.5 벡터 공리 — 덧셈·음수·역원|덧셈 역원]].
> - $a^\mathsf{T}(b+c)=a^\mathsf{T}b+a^\mathsf{T}c$ (분배). 검증: `np.dot(a,b+c)==np.dot(a,b)+np.dot(a,c)`.
>
> **③ §1.4.2 기하적 정의 (식 1-11)**
> $$ \delta = \cos(\theta_{v,w})\,\lVert v\rVert\,\lVert w\rVert $$
> - 식 1-9 = 식 1-11(동치 — **증명은 §📐**, 제2코사인 법칙). 크기는 양수, $\cos\theta\in[-1,1]$ → **내적 부호 = 두 벡터의 각도**가 결정.
> - **직교벡터 내적 = 0** (NOTE·암기, 양방향): ① 직교 ② 내적 0 ③ 각 90° 는 **동치**.
>
> **④ §1.5.1 아다마르곱 (원소별 곱)**
> - $a\odot b$ = 대응 원소끼리 곱, *같은 차원* → 같은 차원 벡터. 예: $[5,4,8,2]\odot[1,0,.5,-1]=[5,0,4,-2]$. python `a*b`. (차원 다르면 에러)
> - 용도: 여러 스칼라를 동시에 곱(가게별 매출 등). *전체 합산은 내적*.
>
> **⑤ §1.5.2 외적 (outer product)**
> - **열벡터 × 행벡터 → 행렬.** $vw^\mathsf{T}$. 각 원소 = 행벡터·열벡터 원소 곱. → **rank 1 행렬**(5장).
> $$ \begin{bmatrix}a\\b\\c\end{bmatrix}\begin{bmatrix}d&e\end{bmatrix}=\begin{bmatrix}ad&ae\\bd&be\\cd&ce\end{bmatrix} $$
> - 내적 $v^\mathsf{T}w$(스칼라) ↔ 외적 $vw^\mathsf{T}$(행렬). 차원 달라도 됨. python `np.outer(v,w)`.
> - ⚠️ 브로드캐스팅과 *비슷하나 다름*: 외적은 두 벡터를 곱하는 *특수 수학 연산*.
>
> **⑥ §1.5.3 교차곱·삼중곱** — 기하·물리용, 기술 응용엔 드묾 → 이름만 언급.

> [!info]- 📛 용어 어원 — 內積 / Dot·Inner·Scalar Product
> - **한자 內積:** 內(안 내) + 積(쌓을 적). *연산 결과가 공간 바깥(새 차원의 벡터)으로 안 나가고, 두 벡터가 속한 체(Field([fiːld], 필드, 필드)=실수·복소수 체계)의 **내부로 떨어져 하나의 스칼라로 누적***된다는 뜻. ↔ 외적(外積)은 *밖으로* 펼쳐 행렬을 만든다(아래 §내적 vs 외적). (內/外 명명의 더 깊은 뿌리는 '내적 공간(Inner Product Space)' 정의.)
> - **영어 3용어 구분:** ① **Dot Product** — 표기 $a\cdot b$ 의 가운데 점(·)에서 온 직관적 이름. ② **Inner Product([ˈɪnər ˈprɒdʌkt], 이너 프로덕트, 이너 프로덕트)** — 추상 선형대수에서 *내적 공간*을 정의하는 더 일반·엄밀한 용어. ③ **Scalar Product([ˈskeɪlər ˈprɒdʌkt], 스케일러 프로덕트, 스칼라 프로덕트)** — 결과가 벡터가 아닌 *스칼라*임을 강조한 이름.
> - **기하 본질 한 줄:** 한 벡터를 다른 벡터 위로 **정사영(Projection([prəˈdʒɛkʃən], 프러젝션, 프로젝션))** 한 뒤, 그 기저 벡터의 **크기만큼 스케일링(Scaling([ˈskeɪlɪŋ], 스케일링, 스케일링))** 한 결합 → 그래서 *방향*과 *크기*가 둘 다 결과에 들어간다(아래 ⚖️).

> [!tip] 🧭 내적이 아직 안 와닿는다면 — 여기부터 (직관 먼저)
> "원소를 곱해서 더한다"가 *왜* 닮음인지 막힐 때, 아래 두 그림(🍿 취향 점수 · 🔦 그림자)으로 먼저 감을 잡고 수식으로 돌아오면 된다. **결론: 내적은 '두 벡터가 같은 쪽을 얼마나 함께 가리키나'를 한 숫자로 만든 점수다.**

### 🍿 직관 1 — 내적 = '취향 일치 점수' (곱→합)
내적을 사람 취향 비교로 보면 단번에 와닿는다. 각 축(장르)에 점수를 매긴 **취향 벡터**가 있다고 하자. 두 사람의 점수를 **축마다 곱하고**(곱), **다 더하면**(합) = **취향 일치 점수**.

![[LA1-7 dot-matching-score.png|760]]

🔄 **재구성** — 나(a)=[SF +2, 액션 +2, 로맨스 −1]. *친구(b)* 와는 축마다 부호가 같아 곱이 전부 +(녹색) → 합 **+8**(취향 정렬). *라이벌(c)* 과는 부호가 반대라 곱이 전부 −(분홍) → 합 **−8**(취향 반대).

- **곱(축별)의 의미 = 그 축에서의 *일치도*.** 둘 다 좋아하면(+×+) 또는 둘 다 싫어하면(−×−) **양수**(합의), 한쪽만 좋아하면(+×−) **음수**(불일치). 0이면 한쪽이 무관심.
- **합의 의미 = 모든 축의 일치/불일치를 합산한 총점.** 정렬될수록 ↑(+), 반대일수록 ↓(−), 무관하면 ≈0.
- 이게 곧 임베딩 유사도(방향 닮음)의 기초. → 아래 🔑. (AI 적용 본론은 🎓 렌즈로)

> [!warning] ⚠️ 정확성 — 곱 한 항 $a_ib_i$ 를 "축별 투영 에너지"라 부르는 건 과한 표현   `#정확성`
> Gemini는 각 $a_ib_i$ 를 "축별 투영 에너지"라 했지만, **투영(projection)은 내적 *전체*가 한 벡터에 대해 갖는 기하적 성질**이지 축마다 따로 있는 게 아니다. 한 항 $a_ib_i$ 는 그냥 *그 축에서의 일치도(부호 곱)*로 보는 게 정확하다. "에너지"도 비유일 뿐(실제 물리 에너지는 아래 🌉 물리 다리의 일·선속 맥락에서만).

### 🔦 직관 2 — 내적 = '그림자(투영) × 길이'
기하로 보면 내적은 **a가 b 방향으로 드리우는 그림자의 길이 × b의 길이**다.

![[LA1-6 dot-projection-intuition.png|760]]

🔄 **재구성** — a를 b 위로 수직으로 떨어뜨린 *그림자*(= $\lVert a\rVert\cos\theta$)에 b의 길이를 곱한 게 내적. 그림자가 길수록(둘이 같은 쪽) 내적↑, 직각이면 그림자=0 → 내적 0, 반대면 그림자가 뒤로 → 음수. (θ([ˈθiːtə], 시터, 세타)는 두 벡터 사잇각)

- **"b 방향으로 a가 얼마나 가나"** 를 잰 뒤 b 크기로 키운 값. 그래서 *방향이 정렬될수록* 그림자가 길어 내적이 크다.
- 이 그림이 곧 식 1-11 $\delta=\cos\theta\,\lVert v\rVert\lVert w\rVert$ 의 그림 버전. **두 직관(🍿 곱→합 = 🔦 그림자×길이)이 같은 값**임은 §📐에서 코사인 법칙으로 증명된다.

### 🔑 내적 = '방향 닮음 측정' → 코사인 유사도의 심장
- 식 1-11을 $\lVert v\rVert\lVert w\rVert$ 로 나누면 **$\cos\theta = \dfrac{v\cdot w}{\lVert v\rVert\lVert w\rVert}$ = 코사인 유사도**. 즉 *내적을 두 크기로 정규화해 방향만 남긴 값*. ([[벡터 — 기하·물리·선형대수를 관통하는 한 뿌리 (피타고라스·삼각함수)|코사인 유사도 한 뿌리]])
- **부호가 의미:** 두 벡터의 내적/코사인 > 0 = 방향 유사, ≈ 0 = 무관(직교), < 0 = 반대.
- **직교 = 정보 독립.** 내적 0 = 두 방향이 *서로 정보를 안 공유*. 각 차원이 직교라야 독립 정보를 담는다([[03 벡터 연산 — 덧셈·뺄셈·스케일링·전치·브로드캐스팅|선형 독립]]와 연결).

### ⚖️ 내적 = 방향(각도) × 크기(Norm) — "내적 큼 ≠ 각도 작음"
식 1-11 $\delta=\cos\theta\,\lVert v\rVert\lVert w\rVert$ 는 **세 인자의 곱**이다. *부호*는 $\cos\theta$(각도)가 정하지만 *크기*는 두 노름(Norm([nɔːrm], 노름, 노름))이 함께 정한다. 그래서 **"내적이 크다 ⇒ 각도가 작다"는 틀린 단정**(흔한 귀인 오류):
- **각이 작아도(예: 10°) 내적이 ~0:** 두 벡터의 크기가 극단적으로 작으면.
- **각이 89°(거의 직교)라도 내적이 거대한 양수:** 두 크기가 비대하면.
- ⇒ **순수 방향(각도)만 보려면 내적이 아니라 코사인 유사도** $\cos\theta=\delta/(\lVert v\rVert\lVert w\rVert)$ — 크기를 나눠 없앤 값. 분모 노름은 *항상 양수*라 **코사인 유사도의 부호 = 내적의 부호**. 내적↑ ⇒ 코사인↑(→ +1, 유사), 내적<0 ⇒ 코사인<0(반대). $\cos\theta$ 는 $\theta\in[0°,180°]$ 에서 **단조 감소**(0°→+1, 90°→0, 180°→−1) → *사잇각이 좁을수록 유사도↑*.

> [!tip] 🧭 각도 · 내적 · 코사인 유사도 매핑 — 한 표로   `#정확성`
> | 사잇각 $\theta$ | 내적 $\delta$ (크기 고정 시) | 코사인 유사도 $\cos\theta$ | 데이터 관계 |
> |---|---|---|---|
> | 0° (일치) | 최대 양수 | +1 (최대 유사) | 방향 완전 일치 |
> | 0°<θ<90° (예각) | 양수 (>0) | 0<cos<1 (높음) | 유사한 성향 공유 |
> | 90° (직교) | 0 | 0 (무관) | 상호 독립·무관 |
> | 90°<θ<180° (둔각) | 음수 (<0) | −1<cos<0 (낮음) | 반대 성향 |
> | 180° (반대) | 최소 음수 | −1 (반대 유사) | 정반대 메커니즘 |
> *내적 열의 부호만 각도가 정하고, **절대 크기는 두 노름이 함께 좌우**한다. 부호 관계를 거꾸로 인지하면 손실(Loss) 계산·임베딩 검색이 정반대로 틀어진다.*

> [!warning] ⚠️ 정확성 — 내적은 '값이 비슷한가(유클리드 거리)'를 보는 게 아니다   `#정확성`
> "두 벡터 값이 비슷할수록 내적이 크다"도 흔한 오해다. 내적은 **부호 정렬(방향) × 크기(에너지)** 이지 *값의 근접*이 아니다.
> - $[10,10]\cdot[-10,-10]=-200$: 절댓값(크기)은 똑같이 닮았지만 **부호가 반대라 큰 음수**.
> - $[1,1]\cdot[1000,1000]=+2000$: 값은 1 vs 1000으로 전혀 안 닮았지만 **부호가 정렬돼 큰 양수**.
> - 같은 내적값이 *다른 (각도,크기) 조합*에서 나온다: $[1,0]\cdot[1,0]=1$ 과 $[10,0]\cdot[0.1,0]=1$ 은 둘 다 1.
> ⇒ 내적이 재는 건 "값이 가까운가"가 아니라 "**같은 쪽으로 함께 큰가(부호 정렬 × 크기)**". *값 거리*가 궁금하면 **유클리드 거리**($\lVert x-y\rVert$ — 정의는 이 원자 아래 §📐, 일반형·비교/정렬에서의 쓰임은 [[T08 유클리드 거리 — 두 점 사이 거리·제곱거리의 순위 불변성|T08]])를, *순수 방향*만 궁금하면 **크기를 나눠 없앤 코사인 유사도**를 써야 한다 → 그래서 임베딩 검색은 흔히 벡터를 **정규화(normalization([ˌnɔːrməlaɪˈzeɪʃən], 노멀라이제이션, 노멀라이제이션))** 한다.
> *(교차검증 팁: Gemini가 든 "사잇각 84.26°, 크기 10·10인데 내적=1" 예시는 산술 오류다. $10\times10\times\cos 84.26°\approx100\times0.1=10$ (≠1). 내적이 1이 되려면 $\cos\theta=0.01$, 즉 θ≈89.4°라야 한다. 수치 예시는 항상 직접 검산할 것.)*

### 📐 왜 '원소곱의 합'이 곧 사잇각 정보인가 — 인과의 역전 (식 1-9 = 식 1-11 증명)
**질문:** "내적을 구하려면 사잇각을 먼저 알아야 하지 않나?" → **인과가 거꾸로다.** 사잇각으로 내적을 만드는 게 아니라, **대수적 내적(원소곱의 합)을 먼저 구한 뒤 그것으로 사잇각을 역산**한다: $\cos\theta=\dfrac{a_1b_1+\cdots+a_nb_n}{\lVert a\rVert\lVert b\rVert}$. 임베딩이 512·4096차원이면 각도기로 잴 길이 없고, 하드웨어가 할 줄 아는 건 *성분 곱셈·덧셈*뿐 — 그 결과가 각도 정보를 그대로 담는다.

**왜 둘이 같은가:** 같은 양 $\lVert a-b\rVert^2$ 을 두 길(기하=제2코사인 법칙 / 대수=성분 전개)로 계산하면 만난다.

> [!example]- 📐 증명 — 제2코사인 법칙(Law of Cosines([lɔː əv ˈkoʊsaɪnz], 로 오브 코사인즈, 로 오브 코사인즈))으로 식 1-9 = 식 1-11
> $c=a-b$ 로 두면 —
> - **기하(코사인 법칙):** $\lVert c\rVert^2=\lVert a\rVert^2+\lVert b\rVert^2-2\lVert a\rVert\lVert b\rVert\cos\theta$
> - **대수(성분 전개, 피타고라스 기반):**
> $$\lVert c\rVert^2=(a_1-b_1)^2+(a_2-b_2)^2=(a_1^2+a_2^2)+(b_1^2+b_2^2)-2(a_1b_1+a_2b_2)=\lVert a\rVert^2+\lVert b\rVert^2-2(a_1b_1+a_2b_2)$$
> - **두 우변을 같다 놓고** $\lVert a\rVert^2+\lVert b\rVert^2$ 소거 후 $-2$ 로 나누면:
> $$\lVert a\rVert\lVert b\rVert\cos\theta = a_1b_1+a_2b_2 = a\cdot b \qquad\blacksquare$$
> *n차원 일반화:* 성분 전개는 차원 수와 무관하게 같은 꼴 → 고차원에선 **이 관계가 곧 '각도의 정의'**(기하적 각도기가 없으니 내적으로 각을 정의).

> [!quote] 🌉 다리 — 각도를 안 구하는 게 하드웨어엔 축복 (FMA)   `#연결/CS`
> $\cos\theta$ 는 GPU/TPU에서 *느린 초월함수(transcendental([ˌtrænsɛnˈdɛntəl], 트랜센덴털, 트랜센덴탈))* 연산. 반면 원소곱의 합은 현대 프로세서가 가장 빨리 처리하는 **FMA([ɛf ɛm eɪ], 에프엠에이, 에프엠에이)(Fused Multiply-Accumulate, $a{\times}b{+}c$ 를 한 명령에)** 그 자체다. 내적에 각도를 먼저 구해야 했다면 데이터마다 역삼각함수를 돌려 AI 연산이 수만 배 느려졌을 것. 그래서 딥러닝은 *각도 없는 성분 연산(행렬곱)* 으로 수렴했다 → [[T06 벡터=상태 vs 행렬=연산자 (기저 재정의·SIMD·GEMM)|T06 산술밀도·GEMM]]

### 🧩 내적($v^\mathsf{T}w$) vs 외적($vw^\mathsf{T}$) — 전치 위치가 가른다
- **내적 $v^\mathsf{T}w$:** (1,N)·(N,1) → **(1,1) 스칼라**. "안으로 접어 한 수로".
- **외적 $vw^\mathsf{T}$:** (N,1)·(1,M) → **(N,M) 행렬**(rank 1). "밖으로 펼쳐 표로".
- *같은 두 벡터, 전치를 어디 두느냐*로 스칼라↔행렬이 갈린다 → [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|shape 규격]]이 결과를 결정.

#### 왜 전치(transpose)를 하는가 — 차원 정합
**내적을 '행렬곱'으로 시키려면 앞 벡터를 눕혀야 한다.** 기본 벡터는 열 $(N{\times}1)$. 그대로 두면 $(N{\times}1)\cdot(N{\times}1)$ = **shape 불일치 에러**(행렬곱 조건: *앞의 열 수 = 뒤의 행 수*). 앞을 전치해 행벡터 $(1{\times}N)$ 로 만들면 $(1{\times}N)\cdot(N{\times}1)=(1{\times}1)$ → 내부 차원 $N$ 이 소거되며 **스칼라**가 나온다. *전치 = 내적이 성립하도록 축을 정렬하는 행위.*

> [!warning] ⚠️ 정확성 — $a^\mathsf{T}b = b^\mathsf{T}a$ 는 맞지만 $a^\mathsf{T}b = ab^\mathsf{T}$ 는 틀렸다   `#정확성`
> 내적의 교환법칙은 $a^\mathsf{T}b=b^\mathsf{T}a$(둘 다 $\sum a_ib_i$ 스칼라)로 적어야 한다. $a^\mathsf{T}b=ab^\mathsf{T}$ 는 **0차원 스칼라 = 2차원 행렬**이라 주장하는 차원 파괴다.
> - $a{=}[1,2],\,b{=}[3,4]$: $a^\mathsf{T}b=1{\cdot}3+2{\cdot}4=11$(스칼라) vs $ab^\mathsf{T}=\begin{bmatrix}3&4\\6&8\end{bmatrix}$(2×2 행렬).
> - **실무 함정(OOM):** `a.T @ b`(스칼라, 4 B) ↔ `a @ b.T`(N×N 행렬). 임베딩 $n{=}4096$ 이면 $ab^\mathsf{T}$ 는 $4096^2\times4\,\text{B}\approx$ **64 MiB** 행렬 → 배치 조금만 커도 **OOM(Out Of Memory)**. 전치 위치 한 끗이 스칼라냐 거대 행렬이냐를 가른다.

> [!quote] 🌉 다리 — 전치와 '연속 메모리'의 진실   `#연결/CS`
> 흔히 "전치하면 메모리가 연속(가로)으로 정렬돼 빨라진다"고 하지만 **부정확**하다. NumPy/PyTorch의 `.T`는 데이터를 옮기지 않고 **strides(보폭)만 맞바꾸는 *view***라 오히려 *비연속(non-contiguous)* 상태가 된다([[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04 strides]]). 물리적 연속 블록을 원하면 `.contiguous()`(복사)가 따로 필요. 즉 **전치의 1차 목적은 '차원 정합'(수학)**이고, 캐시 친화·연합 메모리 액세스(coalesced access)는 *그다음의 물리적 재배치 단계*다 → [[T06 벡터=상태 vs 행렬=연산자 (기저 재정의·SIMD·GEMM)|T06 GEMM]]. 어텐션 $QK^\mathsf{T}$ 의 전치도 본질은 *차원 정합*(AI 적용은 🎓 렌즈).

> [!tip] 🪂 심화 — 외적 = rank-1, 저랭크 분해·LoRA의 씨앗   `#사이드/심화`
> 외적 $vw^\mathsf{T}$ 는 **계수(rank) 1 행렬** — 한 방향 정보로만 채운 표. 거꾸로, 큰 행렬을 *외적들의 합*으로 근사하는 게 **저차원 분해(SVD([ɛs viː diː], 에스브이디, 에스브이디))**·**LoRA([ˈlɔːrɑː], 로라, 로라)**(3단계)의 핵심. "rank 1 벡터곱"이 거대 모델 압축의 기본 벽돌. → [[왜 선형대수인가 — 데이터·알고리즘의 표준 언어|행렬 분해]]

> [!warning] ⚠️ 정확성 — 외적은 '경우의 수(Outcome Count)'가 아니라 '전수 상호작용 지도'   `#정확성`
> **직관은 맞다:** "모든 성분 곱의 조합을 격자로 늘어놓는다"는 외적의 수리적 사실 그대로다. **틀린 건 *용어*다(내 표현 교정):** 이걸 확률통계 용어인 **'경우의 수(Outcome Count([ˈaʊtkʌm kaʊnt], 아웃컴 카운트, 아웃컴 카운트))'**로 부르면 개념 결함이다. 경우의 수는 *가능한 경우를 **세는(counting)*** 것 — 결과가 **개수**. 반면 외적은 세지 않는다: 각 칸은 개수가 아니라 두 성분의 **곱(상호작용 강도) 값** $M_{ij}=v_i w_j$ 다.
> - **정확한 정의:** 외적 = 두 독립 시스템의 성분들 사이 **전수 상호작용(All-to-All Cross Interaction) 지도**를 만드는 *대수적 연산*(집합론의 카테시안 곱(Cartesian product([kɑːrˈtiːʒən ˈprɒdʌkt], 카티잔 프로덕트, 카테시안 프로덕트))을 곱셈으로 확장한 꼴). 결과는 **상호작용 강도의 행렬(Correlation Map)**.
> - **AI 실전:** 선형층 가중치 grad가 바로 외적 — $\Delta W=\delta x^\mathsf{T}$ (출력 오차 $\delta$ 와 입력 $x$ 의 *모든 조합*) → 어떤 시냅스를 얼마나 고칠지 정한다.
> - **용어 함정 — 한국어로 둘 다 "외적":** ① **Outer Product** $vw^\mathsf{T}$ = 임의 차원 → **행렬**(AI에서 쓰는 것). ② **Cross Product([krɒs ˈprɒdʌkt], 크로스 프로덕트, 크로스 프로덕트)** $v\times w$ = **3D 한정** → 두 벡터에 수직인 **벡터**(물리 토크·법선, §1.5.3). 데이터·AI 문맥의 "외적"은 항상 *Outer Product* — 헷갈리면 영어로 분리.

> [!quote] 🌉 타 영역 연결 — 내적 = 합성곱·상관·푸리에의 공통 부품   `#연결/수학`
> 내적(원소곱의 합)은 **합성곱(convolution([ˌkɒnvəˈluːʃən], 컨벌루션, 컨볼루션))·상관관계(correlation([ˌkɒrəˈleɪʃən], 코럴레이션, 코릴레이션))·푸리에 변환(Fourier transform([ˈfʊrieɪ ˈtrænsfɔːrm], 푸리에 트랜스폼, 푸리에 변환))·신호 필터링**의 공통 연산. "두 신호를 곱해 합쳐 닮음을 잰다"가 전부 내적. CNN 필터도 국소 내적. **자세한 전개(커널·슬라이딩·합성곱 vs 상관관계 구분)는 → [[1.10 시계열 필터링과 특징 탐지 — 커널·슬라이딩 내적·합성곱|1.10 시계열 필터링]].** 전역 다리 → [[_지식 연결 허브]]

> [!quote] 🌉 타 영역 연결 — 내적 = 물리의 '유효 기여' (일·선속·전력)   `#연결/물리`
> 물리학은 추상적 내적에 **"한 방향으로 정렬된 성분만이 만드는 실제 성과"**라는 실체를 준다 — 기하 정의 $\delta=\cos\theta\,\lVert v\rVert\lVert w\rVert$ 의 물리적 화신(여기서도 θ 주석은 위 ⚖️ 참조).
> - **일(Work([wɜːrk], 워크, 워크)) $W=\vec F\cdot\vec s$:** 이동 방향과 *정렬된 힘 성분*($\lVert F\rVert\cos\theta$)만 운동에너지로 전환, 수직 성분($\lVert F\rVert\sin\theta$)은 낭비. 부호=가속/감속: 예각 $W{>}0$(가속) · 직교 $W{=}0$(**원운동 위성에 중력이 한 일=0** → 속력 일정) · 둔각 $W{<}0$(마찰=감속).
> - **자기 선속(Flux([flʌks], 플럭스, 플럭스)) $\Phi_B=\vec B\cdot\vec A$:** 면적 벡터 $\vec A$(표면에 수직)와의 내적. 장이 면을 정면 관통(θ=0°)→선속 최대, 면과 평행(θ=90°)→0(뚫고 든 유효 장 없음).
> - **전력(Power([ˈpaʊər], 파워, 파워)) $P=\vec J\cdot\vec E$:** 전류 밀도와 전기장의 정렬도 = 소비 전력 밀도(줄 발열).
> - **AI로의 환원:** *가중치 벡터가 입력의 어떤 성분을 얼마나 통과(동조)시킬지* 정하는 게 곧 내적 — 물리의 '유효 기여' 필터와 동일 메커니즘. 전역 다리 → [[_지식 연결 허브]]

## 🧩 코드 실습 (§1.4~1.5)
> 🎯 목표: 내적(정의·스칼라선형·분배·직교)·아다마르곱·외적을 코드로. (검증: dot=70·700, 분배 True, 직교 0, 외적 (3,2))
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la1_4_dot.py`

```python
import numpy as np
v, w = np.array([1,2,3,4]), np.array([5,6,7,8])
print(np.dot(v, w))                 # 70  = Σ aᵢbᵢ
print(np.dot(10*v, w))              # 700 (스칼라 선형: (σv)·w = σ(v·w))
print(np.dot(-1*v, w))              # -70 (음수=부호만 반전)
# 분배 법칙 aᵀ(b+c)=aᵀb+aᵀc
a,b,c = np.array([0,1,2]), np.array([3,5,8]), np.array([13,21,34])
print(np.dot(a,b+c) == np.dot(a,b)+np.dot(a,c))   # True (둘 다 110)
# 직교 → 내적 0
print(np.dot([1,0],[0,1]))          # 0  (각 90°)
# 아다마르곱(원소별) vs 외적(rank-1 행렬)
print(np.array([5,4,8,2]) * np.array([1,0,.5,-1]))   # [5. 0. 4. -2.]
print(np.outer([1,2,3],[4,5]).shape)                 # (3, 2)
# 내적 vs 외적: 전치 위치 (→ T04)
col=np.array([[1],[2],[3]]); row=np.array([[4],[5],[6]]).T
print((col.T@col).shape, (col@row).shape)            # (1,1) (3,3)
```

관찰 포인트:
- **내적 = 한 숫자**(스칼라), 외적 = 행렬. 전치 위치가 가른다.
- **부호가 각도**(예각+/직각0/둔각−). 코사인 유사도는 여기서 크기를 나눈 것.
- **아다마르곱 ≠ 내적**: 원소별 곱(차원 유지) vs 곱의 합(스칼라).

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **내적** (Dot/Inner/Scalar Product) | 대응 원소 곱의 합 δ=Σaᵢbᵢ; 한 스칼라; 방향 닮음 측정. 內積=체 내부로 누적. Dot(·표기)/Inner(추상 일반)/Scalar(결과 강조) |
| **내적 기하 정의** | δ=cosθ·‖v‖‖w‖; 부호=각도(예각+·직각0·둔각−), **크기는 Norm이 좌우**(내적 큼≠각도 작음) |
| **직교** (Orthogonal) | 내적=0 ⇔ 각 90° ⇔ 직교(정보 독립) |
| **분배 법칙** | aᵀ(b+c)=aᵀb+aᵀc |
| **아다마르곱** (Hadamard) | 원소별 곱 a⊙b(차원 유지); python `a*b` |
| **외적** (Outer Product) | 열×행 vwᵀ → rank-1 행렬; python `np.outer` |
| **rank-1 행렬** | 한 외적으로 만든 행렬; 저차원 분해(SVD·LoRA)의 벽돌 |

| 내적 큼 ⇒ 각도 작음? | 틀림 — 부호만 각도가 정함, 크기는 두 노름이 좌우 |

| 아다마르곱 a⊙b? | 원소별 곱 (차원 유지); python `a*b` |
| 외적 vwᵀ? | 열×행 → rank-1 행렬 (SVD·LoRA의 벽돌); python `np.outer` |
| 외적은 '경우의 수'? | 아니다(용어 오류) — 경우의 수=개수 세기 / 외적=곱 값(상호작용 강도) 지도 Mᵢⱼ=vᵢwⱼ; grad ΔW=δxᵀ |
| Outer vs Cross Product? | Outer abᵀ=임의차원 행렬(AI) / Cross a×b=3D 한정 벡터(물리); 둘 다 한국어 "외적" |
| 내적 vs 외적 차원? | $v^\mathsf{T}w$=(1,1) 스칼라 / $vw^\mathsf{T}$=(N,M) 행렬 |
