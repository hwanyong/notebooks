# 벡터 — 정의·차원·방향

> 📐 **수학 개념원자(SSoT)** — 선형대수. 정의·성질·증명·기하 직관. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[01 벡터 — 정의·차원·방향 (NumPy 생성·시각화)|01(AI)]]

> 🎯 **핵심 (TL;DR)**
> - **벡터** = 순서대로 나열된 수. (이 책은 *수 벡터*만 다룸)
> - **차원(dimensionality([ˌdɪmɛnʃəˈnælɪti], 디멘셔낼리티, 디멘셔낼리티))** = 원소 수. **방향(orientation([ˌɔːriɛnˈteɪʃən], 오리엔테이션, 오리엔테이션))** = 열(세로) vs 행(가로).
> - **`ℝ^N`** = N차원 실수 벡터 (ℝ([ɑːr], 알, 알)=실수, ℂ([siː], 시, 시)=복소수). 예: 원소 2개 → `ℝ²`.
> - ⚠️ **수학 차원 ≠ 파이썬 차원:** 수학 차원=원소 수 / 파이썬(NumPy) 차원=`ndim`(축 개수). 벡터의 원소 수는 파이썬에서 **length·shape([ʃeɪp], 셰이프, 셰이프)**.
> - **NumPy 4가지 생성:** 리스트 / 1차원 배열(방향 X) / 행벡터(1,N) / 열벡터(N,1). 방향은 `shape=(행,열)`로 드러남.
> - **기하 해석:** 벡터 = 크기(magnitude([ˈmæɡnɪtjuːd], 매그니튜드, 매그니튜드)) + 방향(angle([ˈæŋɡl], 앵글, 앵글)). 꼬리(시작)→머리(끝). 꼬리가 원점이면 **기준 위치(standard position([ˈstændərd pəˈzɪʃən], 스탠더드 퍼지션, 스탠더드 포지션))**.

> [!note] 🗺️ 이 노트 읽는 법
> [실전선대] 1장을 *척추*로 선형대수의 첫 대상인 **벡터**를 정식으로 판다. 0단계에서 [[02 텐서 이해하기|텐서]]를 "다차원 컨테이너"로 봤다면, 여기선 그 **1차원 사례인 벡터**를 선형대수 정식으로 다룬다. 수학 표기(ℝ^N)와 NumPy shape를 *나란히* 익히는 게 이 장의 핵심.

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 데이터(수의 묶음)를 수학적으로 다루려면, 먼저 그것을 담을 단일 대상이 필요하다. 그 모든 것의 1번 부품이 **벡터 자체**다.
> 1. 여러 수를 하나로 다뤄야 함 → 한 덩이로 묶을 형태 필요 → **벡터(순서 있는 수의 묶음)**
> 2. "정보가 몇 개냐(특징 수)"를 세야 함 → 원소 개수를 셀 척도 필요 → **차원(dimensionality)**
> 3. 같은 수라도 가로/세로에 따라 곱이 달라짐 → 데이터를 행/열로 세울 규약 필요 → **방향(orientation, 행/열)**
> 4. 칠판 수식과 코드의 '차원'이 어긋나 디버깅 지옥 → 둘을 나란히 볼 도구 필요 → **NumPy shape·ndim**
> **한 줄:** 수를 한 대상으로 묶음 → 벡터 → (개수=차원) + (행/열=방향) + (코드=shape)로 정식화.

> [!abstract] 📘 책 핵심 — [실전선대] §1.1 벡터 생성·시각화
> **① 벡터의 두 특징**
> - **차원(dimensionality):** 벡터가 가진 **원소의 수**.
> - **방향(orientation):** **열 방향**(세로로 세움) vs **행 방향**(가로로 누움).
> - 표기 **`ℝ^N`**: ℝ=실수, N=차원. 원소 2개 → `ℝ²`(R2, R^2로도 씀).
>
> **② 열벡터·행벡터 (식 1-1)**
> $$ x=\begin{bmatrix}1\\4\\5\\6\end{bmatrix},\quad y=\begin{bmatrix}.3\\-7\end{bmatrix},\quad z=\begin{bmatrix}1&4&5&6\end{bmatrix} $$
> - x = 4차원 **열**벡터 (`x∈ℝ⁴`), y = 2차원 열벡터, z = 4차원 **행**벡터.
> - **x와 z는 다른 벡터다.** 같은 순서·같은 원소라도 **방향이 다르면 엄밀히 다름**.
>
> **③ 칠판 수학 vs 코드 수학 (중요한 용어 충돌)**
> - **수학적 차원** = 원소 수. **파이썬 차원** = 출력에 쓰는 *기하학적 차원의 수*(NumPy `ndim`, 축 개수).
> - 모든 벡터는 NumPy에서 **2차원 배열**로 만들 수도, *방향 없는* 수 나열은 **1차원 배열**로도. 수학적 차원(원소 수)은 파이썬에선 **length / shape**.
> - 표기: 벡터 v = 소문자 진한 로마자 **v**(또는 이탤릭 *v*, 화살표 $\vec v$). 행벡터 = **$w^\mathsf{T}$** (T=전치 → [[02 텐서 이해하기|전치]]).
>
> > [!quote]- 📐 NOTE — 벡터 방향이 중요한가? (책)
> > 데이터를 *저장*할 때 방향은 보통 무관. 하지만 **어떤 파이썬 연산은 방향이 틀리면 예기치 않은 결과·오류**(브로드캐스팅 등). "행벡터를 열벡터로 바꿔야 함"을 몰라 30분 디버깅하는 일을 피하려면 방향을 의식해야 한다.
>
> **④ NumPy 생성 4가지 (코드)**
> ```python
> asList  = [1,2,3]                 # 리스트(선대 연산엔 부적합)
> asArray = np.array([1,2,3])       # 1차원 배열 — 방향 없음
> rowVec  = np.array([[1,2,3]])     # 행(1×3)
> colVec  = np.array([[1],[2],[3]]) # 열(3×1)
> ```
> - 방향은 **대괄호**로 지정: 바깥 대괄호=한 객체, 내부 괄호=행. `rowVec`=1행에 전부, `colVec`=숫자 1개짜리 행이 여럿.
> - **shape**: `asList (3,)` · `asArray (3,)` · `rowVec (1,3)` · `colVec (3,1)`. 차수 = **(행, 열)**.
> - 1차원 `asArray`는 `(3,)`(방향 없음), 방향 부여 벡터는 **2차원** `(1,3)`/`(3,1)`.
>
> **⑤ §1.1.1 기하학적 해석**
> - **대수적 해석:** 순서대로 나열된 수 목록.
> - **기하학적 해석:** 특정 **크기(magnitude·길이)** + **방향(angle, 양의 x축 기준)** 을 가진 직선. **꼬리**(시작)→**머리**(끝), 화살표.
> - 벡터 ≠ 좌표(점)지만, **꼬리가 원점에서 시작하면 일치** = **기준 위치(standard position)**.
> - 기하 해석 = 물리·공학(힘)에 유용 / 대수 해석 = 데이터과학(시간별 판매)에 유용.
>
> ![[그림 1-1 다양한 위치의 벡터.png|460]]
> ▲ 그림 1-1 — 모든 화살표는 **동일한 벡터**(크기·방향 같음). 기준 위치 벡터는 꼬리가 원점, 머리가 기하학적 좌표를 가리킨다.

## 🧠 근본 이해 · 융합 · 통찰

### 🔗 0단계 '텐서'와의 연결 — 벡터 = 1차 텐서
[[02 텐서 이해하기|텐서]]에서 "스칼라(0차)·벡터(1차)·행렬(2차)·…"로 봤던 그 **벡터(1차 텐서)** 를 선형대수로 정식화한 것. 0단계는 *컨테이너·shape* 관점, 여기선 *수학적 대상(차원·방향·기하)* 관점. 같은 객체의 두 렌즈.

### ⚠️ 가장 헷갈리는 지점 — '차원'이라는 한 단어, 두 의미
| | 수학적 차원 | 파이썬 차원(`ndim`) |
|---|---|---|
| 뜻 | **원소 수** | **축(axis) 개수** |
| `[1,2,3]` | 3차원 벡터 | `ndim=1` (1차원 배열) |
| `[[1,2,3]]` | 3차원 (행)벡터 | `ndim=2` (2차원 배열, shape (1,3)) |

→ "3차원 벡터"가 NumPy에선 1차원 배열일 수 있다. **수학 차원=원소 수=`size`/`len`, 파이썬 차원=`ndim`.** 이 충돌을 모르면 shape 디버깅에서 헤맨다.

> [!tip] 🌱 탐구 · [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|NumPy 벡터 내부 — 1D 블록·strides·(N,1)]]
> `(3,)`(방향 없음) vs `(3,1)`(열) vs `(1,3)`(행)이 *왜* 다른 객체인지 — ndarray=1D 연속 블록+`shape`/`strides`(=C 포인터 산술)라는 내부 정체와, `.T`·`@`·`np.linalg`가 1D에서 깨지는 이유. **두 책의 열벡터 표기 차이의 근원.** ↳ [[_탐구 MOC]]

### 🧭 방향이 '엄밀히 다른' 이유 — 행/열은 곱셈 유효성을 가른다
x(열)와 z(행)가 원소가 같아도 다른 건 단순 규약이 아니다. **행렬 곱은 (행×열) 차수가 맞아야** 성립하므로, `(1,3)`이냐 `(3,1)`이냐가 *어떤 곱이 가능한지*를 결정한다. → 내적/외적·`Wx+b`에서 방향이 결과를 좌우(다음 절·[[3D 변환 행렬 — 동차좌표·회전·투영 (선형대수 응용)|행렬곱 차수]]).

> [!quote] 🌉 타 영역 연결 — 벡터 = '공간의 점 또는 방향'   `#연결/수학`
> 기하 해석의 *기준 위치(꼬리=원점)* 벡터는 곧 **공간의 한 점(좌표)**. [[3D 변환 행렬 — 동차좌표·회전·투영 (선형대수 응용)|3D 변환 노트]]의 `w=1`(점) / `w=0`(방향) 구분이 바로 이 "벡터=점 vs 방향"의 동차좌표판. 그래픽스(공간)·데이터과학(특징)·신경망(`Wx+b`)이 *같은 벡터·행렬*을 공유. 전역 다리 → [[_지식 연결 허브]]

## 🧩 코드 실습 (§1.1)
> 🎯 목표: 벡터를 4가지로 생성해 **방향이 shape에 어떻게 드러나는지** 확인하고, 같은 벡터를 여러 위치에 그려 **기준 위치** 개념을 시각화.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la1_vectors.py` (검증: shape `(3,)·(3,)·(1,3)·(3,1)`)

```python
import numpy as np
asList  = [1,2,3]
asArray = np.array([1,2,3])       # (3,)   방향 없음(ndim=1)
rowVec  = np.array([[1,2,3]])     # (1,3)  행
colVec  = np.array([[1],[2],[3]]) # (3,1)  열
print(asArray.shape, rowVec.shape, colVec.shape)
```

![[벡터 기하 — 동일 벡터의 여러 위치.png|460]]
▲ **내가 직접 그린 코드 출력**(`la1_vectors.py`) — 같은 벡터 `v=[-1,2]`를 세 위치에. 모두 동일 벡터, 꼬리가 원점인 것이 **기준 위치**. (위 📘 책 핵심의 그림 1-1을 NumPy/matplotlib로 재현해 본 것)

관찰 포인트:
- **`asArray`(1차원)는 방향 없음** — 행도 열도 아닌 순수 수 나열. 방향이 필요하면 `[[...]]`로 2차원화.
- **대괄호 깊이 = 방향** — 내부 괄호가 '행'을 만든다. `(1,3)` vs `(3,1)`.
- **벡터는 위치 무관** — 꼬리를 어디 두든 크기·방향 같으면 같은 벡터. 좌표와 일치하는 건 *기준 위치*뿐.

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **벡터** (Vector) | 순서대로 나열된 수 |
| **차원** (Dimensionality, 수학) | 벡터의 원소 수 (≠ 파이썬 `ndim`) |
| **방향** (Orientation) | 열(세로) vs 행(가로) |
| **`ℝ^N`** | N차원 실수 벡터 공간 (ℂ=복소수) |
| **열벡터 / 행벡터** | 세로(N,1) / 가로(1,N); 전치($w^\mathsf{T}$)로 변환 |
| **shape / length** | (행,열) 차수 / 원소 수 = 수학적 차원 |
| **`ndim`** (파이썬 차원) | NumPy 배열의 축 개수(방향 없는 벡터=1) |
| **크기** (Magnitude) | 벡터의 길이 |
| **기준 위치** (Standard Position) | 꼬리가 원점에 있는 벡터(좌표와 일치) |

## 📌 암기 카드
> 앞면(cue)을 보고 뒷면을 떠올린 뒤 확인. (→ [[_핵심 암기장]])

| 앞면 (cue) | 뒷면 (외울 내용) |
|---|---|
| 벡터 정의? | 순서대로 나열된 수 |
| 수학적 차원 = ? | 원소 수 (= `size`/`len`) |
| 파이썬 차원(`ndim`) = ? | 축(axis) 개수 (방향 없는 벡터=1) |
| `ℝ^N` 뜻? | N차원 실수 벡터 (ℝ=실수, ℂ=복소수) |
| 방향(orientation) = ? | 열(세로, N,1) vs 행(가로, 1,N) |
| shape `(3,)` vs `(1,3)` vs `(3,1)`? | 방향 없음 / 행벡터 / 열벡터 (= (행,열)) |
| 같은 원소·같은 순서인데 행≠열? | 방향이 다르면 엄밀히 다른 벡터 (x≠z) |
| 기준 위치(standard position) = ? | 꼬리가 원점에 있는 벡터 (머리 좌표 = 성분) |
| 행벡터 표기? | $w^\mathsf{T}$ (T=전치) |
