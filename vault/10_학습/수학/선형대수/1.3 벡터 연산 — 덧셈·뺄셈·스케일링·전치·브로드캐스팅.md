# 벡터 연산 — 덧셈·뺄셈·스케일링·전치·브로드캐스팅

> 📐 **수학 개념원자(SSoT)** — 선형대수. 벡터 *산술 연산*의 단일 주인(두 책 통합). AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[03 벡터 연산 — 덧셈·뺄셈·스케일링·전치·브로드캐스팅|03(AI)]] ｜ 토대: [[1.1 벡터 — 정의·차원·방향|벡터 정의]] · 다음: [[1.5 벡터 공리 — 덧셈·음수·역원|벡터 공리]]

> 🎯 **핵심 (TL;DR)**
> - **덧셈** = 머리-꼬리(head-to-tail([hɛd tə teɪl], 헤드 투 테일, 헤드 투 테일)) 연결, 원소별 합. **교환법칙** 성립. **같은 차원에서만.**
> - **뺄셈** = 원소별 차. 기하 = *두 머리를 잇는 선*(둘째 머리→첫째 머리) — 직교분해·회귀의 기초.
> - **스케일링(스칼라곱)** = 각 원소에 스칼라 곱, 길이만 조절. **음수 = 방향 반전이지만 같은 선상**(→ 선형 종속의 씨앗, [[1.8 벡터 확장 — 집합·선형결합·선형독립·부분공간·생성·기저|1.8]]).
> - **전치(transpose)** = 행↔열 뒤집기. `vᵀᵀ=v`. $X^{\mathsf{T}}X$ = 대칭 공분산(PCA의 뿌리).
> - ⚠️ **방향이 결과 차원을 바꾼다:** 열벡터+행벡터 → **브로드캐스팅**으로 (3,3) *행렬*. 스칼라-벡터 덧셈도 수학 불가·파이썬 허용.
> - **벡터 평균** = 다 더하고 `1/N` 곱. "데이터 조작 = 벡터 조작".

---

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 데이터(벡터)를 실제로 *조작*하려면 — 합치고, 차이를 재고, 키우고, 축을 맞추고, 중심을 구해야 한다.
> 1. 벡터 하나로는 여러 데이터를 합칠 수 없음 → 결합 연산 필요 → **덧셈(머리-꼬리)**
> 2. 두 데이터가 얼마나·어느 쪽으로 다른가 → 차이 연산 필요 → **뺄셈(두 머리 잇는 선)**
> 3. 크기를 조절하고 싶음(평균의 1/N, 가중치) → 길이 조절 필요 → **스케일링(음수=반전)**
> 4. 내적·노름($v^{\mathsf{T}}v$)을 행렬곱으로 쓰려는데 행/열이 안 맞음 → 축 눕히기 필요 → **전치**
> 5. 모양 다른 배열을 루프 없이 연산 → 자동 확장 필요 → **브로드캐스팅**(단 방향 틀리면 행렬이 튀어나옴)
> **한 줄:** 결합(덧셈)→차이(뺄셈)→조절(스케일링)→정합(전치)→확장(브로드캐스팅) — "데이터 조작 = 벡터 조작". *이 연산들의 무한 조합이 만드는 공간(스팬)은 [[1.8 벡터 확장 — 집합·선형결합·선형독립·부분공간·생성·기저|1.8]]에서.*

> [!abstract] 📘 책 핵심 — [필수수학] §4.1~4.1.2 덧셈과 스케일링
> **① 벡터 생성 (예제 4-1~4-4)** — 리스트 `[3,2]`/`np.array`/3D/5D. 정의·차원·방향·도판(그림 4-1~4-3)은 [[1.1 벡터 — 정의·차원·방향|1.1]] 소관.
>
> **② §4.1.1 덧셈과 결합 (예제 4-5)** — 두 이동을 **머리-꼬리 연결**, 끝점이 결과.
> $$ \vec v+\vec w=\begin{bmatrix}3\\2\end{bmatrix}+\begin{bmatrix}2\\-1\end{bmatrix}=\begin{bmatrix}5\\1\end{bmatrix} $$
> - **교환 법칙(commutative([kəˈmjuːtətɪv], 커뮤터티브, 커뮤터티브)):** $\vec v+\vec w=\vec w+\vec v$ (그림 4-6). 실제 의미: 데이터 합산.
>
> **③ §4.1.2 스케일링 (예제 4-6)** — **스칼라**(하나의 값)를 각 원소에 곱해 길이 조절.
> $$ 2\vec v=\begin{bmatrix}6\\2\end{bmatrix},\quad 0.5\vec v=\begin{bmatrix}1.5\\0.5\end{bmatrix} $$
> - **크기만 변하고 방향은 그대로.** 음수 스칼라 = 방향 반전(그림 4-9)이지만 *같은 선상*. 데이터 의미: **평균** = 합산 후 `1/N` 스케일링.
>
> *(④ §4.1.3 스팬·선형 종속 → [[1.8 벡터 확장 — 집합·선형결합·선형독립·부분공간·생성·기저|1.8]]로 이관 — 아래 위임 콜아웃)*

> [!abstract] 📘 책 핵심 — [실전선대] §1.2 벡터 연산
> **① §1.2.1 덧셈/뺄셈** — 원소별, *같은 차원에서만* (ℝ([ɑːr], 알, 알)³+ℝ⁵ 불가).
> $$ \begin{bmatrix}4\\5\\6\end{bmatrix}+\begin{bmatrix}10\\20\\30\end{bmatrix}=\begin{bmatrix}14\\25\\36\end{bmatrix},\quad \begin{bmatrix}4\\5\\6\end{bmatrix}-\begin{bmatrix}10\\20\\30\end{bmatrix}=\begin{bmatrix}-6\\-15\\-24\end{bmatrix} $$
> - **방향이 중요(식 1-4):** 열벡터 + 행벡터는? → **브로드캐스팅**으로 *행렬*이 나온다.
>
> **② §1.2.2 덧셈/뺄셈의 기하** — 덧셈=머리-꼬리(그림 1-2 A). **뺄셈=두 머리를 잇는 선**(둘째 머리→첫째 머리, 그림 1-2 B). *직교벡터 분해·선형 최소제곱(회귀)의 기초.*
>
> **③ §1.2.3 스칼라-벡터 곱** — 각 원소에 스칼라 λ([ˈlæmdə], 램더, 람다) 곱. $\lambda=4,\ \lambda w=[36,16,4]^{\mathsf{T}}$. 방향 안 바뀌고 크기만(음수=180° 반전, 같은 선상 — 그림 1-3).
> - ⚠️ **λ 신분 지도 (첫 등장 — 전방 주의):** 이 책은 λ를 "아무 스칼라"의 범용 글자로 재사용한다. ① **계수·배율** — 여기 스케일링, [[1.8 벡터 확장 — 집합·선형결합·선형독립·부분공간·생성·기저|1.8]] 선형결합의 λᵢ, [[2.6 행렬 공간 — 열공간·행공간·영공간|2.6]] 열공간에선 전 실수를 훑는 **자유 변수**로 확장 ② **조율 상수** — [[2.1 행렬 연산 — 덧셈·뺄셈·이동·스칼라곱·아다마르곱|2.1]] 이동 $A+\lambda I$, *사람이 고르는 고정값* ③ **고윳값** — $Av=\lambda v$, *행렬이 정해주는 값* `(🔭 예고)`. 같은 글자, 세 신분 — 맥락으로 구분.
>
> ![[LA1-3 스칼라-벡터 곱 (재구성).png|640]]
> 🔄 재구성 [그림 1-3] — 같은 벡터에 스칼라 σ([ˈsɪɡmə], 시그머, 시그마)를 곱함. σ>1 늘림·0<σ<1 줄임·σ<0 방향 반전(같은 선상).
> > [!quote]- 📖 책 원본 [그림 1-3]
> > ![[LA1-3 스칼라-벡터 곱 (책원본).png|620]]
> > [!quote]- 📐 영벡터(zero vector) — 책 보충
> > 모든 원소 0(굵은 **0**). 영벡터로 푸는 건 **자명한 해(trivial([ˈtrɪviəl], 트리비얼, 트리비얼))** — 제외. 선대는 보통 "0 아닌(비자명) 해를 찾아라".
>
> **④ §1.2.4 스칼라-벡터 덧셈** — *수학적으론 불가*(별도 객체). 파이썬은 **브로드캐스팅**으로 각 원소에 더함: `2 + [3,6] = [5,8]`. **벡터 평균** = 덧셈 + 스칼라곱(1/N).
>
> **⑤ §1.2.5 전치(transpose)** — $m^{\mathsf{T}}_{i,j}=m_{j,i}$, 열벡터↔행벡터. **`vᵀᵀ=v`**. *데이터 행렬 $X^{\mathsf{T}}X$ = **대칭 공분산 행렬*** → PCA([piː siː eɪ], 피시에이, 피시에이)가 직교 회전인 이유.
>
> **⑥ §1.2.6 브로드캐스팅** — 현대 컴퓨터 선대만의 연산(전통 교과서엔 없음). 한 벡터를 다른 벡터의 각 원소로 반복 연산. 행+열 → 행렬. 효율·간결(k-평균 등).

> 음수 곱(−v)은 화살표가 반대로 가지만 **여전히 같은 직선 위**다. 그래서 v와 −v는 *선형 종속*(한 선에 갇힘 → [[1.8 벡터 확장 — 집합·선형결합·선형독립·부분공간·생성·기저|1.8]]). "방향이 진짜 다르다"는 건 *다른 직선*일 때만.

> [!tip] 🌱 탐구 · [[1.5 벡터 공리 — 덧셈·음수·역원|왜 '+'가 방향을 만드나 — 덧셈 역원]]
> "더했는데 왜 방향이?" "음수가 왜 뒤집나"의 근본 답 — **방향은 성분 비율의 *파생값***(저장값 아님), **음수=덧셈 역원**(0으로 되돌리는 짝; (-1)v=-v는 공리적 강제). ↳ [[_탐구 MOC]]

> [!tip] 🪂 심화 — 스칼라 `2` vs 1×1 행렬 `[2]`: 결과 같고 정체 다름   `#사이드/심화`
> 벡터에 *숫자 2*를 곱하는 것과 *1×1 행렬 `[[2]]`*를 곱하는 것은 **결과는 `[6,2]`로 똑같지만 타입·메커니즘이 완전히 다르다.**
>
> | | 스칼라 `2` | 1×1 행렬 `[[2]]` |
> |---|---|---|
> | 정체 | 원시 타입(순수 크기/배율) | 2차원 배열 **객체**(공간 변환기) |
> | `ndim` / `shape` | `ndim=0` | `shape=(1,1)` |
> | 연산 | **브로드캐스팅**(모든 원소에 침투) | **행렬 곱**(규격 맞물림: 앞 가로1=뒤 세로1) |
>
> ```python
> col = np.array([[3],[1]])         # 2×1
> col * 2                           # 브로드캐스팅 → [[6],[2]]
> np.dot(col, np.array([[2]]))      # 행렬곱 (2×1)·(1×1) → [[6],[2]]  (같은 결과)
> np.ndim(2), np.array([[2]]).shape # 0 , (1,1)  ← 정체는 다름
> ```
> - **왜 수학은 둘을 엄격히 구분?** *행렬은 공간을 변환하는 **함수***. `[2]`는 "1차원 공간을 2배 스케일하는 변환기", 스칼라 `2`는 공간 의미 없는 순수 숫자([[3D 변환 행렬 — 동차좌표·회전·투영 (선형대수 응용)|3D 변환]] · [[1.1 벡터 — 정의·차원·방향|1.1]]의 '수학 차원 ≠ ndim' 연장).

### 🔑 전치 vᵀ — 노름(vᵀv)·공분산·PCA로 이어지는 핵심 (실전 §1.2.5)
- [[1.2 벡터 크기와 단위벡터 (노름)|노름]]의 **vᵀv**(제곱합)가 곧 이 전치. 전치로 가로 눕혀 행렬곱 = 제곱합.
- **데이터행렬 XᵀX = 공분산 행렬**(대칭) → PCA가 *데이터 공간의 직교 회전*인 근거. 전치가 *통계 구조*를 만든다.
- `vᵀᵀ=v`(왕복) = 가역 연산.
- **왜 전치하나(차원 정합):** 내적을 행렬곱으로 만들려 앞 벡터를 행으로 눕힘 $(1{\times}N)(N{\times}1){=}(1{\times}1)$; $a^{\mathsf{T}}b{=}b^{\mathsf{T}}a$ (≠ $ab^{\mathsf{T}}$, 외적 행렬) → [[1.6 벡터 내적 — 점곱·기하·직교·아다마르곱·외적|왜 전치하나·OOM 함정]].

> [!tip] 🌱 탐구 · [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|전치가 1D에서 '먹통'인 이유]]
> `.T`는 **2D `(N,1)`** 에서만 눕는다. **1D `(N,)`** 는 `.T`해도 그대로(뒤집을 축 없음) — `vᵀ` 수식이 코드에서 안 먹힘. 그래서 열벡터는 `(N,1)` 명시. NumPy([ˈnʌmpaɪ], 넘파이, 넘파이) 내부(1D 블록+strides([straɪdz], 스트라이즈, 스트라이드))까지 → [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]. ↳ [[_탐구 MOC]]

### 🧩 스칼라-벡터 덧셈 '불가'인데 파이썬은 됨 — 칠판 수학 ≠ 코드 수학 (실전 §1.2.4)
수학적으론 벡터+스칼라 불가(별도 객체)인데, 파이썬은 **브로드캐스팅**으로 허용. 위 '스칼라 2 vs 1×1 행렬'·[[1.1 벡터 — 정의·차원·방향|1.1]]의 '차원 충돌'과 같은 테마. 브로드캐스팅은 *편의*지 수학 정의가 아님 → 의도치 않은 shape 변화 주의.

> [!quote] 🌉 타 영역 연결 — 브로드캐스팅 = '여럿에 같은 연산 살포'   `#연결/CS`
> 브로드캐스팅은 [[GPU 하드웨어 구조 — SIMT·SM·텐서코어 (ARM 관점)|GPU SIMT(하나의 명령을 수천 코어에)]]와 같은 발상의 소프트웨어판 — *루프 없이 벡터화*. 그래서 빠르다. 전역 다리 → [[_지식 연결 허브]]

> [!quote] 🌉 타 영역 연결 — 스케일링·음수반전 = 선형 변환의 특수 경우   `#연결/수학`
> 스칼라 곱·음수 반전은 [[3D 변환 행렬 — 동차좌표·회전·투영 (선형대수 응용)|3D 변환]]의 **크기 행렬·반사**의 1차원 사례. 덧셈·스케일링이 모여 만드는 *선형 결합*([[1.8 벡터 확장 — 집합·선형결합·선형독립·부분공간·생성·기저|1.8]])이 곧 신경망 `Wx+b`([[05 퍼셉트론·편향·은닉층|선형층]])의 본질. 전역 다리 → [[_지식 연결 허브]]

> [!tip] 🪂 사이드 · 심화 — 벡터 평균 = 데이터 평균 (스케일링의 응용)   `#사이드/심화`
> N개 벡터의 평균 = 다 더하고 `1/N` 곱. "동네 주택들의 평균 가격·면적" = 주택 벡터들을 더해 `1/N` 스케일. **데이터 평균이 곧 벡터 연산** — 통계의 평균·중심화(centering([ˈsɛntərɪŋ], 센터링, 센터링))가 선대 위에 있다.

## 🧩 코드 실습 (필수 §4.1 + 실전 §1.2)
> 🎯 목표: 덧셈(교환법칙)·뺄셈·스케일링(음수 반전)·열+행=행렬·전치(vᵀᵀ=v)·평균을 코드로.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/md4_vector_ops.py` · `la1_2_ops.py`

> [!tip] 🧭 매핑 템플릿 — [수학] → [책 1D] → [내 2D]   `#규칙`
> 열벡터 $v=\begin{bmatrix}3\\2\end{bmatrix}$ 를:
> - **책(1D shortcut):** `np.array([3,2])` → `(2,)` *(.T 먹통)*
> - **내 표기(2D 명시):** `np.array([[3],[2]])` → `(2,1)` *(수식 1:1)* — 규칙 출처 [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]

```python
import numpy as np
# 덧셈 (예제 4-5) — 교환법칙 (열벡터 (N,1) 명시)
v = np.array([[3], [2]]); w = np.array([[2], [-1]])
print((v + w).ravel(), np.array_equal(v+w, w+v))      # [5 1] True
# 스케일링 (예제 4-6) — 음수=방향반전(같은 선상)
a = np.array([[3], [1]])
print((2.0*a).ravel(), (0.5*a).ravel(), (-1*a).ravel())  # [6. 2.] [1.5 0.5] [-3 -1]
# 뺄셈 (실전 §1.2.1)
p, q = np.array([4,5,6]), np.array([10,20,30])
print(p - q)                                          # [-6 -15 -24]
# 방향 다르면 행렬! (브로드캐스팅)
col = np.array([[4,5,6]]).T; row = np.array([[10,20,30]])
print((col + row).shape)                              # (3,3)
# 전치 왕복 / 스칼라-벡터 덧셈(broadcasting) / 평균
print(np.array_equal(row.T.T, row))                   # True (vᵀᵀ=v)
print(2 + np.array([3,6]))                            # [5 8]
print((np.array([2,8]) + np.array([4,2])) * (1/2))    # [3. 5.] 평균
```

관찰 포인트:
- **같은 차원만 덧셈/뺄셈** — 다르면 `ValueError`, *방향*이 다르면 브로드캐스팅 **행렬**.
- **`vᵀᵀ=v`** 왕복. 데이터행렬엔 `XᵀX`로 공분산.
- **열벡터는 `(N,1)` 명시** — 1D `(2,)`는 `.T` 먹통([[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]).

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **벡터 덧셈** | 머리-꼬리 결합(원소별 합); 교환법칙 `v+w=w+v`; 같은 차원만 |
| **벡터 뺄셈** | 원소별 차; 기하=두 머리를 잇는 선(직교분해·회귀 기초) |
| **스케일링 / 스칼라** | 스칼라(한 값) 곱으로 길이 조절; 음수=방향 반전(같은 선상) |
| **전치** (Transpose, vᵀ) | 행↔열 뒤집기; `vᵀᵀ=v`; `XᵀX`=공분산(대칭) |
| **브로드캐스팅** | 작은 배열을 전체에 퍼뜨려 연산(행+열→행렬); 컴퓨터 선대 전용 |
| **스칼라-벡터 덧셈** | 수학 불가, 파이썬은 broadcasting([ˈbrɔːdkæstɪŋ], 브로드캐스팅, 브로드캐스팅)으로 허용 |
| **벡터 평균** | 다 더하고 1/N 곱(=데이터 평균) |
| **영벡터 / 자명한 해** | 모든 원소 0 / 0으로 푸는 명백한 해(보통 제외) |
