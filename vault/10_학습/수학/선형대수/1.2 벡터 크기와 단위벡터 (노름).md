# 벡터 크기와 단위벡터 (노름)

> 📐 **수학 개념원자(SSoT)** — 선형대수. 정의·성질·증명·기하 직관. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[02 벡터 크기와 단위벡터 (노름)|02(AI)]]

> 🎯 **핵심 (TL;DR)**
> - **크기/노름 ‖v‖** = $\sqrt{\sum v_i^2}$ (원소 제곱합의 제곱근 = 유클리드 길이). 표기: 이중 수직 막대 ‖v‖.
> - **제곱 크기 ‖v‖²** = 제곱근 생략(일부 응용; sqrt가 비싸서 더 쌈).
> - ⚠️ **또 용어 충돌:** `len()`=수학적 **차원**(원소 수) / `np.linalg.norm()`=기하학적 **크기(길이)**. 책은 혼란 피해 '길이' 대신 **크기/노름** 사용.
> - **단위벡터** ‖v‖=1. 만들기: $\hat v = \frac{1}{\lVert v\rVert}\,v$ (노름의 역수를 스칼라 곱) → 방향 같고 크기만 1.
> - ⚠️ **예외(영벡터):** ‖0‖=0 → 1/0 불가 → **영벡터는 연관 단위벡터가 없다.**

---

> 💡 **이 노름이 사실 '거대해진 피타고라스'** — 기하·물리·선대를 관통하는 한 뿌리: [[벡터 — 기하·물리·선형대수를 관통하는 한 뿌리 (피타고라스·삼각함수)|벡터 융합 통찰]]

> [!question] 🎯 왜 이걸 배우나 — 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 벡터 하나의 *크기*부터 잴 수 있어야, "두 벡터가 얼마나 가까운가/닮았나"도 잴 수 있다.
> 1. 벡터(01)는 생겼는데 "얼마나 큰가·원점에서 얼마나 먼가"를 못 잼 → 길이를 재는 자 필요 → **노름 ‖v‖(유클리드 거리)**
> 2. 매번 제곱근까지 구하는 게 비교·최적화엔 낭비 → 제곱근 생략한 값 필요 → **제곱 크기 ‖v‖²**
> 3. 크기가 제각각이라 "방향만" 깨끗이 비교가 안 됨 → 크기를 1로 표준화 필요 → **단위벡터·정규화 ($\hat v=v/\lVert v\rVert$)**
> 4. 단, 0은 1로 못 만듦(1/0) → 예외 처리 필요 → **영벡터엔 단위벡터 없음**
> **한 줄:** 크기를 재는 자(노름) → 빠른 비교(제곱 크기) → 방향만 남기는 정규화(단위벡터).

> [!abstract] 📘 책 핵심 — [실전선대] §1.3 벡터 크기와 단위벡터
> **① 크기(노름)**
> - 벡터의 **크기**(기하학적 길이 = **노름 norm**) = 벡터의 *꼬리부터 머리까지의 거리* = 표준 **유클리드(Euclidean([juːˈklɪdiən], 유클리디언, 유클리드)) 거리**(원소들의 제곱합에 제곱근). 표기 **‖v‖**.
> > [!info] 📐 꼬리·머리란? — 벡터의 시작점·끝점(영어 정식 명칭)
> > - **꼬리 = 시작점.** 통용어 **tail([teɪl], 테일, 테일)**, 정식 명칭 **초기점(initial point([ɪˈnɪʃəl pɔɪnt], 이니셜 포인트, 이니셜 포인트))**. *기준 위치(standard position([ˈstændərd pəˈzɪʃən], 스탠더드 퍼지션, 스탠더드 포지션))* 에선 꼬리가 **원점 (0,0)**.
> > - **머리 = 끝점(화살촉).** 통용어 **head([hɛd], 헤드, 헤드)** 또는 **tip([tɪp], 팁, 팁)**, 정식 명칭 **종점(terminal point([ˈtɜːrmɪnəl pɔɪnt], 터미널 포인트, 터미널 포인트))**. *좌표값이 가리키는 위치*.
> > - 그래서 **노름 = 초기점→종점 거리**. 꼬리를 원점에 두면 *머리 좌표 = 벡터 성분*, 노름 = 원점→그 점 거리. (기준 위치 정리: [[01 벡터 — 정의·차원·방향 (NumPy 생성·시각화)|벡터 정의]])
> $$ \lVert \mathbf{v} \rVert = \sqrt{\sum_{i=1}^{n} v_i^2} \quad\text{(식 1-7)} $$
> - **제곱 크기 ‖v‖²**: 일부 응용에서 오른편 제곱근 항을 제거하고 사용.
>
> **② 칠판 vs 코드 용어 (또 충돌)**
> - 수학: 벡터의 **차원**=원소 수, **길이**=기하학적 거리. 파이썬: `len()`은 배열의 **차원**(원소 수)을, `np.linalg.norm()`은 **기하학적 길이(크기)** 를 반환.
> - 혼란 방지로 책은 '길이' 대신 **크기(기하학적 길이, 노름)** 라는 용어를 계속 사용.
> ```python
> v = np.array([1,2,3,7,8,9])
> v_dim = len(v)               # 수학의 차원(원소 수) = 6
> v_mag = np.linalg.norm(v)    # 수학적 크기·길이·노름 ≈ 14.42
> ```
>
> **③ 단위벡터 (unit vector([ˈjuːnɪt ˈvɛktər], 유닛 벡터, 유닛 벡터))**
> - 기하학적 길이가 **1**인 벡터. **‖v‖ = 1** 로 정의. 응용: 직교행렬·회전행렬·고유벡터·특이벡터.
> - 대부분의 벡터는 단위벡터가 아니다(단, 단위벡터 집합도 비단위벡터 집합도 둘 다 무한). **모든 비단위벡터는 같은 방향의 단위벡터와 연관** — 노름의 역수를 스칼라 곱하면 만든다.
> $$ \hat{\mathbf{v}} = \frac{1}{\lVert \mathbf{v} \rVert}\,\mathbf{v} \quad\text{(식 1-8)} $$
> - 그림 1-4: 단위벡터(강조색)는 비단위벡터(검은색)를 *가공해* 생성. **두 벡터의 각도(방향)는 동일, 크기만 다름.**
>
> ![[그림 1-4 단위벡터와 비단위벡터.png|460]]
> ▲ 그림 1-4 — 단위벡터(파랑, ‖v̂‖=1)는 비단위벡터(검정, ‖v‖≠1)를 정규화해 만든다. **방향(각도) 같고 크기만 다름** = '단위'는 길이 1 표준일 뿐 쌓는 블록이 아님.
> > [!quote]- 📐 NOTE — '모든 비단위벡터가 단위벡터를 가진다'의 예외
> > 완벽히 맞진 않다. 비단위 길이지만 연관 단위벡터가 **없는** 벡터 = **영벡터(zero vector([zɪəroʊ ˈvɛktər], 지어로우 벡터, 제로 벡터))**. ‖0‖=0이라 $\frac{1}{0}$ 불가. (단위벡터 만드는 파이썬 코드는 책 1장 마지막 연습문제)

## 🧠 근본 이해 · 융합 · 통찰

### 🔢 노름 수식 읽는 법 — Σ([sʌˈmeɪʃən], 서메이션, 서메이션) = for loop([fɔːr luːp], 포루프, 포 루프), vᵀ([trænsˈpoʊz], 트랜스포우즈, 트랜스포즈)v = 행렬곱 (개발자 관점)
> 수식 $\lVert v\rVert=\sqrt{\sum_{i=1}^{n} v_i^2}$ 의 시그마·$v^\mathsf{T}v$ 가 외계어로 보이는 건, "제곱해서 더한 뒤 루트"를 *압축한 기호*일 뿐이라서. 코드로 치환하면 끝난다.

**① 2D 대각선 → N차원 노름 (알파벳이 부족해서 시그마)**
- 2D `√(x²+y²)` → 3D `√(x²+y²+z²)` → 100차원이면 x,y,z 알파벳 부족 → 원소를 $v_1,v_2,\dots,v_n$ 으로 인덱싱 → 긴 덧셈을 **시그마 Σ로 압축**. 즉 **거대해진 피타고라스**([[벡터 — 기하·물리·선형대수를 관통하는 한 뿌리 (피타고라스·삼각함수)|융합 통찰]]).

**② 시그마 Σ = `for` 루프** — 1:1 매핑
| 수식 조각 | 코드 |
|---|---|
| $\sqrt{\phantom{x}}$ | `math.sqrt(...)` (마지막에 루트) |
| $\sum_{i=1}^{n}$ | `for i in range(1, n+1): total += ...` (1~n 누적) |
| $v_i^2$ | `v[i]**2` (i번째 원소 제곱 = 루프 바디) |

```python
def norm(v):
    total = 0
    for x in v:        # Σ
        total += x**2  # vᵢ²
    return math.sqrt(total)   # √
```

**③ $v^\mathsf{T}v$ = 행렬 곱으로 쓴 제곱합** — 시그마조차 귀찮아서
- $v^\mathsf{T}$(전치 → [[02 텐서 이해하기|전치]])는 세로 열벡터를 가로로 눕힘. $v^\mathsf{T}v$ = 가로×세로 **행렬 곱(내적)**:
$$ v^\mathsf{T}v=\begin{bmatrix}v_1&v_2&v_3\end{bmatrix}\begin{bmatrix}v_1\\v_2\\v_3\end{bmatrix}=v_1^2+v_2^2+v_3^2 \;\Rightarrow\; \lVert v\rVert=\sqrt{v^\mathsf{T}v} $$
- **왜 이렇게?** GPU([dʒiː piː juː], 지피유, 지피유)·NumPy([ˈnʌmpaɪ], 넘파이, 넘파이)는 "루프 돌며 하나씩 제곱·합"보다 **`np.dot(v.T, v)` 통짜 행렬곱**이 하드웨어 가속에 훨씬 빠르고 간결하기 때문 → [[GPU 하드웨어 구조 — SIMT·SM·텐서코어 (ARM 관점)|텐서 코어(행렬곱 매크로)]].

> [!tip] 🌱 탐구 · [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|vᵀv를 (N,1)로 적는 이유]]
> $v^\mathsf{T}v$ 를 코드로 *제대로* 쓰려면 열벡터를 **2D `(N,1)`** 로: `col.T @ col` = `(1,N)@(N,1)` = `(1,1)` 스칼라. 1D `v@v`도 같은 값이나 그건 *내적*(행렬곱 규격 아님)이라 `.T`·`np.linalg`에서 어긋난다 → [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]. ↳ [[_탐구 MOC]]

**④ 세 표기 = 똑같은 연산** (코드로 검증: `la1_3_norm.py`)
$$ \underbrace{\sqrt{x^2+y^2+z^2}}_{\text{기하 직관}} \;=\; \underbrace{\sqrt{\textstyle\sum_{i=1}^{n} v_i^2}}_{\text{시그마(대수)}} \;=\; \underbrace{\sqrt{v^\mathsf{T}v}}_{\text{행렬곱(AI)}} $$
→ 표기법만 다를 뿐 *"모든 원소를 제곱해 더한 뒤 루트 = 고차원 대각선 길이"*. (알파벳 부족 → 시그마, 루프도 귀찮 → 행렬곱)

> [!tip] 🪂 사이드 · 심화 — 시그마 위치(인라인 vs 디스플레이)에 속지 말기   `#사이드/심화`
> 같은 시그마라도 **독립 수식(Display([dɪˈspleɪ], 디스플레이, 디스플레이))** 이면 시작값 $i{=}1$ 이 *기호 정확히 아래*·끝값 $n$ 이 *위*에 오고, **문장 속(Inline([ˈɪnlaɪn], 인라인, 인라인))** 이면 *우측 아래(첨자)·우측 위*로 압축된다($\sum_{i=1}^{n}$). 위치만 다를 뿐 **for 루프 구조는 동일**: 루프변수 $i$, 초기값 $i{=}1$, 끝점 $n$, 바디 $v_i^2$. "폰트 정렬 위치만 바뀐 루프문"으로 읽으면 된다.

### 🔀 물리 벡터(극좌표) vs 성분 벡터 — 노름이 "보이는지 안 보이는지"의 차이
> 💭 막힘: *"벡터는 원래 크기가 정해져 있는 거 아닌가? (3,4)로 적든 뭘로 적든 노름은 이미 벡터 안에 있는 값 아닌가?"*

- **표기법이 둘로 갈린다.** 물리 시간엔 벡터를 보통 **극좌표(polar, 크기+방향)** 로 줌 — "5N, 53°"처럼 크기가 표기 자체에 이미 적혀 있음. 반면 선형대수·이 책은 벡터를 **성분(좌표)** 으로 적음 — (3,4). 이 표기엔 크기가 안 보임 — 3과 4를 아무리 들여다봐도 5가 안 나오고, $\sqrt{3^2+4^2}$ 을 실제로 계산해야 5가 나온다. **그 계산 행위 자체가 노름.**
- **벡터의 정의(성분 목록) 자체엔 "크기"라는 필드가 없다.** 노름은 재정의가 아니라 성분에서 크기를 뽑아내는 계산. 증거: 같은 (3,4)에 L2를 쓰면 5, L1을 쓰면 7, L∞를 쓰면 4(→ 아래 "노름은 하나가 아니다") — 크기가 벡터 안에 고정값으로 박혀 있었다면 이런 일은 불가능하다.
- **힘 합성 예시도 사실은 이미 노름을 쓰고 있었다.** 동쪽 3N + 북쪽 4N을 더하면 방향(53°)만 나오는 게 아니라 크기(5N)도 나와야 완전한 답 — 그 5를 구하는 계산 √(3²+4²)이 정확히 노름. 물리에서 "합력의 크기"라 부르던 계산이 노름과 같은 것이었다.

![[극좌표 vs 성분 벡터 (노름 가시성).png|620]]
▲ 같은 벡터, 두 표기법. 극좌표(왼쪽)는 크기가 표기 자체에 이미 적혀 있고, 성분(오른쪽)은 노름을 계산해야만 크기가 나온다. 이 그림이 실제로 이 막힘을 풀어준 결정적 순간.

> 한 줄: 물리의 "이미 아는 크기"는 극좌표라서 공짜로 보이는 것이고, 성분 표기에선 노름을 계산해야만 그 크기가 나온다. 노름은 뒤늦게 발명된 군더더기가 아니라 표기법 차이가 만드는 필연적 계산이다.

### 🔑 노름은 다대일(多對一) — 원래 벡터를 못 되돌리는 이유
> 💭 막힘: *"노름 값만 알면 원래 벡터로 되돌릴 수 있는 거 아닌가?"*

- **아니다 — 노름은 정보를 잃는 연산이다.** 노름=5를 만드는 벡터는 (3,4) 하나가 아니다: (5,0),(0,5),(4,3),(-3,4)... 정수만 추려도 여러 개고, 실수까지 허용하면 **반지름 5인 원 위의 모든 점** — 무한개다. 노름 하나의 값에서 원래 성분을 복원할 방법은 없다.
- **왜 하필 무한한가 — 자유도 압축.** 독립된 숫자 n개(자유도 n)를 숫자 1개(자유도 1)로 누르는 연산이 노름이다. 미지수 2개짜리 방정식 $x^2+y^2=25$ 하나의 해는 점 하나가 아니라 곡선(원) 전체 — n≥2인 이상 여러 입력이 같은 출력에 몰리는 건 우연이 아니라 구조적으로 피할 수 없다.
- **이 그림, 이미 위에 있다.** 노름=5인 벡터들의 집합은 위 "단위원→단위구→초구면"과 똑같은 도형이다 — 반지름만 1 대신 5일 뿐. 노름이 일정한 벡터들은 항상 원점 중심의 구(球) 껍질 위에 산다.

> 한 줄: 노름은 "크기"만 남기고 "어느 성분 조합이었는지"는 의도적으로 버린다. 그래서 노름만으론 원래 벡터를 복원 못 하지만, 크기 비교라는 목적엔 그걸로 충분하다.

### 📐 노름은 하나가 아니다 — L1·L2(유클리드)·L∞, 그리고 거리와의 관계
> 막혀 있던 TODO("L1·L∞ 등 다른 노름과 L2의 차이") 해소.

- **노름의 자격 조건(공리) 3개** — 어떤 함수가 "노름"이라 불리려면: ① $\lVert v\rVert\ge0,\ \lVert v\rVert=0\Leftrightarrow v=0$(양의 정부호성) ② $\lVert \alpha v\rVert=|\alpha|\lVert v\rVert$(절대 동차성) ③ $\lVert u+v\rVert\le\lVert u\rVert+\lVert v\rVert$(삼각부등식). 이 노트가 쓰는 $\sqrt{\sum v_i^2}$(L2)는 이 조건을 만족하는 *여러 노름 중 하나*일 뿐이다.
- **일반형(Minkowski/Lp 노름):** $\lVert v\rVert_p=\left(\sum_i|v_i|^p\right)^{1/p}$
  - **L1(맨해튼):** $p{=}1$, $\sum|v_i|$ — 좌표축 방향으로만 이동(도시 블록 거리)
  - **L2(유클리드, 이 노트의 정의):** $p{=}2$, $\sqrt{\sum v_i^2}$ — 직선 최단거리
  - **L∞(체비셰프):** $p\to\infty$, $\max_i|v_i|$ — 가장 큰 성분 하나만 봄
  - 참고: $0<p<1$은 삼각부등식이 깨져 노름이 아니다(볼록성 상실).
- **거리(distance)와의 관계 — 노름은 벡터 1개, 거리는 점 2개.** 둘을 잇는 식: $d(x,y)=\lVert x-y\rVert$("거리 = 차벡터의 노름"). 이 노트의 "노름=꼬리→머리 거리"는 사실 $y=0$(원점)인 특수 케이스이고, 두 점이 둘 다 원점이 아니면 일반형 $\lVert x-y\rVert$ 를 써야 한다.

> 한 줄: L2(유클리드)는 유일한 정답이 아니라 여러 노름 중 책이 고른 기본값. 노름을 알아야 비로소 "두 벡터가 얼마나 다른가(거리)"도 정의된다.

> [!example] 🖼️ 단위 노름 구(球) 시각화 — 포함관계 $L_1\subseteq L_2\subseteq L_\infty$
> "노름=1"을 만족하는 벡터들을 좌표평면에 모으면 노름 종류마다 다른 도형이 된다. 셋을 겹쳐 그리면 크기 순서가 눈으로 보인다.
>
> ![[LA1-3 노름 단위구 포함관계 L1 L2 Linf (재구성).png|420]]
> ▲ $\lVert x\rVert_1=1$(마름모, 주황) ⊂ $\lVert x\rVert_2=1$(원, 파랑) ⊂ $\lVert x\rVert_\infty=1$(정사각형, 핑크). 네 점 $(\pm1,0),(0,\pm1)$에서만 셋이 동시에 접한다 — 그 방향(축 정렬, 성분 하나만 존재)에서만 세 노름이 전부 같은 값 1이기 때문.
>
> - **왜 마름모가 원 안에, 원이 정사각형 안에 있나?** 같은 벡터 $x$에 대해 항상 $\lVert x\rVert_1\ge\lVert x\rVert_2\ge\lVert x\rVert_\infty$(바로 아래 부등식). "노름=1"이라는 *같은 조건*을 걸면, 가장 엄격히 재는 자(L1)는 더 짧은 벡터에서 벌써 1에 도달 → 그 벡터들의 집합(도형)이 작다. 가장 후하게 재는 자(L∞)는 더 긴 벡터라야 1에 도달 → 도형이 크다. "잣대가 헐거울수록 도형이 커진다"는 인과 관계.

> [!note] 📏 같은 벡터, 노름 값 순서 — $\lVert x\rVert_1\ge\lVert x\rVert_2\ge\lVert x\rVert_\infty$
> 위 "물리 벡터 vs 성분 벡터" 절에서 이미 구한 $(3,4)$ 값을 순서로 재배열: $\lVert x\rVert_1=7 > \lVert x\rVert_2=5 > \lVert x\rVert_\infty=4$.
> - **일반 부등식(모든 $x\in\mathbb{R}^n$):** $\lVert x\rVert_1\ge\lVert x\rVert_2\ge\lVert x\rVert_\infty$ — 표본 하나가 아니라 *모든* 벡터에서 성립하는 구조적 사실.
>   - $\lVert x\rVert_2\ge\lVert x\rVert_\infty$ 증명: $\lVert x\rVert_2^2=\sum x_i^2 \ge \max_i x_i^2 = \lVert x\rVert_\infty^2$(다른 항은 모두 $\ge0$이니 최댓값 항 하나보다 합이 작을 수 없음).
>   - $\lVert x\rVert_1\ge\lVert x\rVert_2$ 증명: $\lVert x\rVert_1^2=(\sum|x_i|)^2=\sum x_i^2+2\sum_{i<j}|x_i||x_j|=\lVert x\rVert_2^2+2\sum_{i<j}|x_i||x_j|\ge\lVert x\rVert_2^2$(교차항이 항상 $\ge0$). 이 교차항이 정확히 등호 조건(아래)을 결정한다 — 0 아닌 성분이 2개 이상이면 교차항 중 최소 하나가 $>0$.
> - **⚠️ 등호 조건 — "항상 $>$"는 과장이다.** 0이 아닌 성분이 **2개 이상**일 때만 엄격한 부등호($>$). 0 아닌 성분이 1개뿐인 벡터(예: $(5,0)$)는 세 노름이 전부 5로 같다 — 위 그림에서 세 도형이 만나는 축 위 접점과 정확히 대응.

> [!info] 🔤 왜 하필 "L1·L2"라 부르나 — 이름의 유래 (2026-07-12 웹 교차검증)
> 표기 $L_p,\ \lVert\cdot\rVert_p$ 의 **L은 앙리 르베그(Henri Lebesgue([ɑ̃ʁi ləˈbɛɡ], 앙리 르베그, 앙리 르베그))** 를 가리킨다. 단, **"르베그가 $L^p$ 공간을 직접 정의·명명했다"는 아니다** — 정확히는 "르베그 적분(Lebesgue integral) 이론 위에 세워진 공간이라 그의 이름을 헌정했다"는 뜻이다.
> - **공간을 실제로 형식화한 사람은 리스(Frigyes Riesz, 1910)** — Bourbaki 학파 기록 기준. 르베그의 측도·적분 이론을 발판 삼아 리스가 $L^p$ 공간(가측함수 공간)을 정규화된 벡터공간으로 처음 구성했다. 즉 "이름의 뿌리(적분 이론)=르베그, 공간의 형식화=리스"로 나뉜다 — 원 출처가 이 구분 없이 "르베그가 만들었다"고 단순화한 부분은 부정확.
> - 아래 첨자 $1,2,p$는 노름 공식의 지수 $p$ 그 자체 — $\lVert x\rVert_p=(\sum|x_i|^p)^{1/p}$ 에 $p{=}1,2$를 대입한 표기.
> - **대문자 $L^p$ vs 소문자 $\ell^p$(리틀 엘):** 함수해석학 관행상 연속(적분 기반, 함수 공간)엔 대문자 $L^p$, 이산(합 기반, 벡터·수열)엔 소문자 $\ell^p$를 구분해 쓴다. 이 노트가 다루는 유한차원 벡터 노름은 엄밀히는 $\ell^p$가 맞는 표기이나, 머신러닝 문헌은 관행적으로 대문자 $L1$·$L2$로 뭉뚱그린다(정규화·손실함수 이름 포함) — 표기 편의가 굳어진 관행이지 더 정밀한 표기라서가 아니다.
> - 출처: [Lp space – Wikipedia](https://en.wikipedia.org/wiki/Lp_space) 등 교차확인. 원 자료(블로그·AI 답변)의 "르베그가 만들었다"는 단순화는 위와 같이 보정.

### 🧭 단위벡터 = '방향만 남기기' (크기/방향 분해)
벡터 = 크기 × 방향. 단위벡터화($\hat v$)는 **크기를 1로 떼어내 방향만 추출**하는 연산. [[3D 변환 행렬 — 동차좌표·회전·투영 (선형대수 응용)|3D 변환 노트]]의 *방향 벡터(w=0)* 와 같은 발상 — 위치·크기를 버리고 방향만. 회전행렬·고유벡터가 단위벡터인 이유도 "방향(축)만 의미 있고 크기는 1로 표준화"라서.

> [!tip] 🪂 핵심 오해 교정 — '단위'는 쌓는 블록이 아니다   `#사이드/심화`
> **오해:** '단위(Unit)' = 쪼갤 수 있는 최소 격자(1원·1g처럼) → *"큰 벡터 = 단위벡터를 레고처럼 여러 개 이어 붙인 것"* ❌. (이러면 "세분화하면 다 단위벡터"라는 혼란)
> **정정:** 수학의 '단위'는 *블록*이 아니라 **"길이를 1로 맞춘 표준 상태"**. 단위벡터 = 쌓는 재료가 아니라 **방향만 가리키는 나침반 바늘(길이 1 표지판)**.
> - 🔦 **레이저 비유:** 1m 레이저(단위) vs 5m 레이저(비단위). 5m는 1m 5개를 *이어붙인* 게 아니라 **방향 같고 길이만 긴 하나의 빛줄기**. 5m 대가리를 1m로 줄이면 1m와 *정확히 겹침*.
> - **"모든 비단위벡터는 하나의 단위벡터와 연관" 재독해:** 길이 5인 `(3,4)` · 50인 `(30,40)` · 500인 `(300,400)` 화살표를 전부 정규화(길이 1)하면 **모두 `[0.6, 0.8]` 하나로 수렴**. 방향을 담당하는 나침반은 *딱 하나*.
> → 결론: **벡터 안에 단위벡터가 들어있는 게 아니다.** 단위벡터 = 그 벡터의 *순수 방향 성분 그 자체*. 앞으로 '단위벡터'는 **"방향만 알려주는 길이 1짜리 미니 화살표"** 로 읽으면 된다.

> [!tip] 🪂 왜 하필 '길이 1'? — 1 = 곱셈의 항등원   `#사이드/심화`
> 단위벡터 길이가 *하필 1*인 건 우연이 아니다. **1은 곱셈의 항등원(Multiplicative Identity([ˌmʌltɪplɪˈkeɪtɪv aɪˈdɛntɪti], 멀티플리케이티브 아이덴티티, 멀티플리커티브 아이덴티티))** — 어떤 수에 곱해도 그 크기를 *그대로 보존*한다(5×1=5). 그래서 벡터를 **크기 × 방향**으로 분리할 때:
> $$ v = \lVert v\rVert \cdot \hat v \qquad (\text{물리량(크기)} \times \text{단위벡터(방향)}) $$
> 방향 화살표(단위벡터) 길이가 1이라야, 크기 스칼라를 곱할 때 **크기를 왜곡 없이** 싣고 방향만 전달한다.
> - 만약 방향 화살표 크기가 2였다면: 힘 5 × 2 = **10** (크기 왜곡 = 버그). 1이라야 5×1=5 보존.
> - `5 × [동쪽 단위벡터]` = 크기 5 동쪽 / `100 × [북쪽 단위벡터]` = 크기 100 북쪽 → **물리량 보존 + 방향만 복사**.
> → 정규화 $\hat v=v/\lVert v\rVert$ 가 '방향 추출'이면, $v=\lVert v\rVert\,\hat v$ 는 그 역인 '크기×방향 재조립'. **1의 마법(항등원)** 이 크기·방향을 깔끔히 분리·재결합하게 해준다. (이 *크기×방향* 분해가 곧 §1.2 스칼라-벡터 곱의 의미)

> [!example] 🧭 정규화 = '자기 자신으로 나누기' → 약분으로 방향만 남김   `#사이드/비유`
> 정규화 $\hat v=v/\lVert v\rVert$ 를 *크기×방향*으로 풀면, **자기 크기로 자기를 나누는** 것이라 크기가 약분된다:
> $$ \hat v=\frac{v}{\lVert v\rVert}=\frac{\text{크기}\times\text{방향}}{\text{크기}}=\underbrace{\frac{\text{크기}}{\text{크기}}}_{=\,1}\times\text{방향}=\text{방향} $$
> 크기끼리 치고받다 **1로 증발**하고 분자엔 *방향*만 남는다. ("어떤 수든 자기 자신으로 나누면 1" 산수 규칙을 벡터에 그대로 대입한 것.)
> - 📏 **5cm 막대 비유:** 동쪽 5cm 막대 ÷ 자기 길이 5 = 동쪽 **1cm 막대**. 토막나거나 방향이 바뀌는 게 아니라 *크기만 1로 리셋*.
> → 정규화 = "전체 물리량을 자기 자신으로 나눠 크기를 1로 증발시키고 **순수 방향만** 남기는" 연산. (위 항등원 콜아웃 $v=\lVert v\rVert\hat v$ 의 역방향)

> [!tip] 🪂 위치벡터 + 단위원→구→초구면 (단위벡터들이 사는 곳)   `#사이드/심화`
> 책의 "단위벡터" 직관을 기하로 굳히기. **위치벡터(Position Vector([pəˈzɪʃən ˈvɛktər], 퍼지션 벡터, 포지션 벡터)):** 선형대수의 대전제 = *모든 벡터의 꼬리(시작점)를 원점으로 끌어와 겹침*. 시작점이 다르면 덧셈·곱이 안 되므로(물리처럼 화살표가 떠돌면 X). → "벡터 중심을 원점에 맞춘다"는 직관이 정확.
> - **단위벡터 전체의 집합** = 원점 중심 길이 1 화살표를 사방으로 무한히 그린 껍질:
>   - 2D → **단위원** $x^2+y^2=1$ · 3D → **단위구(Unit Sphere([ˈjuːnɪt sfɪər], 유닛 스피어, 유닛 스피어))** $x^2+y^2+z^2=1$ · N차원 → **초구면(Hypersphere([ˈhaɪpərsfɪər], 하이퍼스피어, 하이퍼스피어))**.
> - **모든 비단위벡터** = 원점에서 이 껍질을 *뚫고 길어지는 빛줄기*. → 고차원도 결국 "공 모양 껍질 위 방향들 + 뻗어나간 길이" 이야기.
>

> [!quote] 🌉 타 영역 연결 — '무한끼리 같은 크기'는 칸토어 집합론   `#연결/수학기초`
> 책의 *"무한 단위벡터 수 = 무한 비단위벡터 수"* 가 찝찝한 이유 = 무한끼리 비교가 되나? → **칸토어:** 두 무한집합 크기는 *"남김없이 짝지을 수 있나(1대1 대응/전단사)"* 로 비교한다.
> - **직관(방향 짝):** 원점에서 뻗는 광선마다 단위원 위 점 하나를 지난다 → 모든 비단위벡터에 *방향을 담당하는 단위벡터*가 있다(정규화 $v/\lVert v\rVert$).
> - 📐 **정밀 보강:** 단, 이 광선 투영은 **1대1이 아니라 '여럿→하나'**(한 광선 위 무수한 비단위벡터가 *같은* 단위벡터 1개에 대응). 그래서 이것만으론 엄밀한 1대1 대응이 아니다. **두 무한이 진짜 '같은 크기'인 근거는 둘 다 연속체 농도(continuum([kənˈtɪnjuəm], 컨티뉴엄, 콘티뉴움), $\mathfrak{c}=2^{\aleph_0}$)** 라서 — 칸토어가 증명한 *"선분 위 점 = 평면 위 점 = 같은 무한"* 의 사례. ('대부분 벡터는 단위벡터 아님' = 연속 실수에서 길이가 정확히 1.000…일 확률 ≈ 0)
>
> ![[단위원과 광선 투영 (여럿→하나).png|520]]
> ▲ 원점에서 뻗는 한 광선 위의 여러 비단위벡터(주황·빨강 `(3,4)` 방향 등)가 단위원 위 **같은 단위벡터 `(0.6,0.8)` 하나**로 투영됨 = *여럿→하나*. 모든 꼬리는 원점(위치벡터). (`la1_3_unit_geometry.py`)
> → 한 줄: "복잡한 크기는 다 버리고 **길이 1 표준 나침반(단위벡터)** 만 갖고 생각하자"가 책의 진짜 메시지. 전역 다리 → [[_지식 연결 허브]]

### ⚡ 제곱 크기(‖v‖²)를 쓰는 이유 — sqrt는 비싸다
‖v‖² = 제곱근을 안 한 형태. 비교·최적화에선 제곱근이 **불필요한 비용**(단조 함수라 대소 비교는 제곱만으로 충분)이라 자주 생략한다. → MSE([ɛm ɛs iː], 엠에스이, 엠에스이) 손실·그레디언트·거리 비교에서 ‖v‖² 사용. (제곱근 한 번이 곱셈 여러 번보다 느림 — CS 연산 비용)

> [!tip] 🪂 사이드 · 심화 — '차원'에 이어 '길이'도 용어가 충돌   `#사이드/심화`
> [[01 벡터 — 정의·차원·방향 (NumPy 생성·시각화)|§1.1]]에서 *차원*(수학=원소 수 vs 파이썬 ndim)이 충돌했는데, §1.3에선 **길이**도 충돌한다: 수학의 '길이'=노름(크기), 파이썬 `len()`='차원'(원소 수). 즉 **수학의 길이 ≠ 파이썬 len**. 책이 '길이' 대신 '크기/노름'을 고집하는 이유. (코드: 크기는 `np.linalg.norm`, 원소 수는 `len`)

## 🧩 코드 실습 (§1.3)
> 🎯 목표: 노름(크기)·제곱 크기를 구하고, 단위벡터를 만들어 **크기가 1·방향 동일**을 검증. 영벡터 예외도 확인.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la1_3_norm.py` (검증: ‖v‖≈14.42, ‖v̂‖=1.0)

```python
import numpy as np
v = np.array([1,2,3,7,8,9])
v_mag = np.linalg.norm(v)          # ‖v‖ = sqrt(Σvᵢ²) ≈ 14.42
sq    = np.dot(v, v)               # ‖v‖² = 208 (제곱근 생략)
v_hat = v / np.linalg.norm(v)      # 식 1-8: 단위벡터
print(np.linalg.norm(v_hat))       # 1.0
# 영벡터: ‖0‖=0 → 1/0 불가 → 단위벡터 없음
```

관찰 포인트:
- **`norm` vs `len`** — `norm`=크기(14.42), `len`=차원(6). 헷갈리지 말 것.
- **`v / ‖v‖`** 한 줄로 정규화 — 크기 1, 방향 그대로(`v̂·‖v‖ == v` 확인).
- **영벡터는 정규화 불가** — `1/0`. 코드에서 normalize 전 `if norm > 0` 가드가 필요한 실전 함정.

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **크기 / 노름** (Magnitude/Norm, ‖v‖) | 꼬리→머리 거리 = $\sqrt{\sum v_i^2}$ (유클리드 길이) |
| **유클리드 거리** | 원소 제곱합의 제곱근(표준 거리) |
| **제곱 크기** (‖v‖²) | 제곱근 생략한 노름(비교·최적화서 비용 절감) |
| **단위벡터** (Unit Vector) | 크기가 1인 벡터(‖v‖=1); 방향만 의미 |
| **정규화** (Normalize) | $\hat v=\frac{1}{\lVert v\rVert}v$ — 크기를 1로 만들기 |
| **영벡터** (Zero Vector) | 모든 원소 0; ‖0‖=0 → 단위벡터 없음(정규화 불가) |
| **시그마 (Σ)** | 1~n 합산 압축 기호 = `for` 루프(초기값 아래·끝점 위·바디 우측) |
| **$v^\mathsf{T}v$** | 전치 후 행렬곱 = 제곱합(Σvᵢ²); √(vᵀv)=노름. GPU에 빠름 |
| **`np.linalg.norm`** | 벡터 크기(노름) 반환 (≠ `len`=원소 수) |
