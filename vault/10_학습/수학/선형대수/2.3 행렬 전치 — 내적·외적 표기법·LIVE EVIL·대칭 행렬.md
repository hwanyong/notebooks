# 행렬 전치 — 내적·외적 표기법·LIVE EVIL·대칭 행렬

> 📐 **수학 개념원자(SSoT)** — 선형대수. 행렬 전치와 *곱셈과의 상호작용*(표기법·순서 규칙)의 단일 주인. 벡터 전치 자체는 [[1.3 벡터 연산 — 덧셈·뺄셈·스케일링·전치·브로드캐스팅|1.3]] 소관.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[13 행렬 전치 — 내적·외적 표기법·LIVE EVIL·대칭 행렬|13 전치(AI)]] ｜ 토대: [[2.2 행렬 곱셈·행렬식|행렬 곱셈(유효성)]] · 다음: [[2.4 역행렬·연립방정식 — 특수 행렬·AX=B 풀이|역행렬]]

> 🎯 **핵심 (TL;DR)**
> - **행렬 전치 = 행↔열 교환** (벡터와 원리 동일, 1장). 식 4-2: $a^{\mathsf{T}}_{i,j}=a_{j,i}$. **이중 전치 = 원래 행렬** ($C^{\mathsf{TT}}=C$). 📌
> - 코드: `A.T`(메서드) / `np.transpose(A)`(함수). ⚠️ **1D 배열엔 `.T`가 아무 일도 안 함** — 경고·오류 없이 그대로 반환(책 각주 3 = [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]의 공식 인증).
> - **내적을 $a^{\mathsf{T}}b$ 로 쓰는 이유** = 유효성 규칙: $(1{\times}M)(M{\times}1)$ → 내부 일치, 외부 $1{\times}1$ = 스칼라. **외적 $ab^{\mathsf{T}}$**: $(M{\times}1)(1{\times}N)$ → $M{\times}N$ 행렬.
> - **LIVE EVIL 📌: 곱의 전치 = 각각 전치하고 순서 뒤집기.** $(LIVE)^{\mathsf{T}}=E^{\mathsf{T}}V^{\mathsf{T}}I^{\mathsf{T}}L^{\mathsf{T}}$ — 임의 개수 행렬에 적용, **유일무이한 방법**.
> - **대칭 행렬(symmetric([sɪˈmetrɪk], 시메트릭, 시메트릭)) = 자신의 전치와 같음: $A^{\mathsf{T}}=A$.** 📌 정방만 가능(비정방은 전치 크기가 다름).
> - **어떤 행렬이든 $A^{\mathsf{T}}A$·$AA^{\mathsf{T}}$ 는 정방 대칭** (곱셈 기법) — 단 **둘은 서로 다른 행렬**(비정방이면 크기부터 N×N vs M×M). 📌 통계 모델·특잇값 분해의 핵심.

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 행렬곱은 내부 차원이 맞아야만 한다([[2.2 행렬 곱셈·행렬식|2.2]]) — 그런데 데이터의 행/열이 안 맞으면?
> 1. 행과 열이 안 맞아 곱셈이 무효 → 축을 뒤집는 연산 필요 → **전치 $a^{\mathsf{T}}_{i,j}=a_{j,i}$** (되돌리기 보장: $C^{\mathsf{TT}}=C$)
> 2. "내적을 왜 $a^{\mathsf{T}}b$ 로 쓰지?"가 관례로만 보임 → 근거 필요 → **유효성 규칙이 정당화**: 전치해야 $(1{\times}M)(M{\times}1)={}$스칼라가 성립
> 3. 곱 $AB$ 를 통째로 전치하면 각각 전치한 것과 같을 줄 알았는데 크기가 안 맞음 → 순서 규칙 필요 → **LIVE EVIL: $(AB)^{\mathsf{T}}=B^{\mathsf{T}}A^{\mathsf{T}}$** (순서 반전)
> **한 줄:** 축 뒤집기(전치) → 그 덕에 내적/외적 표기가 유효성 규칙 위에서 *필연*이 되고 → 곱을 통째로 전치하면 순서가 뒤집힌다(LIVE EVIL).

> [!abstract] 📘 책 핵심 — [실전선대] §4.4 행렬 전치 · §4.4.1 표기법 · §4.5 LIVE EVIL
> **① §4.4 — 행렬 전치**
> - 벡터 전치(1장)와 원리 동일 — **단순히 행과 열을 바꾸는 것**. 표기: 위 첨자 $[\ ]^{\mathsf{T}}$ ($C^{\mathsf{T}}$). **이중 전치 = 원래 행렬** ($C^{\mathsf{TT}}=C$).
> - 공식 정의(식 4-2, 이전 장과 동일): $a^{\mathsf{T}}_{i,j}=a_{j,i}$ — "행과 열을 교환한다"로 기억하는 게 쉽다.
> - 예제: $\begin{bmatrix}3&0&4\\9&8&3\end{bmatrix}^{\mathsf{T}}=\begin{bmatrix}3&9\\0&8\\4&3\end{bmatrix}$
> ```python
> A = np.array([ [3,4,5],[1,2,3] ])
> A_T1 = A.T              # 메서드
> A_T2 = np.transpose(A)  # 함수
> ```
> - 📌 **각주 3:** 1D 배열로 인코딩된 벡터에 전치 메서드를 적용하면? — **아무런 일도 일어나지 않는다.** NumPy는 벡터를 변경하지 않고 *경고나 오류 없이* 동일한 1차원 배열을 반환.
>
> **② §4.4.1 — 내적과 외적 표기법**
> - $M{\times}1$ 열벡터 둘 중 **첫째만 전치**하면 $1{\times}M$ 과 $M{\times}1$ 두 '행렬' → **내부 차원 일치, 외부로 결과 $1{\times}1$ = 스칼라.** **이것이 내적을 $a^{\mathsf{T}}b$ 로 표기하는 이유.**
> - 외적도 동일 추론: $M{\times}1$ 열벡터 × $1{\times}N$ 행벡터 → 내부(1) 일치, 결과 **$M{\times}N$**.
>
> **③ §4.5 — LIVE EVIL (연산 순서)**
> - LIVE EVIL은 **팰린드롬(palindrome([ˈpælɪndroʊm], 팰린드롬, 팰린드롬), 뒤집어도 철자 동일)**이면서 곱셈 전치의 순서를 기억하는 귀여운 연상법.
> - **여러 행렬의 곱을 전치하면 = 개별 행렬을 전치하고 곱하되 순서는 뒤바뀐다** (식 4-3):
> $$ (LIVE)^{\mathsf{T}} = E^{\mathsf{T}}V^{\mathsf{T}}I^{\mathsf{T}}L^{\mathsf{T}} $$
> - 4개뿐 아니라 **임의의 개수** 행렬 곱셈에 적용. 이상해 보여도 **곱을 전치하는 유일무이한 방법**. (연습 문제 4-7에서 직접 테스트)

> [!abstract] 📘 책 핵심 — [실전선대] §4.6 대칭 행렬 · §4.6.1 비대칭으로부터 생성
> **① §4.6 — 대칭 행렬이란**
> - 여러 작업에 편리한 특성이 많고 **수치적으로 안정적** — 컴퓨터 알고리즘 처리에 유리(특별한 속성은 책 전반에서 — `(🔭 예고)`).
> - **정의: 대응되는 행과 열이 같다** = 행↔열 바꿔도 무변화 = **자신의 전치와 같다: $A^{\mathsf{T}}=A$.** 📌
> - 식 4-4(각 행 = 대응 열): $\begin{bmatrix}a&e&f&g\\e&b&h&i\\f&h&c&j\\g&i&j&d\end{bmatrix}$
> - **비정방은 대칭 불가:** $M{\times}N$ 의 전치는 $N{\times}M$ — $M{\ne}N$ 이면 크기부터 다르다. **정방이 아니면 대칭일 수 없다.**
>
> **② §4.6.1 — 비대칭 행렬로부터 대칭 행렬 생성 (곱셈 기법)**
> - **어떤 행렬이든**(비정방·비대칭이라도) 자신의 전치를 곱하면 **정방 대칭**: $A^{\mathsf{T}}A$ 와 $AA^{\mathsf{T}}$ 모두. (타이핑 편의 표기: `AtA`·`A'A`)
> - **증명 ①(정방):** $A$ 가 $M{\times}N$ → $A^{\mathsf{T}}A=(N{\times}M)(M{\times}N)=N{\times}N$ 정방. $AA^{\mathsf{T}}$ 도 동일 논리($M{\times}M$). *(선형대수 증명은 미적분 증명보다 짧고 쉬운 경향 — 연습하기 좋다)*
> - **증명 ②(대칭, LIVE EVIL 이용):**
> $$ (A^{\mathsf{T}}A)^{\mathsf{T}} = A^{\mathsf{T}}A^{\mathsf{TT}} = A^{\mathsf{T}}A $$
> 첫 항 = 끝 항: 전치가 자신과 같으므로 **대칭**. ∎ ($AA^{\mathsf{T}}$ 는 직접 증명해 볼 것 — 개념 기억에 도움)
> - 📌 **$A^{\mathsf{T}}A$ 와 $AA^{\mathsf{T}}$ 는 모두 정방 대칭이지만 서로 같은 행렬이 아니다!** $A$ 가 비정방이면 **크기부터 다르다**($N{\times}N$ vs $M{\times}M$).
> - $A^{\mathsf{T}}A$ 방식 = **곱셈 기법(multiplicative method)**. 정방+비대칭일 때 유효한 **덧셈 기법**도 있으나 응용 가치가 적어 책이 생략 `(🔭 예고 — 연습 문제 4-9)`.

## 🧠 근본 이해 · 융합 · 통찰

### 🔑 각주 3 = T04의 공식 인증 — "1D에 .T는 먹통"
- 책이 각주로 확인해 준다: **1D 배열에 `.T`를 적용하면 경고도 오류도 없이 그대로** — 조용히 실패하는 최악의 유형. [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]에서 파헤친 "1D는 뒤집을 축이 없다"의 책 공식 버전이고, **우리가 열벡터를 명시적 $(N,1)$로 쓰는 규칙의 최종 근거**다.

### 🔑 $a^{\mathsf{T}}b$ 는 관례가 아니라 필연 — 유효성 규칙의 귀결
- [[1.6 벡터 내적 — 점곱·기하·직교·아다마르곱·외적|1.6]]에서 "왜 전치하나(차원 정합)"로 배운 내용이 §4.4.1에서 **완결**된다: 내적 표기 $a^{\mathsf{T}}b$ 는 취향이 아니라 **행렬곱 유효성 규칙(내부 일치·외부 결과)이 강제하는 유일한 쓰기 방식**이다. 외적 $ab^{\mathsf{T}}$ 가 $M{\times}N$ 행렬이 되는 것도 같은 규칙 한 줄에서 나온다.
- 즉 **내적·외적·행렬곱이 전부 한 규칙(내부/외부)의 사례**로 통일된다 — 용어집 곱 지도의 수학적 등뼈.

### 🔑 왜 순서가 뒤집히나 — LIVE EVIL의 이유 (연상법 너머)
- **크기 논증(왜 뒤집힐 수밖에 없나):** $A(M{\times}N)\,B(N{\times}K)$ 에서 $(AB)^{\mathsf{T}}$ 는 $K{\times}M$. 그런데 $A^{\mathsf{T}}B^{\mathsf{T}}$ 는 $(N{\times}M)(K{\times}N)$ — **내부가 안 맞아 아예 무효**. $B^{\mathsf{T}}A^{\mathsf{T}}=(K{\times}N)(N{\times}M)=K{\times}M$ ✔️ — 유효성 규칙이 순서 반전을 강제한다.
- **원소 논증(왜 값도 맞나):** $[(AB)^{\mathsf{T}}]_{ij}=(AB)_{ji}=(A\ j\text{행})\cdot(B\ i\text{열})=(B^{\mathsf{T}}\ i\text{행})\cdot(A^{\mathsf{T}}\ j\text{열})=[B^{\mathsf{T}}A^{\mathsf{T}}]_{ij}$.

> [!example] 🧭 체감 비유 — 옷 입기/벗기 (연산 취소는 역순)   `#사이드/비유`
> 양말 → 신발 순서로 입었으면, 벗을 땐 **신발 → 양말** 역순. "적용을 되돌리는(전치·역행렬) 연산은 순서를 뒤집는다"의 생활 버전 — 역행렬도 같은 패턴: $(AB)^{-1}=B^{-1}A^{-1}$ ([[2.4 역행렬·연립방정식 — 특수 행렬·AX=B 풀이|2.4]], 같은 LIVE EVIL 계열).

### 🔑 대칭 증명 = LIVE EVIL의 첫 수확 — 배운 규칙이 바로 다음 절의 도구
- $(A^{\mathsf{T}}A)^{\mathsf{T}}=A^{\mathsf{T}}A^{\mathsf{TT}}=A^{\mathsf{T}}A$ — 이 3단 증명의 첫 등호가 **LIVE EVIL**(순서 반전), 둘째 등호가 **이중 전치**($C^{\mathsf{TT}}{=}C$)다. §4.5에서 배운 두 규칙이 §4.6에서 즉시 증명 도구로 쓰인다 — "연상법"이 실전 대수 조작이 되는 순간.
- **크기 직관:** $A^{\mathsf{T}}A$ 는 "전치 × 원본" 샌드위치라 내부($M$)가 소거되고 양끝($N$)만 남는다 — 정방이 *강제*되는 구조. $AA^{\mathsf{T}}$ 는 반대로 $M{\times}M$. **같은 재료, 다른 요리** — 절대 동일시하지 말 것. 📌

### 🔑 4장 전체가 $A^{\mathsf{T}}A$ 로 수렴한다 — 이미 세 번 만난 행렬
지금까지 흩어져 있던 조각들이 전부 **"$A^{\mathsf{T}}A$ 는 대칭"** 한 사실의 사례였다:
- **공분산 행렬 $X^{\mathsf{T}}X$** ([[1.3 벡터 연산 — 덧셈·뺄셈·스케일링·전치·브로드캐스팅|1.3]] "XᵀX=대칭 공분산") — 이제 *왜* 대칭인지 증명 완료.
- **릿지 정규화 $X^{\mathsf{T}}X+\lambda I$** ([[2.1 행렬 연산 — 덧셈·뺄셈·이동·스칼라곱·아다마르곱|2.1]] 이동) — 대칭 행렬에 이동을 얹는 구조.
- **PCA $(Y^{\mathsf{T}}Y)v$** ([[2.2 행렬 곱셈·행렬식|2.2]] 행렬-벡터 곱 응용) — 고유벡터를 꺼내는 대상이 바로 이 대칭 행렬.
- 책이 예고한 "통계 모델과 특잇값 분해의 핵심"이 이 수렴점이다 — 대칭이 왜 좋은가(고윳값 성질 등)는 후속 장에서 `(🔭 예고)`.

## 🧩 코드 (§4.4~4.5)
> 🎯 목표: 전치 예제 + 1D `.T` 먹통 확인 + LIVE EVIL 수치 검증.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la4_matrices.py`

```python
import numpy as np
A = np.array([[3, 0, 4], [9, 8, 3]])
print(A.T)                                   # [[3,9],[0,8],[4,3]] (식 4-2 예제)
print(np.array_equal(A.T.T, A))              # True — 이중 전치 = 원래 (Cᵀᵀ=C)
v1d = np.array([3, 4, 5])
print(v1d.T.shape)                           # (3,) — 각주 3: 아무 일도 없음! (T04)
# LIVE EVIL: (AB)ᵀ = BᵀAᵀ  (AᵀBᵀ는 무효)
L = np.random.randn(2, 3); V = np.random.randn(3, 5)
print(np.allclose((L @ V).T, V.T @ L.T))     # True
# (L.T @ V.T)                                # ValueError — (3,2)(5,3) 내부 불일치
```

| **이중 전치** | $C^{\mathsf{TT}}=C$ — 전치 두 번 = 원래(가역 연산) |
| **1D `.T` 먹통** | 1D 배열엔 아무 일도 없음(경고·오류 없이 그대로) — 명시적 (N,1) 규칙의 근거(책 각주 3=T04) |
| **$a^{\mathsf{T}}b$ 표기의 이유** | 유효성 규칙의 필연: (1×M)(M×1)→1×1 스칼라; 외적 $ab^{\mathsf{T}}$=(M×1)(1×N)→M×N |
| **LIVE EVIL** | 곱의 전치 = 각각 전치 + **순서 반전**: $(AB)^{\mathsf{T}}=B^{\mathsf{T}}A^{\mathsf{T}}$; 임의 개수 적용; 유일한 방법 |
| **팰린드롬** (palindrome) | 뒤집어도 같은 단어/구 — LIVE EVIL 연상법의 장치 |
| **대칭 행렬** (symmetric) | $A^{\mathsf{T}}=A$ — 행=대응 열; 정방만 가능; 수치적으로 안정 |
| **곱셈 기법** (multiplicative) | 어떤 행렬이든 $A^{\mathsf{T}}A$(N×N)·$AA^{\mathsf{T}}$(M×M) = 정방 대칭; **둘은 서로 다름**; 공분산·SVD의 핵심 |
| **덧셈 기법** (예고) | 정방+비대칭 전용 대칭화; 응용 가치 적어 책 생략(연습 4-9) |
