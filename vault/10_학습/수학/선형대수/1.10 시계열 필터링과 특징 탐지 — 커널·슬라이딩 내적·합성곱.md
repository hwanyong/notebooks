# 시계열 필터링과 특징 탐지 — 커널·슬라이딩 내적·합성곱

> 📐 **수학 개념원자(SSoT)** — 선형대수(신호처리 응용). 정의·성질·기하 직관. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[08 벡터 응용 — 시계열 필터링과 특징 탐지|08(AI)]]

> **필터링(filtering([ˈfɪltərɪŋ], 필터링, 필터링))** = 내적의 또 다른 응용. 짧은 패턴(**커널**)을 긴 시계열 위에서 한 칸씩 옮기며 매번 내적을 구하는 절차 — 이 책은 이를 **합성곱(convolution)**이라 부른다. 결과 신호의 값이 클수록 그 위치가 커널이 찾는 패턴과 닮았다는 뜻 → **특징 탐지**.

> 🎯 **핵심 (TL;DR)**
> - **메커니즘 = 커널과 신호 조각의 내적을 반복.** 커널 길이만큼 신호를 잘라(윈도우) 내적 → 한 점 생성 → 커널을 오른쪽으로 한 칸 이동 → 반복.
> - **커널(kernel)** = 찾고 싶은 패턴을 담은 짧은 템플릿 벡터. 매끄러운 변동·날카로운 에지·특정 파형 등, 목적에 맞게 설계.
> - 결과 신호의 각 점 = "이 위치가 커널과 얼마나 닮았나"(국소 내적값) → 값이 튀는 곳 = **특징(feature)이 있는 위치**.
> - ⚠️ 이 책이 "합성곱"이라 부르는 절차는 커널을 뒤집지 않으므로 엄밀히는 **상관관계(correlation)** 다 — 아래 정확성 참조.

---

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* [[1.6 벡터 내적 — 점곱·기하·직교·아다마르곱·외적|내적(1.6)]]은 "미리 정해진 두 벡터"가 닮았는지 한 숫자로 답했다. 그런데 신호(음성·주가·센서 데이터)에는 비교할 '두 번째 벡터'가 애초에 없다 — 대신 "이 긴 신호 어딘가에 내가 찾는 짧은 패턴이 있는가"를 물어야 한다.
> 1. 비교 대상이 없음 → 찾으려는 패턴을 직접 짧은 벡터로 만들어야 할 필요 → **커널(패턴 벡터)**.
> 2. 신호가 커널보다 훨씬 길어 한 번에 통째로 내적할 수 없음(내적은 같은 차원끼리만) → 커널 길이만큼만 **국소적으로** 잘라 비교할 필요 → **윈도우(window) 슬라이딩**.
> 3. 창을 한 칸씩 옮기며 내적을 계속 구하면 값들이 새 시계열을 이룸 → 이 반복 절차에 이름이 필요 → **합성곱(convolution)**(→ 아래 ⚠️ 정확성: 엄밀히는 상관관계).
> 4. 결과값이 큰 위치 = 그 자리가 커널 패턴과 닮았다는 뜻 → 커널을 원하는 패턴(에지·파형)에 맞게 설계하면 그 패턴이 있는 위치에서 결과가 튐 → **특징 탐지**.
> **한 줄:** 내적(닮음 한 숫자, 1.6) → 비교 대상이 신호 속 '어딘가'로 바뀜(커널) → 국소 윈도우 슬라이딩 → 반복된 내적(합성곱) → 결과값 크기 = 특징 존재 정도.

---

> [!abstract] 📘 책 핵심 — [실전선대] §3.2 시계열 필터링과 특징 탐지
> - 필터링은 본질적으로 **특징 탐지(feature detection([ˌfiːtʃər dɪˈtekʃən], 피처 디텍션, 피처 디텍션)) 기법**. 커널(kernel([ˈkɜːrnəl], 커널, 커널) — 필터링 세계의 "템플릿")이 시계열 신호의 어느 부분과 일치하는지 찾는다.
> - **메커니즘:** 커널과 신호 조각(커널과 같은 길이) 사이의 **내적**을 계산 → 필터링된 신호의 한 점 생성 → 커널을 **오른쪽으로 한 구간씩 이동**시키며 다른(또는 겹치는) 조각과 내적을 반복. 이 절차의 공식 명칭 = **합성곱(convolution([ˌkɒnvəˈluːʃən], 컨벌루션, 컨볼루션))**.
> - 커널은 매끄러운 변동, 날카로운 **에지(edge([ɛdʒ], 에지, 에지))**, 특정 파형 모양 등 원하는 기준에 최적화되도록 설계한다.
> - 시계열 필터링은 과학·공학 전반의 핵심 주제 — 음악·라디오·통신·위성 등이 이것 없이 존재할 수 없다고 책은 강조한다.
> - 🔭 예고 — 3장 연습문제: 에지 탐지와 시계열 **평활화(equalization([ˌiːkwəlaɪˈzeɪʃən], 이퀄라이제이션, 이퀄라이제이션))**에 내적을 어떻게 쓰는지는 챕터 끝 연습문제에서 다룬다. (지금은 skim-to-anchor만 — [[forward-reference-study-method|전방 참조]])

## 🧠 근본 이해 · 융합 · 통찰

![[LA3-2 시계열 필터링 도식화 (재구성).png]]
🔄 **재구성** [그림 3-2] — 원본 신호(파랑)의 한 윈도우(주황 음영)와 커널(주황 막대)이 정렬돼 있고, 각 점의 내적이 Σ로 모여 필터링된 신호(초록)의 점 하나를 만든다. 이 윈도우가 오른쪽으로 한 칸씩 이동하며 전체 결과 신호를 완성한다. 실제 numpy로 계산한 슬라이딩 내적 값(대칭 평활화 커널 `[1,2,3,2,1]/9`)을 그대로 그렸다 — 코드는 아래 🧩.
> [!note]- 📖 책 원본 [그림 3-2] 관련 메모
> 책의 원본 캡처는 붙여넣기 컨텍스트에만 존재해 크롭 저장이 불가했다. 다만 원본 그림의 원본 신호는 **노이즈성 예시 곡선**(정밀 수치를 읽어낼 대상이 아님)이라 [[figure-regeneration-policy|재구성 정책의 모사 예외 조항]](노이즈 등은 전사가 무의미 — 재구성과 정보 동일)에 해당 — 그래서 별도 "책원본 모사"를 만들지 않고 위 재구성 하나로 대체한다. 구조(원본→커널·Σ→필터링된 신호의 3단 레이아웃)는 책과 1:1로 맞췄다.

> [!warning] ⚠️ 정확성 — 이 책의 "합성곱"은 엄밀히는 상관관계(correlation)다   `#정확성`
> **합성곱(convolution)의 정의**는 커널을 **뒤집은(반전, flip)** 뒤 슬라이딩하며 내적하는 것이다: $(f*g)[n]=\sum_m f[m]\,g[n-m]$. 반면 이 책이 설명한 절차(커널을 뒤집지 않고 그대로 슬라이딩하며 내적)는 **상관관계(cross-correlation)**의 정의다: $(f\star g)[n]=\sum_m f[m]\,g[n{+}m]$.
> - **커널이 대칭(좌우 대칭)이면 둘은 정확히 같다** — 뒤집어도 커널이 변하지 않으므로. 위 재구성에 쓴 평활화 커널 `[1,2,3,2,1]`이 바로 이 경우라 책의 설명이 *이 예시에서는* 수치적으로 정확하다.
> - **커널이 비대칭이면 다르다** — 예: 에지 탐지에 흔한 `[-1,0,1]`류 커널은 뒤집으면 부호가 반전된 `[1,0,-1]`이 되어 결과가 달라진다. 책이 예고한 "챕터 끝 에지 탐지 연습문제"가 바로 비대칭 커널을 쓸 가능성이 높은 지점이라, 그때는 이 구분이 실질적으로 중요해진다.
> - **이게 책만의 느슨함은 아니다:** CNN(합성곱 신경망)도 관용적으로 "convolution"이라 부르지만 대부분의 딥러닝 프레임워크는 실제로 **상관관계**를 구현한다(커널을 학습으로 찾으므로 뒤집어봐야 최적화 결과는 동일해 구현에서 생략하는 관행 — Goodfellow·Bengio·Courville, *Deep Learning*에 명시된 잘 알려진 사실). 즉 이 책의 표현은 신호처리·딥러닝 업계의 흔한 관용 표기를 따른 것.

> [!quote] 🌉 다리 — 이 슬라이딩 내적이 CNN 커널의 원형   `#연결/CS`
> 여기서 "커널을 신호 위에서 슬라이딩하며 국소 내적을 구한다"는 정확히 **합성곱 신경망(CNN)의 합성곱 층**이 이미지·시퀀스 위에서 하는 연산과 같은 골격이다. 차이는 커널 값을 사람이 설계(에지 검출기 등 고전 신호처리)하느냐, 데이터로부터 학습(딥러닝)하느냐뿐 — 연산 자체(윈도우 내적의 반복)는 동일하다. AI 필요성은 → [[08 벡터 응용 — 시계열 필터링과 특징 탐지|08 렌즈]].

## 🧩 코드 실습 (§3.2)
> 🎯 목표: 커널과 신호 조각의 슬라이딩 내적으로 필터링된 신호를 만든다. (검증: 출력 길이 = n−klen+1, 대칭 커널이라 합성곱=상관관계)
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la3_2_filtering.py`

```python
import numpy as np

rng = np.random.default_rng(3)
n = 40
t = np.arange(n)
signal = np.sin(t/5) + 0.4*np.sin(t/2.3) + rng.normal(0, 0.35, n)

kernel = np.array([1, 2, 3, 2, 1], dtype=float)
kernel = kernel / kernel.sum()          # 합=1인 대칭 평활화 커널
klen = len(kernel)

# 슬라이딩 내적 — 이 책의 "합성곱"(커널을 뒤집지 않음 → 엄밀히는 상관관계)
filtered = np.array([np.dot(signal[i:i+klen], kernel) for i in range(n - klen + 1)])
print(len(filtered), n - klen + 1)      # 36 36 (윈도우 개수)
print(kernel.sum())                     # 1.0 (평활화 커널 = 가중평균 → 신호 평균 보존)
```

관찰 포인트:
- **필터링 = 반복된 내적.** 매 위치마다 `np.dot(signal[i:i+klen], kernel)` 한 번.
- 출력 길이가 입력보다 `klen-1`만큼 짧다("valid" 모드) — 커널이 신호 양 끝을 벗어나지 않는 위치만 계산했기 때문.
- 커널 합이 1이면 각 출력값은 그 구간의 **가중평균** → 잡음이 깎이는 평활화 효과가 나온다(🔭 예고 — 평활화 연습문제와 직결).

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **필터링** (Filtering) | 커널과 신호 조각의 내적을 반복해 새 시계열을 만드는 절차; 본질은 특징 탐지 |
| **커널** (Kernel, 필터링) | 찾으려는 패턴을 담은 짧은 템플릿 벡터. ⚠️ GPU의 "커널"(병렬 실행 함수, [[GPU 하드웨어 구조 — SIMT·SM·텐서코어 (ARM 관점)\|GPU 하드웨어]])과 동음이의 — 문맥으로 구분 |
| **특징 탐지** (Feature Detection) | 필터링된 신호값이 튀는 위치 = 커널이 찾는 패턴이 있는 자리 |
| **합성곱** (Convolution) | 커널을 뒤집어 슬라이딩하며 내적하는 연산; 이 책·CNN 관용 표기는 실은 상관관계인 경우가 흔함 |
| **상관관계** (Cross-correlation) | 커널을 뒤집지 않고 슬라이딩하며 내적; 대칭 커널이면 합성곱과 동일 |
| **에지** (Edge) | 신호가 급격히 변하는 지점; 필터링으로 탐지하는 대표적 특징 (🔭 예고) |
| **평활화** (Equalization/Smoothing) | 잡음을 깎아 신호를 매끄럽게 만드는 필터링 응용 (🔭 예고 — 3장 연습문제) |
