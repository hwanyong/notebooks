# 행렬 — 정의·크기·인덱싱·슬라이싱

> 📐 **수학 개념원자(SSoT)** — 선형대수. 정의·표기·인덱싱 규약. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[10 행렬 — 정의·생성·시각화·인덱싱·슬라이싱|10 행렬(AI)]] ｜ 토대: [[1.1 벡터 — 정의·차원·방향|벡터]] · 다음: [[2.2 행렬 곱셈·행렬식|행렬 곱셈]]

> 🎯 **핵심 (TL;DR)**
> - **행렬(matrix([ˈmeɪtrɪks], 메이트릭스, 매트릭스)) = 벡터를 한 차원 더 끌어올린 것.** 표기는 굵은 대문자 $\mathbf{A},\ \mathbf{M}$.
> - **크기 규칙 = (행, 열).** 행 먼저, 열 나중 — $3\times 5$ = 행 3·열 5. 📌
> - **원소 참조:** $a_{3,4}$ = 3행 4열. **수학은 1-기반, 파이썬은 0-기반** → $a_{3,4}$ = `A[2,3]`. 📌
> - **슬라이싱 = 행·열 부분 집합 추출.** `A[1:4:1, 0:5:1]` = 행 2~4 × 열 1~5 (stop 미포함).
> - **큰 행렬은 이미지로 시각화** — 원소 수치 ↔ 색. 색 매핑은 임의라 색 자체는 무의미.

---

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 벡터 하나는 관측 *한 개*(한 고객, 한 신호)만 담는다. 현실 데이터는 수천 관측 × 수백 특징이다.
> 1. 벡터 여러 개를 낱개로 들고 다닐 수 없음 → 벡터들을 *한 객체*에 담을 그릇 필요 → **행렬(벡터를 한 차원 위로)**
> 2. 행렬 하나에 여러 벡터가 들어가니 "몇 번째의 몇 번째"를 부를 이름 필요 → **크기 (행, 열) 규약 + 원소 인덱싱 $a_{i,j}$**
> 3. 수식(1-기반)과 코드(0-기반)가 어긋나 오프바이원 버그 발생 → 변환 규칙 필요 → **$a_{i,j}$ = `A[i-1, j-1]`**
> 4. 전체가 아니라 *일부 행·열*만 꺼내 조작해야 함 → 부분 집합 추출 필요 → **슬라이싱 `start:stop:step`**
> 5. 원소가 수십억 개면 숫자 출력으로는 볼 수 없음 → 한눈 요약 필요 → **이미지 시각화(원소↔색)**
> **한 줄:** 벡터 여러 개를 담는 그릇(행렬) → 부르는 규약(크기·인덱싱) → 꺼내는 도구(슬라이싱) → 보는 도구(이미지 시각화).

> [!abstract] 📘 책 핵심 — [실전선대] 4장 도입 · §4.1 · §4.1.1
> **① 4장 도입 — 행렬이란**
> - 행렬 = **벡터를 한 차원 더 끌어올린 것**. 매우 다채로운 수학적 객체.
> - 저장 예: 일련의 방정식 · 기하학적 변형 · 시간에 따른 입자 위치 · 재무 기록.
> - 데이터 과학 관행: 행렬 = **관측치 행(예: 고객) × 특징 열(예: 구매)** 데이터 테이블.
>
> **② §4.1 — 문맥에 따른 행렬의 세 가지 정의**
> - 옆으로 늘어놓은 **열벡터 집합** (예: 특징별 관측치 테이블)
> - 위로 쌓아 놓은 **행벡터 집합** (예: 각 행이 서로 다른 채널의 시계열인 다중 센서 데이터)
> - **개별 원소가 정렬된 집합** (예: 각 원소가 단일 이미지의 픽셀 강도 값)
>
> **③ §4.1.1 — 시각화·인덱싱·슬라이싱**
> - 작은 행렬은 전체 출력 가능: $\begin{bmatrix}1&2\\ \pi&4\\ 6&7\end{bmatrix},\ \begin{bmatrix}-6&1/3\\ \mathrm{e}^{4.3}&-1.4\\ 6/5&0\end{bmatrix}$ — 하지만 확장성이 없다(데이터 과학 행렬은 수십억 원소).
> - **큰 행렬은 이미지로 시각화** — 각 원소의 수치가 색상에 대응(matplotlib, 그림 4-1). 매핑이 임의라 **색은 중요하지 않음**.
> - 표기: 굵은 대문자 $\mathbf{A}$, $\mathbf{M}$. 크기는 **(행, 열)** — 행 3·열 5면 $3\times 5$.
> - 인덱싱: $a_{3,4}$ = 3행 4열 원소(예제 행렬에선 $a_{3,4}=8$). 📌 **중요한 복습: 수학은 1-기반, 파이썬은 0-기반** → $a_{3,4}$는 `A[2,3]`.
> - 슬라이싱: 시작·마지막 행/열을 지정해 1칸씩 진행하며 부분 행렬 추출.
> ```python
> A = np.arange(60).reshape(6,10)
> sub = A[1:4:1, 0:5:1]   # 행 2~4 × 열 1~5 → [[10..14],[20..24],[30..34]]
> ```

## 🧠 근본 이해 · 융합 · 통찰

### 🔑 행렬 = 그릇의 계층에서 한 칸 위
- 스칼라(수 1개) → 벡터(수의 나열, 1차원) → **행렬(벡터의 나열, 2차원)** → 텐서(3차원+). 각 단계는 *아래 단계를 원소로 담는 그릇*이다.
- "벡터를 한 차원 끌어올림"의 실체: 벡터는 인덱스 1개($v_i$)로, 행렬은 인덱스 2개($a_{i,j}$)로 원소를 지목한다. **차원 = 필요한 인덱스 개수.** ([[1.1 벡터 — 정의·차원·방향|1.1]]의 "수학 차원 ≠ ndim" 구분과 연결 — 여기서 말하는 건 ndim 쪽)

### 🔑 같은 행렬, 세 가지 읽기 — 열 나열 · 행 쌓기 · 원소 격자
같은 $\mathbf{A}$ 를 어떻게 *썰어 읽느냐*가 문맥별 정의 3종의 정체다.
- **열 읽기:** $\mathbf{A} = [\,\mathbf{a}_1 \mid \mathbf{a}_2 \mid \cdots\,]$ — 열벡터를 옆으로 나열. 특징별 관측치 테이블. (열벡터는 $(N,1)$ — 표기 규칙은 [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]])
- **행 읽기:** 행벡터 $(1,N)$ 를 위로 쌓기. 채널별 시계열(다중 센서).
- **격자 읽기:** 벡터 구조 없이 원소 $a_{i,j}$ 의 정렬된 집합. 이미지 픽셀 강도.
- **통찰:** 읽기 방식은 *데이터의 의미*가 정하는 것이지 행렬 자체에 새겨져 있지 않다 — 저장은 하나, 해석이 셋. 이 구분이 나중에 행렬곱의 두 시점([[2.2 행렬 곱셈·행렬식|2.2]] 칸=내적/열=선형결합)을 읽는 눈이 된다.

### 🔑 인덱싱 — 오프바이원의 근원과 half-open의 이득
- 수학 $a_{i,j}$(1-기반)와 파이썬 `A[i-1, j-1]`(0-기반)의 어긋남은 **관례 충돌**이지 수학적 내용이 아니다. 변환은 기계적으로: *각 인덱스에서 1을 뺀다.*
- 파이썬 슬라이스 `start:stop:step`은 **stop 미포함(half-open, $[\text{start},\ \text{stop})$)**. 이게 불편해 보이지만: **길이 = stop − start** 가 즉시 나오고, `A[:k]`와 `A[k:]`가 겹침·누락 없이 이어진다 — 오프바이원을 *줄이는* 설계다.
- 책 예제 검산: `A[1:4:1, 0:5:1]` → 0-기반 행 {1,2,3} = 1-기반 행 2~4, 열 {0..4} = 열 1~5. ✔️

### 🔑 왜 이미지로 보나 — 시각화는 요약이다
- 수십억 원소는 출력도 육안 검사도 불가 → 원소↔색 매핑으로 **구조(블록·노이즈·그레이디언트·대각 패턴)** 만 요약해 본다.
- 색 매핑이 임의라는 것의 의미: 이미지에서 읽을 정보는 *색 자체*가 아니라 **패턴·대비**다.

> [!tip] 🪂 심화 — 열 하나를 뽑을 때 (N,1)을 지키는 법   `#사이드/심화`
> `A[:, 1]` 은 **1D `(N,)`** 로 떨어진다(열벡터 아님). 열벡터 $(N,1)$ 을 유지하려면 **`A[:, [1]]`** 또는 **`A[:, 1:2]`**. 행도 동일: `A[[1], :]` = $(1,N)$. 근거·내부 원리는 [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]] · 표기 규칙 [[_용어집#📐 벡터 행/열 표기 규칙|용어집]].

## 🧩 코드 (§4.1.1)
> 🎯 목표: 크기 (행,열) 확인 + $a_{3,4}$=`A[2,3]` 오프바이원 검증 + 슬라이싱 half-open 확인.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la4_matrices.py`

```python
import numpy as np
A = np.arange(60).reshape(6, 10)   # 6행 10열 — shape = (행, 열)
print(A.shape)                     # (6, 10)
print(A[2, 3])                     # 23 — 수학 a_{3,4} (1-기반→0-기반: 각 -1)
sub = A[1:4:1, 0:5:1]              # 행 2~4 × 열 1~5 (stop 미포함)
print(sub)                         # [[10..14],[20..24],[30..34]] → shape (3,5)
col = A[:, [1]]                    # (6,1) 열벡터 유지 — A[:,1]은 (6,)
```
