# k-평균 클러스터링 — 브로드캐스팅·argmin·거리 척도

> 📐 **수학 개념원자(SSoT)** — 선형대수(머신러닝 응용). 알고리즘·구현·기하 직관. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[09 k-평균 클러스터링 — 거리 기반 비지도 학습|09(AI)]]

> **k-평균 클러스터링(k-means clustering)** 은 그룹 중심까지의 거리를 최소화하도록 다변량 데이터를 상대적으로 적은 수($k$)의 그룹으로 분류하는 **비지도(unsupervised)** 기법이다. 이 절의 진짜 주인공은 클러스터링 자체가 아니라 — **거리를 어떻게 재고, 그 계산을 어떻게 벡터화하는가**다.

> 🎯 **핵심 (TL;DR)**
> - **5단계 알고리즘:** ① 중심 $k$개 무작위 초기화 → ② 모든 점-중심 거리 계산 → ③ 각 점을 최근접 중심에 배정(`argmin`) → ④ 각 중심을 배정된 점들의 평균으로 갱신 → ⑤ 수렴할 때까지(또는 $N$회) ②~④ 반복.
> - **거리 = 유클리드 거리** — 정의·"제곱근을 생략해도 되는 이유"는 [[T08 유클리드 거리 — 두 점 사이 거리·제곱거리의 순위 불변성|T08]] 소관(여기선 재유도 안 하고 적용만).
> - **이중 for문(점×중심) 대신 브로드캐스팅**으로 벡터화 — `(150,2) - (1,2)` 형식상 불가능해 보이지만 파이썬이 자동으로 150회 반복해 뺀다.
> - **`np.argmin`** — 값이 아니라 "가장 가까운 중심의 **인덱스**"를 반환. `np.min`(값)과 혼동 주의.

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 라벨 없는 다변량 데이터(예: 150개 관측치 × 2특징)를 몇 개의 자연스러운 그룹으로 나누고 싶다 — 정답(라벨)은 없다.
> 1. "그룹"을 정의할 기준이 없음 → 그룹의 대표(중심)까지 **가장 가까운 점들의 집합**이라는 기준이 필요 → **중심점(centroid)** 개념.
> 2. 가깝다·멀다를 재려면 척도가 필요 → [[T08 유클리드 거리 — 두 점 사이 거리·제곱거리의 순위 불변성|유클리드 거리]] $\delta_{i,j}=\lVert d_i-c_j\rVert$.
> 3. 관측치 150개 × 중심 $k$개 = 수백 번의 거리 계산이 필요 → 이중 for문은 느리고 장황함 → [[03 벡터 연산 — 덧셈·뺄셈·스케일링·전치·브로드캐스팅|브로드캐스팅]]으로 한 번에.
> 4. 거리들 중 "값"이 아니라 "어느 중심이 가장 가까운가"(위치)가 필요 → **`argmin`**.
> 5. 처음 중심은 무작위라 최적이 아님 → 배정된 점들의 평균으로 중심을 **갱신**하고 반복하면 점점 개선 → 수렴.
> **한 줄:** 거리 재기(T08) → 전수 비교를 벡터화(브로드캐스팅) → 최솟값 위치 찾기(argmin) → 대표를 갱신(평균) → 반복하면 자연스러운 그룹이 드러난다.

> [!abstract] 📘 책 핵심 — [실전선대] §3.3 k-평균 클러스터링
> **① 정의:** 그룹 중심까지의 거리를 최소화하도록 다변량 데이터를 상대적으로 적은 수($k$)의 그룹 또는 범주로 분류하는 비지도 기법. 여기서는 간단한 형태를 구현하며, 정교한 변형도 존재하지만 **모든 k-평균 기법은 본질적으로 이 알고리즘의 확장**이다.
>
> **② 5단계:**
> 1. 데이터 공간에서 임의의 $k$개 중심점을 초기화 (중심 = 클래스/범주, 임의 차원으로 일반화 가능).
> 2. 각 관측치와 각 중심 사이의 유클리드 거리를 계산.
> 3. 각 관측치를 가장 가까운 중심의 그룹에 할당.
> 4. 각 중심을 해당 그룹에 할당된 모든 관측치의 평균으로 갱신.
> 5. 수렴 기준을 만족할 때까지 또는 $N$회까지 2~4단계 반복.
>
> **③ 1단계 구현 — 중심 초기화:** $k$개의 데이터 샘플을 무작위로 선택해 초기 중심으로 삼는다(다른 초기화법도 있음). $k$는 데이터 과학에서 최적값 결정이 어려운 하이퍼파라미터 — 책은 $k=3$으로 고정.
> ```python
> k = 3
> ridx = np.random.choice(range(len(data)), k, replace=False)
> centroids = data[ridx, :]   # data 행렬은 특징별 샘플입니다.
> ```
>
> **④ 2단계 구현 — 거리 계산과 브로드캐스팅:** 수식으로는 $\delta_{i,j}=\sqrt{(\mathrm d_i^x-\mathrm c_j^x)^2+(\mathrm d_i^y-\mathrm c_j^y)^2}$ ($i$=관측치, $j$=중심). 이중 for문(중심 루프 × 관측치 루프)으로 구현할 수도 있지만 벡터·브로드캐스팅을 쓰면 간결·효율적이다:
> ```python
> dists = np.zeros((data.shape[0], k))
> for ci in range(k):
>     dists[:, ci] = np.sum((data - centroids[ci, :])**2, axis=1)
> ```
> `data`는 $150\times2$, `centroids[ci,:]`는 $1\times2$ — **형식상 두 벡터는 뺄 수 없다.** 그러나 파이썬 브로드캐스팅이 중심을 150회 반복해서 각 관측치마다 뺀다. `**2`는 원소별 적용, `axis=1`은 열에 걸쳐(행마다 분리해서) 합을 구하라는 명령 — 출력은 중심 `ci`에 대한 각 점의 거리를 인코딩하는 $150\times1$ 배열.
>
> **⑤ 거리 공식과 코드의 불일치 — "정답은 나중에":** 유클리드 거리 공식엔 제곱근이 있는데 코드엔 빠져 있다. 책은 이 자리에서 "코드가 잘못된 것일까요?" 라고 묻고 답을 미룬 뒤, 3단계 설명 직후 이렇게 답한다: **거리와 거리 제곱은 단조로운 관계이므로 두 메트릭의 결과는 같다 → 제곱근을 추가해도 결과는 그대로면서 코드 복잡성·계산 시간만 늘어남 → 되도록 생략.** (증명·"모노토닉"의 정확한 의미는 [[T08 유클리드 거리 — 두 점 사이 거리·제곱거리의 순위 불변성|T08]] 참조 — 이 원자는 재유도하지 않는다.)
>
> **⑥ 3단계 구현 — 최근접 중심 배정:**
> ```python
> groupidx = np.argmin(dists, axis=1)
> ```
> 최솟값을 반환하는 `np.min`과 최솟값의 **인덱스**를 반환하는 `np.argmin`의 차이를 기억할 것 — 필요한 건 "거리가 얼마인가"가 아니라 "어느 중심이 가장 가까운가"이므로 `argmin`.
>
> **⑦ 4단계 구현 — 중심 갱신:**
> ```python
> for ki in range(k):
>     centroids[ki, :] = [np.mean(data[groupidx==ki, 0]),
>                          np.mean(data[groupidx==ki, 1])]
> ```
> 불리언 마스크(`groupidx==ki`)로 해당 클러스터에 배정된 모든 점을 찾고, 특징(열)별로 평균을 낸다.
>
> **⑧ 5단계 — 반복:** 실무에서는 중심이 더 이상 이동하지 않는 것 등을 종료 기준으로 삼아 수렴할 때까지 반복한다. 책은 그림을 균형 있게 그리기 위해 **3회로 고정**해 반복한다(그림 3-3: 반복 0=초기화, 1~3=갱신 결과).
>
> **⑨ 마무리:** 중심 초기화·종료 기준의 정교한 방법과 최적 $k$를 고르는 정량적 기법은 더 연구하면 배울 수 있다. 그럼에도 모든 k-평균 기법은 본질적으로 이 알고리즘의 확장이며, **선형대수(벡터·노름·브로드캐스팅)가 그 구현의 중심에 있다.**

### 🔑 브로드캐스팅이 여기서 정확히 뭘 하는가 — shape로 추적
$(150,2)$ 인 `data`에서 $(2,)$ 인 `centroids[ci,:]`를 빼는 연산은, 선형대수적으로는 "행렬에서 벡터를 빼는" 형식상 정의되지 않은 연산이다. 그런데 넘파이는 **작은 배열(`centroids[ci,:]`)의 shape을 큰 배열(`data`)에 맞춰 가상으로 복제**해 $(150,2)$로 만든 뒤 원소별로 뺀다(실제 메모리 복제는 없음, 뷰 연산). 그 결과 `(150,2)` 차이 행렬을 얻고, `**2`(원소별 제곱) → `np.sum(..., axis=1)`(행마다 두 특징을 더함, $150\times2\to150\times1$)을 거치면 정확히 "관측치 150개 각각의 (해당 중심까지) 제곱거리"가 나온다. **이중 for문 두 겹이 브로드캐스팅+`axis=1` 합으로 완전히 대체된 것.**

> [!warning] ⚠️ 정확성 — $k$는 데이터가 알려주지 않는다   `#정확성`
> 책이 명시하듯 "데이터 과학 분야에서 최적 $k$를 결정하는 것은 어려운 문제"다. 이 원자·코드는 $k=3$을 **미리 정해 놓고** 시작한다 — 즉 "3개의 자연스러운 무리가 있다"는 가정이 이미 답의 일부로 주어져 있다(합성 데이터라 정답을 알고 만들었기 때문). 실전 데이터에서 $k$를 정하는 정량적 기법(예: elbow method, 실루엣 계수)은 책이 다루지 않으며 이 원자도 다루지 않는다 — 필요성이 생기기 전까지 확장하지 않는다(선행 학습 금지 원칙).

> [!quote] 🌉 다리 — argmin은 "순위만 필요한 모든 곳"에서 반복된다   `#연결/CS`
> `np.argmin(dists, axis=1)`의 패턴 — "여러 후보 중 값이 아니라 위치가 필요하다" — 은 k-평균에 국한되지 않는다. 분류기의 `np.argmax(확률들)`(가장 높은 확률의 클래스), 빔서치의 최선 후보 선택 등도 같은 골격이다. [[T08 유클리드 거리 — 두 점 사이 거리·제곱거리의 순위 불변성|T08]]의 모노토닉 논증이 여기서도 그대로 적용된다 — "값 자체가 아니라 순위만 필요한 연산은, 순서를 보존하는 어떤 변환을 앞뒤로 붙여도 결과가 같다."

## 🧩 코드 실습 (§3.3)
> 🎯 목표: 5단계 알고리즘 전체를 구현하고, 무작위 초기화에서 시작해 실제로 세 군집으로 수렴하는지 검증.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la3_3_kmeans.py`

```python
import numpy as np

rng = np.random.default_rng(0)

# 합성 데이터 — 세 개의 자연스러운 무리(정답을 알고 만듦, 검증용)
c1 = rng.normal([2, 2], 0.6, (50, 2))
c2 = rng.normal([6, 6], 0.6, (50, 2))
c3 = rng.normal([2, 7], 0.6, (50, 2))
data = np.vstack([c1, c2, c3])
rng.shuffle(data)

# ① 중심 초기화
k = 3
ridx = rng.choice(range(len(data)), k, replace=False)
centroids = data[ridx, :].copy()

for iteration in range(3):
    # ② 거리 계산 (브로드캐스팅, 제곱근 생략 — T08 모노토닉 논증)
    dists = np.zeros((data.shape[0], k))
    for ci in range(k):
        dists[:, ci] = np.sum((data - centroids[ci, :])**2, axis=1)
    # ③ 최근접 중심 배정
    groupidx = np.argmin(dists, axis=1)
    # ④ 중심 갱신
    for ki in range(k):
        centroids[ki, :] = [np.mean(data[groupidx==ki, 0]),
                             np.mean(data[groupidx==ki, 1])]

print("수렴한 중심:\n", centroids)                 # [2,2] [6,6] [2,7] 근처로 수렴
print("클러스터 크기:", [np.sum(groupidx==ki) for ki in range(k)])  # [50, 50, 50]
```

관찰 포인트:
- 정답을 알고 만든 합성 데이터(중심 $[2,2],[6,6],[2,7]$)로 검증 — 3회 반복만으로 알고리즘이 수렴한 중심이 실제 정답 근처(오차 < 0.3)로, 클러스터 크기가 정확히 $50/50/50$으로 맞아떨어진다.
- **라벨을 한 번도 쓰지 않았다** — `groupidx`는 알고리즘이 거리만으로 스스로 찾아낸 그룹. 이게 "비지도"의 의미.
- 무작위 초기화(`ridx`)가 매번 달라도 3회 반복 안에 수렴하는 이유는 데이터가 뚜렷이 분리된 무리를 이루기 때문 — 무리가 겹치거나 $k$가 실제 무리 수와 다르면 더 많은 반복이나 다른 결과가 나올 수 있다(🔭 다루지 않음).

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **k-평균 클러스터링** (k-means Clustering) | 중심까지의 거리 최소화 기준으로 데이터를 $k$개 그룹으로 나누는 비지도 기법 |
| **중심점** (Centroid) | 한 클러스터를 대표하는 점; 배정된 관측치들의 평균으로 갱신 |
| **비지도 학습** (Unsupervised Learning) | 정답 라벨 없이 데이터 구조(그룹·패턴)를 스스로 찾는 학습 |
| **`np.argmin`** | 최솟값이 아니라 최솟값의 **인덱스**를 반환 (↔ `np.min`=값 자체) |
| **수렴** (Convergence) | 반복해도 중심이 더 이상 (유의미하게) 이동하지 않는 상태 |
