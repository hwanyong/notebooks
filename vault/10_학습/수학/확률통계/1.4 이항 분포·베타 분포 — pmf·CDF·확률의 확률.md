# 이항 분포·베타 분포 — pmf·CDF·확률의 확률

> 📐 **수학 개념원자(SSoT)** — 첫 확률 **분포** 두 개. 같은 엔진 테스트(8/10 성공)를 두 방향에서 본다: **이항 = p를 고정하고 k의 분포**, **베타 = k를 고정하고 p의 분포** — [[1.1 확률 — 정의·가능도·오즈|1.1]] 확률/가능도 구분의 실체화. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[04 이항 분포·베타 분포 — pmf·CDF·확률의 확률 (평가 지표의 불확실성)|04 분포(AI)]] ｜ 토대: [[1.1 확률 — 정의·가능도·오즈|1.1]] · [[1.2 확률 계산 — 주변·결합·합·덧셈 법칙|1.2]]

> 🎯 **핵심 (TL;DR)**
> - **이항 분포(binomial([baɪˈnoʊmiəl], 바이노미얼, 이항) distribution([ˌdɪstrɪˈbjuːʃən], 디스트리뷰션, 분포)) = 확률이 $p$일 때 $n$번 시도 중 $k$번 성공할 가능성** 📌. 엔진 테스트: $p{=}0.9$ 가정 시 $P(k{=}8) = 0.1937$, **$P(k \le 8) = 0.2639$** — 기본 성공률이 90%여도 8회 이하가 26.39%라 "테스트 더 하자"는 엔지니어가 맞을 수도.
> - 최빈 성공 횟수는 9회(0.3874). $k{=}1$은 $9\times10^{-9}$ — 막대가 안 보일 만큼 희박.
> - 계산: `binom.pmf(k, n, p)` — **pmf = 확률 질량 함수**(이산용; 연속은 밀도 함수 `🔭 102쪽`). 밑바닥 구현은 부록 A.2 `(🔭 예고)`.
> - **베타 분포 = α번 성공·β번 실패가 주어졌을 때 가능한 기본 확률(underlying([ˌʌndərˈlaɪɪŋ], 언더라잉, 기본) probability)들의 가능성** 📌 — 질문을 뒤집어 **"확률의 확률"(메타 확률)**. x축 = 기본 성공률 0~1, y축 = 그 성공률의 **가능도**(밀돗값 ≠ 확률).
> - 확률은 **곡선 아래 면적**으로: 전체 면적 = 1.0(확률 분포의 자격) 📌. **CDF(누적 분포 함수(cumulative([ˈkjuːmjələtɪv], 큐뮬러티브, 누적)))** = 주어진 $x$까지의 면적(왼쪽만).
> - 8/10 결과에서: $P(p \le 0.9) =$ `beta.cdf(0.9, 8, 2)` $= 0.7748$ / $P(p \ge 0.9) = 1 - 0.7748 = 0.225$ 📌 / 중간 $P(0.8 \le p \le 0.9) = \text{cdf}(0.9)-\text{cdf}(0.8) = 0.3386$.
> - **데이터 추가 = 분포가 좁아짐**: 30성공 6실패로 갱신하면 $P(p \ge 0.9)$가 22.5% → **13.16%로 감소** — "새 정보로 믿음 업데이트"([[1.3 조건부 확률·베이즈 정리 — 방향·뒤집기·조건부 결합·합|1.3]] 베이즈)의 실연.
> - ⚠️ 책의 $\text{Beta}(a{=}$성공$, b{=}$실패$)$ 매핑에서 **정점은 관측률 0.8이 아니라 0.875** — 아래 🔑 심화.

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 엔진 테스트 8/10 성공 — "성공률 90% 기대는 실패"인가? 동전 10번에 앞면 8개가 나와도 동전의 확률이 80%인 건 아니다(엔지니어의 반론).
> 1. $p{=}0.9$가 참이라면 8/10이 나올 가능성은 얼마나 되나 — **n번 중 k번의 분포**가 필요 → **이항 분포** `binom.pmf` → $P(k \le 8) = 26.39\%$: 드물지 않다.
> 2. 그런데 이 답은 "$p = 0.9$"라는 **가정** 위에 서 있다 — 90%가 아니라면? 80%, 70%, 30%마다 이항 분포를 다시 만들 수는 없음 → **질문을 뒤집어 8/10을 고정하고 $p$ 자체의 분포**를 보자 → **베타 분포**(확률의 확률).
> 3. 연속 곡선의 밀돗값은 확률이 아님 → 확률을 읽을 도구 필요 → **면적 = 확률, CDF**(왼쪽 누적) + 공리(전체=1)로 오른쪽·중간 면적 유도.
> 4. 판단엔 갱신이 필요 → 테스트 추가(30/6) → 분포가 좁아지고 $P(p{\ge}0.9)$ 하락(22.5→13.16%) → **중단 결정**의 근거.
> **한 줄:** k의 분포(이항, p 가정) → 가정이 의심되면 뒤집어 p의 분포(베타) → 면적=확률(CDF) → 데이터로 갱신(좁아짐).

> [!abstract] 📘 책 핵심 — [필수수학] §2.3 이항 분포
> - §2.3~2.4는 두 확률 분포(이항·베타)를 설명 — 이 책의 다른 부분에서 쓰진 않지만 그 자체로 유용하고, **여러 번의 시도에서 사건이 어떻게 발생하는지** + 3장에서 많이 쓸 확률 분포 이해의 기초 `(🔭 예고 — 3장)`.
> - **제트 엔진 사례**: 10번 테스트 → ✓✓✓✓✓✗✓✗✓✓ (8성공 2실패). 기대 90%인데 80%라 실패로 보고 재설계 결정 — 그러나 엔지니어: "테스트를 더 해야 확신. **동전 10번에 앞면 8개 나왔다고 동전의 확률이 80%인 건 아니다.**"
> - 일리 있음: 공정한 동전도 10번이면 5번이 최빈일 뿐 3·4·6·7번도 나오고 극히 드물게 10번 전부도. 그렇다면 **기본 확률이 90%라고 가정할 때 80% 성공에 대한 가능도**는? → **이항 분포: 확률 $p$일 때 $n$번 중 $k$번 성공할 가능성 측정.**
> - [그림 2-1] $n{=}10$, $p{=}0.9$ 막대그래프: $P(k{=}8) = 0.1937$, $P(k{=}1) = 0.000000008999$(막대 안 보임). **8회 이하 막대 합 = 0.2639.**
> - 구현: 밑바닥부터(부록 A.2 `🔭`) 또는 사이파이 [예제 2-2] `binom.pmf(k, n, p)`로 k=0~10 전부 출력(pmf = probability **mass** function, 확률 질량 함수; 옮긴이 각주 4 — 이산 확률 변수용, 연속은 확률 **밀도** 함수 `🔭 102쪽`). $n$=시도 횟수, $p$=시도당 성공 확률, $k$=알고 싶은 성공 횟수.
> - 출력에서 최빈 성공 횟수 = **9회**(0.3874). $k{\le}8$ 합 = 0.2639 — **기본 성공률 90%여도 8회 이하 성공이 26.39%**. "26.39%는 아무것도 아닌 게 아니며 확실히 일어날 수 있는 일" → 엔지니어가 맞을 수도.
> - 하지만 **모델에 하나의 가정($p{=}90\%$)을 했다** → 다음 절에서 논의.

> [!abstract] 📘 책 핵심 — [필수수학] §2.4 베타 분포
> - 이항 모델의 문제 후보 = **기본 성공률 90% 가정**. 모델이 쓸모없단 뜻은 아님 — "90%라면 8회 이하가 26.39%"를 보였을 뿐이라 **엔지니어 말이 틀린 건 아니다**.
> - **질문 뒤집기**: 90%가 아닌 다른 기본 성공률로도 8/10이 나온다면? 80%면? 70%나 30%는? **8/10을 고정하고 가능한 기본 확률들을 탐색** — 무수한 이항 분포를 만드는 대신 → **베타 분포: 알파(α)번의 성공과 베타(β)번의 실패가 주어졌을 때 사건이 발생할 수 있는 다양한 기본 확률의 가능성.**
> - [그림 2-2] 8성공·2실패의 베타 분포. [TIP] 데스모스 인터랙티브 그래프: <https://oreil.ly/pN4Ep>
> - **x축 = 0.0~1.0의 모든 기본 성공률, y축 = 8성공 2실패가 주어졌을 때 해당 확률의 가능도** — "**메타 확률**(확률의 확률)"로 생각하며 천천히 이해할 것.
> - 베타는 **연속 함수**(이항의 불연속 정수와 대비). **y축 밀돗값은 확률이 아니므로** 수학적으로 조금 더 어렵다 — 이 책은 **곡선 아래 면적**으로 확률을 계산.
> - 베타 분포는 **확률 분포**의 일종 → **전체 곡선 아래 면적 = 1.0** 📌. 확률 = 특정 범위의 면적: [그림 2-3] 0.9~1.0 면적 = **0.225** (90% 이상의 기본 성공률에서 8/10이 나올 확률 22.5%).
> - 모든 연속 확률 분포에는 주어진 $x$까지의 면적을 계산하는 **누적 분포 함수(CDF)** 가 있다: [그림 2-4] 0~0.9 면적 → [예제 2-3] `beta.cdf(0.90, a=8, b=2)` = **0.7748** — 기본 성공 확률이 90% 이하일 가능성 77.48%.
> - **CDF는 경계의 왼쪽만 계산** — 0.9 *이상*은 여사건으로: [그림 2-5·2-6] $1.0 - \text{cdf}(0.9) = $ **0.2252** [예제 2-4].
> - 22.5%에 도박: CFO(최고 재무 관리자)가 26번 추가 테스트 승인 → 최종 **30성공 6실패** [그림 2-7]: **분포가 좁아져** 기본 성공률 범위 축소 — $P(p \ge 0.9)$가 22.5% → **13.16%** [예제 2-5]. "13.16%에 계속 도박을 걸며 정점이 오른쪽으로 이동하길 바라지 않는다면 **테스트 중단**이 좋다."
> - **중간 영역** [그림 2-8·2-9]: $0.8 \le p \le 0.9$ = $\text{cdf}(0.9) - \text{cdf}(0.8)$ = **0.3386** (33.86%) [예제 2-6].
> - 정리: 베타 분포 = 제한된 표본으로 사건 발생/미발생 확률을 측정 — **확률의 확률을 추론하고 새 데이터로 업데이트**. 가설 검정에도 쓸 수 있지만 3장은 정규·t 분포에 초점 `(🔭 예고)`. 밑바닥 구현은 부록 A.3 `(🔭 예고)`.

## 🧠 근본 이해 · 융합 · 통찰

### 🔑 1.1 설계도 예고의 회수 — 두 분포는 같은 데이터의 두 방향이 맞았다
- [[1.1 확률 — 정의·가능도·오즈|1.1]]에서 "*이항 = 확률 방향, 베타 = 가능도 방향일 것*"이라 추정(🔭)했다 — **적중**: 이항은 $p$를 고정하고 $k$의 분포($P(k \mid p{=}0.9)$, 미래 예측), 베타는 관측(8성공 2실패)을 고정하고 **$p$의 가능도**(과거 데이터로 파라미터 추론). 책 y축 라벨이 정확히 "8/10 성공의 **가능도**"다.
- [[1.1 확률 — 정의·가능도·오즈|1.1]] 🪂 심화의 "가능도는 파라미터의 함수 $L(\theta \mid D)$"가 여기서 실체를 얻었다 — 베타 곡선이 바로 $\theta{=}p$를 변수로 한 그 함수(정규화된 형태). **"가능도 합=1 규칙 없음"(1.1)과 "밀돗값은 확률이 아니다"(이 절)는 같은 말**이다.

### 🔑 이항 pmf의 부품은 전부 배운 것 — 1.2 곱셈 정리의 집대성
- 책은 공식을 안 보여주지만 pmf의 정체는 $P(k) = \binom{n}{k} p^k (1-p)^{n-k}$: **독립 시행의 AND 곱**([[1.2 확률 계산 — 주변·결합·합·덧셈 법칙|1.2]] 곱셈 정리)이 $p^k(1-p)^{n-k}$, **"몇 가지 배열로 일어나나"**(순열·조합, 1.2 열거의 일반화)가 $\binom{n}{k}$. 검산: $\binom{10}{8}(0.9)^8(0.1)^2 = 45 \times 0.4305 \times 0.01 = 0.19371$ ✓ (코드 §).
- 상호 배타 합([[1.1 확률 — 정의·가능도·오즈|1.1]] 합=1 규칙): 막대 11개의 합 = 1, 그래서 "$k \le 8$" = 막대 합 = $1 - P(9) - P(10)$ 로도 나온다(코드 §에서 두 경로 일치 확인).

### 🔑 ⚠️ 책의 Beta(a=성공, b=실패) — 정점은 0.875이지 관측률 0.8이 아니다
- $\text{Beta}(8,2)$의 모드 = $\frac{a-1}{a+b-2} = \frac{7}{8} = 0.875$. **"8/10 관측이니 정점이 0.8"이라고 읽기 쉽지만 그림 2-2의 정점은 0.875다**(재구성 그림에 점선 표시). 관측률 0.8에 정점이 오는 건 $\text{Beta}(s{+}1, f{+}1) = \text{Beta}(9,3)$ — 균등 사전분포를 둔 베이즈 사후분포의 표준 매핑(모드 $\frac{8}{10} = 0.8$ 정확히).
- 즉 책의 $a{=}$성공, $b{=}$실패 직결 매핑은 **입문용 단순화**: "$s$번 성공 $f$번 실패의 가능도 곡선" $p^s(1-p)^f$ 자체는 $\text{Beta}(s{+}1, f{+}1)$에 비례한다. 책 수치(0.225 등)는 책의 매핑 기준으로 전부 정확하므로 그대로 외우되, **정점 위치를 관측 비율로 착각하지만 말 것.** 비교 그림 ↓
- (심화 예고) 이 구분은 부록 A.3(밑바닥 베타 구현)·베이즈 통계에서 사전분포가 등장하면 재론될 것 `(🔭)`.

### 🔑 CDF 하나 + 공리 둘로 면적 3종이 다 나온다 — 1.1·T09 회수
- 도구는 `cdf(x)`(왼쪽 면적) 하나뿐인데 오른쪽은 $1 - \text{cdf}$(여사건, [[1.1 확률 — 정의·가능도·오즈|1.1]] $1-P$), 중간은 $\text{cdf}(b) - \text{cdf}(a)$(포함-배제의 1차원판, [[1.2 확률 계산 — 주변·결합·합·덧셈 법칙|1.2]]) — **"전체 면적=1" 공리([[T09 확률은 왜 0.0~1.0인가 — 공리·곱셈 닫힘·부동소수점 팩트체크·로그 공간|T09]] $P(\Omega){=}1$)가 장식이 아니라 계산 부품**으로 쓰이는 첫 장면.
- 그림 2-6·2-9의 "빼기 3연"이 그 시각화 — 확률 규칙이 면적 산수로 번역된다.

### 🔑 갱신의 방향 — 데이터가 늘면 분포는 좁아지고, 희망은 검증 가능해진다
- 8/10(22.5%) → 30/36(13.16%): 표본이 3.6배 늘자 분포 폭이 줄고 **"p≥0.9" 희망의 확률이 절반 가까이 깎였다**. 데이터 추가는 중립적 행위가 아니라 **불확실성을 압축**하는 행위 — 정점이 이동하지 않는 한 꼬리 확률은 계속 준다("정점이 오른쪽으로 이동하길 바라지 않는다면 중단"의 수학적 근거).
- [[1.1 확률 — 정의·가능도·오즈|1.1]] §2.1.1의 "통계학자: 30번 이상 굴려라"가 여기서 정량화됐다 — 30번대 표본이 실제로 판단을 뒤집을 만큼 분포를 좁혔다. [[1.3 조건부 확률·베이즈 정리 — 방향·뒤집기·조건부 결합·합|1.3]] "새 정보로 믿음 업데이트"의 첫 수치 실연.

## 🧩 코드 (§2.3~2.4)
> 🎯 목표: ① 예제 2-2 pmf 재현 + 공식 $\binom{n}{k}p^k(1-p)^{n-k}$ 일치 ② $P(k{\le}8)$ 두 경로(합 vs $1{-}P(9){-}P(10)$) = 0.2639 ③ 예제 2-3~2-6 CDF 4수치 ④ ⚠️ 모드 0.875 vs 0.8 확인.
> 코드: `~/LOCAL/03-00_STUDIES/AI/02_optimization/md2_4_binomial_beta.py`

```python
from scipy.stats import binom, beta
from math import comb, isclose

n, p = 10, 0.9
pmf8 = binom.pmf(8, n, p)
assert isclose(pmf8, comb(10, 8) * 0.9**8 * 0.1**2)     # 곱셈 정리 × 경우의 수
print(pmf8)                                              # 0.19371024449999982

le8 = sum(binom.pmf(k, n, p) for k in range(9))          # 막대 합
assert isclose(le8, 1 - binom.pmf(9, n, p) - binom.pmf(10, n, p))  # 여사건 경로
print(le8)                                               # 0.2639010708999999

print(beta.cdf(0.90, 8, 2))                              # 0.7748409780000002 (예제 2-3)
print(1.0 - beta.cdf(0.90, 8, 2))                        # 0.22515902199999982 (예제 2-4)
print(1.0 - beta.cdf(0.90, 30, 6))                       # 0.13163577484183697 (예제 2-5)
print(beta.cdf(0.90, 8, 2) - beta.cdf(0.80, 8, 2))       # 0.33863336199999994 (예제 2-6)

mode = lambda a, b: (a - 1) / (a + b - 2)                # ⚠️ 정점 위치
print(mode(8, 2), mode(9, 3))                            # 0.875 (책 매핑) vs 0.8 (s+1, f+1)
```

## 🆕 용어
| 용어 | 한 줄 정의 |
|---|---|
| **이항 분포** (binomial distribution) | 확률 $p$일 때 $n$번 시도 중 $k$번 성공할 가능성의 분포 — `binom.pmf(k, n, p)`; 최빈 $k$ ≈ $np$ |
| **기본 확률** (underlying probability) | 관측 이면에 있다고 가정하는 시행당 참 성공률 $p$ — 이항은 이걸 *가정*하고, 베타는 이걸 *추론*한다 |
| **확률 질량 함수** (pmf) | 이산 확률 변수의 각 값에 확률을 주는 함수(막대 높이 = 확률) ↔ 연속은 밀도 함수 `🔭 102쪽` |
| **베타 분포** (beta distribution) | α번 성공·β번 실패가 주어졌을 때 기본 확률들의 가능도 곡선 — "확률의 확률"(메타 확률); ⚠️ 정점은 $\frac{a-1}{a+b-2}$(관측 비율 아님) |
| **확률 분포** (probability distribution) | 전체 (막대 합·곡선 아래 면적) = 1.0인 분포 — 확률은 범위의 면적으로 |
| **누적 분포 함수** (CDF) | 주어진 $x$까지(왼쪽) 면적 — 오른쪽은 $1{-}\text{CDF}$, 중간은 CDF 차 |
| **밀돗값 ≠ 확률** | 연속 분포의 y값은 가능도(밀도)일 뿐 — 확률은 면적에서만 나온다 (1.1 "가능도 합=1 없음"의 연속판) |
