# A/B 테스트 — 분할 테스트·단일·이중 맹검

> ⚠️ **파일명 표기 안내**: 파일명은 "A·B"(가운뎃점)를 쓴다 — "A/B"의 슬래시(`/`)는 파일시스템 경로 구분자로 해석돼 폴더가 생기는 문제가 있었다(2026-07-18 발견·수정). 본문·용어에서는 원래 표기 "A/B 테스트"를 그대로 쓴다.

> 📐 **수학 개념원자(SSoT)** — [[1.7 확률 분포 카탈로그 — 지수족·생존분석·기타 13종|1.7]]까지 "분포를 어떻게 모델링하나"를 다뤘다면, 이 노트는 "두 분포(대조군 vs 실험군)를 어떻게 **비교**해 결정을 내리나"의 입문이다. [필수수학AI] 2장의 마지막 절. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.

> 🎯 **핵심 (TL;DR)**
> - **A/B 테스트 = 분할 테스트(split test)** — 전체 인구를 두 그룹으로 나눠, 한쪽(실험군)에만 새 버전을 배포하고 다른 쪽(대조군)은 그대로 유지, 결과를 비교.
> - **단일 맹검** = 피험자만 자기 그룹을 모름(실험자는 앎). **이중 맹검** = 피험자·실험자 **둘 다** 모름.
> - 목적은 언제나 같다: 버전 차이 때문에 생긴 차이인지, 우연/편향 때문인지 가려내는 것.

---

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 새 웹 페이지 디자인이 "더 낫다"고 느껴져도, 그 느낌만으로는 **진짜 효과**인지 우연인지 구분할 수 없다.
> 1. 전체 사용자에게 한꺼번에 바꾸면 비교 대상(바꾸지 않았을 때)이 사라짐 → **대조군**이 필요 → 인구를 **두 그룹으로 분할**.
> 2. 피험자가 "내가 새 버전을 쓰고 있다"는 걸 알면 행동이 달라짐(플라시보·자각 효과) → 결과가 오염됨 → **단일 맹검**(피험자에게 감춤).
> 3. 실험자조차 어느 피험자가 어느 그룹인지 알면 무의식적으로 다르게 대하거나 결과를 다르게 해석할 위험 → **이중 맹검**(양쪽 다 감춤).
> 4. 대상이 사람이 아니어도(신약, 정치 광고, 웹 UI) 같은 논리 — 마이크로소프트·아마존·링크드인·구글이 매년 수천 건을 수행하는 이유.
> **한 줄:** 비교할 대상이 없으면 효과를 못 잼(대조군) → 아는 것 자체가 결과를 오염시킴(맹검) → "안다"의 주체가 누구냐에 따라 단일/이중으로 세분.

> [!abstract] 📘 책 핵심 — [필수수학AI] §2.15 A/B 테스트
> 분할 테스트(split test) 또는 무작위 단일 맹검 시험(randomized single-blind trial([ˈtraɪəl], 트라이얼, 트라이얼))/이중 맹검 시험(double-blind trial)이라고도 하는 A/B 테스트. 데이터 과학자에게 매우 중요하며, 수많은 기업이 사용자 참여·매출·고객 만족도를 위해 수행한다. 마이크로소프트·아마존·링크드인·구글은 매년 수천 건의 A/B 테스트를 수행한다.
> - **기본 개념**: 전체 인구를 두 그룹으로 나눔 → 테스트 항목(새 웹 페이지 디자인, 다른 글꼴 크기, 새 의약품, 새 정치 광고)의 버전을 **테스트 그룹**에 배포, 다른 그룹은 **대조군(control group)**으로 유지 → 두 그룹의 결과 데이터를 비교.
> - **단일 맹검 시험(single-blind([blaɪnd], 블라인드, 블라인드) trial)**: 피험자는 자신이 어느 그룹인지 모름(실험에 참여하는지조차 모를 수 있음) — 실험자는 앎.
> - **이중 맹검 시험(double-blind trial)**: 실험자·피험자 **모두** 자신이 어느 그룹과 상호작용하는지 모름.

## 🧠 근본 이해 · 융합 · 통찰

### 🧩 왜 "분할+맹검"이 세트인가 — 편향 통제의 두 축
- **분할(무작위 배정)**은 "차이를 잴 자(scale)"를 만드는 작업이고, **맹검**은 그 자를 "오염시키지 않는" 작업이다. 분할만 하고 맹검을 안 하면, 측정 자체는 가능해도 측정값이 왜곡될 수 있다(피험자가 "나는 새 버전이다"를 알고 평소와 다르게 행동 — 호손 효과 계열).
- 이중 맹검이 항상 가능한 건 아니다 — 웹 페이지 디자인 A/B 테스트에서 "실험자가 어느 사용자가 어느 버전을 보는지 모르게" 하기는 쉽지만(자동화된 배포), 신약 임상시험처럼 의사(실험자)가 처치 여부를 알아야 부작용에 대응할 수 있는 경우엔 완전한 이중 맹검이 윤리적으로 제한되기도 한다 `(🔭 예고 — 실무 트레이드오프, 책 범위 밖)`.
- [[19 실제 vs 시뮬레이션 데이터 — 키-체중·조건부 층화·노이즈|19]]의 "조건화=속임수 없는 공정한 게임" 테마와 같은 뿌리 — 대조군·맹검 둘 다 "비교가 공정하도록" 만드는 장치다.

### ⚠️ 이 절이 다루지 않는 것 — 통계적 유의성
책은 "두 그룹의 결과 데이터를 비교한다"까지만 말하고, **그 비교가 통계적으로 유의미한지**(우연히 차이 나 보이는 게 아닌지) 판단하는 도구(가설검정·p-값·신뢰구간)는 이 절 범위 밖이다 `(🔭 예고 — 3장 이후 예상, 카이제곱·스튜던트 t가 그 도구가 될 가능성 — [[1.7 확률 분포 카탈로그 — 지수족·생존분석·기타 13종|1.7]] 참조)`.

### 🔍 직접 예시 — 그룹 분할·비교의 최소 골격 (책에 코드 없음, 개념 예시로 직접 작성)
```python
import numpy as np
rng = np.random.default_rng(0)

# 대조군(A) vs 실험군(B) — 예: 클릭률(가상의 관찰 데이터)
n = 1000
group_A = rng.binomial(1, 0.10, n)   # 대조군: 클릭률 10%
group_B = rng.binomial(1, 0.12, n)   # 실험군: 클릭률 12% (새 디자인)

rate_A, rate_B = group_A.mean(), group_B.mean()
print(f"대조군 클릭률={rate_A:.3f}  실험군 클릭률={rate_B:.3f}  차이={rate_B-rate_A:+.3f}")
```
실행 결과(seed=0로 검증): `대조군 클릭률=0.096  실험군 클릭률=0.118  차이=+0.022` — 이 차이가 "진짜"인지 우연인지 판단하려면 위 🔭 예고의 유의성 검정이 필요하다(이 노트는 여기까지만).

## 🆕 용어
| 용어 | 한 줄 정의 |
|---|---|
| **A/B 테스트 / 분할 테스트** split test | 전체 인구를 두 그룹(대조군·테스트군)으로 나눠 버전 차이의 효과를 비교하는 실험 설계 |
| **대조군** control group([kənˈtroʊl ɡruːp], 컨트롤 그룹, 컨트롤 그룹) | 새 버전을 적용하지 않고 기존 상태로 유지하는 비교 기준 그룹 |
| **단일 맹검 시험** single-blind trial | 피험자만 자신의 그룹을 모름(실험자는 앎) |
| **이중 맹검 시험** double-blind trial | 피험자·실험자 모두 자신이 어느 그룹과 상호작용하는지 모름 |
