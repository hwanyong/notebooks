# 확률 — 정의·가능도·오즈

> 📐 **수학 개념원자(SSoT)** — 확률·통계 관점 첫 원자. 확률·가능도·오즈의 단일 주인(불확실성 정량화의 출발점, `수학/확률통계/` 폴더 개설). AI 적용·연결은 코스 렌즈가 끌어다 쓴다.

> 🎯 **핵심 (TL;DR)**
> - **확률(probability([ˌprɑːbəˈbɪləti], 프라버빌러티, 프로버빌리티)) $P(X)$ = 사건(event([ɪˈvent], 이벤트, 이벤트)) $X$가 일어날 것이라고 믿는 정도.** 표현은 백분율(70%)이 흔하지만 **작업은 0.0~1.0 소수**(0.70) — 같은 값의 표시 형식 차이일 뿐.
> - 범위는 엄밀히 $[0,1]$ 📌, 여사건은 $P(\text{not } X) = 1 - P(X)$ (0.70 → 0.30).
> - **가능도(likelihood([ˈlaɪklihʊd], 라이클리후드, 라이클리후드)) ≠ 확률** 📌 — 확률 = *아직 일어나지 않은* 사건 예측의 정량화(미래) / 가능도 = *이미 발생한* 사건의 빈도 측정(과거). 통계·ML은 **가능도(과거 데이터)로 확률(미래)을 예측**한다.
> - 차이 ②: 한 사건의 **상호 배타적** 결과들의 확률 총합 = 1.0 📌 — **가능도에는 이 규칙이 없다.**
> - **오즈(odds([ɑːdz], 아즈, 오즈)) $O(X)$** = 믿음의 비율 표현(7:3 = 7/3 = 2.333…). 변환 📌: $P(X)=\dfrac{O(X)}{1+O(X)}$ ↔ $O(X)=\dfrac{P(X)}{1-P(X)}$ — 서로 역함수.
> - 오즈 2.0 = 일어날 확률이 일어나지 않을 확률의 **2배** — 66.666…% 백분율보다 주관적 신념(베팅)을 직관적으로 표현. 로그 오즈 `(🔭 예고 — 6장 로지스틱 회귀)`·베이즈(Bayes([beɪz], 베이즈, 베이즈)) 통계 `(🔭 예고)`의 부품.

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 세상은 측정하고 싶은 불확실성으로 가득하다(주식 실적·선거 결과·비행기 정시 도착·제품 결함) — 그런데 "불확실한 것"을 잴 자가 없다.
> 1. 사건이 일어난다는 **믿음의 정도를 수 하나로** 만들어야 함 → **확률 $P(X) \in [0,1]$** (백분율은 표시 형식; *왜 하필 0~1인지*의 뿌리는 [[T09 확률은 왜 0.0~1.0인가 — 공리·곱셈 닫힘·부동소수점 팩트체크·로그 공간|T09]] — 비율 정의·공리·곱셈 닫힘).
> 2. 수치화하고 보니 "**앞으로 일어날** 예측"과 "**이미 관측된** 빈도"가 한 단어에 뒤섞임(사전조차 혼동) → 구분 발명 → **확률(미래) vs 가능도(과거)** — ML의 엔진: 과거의 가능도로 미래의 확률을 추정.
> 3. 사람의 주관적 믿음(도박·베팅)은 "될 쪽 대 안 될 쪽" **비율**이 더 자연스러움 → **오즈 $O(X)$** + 확률과의 왕복 변환 공식.
> **한 줄:** 불확실성을 잴 자 없음(결핍) → 믿음의 수치화 $P(X)$(발명) → 미래/과거 구분(가능도) → 믿음의 비율 옷(오즈).

> [!abstract] 📘 책 핵심 — [필수수학] 2장 도입 (확률)
> - 제비뽑기·주사위(두 개에서 같은 숫자) 같은 도박부터 주식 실적·선거 결과·비행기 정시 도착 예측까지 — **세상은 우리가 측정하고 싶은 불확실성으로 가득**하다. 관심 단어는 바로 **'불확실성'**: 불확실한 것을 어떻게 측정할까?
> - **확률 = 어떤 사건이 일어날 확실성을 측정하는 이론적 연구 분야.** 통계·가설 검정·머신러닝 등 이 책 전체의 기초.
> - 많은 사람이 확률을 당연하게 여기지만 **생각보다 훨씬 미묘하고 복잡**하다. 확률의 정리·개념 자체는 수학적으로 잘 정리돼 있으나, **데이터를 도입하고 통계학으로 들어가면 더 복잡**해진다 `(🔭 예고 — 3장 통계와 가설 검정)`.
> - 이 장의 경로: 확률이 무엇인지 → 확률의 수학 개념 → 베이즈 정리 → 이항 분포 → 베타 분포 `(🔭 예고 — 이 장 후반)`.

> [!abstract] 📘 책 핵심 — [필수수학] §2.1 확률 이해하기
> **① 확률의 정의와 표기**
> - **확률 = 어떤 사건이 일어날 것이라고 믿는 정도**, 종종 백분율로 표현. 확률로 답해야 할 질문들: 동전 10번 중 앞면 7번이 나올 확률? 선거 승리 가능성? 비행기 지연 확률? 제품 결함 확신도?
> - 표기 $P(X)$ — $X$는 관심 대상인 사건. 가장 일반적인 표현은 백분율("제가 탈 항공편이 연착될 확률은 70%") — **작업할 때는 0.0~1.0 사이 소수**(이 경우 0.70).
>
> **② 가능도와의 구분**
> - 가능도는 확률과 비슷해 혼동하기 쉽다(많은 사전도 혼동). 일상 대화에선 혼용해도 무방하지만 차이는 정확히 알아야 한다:
>   **확률 = 아직 일어나지 않은 사건에 대한 예측의 정량화 / 가능도 = 이미 발생한 사건의 빈도 측정.** 📌
> - 통계와 머신러닝에서는 **확률(미래)을 예측하기 위해 데이터 형태에 대한 가능도(과거)를 사용**한다.
> - 확률은 엄밀히 0%~100%, 즉 $0.0 \le P(X) \le 1.0$. 발생하지 않을 확률: $P(\text{not } X) = 1 - P(X)$ ($= 1 - 0.70 = 0.30$).
> - **또 다른 차이:** 한 사건에 대해 가능한 모든 **상호 배타적** 결과(여러 개가 아니라 오직 하나만 나오는)의 확률을 모두 더하면 **1.0(100%)**. **가능도에는 이 규칙이 적용되지 않는다.** 📌
>
> **③ 오즈**
> - 확률을 **오즈 $O(X)$** 로도 표현 가능 — 7:3, 7/3, $2.\overline{333}$ 같은 형태.
> - **오즈 → 확률:** $P(X)=\dfrac{O(X)}{1+O(X)}$ — 예: $O=\frac{7}{3}$ 이면 $P(X)=\frac{7/3}{1+7/3}=0.7$.
> - **확률 → 오즈:** $O(X)=\dfrac{P(X)}{1-P(X)}$ — 예: $\frac{0.70}{1-0.70}=\frac{7}{3}$.
>
> **④ [박스] 오즈의 유용함**
> - 백분율·비율이 편한 사람이 많지만 오즈도 유용한 도구 — **오즈 2.0 = 일어날 확률이 일어나지 않을 확률보다 두 배 높다**는 뜻. $66.\overline{666}\%$ 같은 백분율보다 신념을 직관적으로 설명.
> - 그래서 **도박·베팅에서 주관적 믿음을 정량화**하는 데 유용. 로그 오즈를 사용한 로지스틱 회귀 `(🔭 예고 — 6장)`와 베이즈 요인(Bayes Factor)을 포함한 베이즈 통계 `(🔭 예고)`에서도 중요한 역할.

## 🧠 근본 이해 · 융합 · 통찰

### 🔑 확률 vs 가능도 — 갈라놓는 축은 '시간 방향'이고, 합=1 규칙이 리트머스다
- 같은 동전을 두 방향에서 볼 수 있다: **"앞면이 나올까?"**(미래 예측 = 확률) vs **"10번 중 7번 나왔다"**(과거 관측 빈도 = 가능도). ML의 학습 루프가 정확히 이 다리다: **과거 데이터 —(가능도 측정)→ 모델 —(확률 예측)→ 미래.**
- **왜 가능도엔 합=1 규칙이 없나(내 해석):** 확률은 *한 상황* 아래 가능한 결과들에 확률 질량을 나눠 뿌린 것(상호 배타적이니 총합 1). 가능도는 반대로 *이미 고정된 데이터*를 서로 다른 후보 설명들이 각각 얼마나 잘 설명하나의 점수 — 예: "기침" 관측에 대해 감기의 가능도 0.8, 폐렴의 가능도 0.6 — **경쟁 후보들의 점수라 합쳐서 1이 될 이유가 없다.**

> [!tip] 🪂 심화 — 통계학의 정식 가능도는 '파라미터의 함수'   `#사이드/심화`
> 책의 "이미 발생한 사건의 빈도 측정"은 입문용 근사. 정식 정의에서 가능도는 **고정된 데이터 $D$를 놓고 파라미터 $\theta$를 변수로 보는 함수** $L(\theta \mid D) = P(D \mid \theta)$ — 같은 수식을 반대 방향으로 읽는 것이다. 이 정밀화가 최대 가능도 추정·베타 분포 `(🔭 예고 — 이 장 후반)`에서 실체를 얻을 예정. 지금은 "미래 예측 vs 과거 관측"의 방향 감각만 확실히.

### 🔑 확률·오즈·로그 오즈 — 같은 정보의 세 벌 옷 (값역만 다르다)
- 7:3(비율) = $7/3$(분수) = $2.\overline{333}$(소수) — 전부 **같은 오즈**의 표기 차이. 그리고 오즈 ↔ 확률은 **1:1 단조 변환**이라 정보 손실이 없다: $O=\frac{P}{1-P}$ 를 $P$에 대해 풀면 그대로 $P=\frac{O}{1+O}$ — **두 공식은 서로 역함수라 하나만 외우면 된다.**
- 값역 지도: $P \in [0,1]$ ↔ $O \in [0,\infty)$ ↔ $\log O \in (-\infty,\infty)$. **로그 오즈만 전체 실수축을 덮는다** — 선형식 $z = wx+b$(출력이 임의 실수)를 확률과 잇는 다리가 되는 이유 `(🔭 예고 — 6장 로지스틱 회귀)`. 미리 검산해 보면: $O=e^z$ 를 $P=\frac{O}{1+O}$ 에 대입하면 $P=\frac{e^z}{1+e^z}=\frac{1}{1+e^{-z}}$ — 시그모이드가 **바로 이 절의 오즈→확률 공식**이다(코드 §에서 수치 확인).
- 오즈의 존재 이유는 **분모 교체**: 확률의 분모는 "전체 경우", 오즈의 분모는 "안 일어날 경우" — 기준을 상대 쪽으로 바꾸면 "몇 배나 유리한가"가 즉답된다(오즈 2.0 = 2배).

### 🔑 이 장의 설계도 — 두 방향(확률·가능도)이 두 분포(이항·베타)로 발전한다 (내 추정)
- 책 도입의 예고(베이즈 정리 → 이항 분포 → 베타 분포)를 §2.1의 프레임에 겹치면: 동전 질문 "10번 중 앞면 7번이 나올 확률은?"이 **확률 방향(이항 분포)**, 그 반전 "10번 던져 7번 나왔다면 이 동전은 얼마나 공정한가?"가 **가능도 방향(베타 분포)** 으로 보인다 `(🔭 예고 — 이 장 후반 검증 예정)`. §2.1의 확률/가능도 구분은 잡담이 아니라 장 전체의 목차 예고편인 셈.

## 🧩 코드 (§2.1)
> 🎯 목표: ① 오즈↔확률 왕복이 정확히 복원되나(7/3 → 0.7 → 7/3, `Fraction`으로 오차 없이) ② 상호 배타적 결과 합=1 ③ (🔭 예고 검증) 시그모이드 = 오즈→확률 공식에 $O=e^z$ 대입과 동일.
> 코드: `~/LOCAL/03-00_STUDIES/AI/02_optimization/md2_1_probability_odds.py`

```python
from fractions import Fraction
import math

def odds_to_p(o): return o / (1 + o)      # P = O/(1+O)   (식: 오즈→확률)
def p_to_odds(p): return p / (1 - p)      # O = P/(1−P)   (식: 확률→오즈)

o = Fraction(7, 3)                        # 책 예제: 오즈 7/3
p = odds_to_p(o)
print(p)                                  # 7/10 — 정확히 0.7 (책과 일치)
print(p_to_odds(p) == o)                  # True — 왕복 복원(서로 역함수)

print(1 - 0.70)                           # 0.30 — P(not X) = 1 − P(X)

print(sum([Fraction(1, 6)] * 6))          # 1 — 상호 배타적 결과(주사위 6면) 합 = 1.0

z = 1.234                                 # (🔭 6장 예고) 시그모이드 = 오즈 변환
print(math.exp(z) / (1 + math.exp(z)))    # 0.774518...  ← O=e^z 를 P=O/(1+O)에
print(1 / (1 + math.exp(-z)))             # 0.774518...  ← 표준 시그모이드 — 동일
```

| **확률** (probability, $P(X)$) | 사건 $X$가 일어날 것이라고 믿는 정도; 엄밀히 $[0,1]$; 백분율은 표시 형식 |
| **사건** (event) | 확률의 관심 대상 $X$ (동전 앞면 7번·선거 승리·비행기 지연·제품 결함…) |
| **가능도** (likelihood) | *이미 발생한* 사건의 빈도 측정(과거) ↔ 확률(미래 예측); 상호 배타 합=1 규칙 없음; 정식으론 $L(\theta \mid D)$(🪂 심화) |
| **여사건 확률** | $P(\text{not } X) = 1 - P(X)$ |
| **상호 배타적** (mutually exclusive) | 여러 결과 중 오직 하나만 나올 수 있는 관계 — 이 결과들의 확률 총합 = 1.0 |
| **오즈** (odds, $O(X)$) | 일어날 확률 대 일어나지 않을 확률의 비 — $O(X)=\frac{P(X)}{1-P(X)}$; 역변환 $P(X)=\frac{O(X)}{1+O(X)}$; 표기 7:3 = 7/3 = $2.\overline{333}$ |

## 📌 암기 카드
> (→ [[_핵심 암기장]])

| 앞면 (cue) | 뒷면 (외울 내용) |
|---|---|
| 확률의 정의? | 사건이 일어날 것이라고 **믿는 정도** — $P(X)$, 작업은 0.0~1.0 소수 |
| 확률의 엄밀한 범위? 📌 | $[0,1]$ (0%~100%) — 백분율/소수는 같은 값의 표시 형식 (근거는 T09: 비율·공리·곱셈 닫힘) |
| $P(\text{not } X)$? | $1 - P(X)$ — 0.70 → 0.30 |
| 확률 vs 가능도 차이 ①? 📌 | 확률 = *아직 안 일어난* 사건 예측의 정량화(미래) / 가능도 = *이미 발생한* 사건의 빈도 측정(과거) |
| 확률 vs 가능도 차이 ②? 📌 | 상호 배타적 결과들의 **확률 총합 = 1.0**; **가능도엔 이 규칙 없음**(경쟁 후보들의 점수) |
| 통계·ML에서 둘의 관계? 📌 | **확률(미래)을 예측하기 위해 데이터 형태에 대한 가능도(과거)를 사용** |
| 오즈 → 확률 공식? 📌 | $P(X)=\dfrac{O(X)}{1+O(X)}$ — 7/3 → 0.7 |
| 확률 → 오즈 공식? 📌 | $O(X)=\dfrac{P(X)}{1-P(X)}$ — 0.70 → 7/3; 두 공식은 **서로 역함수**(하나만 외우면 됨) |
| 오즈 2.0의 뜻? | 일어날 확률이 일어나지 않을 확률보다 **2배** 높음 |
| 오즈의 세 표기? | 7:3(비율) = 7/3(분수) = $2.\overline{333}$(소수) — 전부 같은 값 |
| 확률·오즈·로그오즈 값역? | $P\in[0,1]$ · $O\in[0,\infty)$ · $\log O\in(-\infty,\infty)$ — 1:1 단조 변환(정보 동일); 실수축 전체를 덮는 건 로그 오즈뿐(🔭 6장) |
| 오즈가 유용한 곳? | 주관적 믿음 정량화(도박·베팅) · 로그 오즈 로지스틱 회귀(🔭 6장) · 베이즈 요인/베이즈 통계(🔭) |
