# 확률 계산 — 주변·결합·합·덧셈 법칙

> 📐 **수학 개념원자(SSoT)** — 단일 사건을 넘어 사건들을 **결합**하는 첫 계산 규칙(AND=곱, OR=합−교집합). AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[02 확률 계산 — 주변·결합·합·덧셈 법칙 (AND·OR 확률 연산)|02 확률 계산(AI)]] ｜ 토대: [[1.1 확률 — 정의·가능도·오즈|1.1 확률 정의]]

> 🎯 **핵심 (TL;DR)**
> - **주변 확률(marginal probability([ˈmɑːrdʒɪnəl], 마지널, 마지널)) = 사건 $P(X)$의 단일 확률** — 여기까진 간단. **서로 다른 사건의 확률을 결합하는 순간 직관이 무너진다.**
> - **결합 확률(joint probability([dʒɔɪnt], 조인트, 조인트)) = AND** 📌: $P(A \text{ AND } B) = P(A) \times P(B)$ — **곱셈 정리(product rule)**. 단, **두 사건이 독립이라는 가정** 하에서(책 명시, 상세는 나중에 `🔭`). 예: $P(\text{앞면 AND } 6) = \frac{1}{2} \times \frac{1}{6} = \frac{1}{12}$.
> - 곱의 근거는 **전수 열거**: 동전(H/T)×주사위(1~6) = 12가지 조합 중 H6은 1개 → $\frac{1}{12}$. 곱셈은 세기의 지름길.
> - **합 확률(union probability([ˈjuːniən], 유니언, 유니언)) = OR**: 상호 배타적([[1.1 확률 — 정의·가능도·오즈|1.1]])이면 단순 합 — $P(4 \text{ OR } 6) = \frac{1}{6} + \frac{1}{6} = \frac{1}{3}$.
> - 비배타 사건에 단순 합을 쓰면 **교집합 이중 계산** — $P(\text{앞면 OR } 6) = \frac{1}{2}+\frac{1}{6} = 0.\overline{666}$(오답, 참값 $\frac{7}{12}$); 극단 반례 $\frac{1}{2}+\frac{5}{6} = 1.\overline{333}$ — **확률이 1.0을 초과 = 확실한 버그 신호.**
> - 처방 = **확률의 덧셈 법칙(sum rule)** 📌: $P(A \text{ OR } B) = P(A) + P(B) - P(A \text{ AND } B)$ — 결합 사건을 한 번만 계산. 배타면 $P(A \text{ AND } B)=0$이라 단순 합으로 환원(일반형이 전 경우 포괄).
> - ⚠️ $-P(A){\times}P(B)$ 대입형은 **독립 전용** — 상호 배타(≠독립!)에 쓰면 오답($\frac{11}{36} \ne \frac{1}{3}$, 아래 🔑).

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* [[1.1 확률 — 정의·가능도·오즈|1.1]]이 단일 사건의 $P(X)$(주변 확률)를 줬다 — 그런데 현실 질문은 대부분 **사건들의 조합**이다: "앞면이 나오*면서* 6이 나올까?"(AND), "4 *또는* 6이 나올까?"(OR).
> 1. AND의 정답은 **모든 조합을 열거**해 세면 나온다(12칸 중 1칸) — 그러나 사건이 늘면 조합이 폭발 → 지름길 필요 → **곱셈 정리** $P(A)\times P(B)$ (독립 가정).
> 2. OR은 배타 사건이면 덧셈으로 충분 → 비배타에 그대로 쓰니 **교집합을 두 번 세는 결함** 발견(0.666 오답, 심하면 1.333 — 확률 공리 위반) → 결핍.
> 3. 두 번 센 것을 한 번으로 → **교집합을 빼는 보정** → **덧셈 법칙** $P(A)+P(B)-P(A \text{ AND } B)$.
> **한 줄:** 단일 확률(1.1) → 조합 질문 등장 → 열거 폭발→곱(AND) → 덧셈의 이중 계산→교집합 빼기(OR).

> [!abstract] 📘 책 핵심 — [필수수학] §2.2 확률 계산 (도입)
> - 사건 $P(X)$의 단일 확률 = **주변 확률** — 앞서 다룬 것처럼 매우 간단하다.
> - 하지만 **서로 다른 사건의 확률을 결합하기 시작하면 직관적으로 이해하기 어려워진다** — 이 절이 그 결합 규칙(AND·OR)을 다룬다.

> [!abstract] 📘 책 핵심 — [필수수학] §2.2.1 결합 확률
> - 공정한 동전 + 공정한 6면 주사위: "동전 앞면 **AND** 주사위 6"이 *함께* 발생할 확률 = **결합 확률**. 결합 확률은 **AND 연산자**로 생각하라.
> - 동전은 양면, 주사위는 6면 → $P(\text{앞면})=\frac{1}{2}$, $P(6)=\frac{1}{6}$. 두 사건이 모두 발생할 확률은 **단순히 두 확률을 곱하면 된다** (여기서는 **두 사건이 독립이라고 가정** — 나중에 자세히 `🔭 예고`):
> $$P(A \text{ AND } B) = P(A) \times P(B), \qquad P(\text{앞면 AND } 6) = \frac{1}{2} \times \frac{1}{6} = \frac{1}{12} = 0.08\overline{333}$$
> - **왜 곱인가?** 가능한 모든 사건 조합을 생성해 보면 확률 규칙들이 발견된다 — 순열·조합(이산 수학)에서 유래. 동전 H/T × 주사위 1~6의 전 조합 12가지 중 관심 결과(H6)는 하나뿐:
>
> `H1 H2 H3 H4 H5 *H6* T1 T2 T3 T4 T5 T6`
>
> - 12가지 중 조건을 만족하는 결과가 1개 → $\frac{1}{12}$. **조합 전부를 생성·헤아리는 대신 곱셈으로 빠르게** — 이것이 **곱셈 정리**(product rule([ˈprɒdʌkt ruːl], 프로덕트 룰, 프로덕트 룰); 옮긴이 주: 곱셈 규칙·곱셈 법칙이라고도).

> [!abstract] 📘 책 핵심 — [필수수학] §2.2.2 합 확률
> - "사건 A **OR** B가 발생할 확률" = **합 확률**.
> - **상호 배타적**(동시 발생 불가 — 주사위 한 번에 4와 6은 함께 못 나옴)이면 **단순 덧셈**:
> $$P(4 \text{ OR } 6) = \frac{1}{6} + \frac{1}{6} = \frac{1}{3}$$
> - **비배타 사건**(앞면 OR 6)은? 전 조합을 다시 열거하면 관심 결과는 H1~H6 전부 + T6 = **12가지 중 7가지** → $\frac{7}{12} = 0.58\overline{333}$.
> - 그런데 단순 덧셈하면 $\frac{1}{2}+\frac{1}{6} = \frac{4}{6} = 0.\overline{666}$ — **잘못된 값**. 이유: **H6과 T6에서 6이 나올 확률을 두 번 계산**했기 때문!
> - 더 분명한 반례: $P(\text{앞면 OR 1~5}) = \frac{1}{2}+\frac{5}{6} = \frac{8}{6} = 1.\overline{333}$ (133.333%) — **확률은 1.0을 넘을 수 없으므로 명백한 오류.** 역시 중복 헤아림 때문.
> - 이중 계산을 제거하는 논리적 방법 = **결합 확률을 빼기** = **확률의 덧셈 법칙(sum rule([sʌm ruːl], 섬 룰, 섬 룰) of probability)** — 모든 **결합 사건(joint event)** 이 한 번만 계산되게:
> $$P(A \text{ OR } B) = P(A) + P(B) - P(A \text{ AND } B) \;\overset{\text{독립}}{=}\; P(A) + P(B) - P(A){\times}P(B)$$
> $$P(\text{앞면 OR } 6) = \frac{1}{2} + \frac{1}{6} - \left(\frac{1}{2} \times \frac{1}{6}\right) = 0.58\overline{333} \;✓$$
> - **상호 배타적 사건에도 적용된다**: 그 경우 $P(A \text{ AND } B) = 0$이라 공식에서 사라져 단순 덧셈으로 환원.
> - **요약:** 상호 배타적이지 않은 사건 간 합 확률은 **결합 확률을 빼서 이중 계산을 방지**하라.

## 🖼️ 시각화 — 12칸 격자 하나로 두 법칙이 다 보인다

|        | 1  | 2  | 3  | 4  | 5  | 6  | **합(주변)** |
|--------|----|----|----|----|----|----|----|
| **H (½)** | H1 | H2 | H3 | H4 | H5 | ⭐**H6** | **6/12 = ½** |
| **T (½)** | T1 | T2 | T3 | T4 | T5 | T6 | **6/12 = ½** |
| **합(주변)** | 2/12 | 2/12 | 2/12 | 2/12 | 2/12 | 2/12 | 12/12 = 1 |

- **AND = 교차 칸 1개:** "H행 ∩ 6열" = H6 하나 → $\frac{1}{12}$. 곱셈 정리의 정체 = **행 비율(½) × 열 비율(⅙) = 칸의 면적 비율** — 독립이란 "행 선택이 열 분포를 안 바꾼다"는 뜻.
- **OR = 행 전체 + 열 전체 − 겹친 칸:** H행 6칸 + 6열 2칸 − 겹침(H6) 1칸 = **7칸** → $\frac{7}{12}$. 단순 덧셈의 오류는 H6을 두 번 센 것.
- **오른쪽 끝 열·아래 끝 행 = 문자 그대로 '주변(margin)':** 각 칸을 결합 확률 p(x,y)라 하면, 오른쪽 여백은 y를 다 더해 지운 p(H)·p(T)(동전의 주변 확률), 아래 여백은 x를 다 더해 지운 p(1)~p(6)(주사위의 주변 확률) — 표의 안쪽=결합, 표의 여백=주변. 아래 🔑에서 이 이름의 유래를 검증.

## 🧠 근본 이해 · 융합 · 통찰

### 🔑 주변 확률의 '주변'이 문자 그대로였다 — 주변화 공식 ([필수수학AI] §2.8.3 다권 보강)
- MA §2.8.3이 이 원자의 "주변 확률"에 분포 버전을 보탠다: **주변 확률 분포 = 결합 확률 분포 표의 가장자리(주변)에 기록되는 분포** — 이름의 유래가 문자 그대로 표의 여백이다(옮긴이 37).
- **주변화(marginalization)**: 결합을 전부 알면 덧셈 규칙으로 $p(x) = \sum_{y \in \text{states of } y} p(x, y)$ — 이 원자의 12칸 격자로 검산: $p(\text{H}) = \sum_{d=1}^{6} p(\text{H}, d) = 6 \times \frac{1}{12} = \frac{1}{2}$ ✓ (H행 전체를 더하면 행의 주변 확률). ⚠️ 책 조판은 등호가 누락된 형태로 인쇄된 듯 — 올바른 진술은 등호 포함([[22 확률·통계 용어 지도 — 확률 변수·분포·기대값·공분산·마르코프|22]] 🔑).
- **외부 검증(2026-07-18):** 옮긴이 37의 "표 여백에 기록"은 통계학사와 일치한다 — "marginal"이라는 이름은 20세기 초 칼 피어슨(Karl Pearson, ~1900년) 이래의 분할표(contingency table) 관행에서 나왔다. 결합 분포를 X(행)×Y(열) 표로 그린 뒤 각 행·열의 합을 표 바깥 **여백(margin)**에 적어 넣던 것이 그대로 이름이 됐다(Wikipedia "Marginal distribution"; Britannica) — **공간적 은유가 아니라 표 레이아웃의 역사적 명명.** 위 🖼️ 시각화에 합 행·합 열을 추가해 이 어원을 그대로 확인할 수 있다.
- "분포(distribution)" 자체의 어원("퍼져 있다"가 아니라 확률 변수 값들에 총 확률 1을 나눠주는 **배분**)은 이 원자의 범위 밖(사건 $P(X)$ 단위) — [[22 확률·통계 용어 지도 — 확률 변수·분포·기대값·공분산·마르코프|22]] 🔑에서 다룸.

### 🔑 공식은 지름길, 진리는 세기 — 두 법칙 모두 열거에서 나왔다
- 책의 논증 순서가 핵심이다: **먼저 12가지 전 조합을 나열해 세고**(H6 = 1개, OR = 7개), **그 다음** 곱셈·덧셈 공식이 같은 답을 내는 지름길임을 보인다. 곱셈 정리 = 카티전 곱 격자의 면적 비율, 덧셈 법칙 = 집합 세기의 **포함-배제**(inclusion–exclusion, $|A \cup B| = |A|+|B|-|A \cap B|$)의 확률판.
- 공식이 헷갈리면 언제든 격자로 후퇴해 세면 된다 — 코드 §의 전수 열거가 그 검산기.

### 🔑 ⚠️ 덧셈 법칙의 두 형태는 동급이 아니다 — $-P(A{\times}B)$형은 독립 전용
- **일반형** $P(A)+P(B)-P(A \text{ AND } B)$: 항상 성립. / **곱 대입형** $P(A)+P(B)-P(A){\times}P(B)$: $P(A \text{ AND } B)=P(A)P(B)$일 때, 즉 **독립일 때만** 성립.
- **상호 배타는 독립이 아니다** — 오히려 극단적 *의존*("A가 나면 B는 확실히 안 남"): 배타는 $P(A \cap B)=0$, 독립은 $P(A \cap B)=P(A)P(B)$ — 둘 다 성립하려면 한쪽 확률이 0이어야 한다.
- **수치 반례:** 배타 사건 $P(4 \text{ OR } 6)$에 곱 대입형을 쓰면 $\frac{1}{6}+\frac{1}{6}-\frac{1}{36} = \frac{11}{36} \ne \frac{1}{3}$ — 오답. 책의 "이 공식은 상호 배타에도 적용"은 **일반형** 기준으로 읽어야 하고, 같은 페이지의 곱 대입형과 섞어 읽으면 모순이 생긴다. **외울 것은 일반형 하나.**
- ✅ **회수(2026-07-16, §2.2.4):** 책이 한 절 뒤에서 스스로 해결 — 진짜 일반형은 **조건부** $P(A \text{ OR } B)=P(A)+P(B)-P(A \mid B){\times}P(B)$: 배타면 $P(A \mid B)=0$이라 자동 환원, 종속이면 정확한 교집합. 상세 → [[1.3 조건부 확률·베이즈 정리 — 방향·뒤집기·조건부 결합·합|1.3]].

### 🔑 "독립 가정"이 이 절의 숨은 기둥 — 다음 결핍의 예고
- 곱셈 정리는 동전·주사위처럼 **서로 물리적 영향이 없는** 사건에서 자연스럽다. 그러나 책 스스로 "독립이라고 가정, 나중에 자세히"라고 못 박았다 — 일반형은 $P(A \text{ AND } B) = P(A) \times P(B \mid A)$(조건부 확률)가 될 것 `(🔭 예고 — 이 장 후반 추정)`. 의존 사건(비 오는 날 AND 우산 판매↑)에서 단순 곱이 무너지는 지점이 다음 절의 결핍-필요 사슬.
- 1.0 초과 오류(133.333%)를 **잡아준 것이 콜모고로프 공리 $P(\Omega)=1$** ([[T09 확률은 왜 0.0~1.0인가 — 공리·곱셈 닫힘·부동소수점 팩트체크·로그 공간|T09]]) — 공리가 이론 장식이 아니라 **실전 버그 검출기**로 작동한 첫 사례.

## 🧩 코드 (§2.2~2.2.2)
> 🎯 목표: ① 전수 열거(12조합)와 곱셈 정리·덧셈 법칙이 일치하나 ② 단순 덧셈의 오답(0.666·1.333) 재현 ③ 배타 사건에 곱 대입형을 쓰면 오답($\frac{11}{36}$)임을 확인.
> 코드: `~/LOCAL/03-00_STUDIES/AI/02_optimization/md2_2_prob_calc.py`

```python
from fractions import Fraction
from itertools import product

space = list(product("HT", range(1, 7)))       # 카티전 곱 — 12조합 전수 열거
assert len(space) == 12
F = Fraction

# ① 결합(AND): 열거 vs 곱셈 정리
joint = F(sum(c == "H" and d == 6 for c, d in space), 12)
print(joint, joint == F(1,2) * F(1,6))          # 1/12 True

# ② 합(OR): 열거 7/12 vs 덧셈 법칙
union = F(sum(c == "H" or d == 6 for c, d in space), 12)
print(union)                                    # 7/12
print(F(1,2) + F(1,6) - F(1,2)*F(1,6) == union) # True — 덧셈 법칙 일치
print(float(F(1,2) + F(1,6)))                   # 0.666... — 단순 덧셈 오답
print(float(F(1,2) + F(5,6)))                   # 1.333... — 공리 P(Ω)=1 위반 신호

# ③ 배타 사건(4 OR 6): 일반형 vs 곱 대입형(독립 전용 — 오용 반례)
print(F(1,6) + F(1,6) - 0)                      # 1/3 — 일반형(교집합 0) ✓
print(F(1,6) + F(1,6) - F(1,6)*F(1,6))          # 11/36 — 곱 대입형 오답 ✗
```

## 🆕 용어
| 용어 | 한 줄 정의 |
|---|---|
| **주변 확률** (marginal probability) | 사건 $P(X)$의 **단일** 확률 — 결합 이전의 기본형 |
| **결합 확률** (joint probability) | 두 사건이 **함께** 발생할 확률 = **AND 연산** — $P(A \text{ AND } B)$ |
| **곱셈 정리** (product rule) | $P(A \text{ AND } B) = P(A) \times P(B)$ (**독립 가정** 하) — 전 조합 열거·세기의 지름길; 옮긴이: 곱셈 규칙·법칙 |
| **합 확률** (union probability) | 사건 A **또는** B가 발생할 확률 = **OR 연산** — $P(A \text{ OR } B)$ |
| **확률의 덧셈 법칙** (sum rule) | $P(A \text{ OR } B) = P(A)+P(B)-P(A \text{ AND } B)$ — 결합 사건 이중 계산 제거; 배타면 단순 합으로 환원 |
| **결합 사건** (joint event) | 두 사건이 함께 일어난 결과(H6 같은 교집합 칸) — 덧셈 법칙이 "한 번만" 세도록 강제하는 대상 |
| **독립** (가정, 🔭) | 이 절에선 *가정*으로만 등장("나중에 자세히") — 한 사건이 다른 사건의 확률을 바꾸지 않음; 정식 정의는 조건부 확률에서 |
| **포함-배제** (inclusion–exclusion, 사이드) | $\lvert A \cup B\rvert = \lvert A\rvert + \lvert B\rvert - \lvert A \cap B\rvert$ — 덧셈 법칙의 집합 세기 원형 |

| $P(\text{앞면 OR } 6)$? | $\frac{1}{2}+\frac{1}{6}-\frac{1}{12}=\frac{7}{12}=0.58\overline{333}$ (열거 7/12와 일치) |
