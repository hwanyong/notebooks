# 강화학습 패러다임 — 정적 데이터 vs 실시간 상호작용

> 강화학습(RL([ɑːr ɛl], 알엘, 알엘)) 코드가 지도학습/LLM([ɛl ɛl ɛm], 엘엘엠, 엘엘엠) 코드와 *완전히 다르게 생긴* 이유 — 고정 데이터셋이 없고, 에이전트가 **환경과 실시간 상호작용**하며 데이터(보상)를 스스로 만들기 때문. 정답(label([ˈleɪbəl], 레이블, 레이블))이 아니라 **보상(reward([rɪˈwɔːrd], 리워드, 리워드))** 으로 배운다.

> 🐍 **AI 영역 노트(코스 외 주제)** — 책 부록 A(지도학습) 밖, 패러다임 비교용 입문.
> 🌐 영역 간 연결: [[_지식 연결 허브]] ｜ 📖 [[_AI-ML MOC]]
> 💡 메타 원리: [[정적 vs 동적 — 컴파일타임과 런타임이 가르는 것들|정적 vs 동적]] — *RL = 동적(런타임) 쪽*

> 🎯 **핵심 (TL;DR)**
> - **정답 없음, 보상만:** label이 아니라 행동을 잘했을 때 주는 `reward`로 학습.
> - **데이터를 스스로 생성:** 고정 데이터셋 ✗. agent([ˈeɪdʒənt], 에이전트, 에이전트)가 `env`와 주고받으며 `state`·`reward`를 런타임에 생성.
> - **코드 구조:** `nn.Module`·`backward` 대신 `env.reset`·`env.step`·점수판(`Q-table`/`V`) 갱신 루프.
> - **반복 단위:** 에포크(전체 데이터 순회)가 아니라 **에피소드**(게임 1판) 또는 상태 전수 갱신.

> [!note] 🗺️ 이 노트 읽는 법
> 지도학습([[08 훈련 루프 (Training Loop)|훈련 루프]])과 **대비**하며 "왜 코드가 다른가"를 잡는 입문. 깊은 수학(벨만)·구현은 나중에. *왜 정적/동적으로 갈리나*의 메타는 [[정적 vs 동적 — 컴파일타임과 런타임이 가르는 것들|정적 vs 동적]].

## 1. 지도학습 vs 강화학습 — 무엇이 다른가
> 머신러닝의 세 패러다임 — **비지도**(레이블 없이 군집), **지도**(레이블로 분류·예측), **강화**(보상으로 행동 학습). 이 노트는 셋 중 *강화학습*이 왜 코드가 다른지를 다룬다.

![[ML 세 패러다임 (비지도·지도·강화).png|560]]
▲ 비지도(군집) · 지도(경계선 분류) · 강화(상태→행동→보상 순환). _출처: Getty Images_

| | 지도학습 / LLM | 강화학습(RL) |
|---|---|---|
| 데이터 | 고정 데이터셋(label 있음) | **없음** — agent가 환경과 만들어냄 |
| 학습 신호 | 예측-정답 **오차(loss)** | 행동에 대한 **보상(reward)** |
| 핵심 코드 | `DataLoader`·`loss.backward()`·`optimizer.step()` | `env.reset()`·`env.step()`·점수판 갱신 |
| 반복 단위 | 에포크(전체 1회 순회) | **에피소드**(게임 1판) |
| 비유 | 정답지 보고 오답 노트 고치기 | 세상에 던져져 부딪히며 보상표 채우기 |

## 2. 구조 — Agent · Environment · Reward
- **에이전트(Agent):** 행동을 고르는 주체(컴퓨터).
- **환경(Environment([ɪnˈvaɪrənmənt], 인바이런먼트, 인바이런먼트), `env`):** 게임판·시뮬레이터. 행동을 받아 다음 상태·보상을 돌려줌.
- 루프: `state` → agent가 `action` 선택 → `env.step(action)` → `next_state` + `reward` → 점수판 갱신 → 반복.
- **피드백 루프:** 행동→환경변화→보상→다음행동. (지도학습의 forward→loss→backward와 자리는 비슷하나 *환경이 데이터를 생성*)

![[강화학습 구조 (에이전트·환경·보상).png|520]]
▲ 에이전트(Agent)가 행동(Actions)을 환경(Environment)에 가하고 보상(Rewards)을 받는 **피드백 루프**. _출처: Getty Images_

## 3. 코드가 다른 이유 — loss/gradient 대신 '점수판'
- 강화학습 기초(동적 계획법·Q-러닝)에는 **미분으로 오차를 역전파하는 과정이 없다.**
- 대신 *"이 상태에서 이 행동을 하면 몇 점?"* 를 가리키는 **점수판(Q-table([kjuː ˈteɪbəl], 큐 테이블, 큐 테이블) / 가치 배열 `V`)** 을 만들고, 정해진 수식(**벨만 방정식**)으로 점수판 숫자를 직접 갱신.
- 그래서 코드가 신경망 연산이 아니라 **단순 행렬·루프문 + 상태 인덱싱** 위주가 된다.

```python
# 강화학습 루프 골격 (개념)
for episode in range(N):           # 게임 1판 = 에피소드
    state = env.reset()            # 환경 초기화
    done = False
    while not done:
        action = policy(state)              # 행동 선택
        next_state, reward, done = env.step(action)  # 환경 상호작용
        # 점수판(Q) 갱신 (벨만): Q[s,a] ← Q[s,a] + lr·(reward + γ·maxQ[s'] − Q[s,a])
        state = next_state
```

> [!quote] 🌉 타 영역 연결 — RL 코드 = 동적 시뮬레이션(런타임)   `#연결/CS`
> 지도학습이 "정적·거대 행렬 연산 덩어리(컴파일타임 친화)"라면, RL은 런타임에 조건 분기·환경 상태를 추적하는 **동적 시뮬레이션 루프**. `env.reset`/`env.step`·외부 환경(`.so`처럼 동적 링킹)·점수판 mutable 갱신이 그 증거. → [[정적 vs 동적 — 컴파일타임과 런타임이 가르는 것들|정적 vs 동적]]

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **강화학습** (RL) | 정답 없이 보상으로 학습; 환경 상호작용으로 데이터 생성 |
| **에이전트 / 환경** (Agent/Env) | 행동 주체 / 행동을 받아 상태·보상을 주는 세계 |
| **상태·행동·보상** (state/action/reward) | 현재 상황 / 고른 행동 / 행동의 점수 |
| **에피소드** (Episode) | 게임 시작~끝 1판(RL의 반복 단위) |
| **Q-table / 가치(V)** | "상태×행동 → 점수" 점수판 |
| **벨만 방정식** | 점수판을 갱신하는 핵심 수식 |
| **정책** (Policy) | 상태 → 행동을 고르는 규칙 |
