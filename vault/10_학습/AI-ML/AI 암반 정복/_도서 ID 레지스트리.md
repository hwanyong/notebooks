# 📚 도서 ID 레지스트리

> 노트 파일명에 쓰는 **책ID(BID)** ↔ 정식 도서명 대응표. 여러 책 목차를 섞어 써도 챕터 번호가 충돌하지 않도록 책을 ID로 네임스페이스화한다.

_새 책을 추가하면 여기 한 줄. BID는 2글자 알파벳 권장._

## 📁 파일명 규칙 (2026-06 경량화)
**형식:** `NN 제목` — 책ID·챕터/절은 파일명에 넣지 않고 **frontmatter `도서:`** 로 기록.

| 자리 | 뜻 | 비고 |
|---|---|---|
| `NN` | **정렬 번호**(내 관리) | 2자리. 공부/복습 순서. 책 순서와 달라도 됨. 중간 삽입은 재번호 또는 허브 학습경로 표로 흡수 |
| `제목` | 노트 제목 | 개념 중심. 다권 통합 노트도 한 제목 |

- **책·절 출처는 frontmatter `도서:`** 에 기록(예: `도서: "[실전선대] §1.4 · [필수수학] §4.1.3"`). **다권 통합 노트는 여러 출처를 나열** — 파일명이 한 책에 결박되지 않는다.
- ⚠️ **콜론(`:`) 금지** — 맥/iCloud 파일명에서 깨짐.
- **탐구**: `T01 제목`(정렬 번호만, `탐구/`). 통찰·주제·CS·수학 노트는 번호 없이 `제목`.
- **왜 바꿨나:** (구) `NN-BID-챕터.서브`는 ① 절번호가 frontmatter `도서:`와 **중복** ② 한 책에 **결박**돼 다권 통합 노트(예 `05` = §1.4+§1.5)에서 절번호가 거짓 ③ 탐구·통찰 등 나머지 노트와 규칙 불일치. → **개념 단위 통합**을 지원하려 경량화.
- **이관 이력:** 2026-06, 0·1단계 척추 노트 15개를 `NN 제목`으로 일괄 리네임 + 전 볼트 링크 갱신(백업 `90_보관`은 동결).
- (참고) 위 `BID` 표는 책 식별·`도서:` 약칭 참조용으로 유지 — 더 이상 파일명엔 쓰지 않음.

**예시**
```
03 계산 그래프                                  (도서: [밑바닥LLM] 부록 A.3)
05 벡터 내적 — 점곱·기하·직교 + 아다마르곱·외적        (도서: [실전선대] §1.4·§1.5)
06 벡터 확장 — 집합·선형독립·부분공간·기저            (도서: [실전선대] §2.1~2.6)
T01 경사 하강법과 훈련 루프                        (탐구)
```
