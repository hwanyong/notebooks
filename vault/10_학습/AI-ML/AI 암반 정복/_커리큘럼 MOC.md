# 🛠️ AI 암반 정복 — 5권 융합 커리큘럼

> 전통적 개발자의 시각에서 **[개념적 원리 ➔ 수리적 시각화 ➔ 밑바닥 코드 구현 ➔ 시스템적 확장]** 흐름으로 5권을 관통하는 코스.

## 📚 5권 책 매핑 (약칭)

노트·체크리스트에서는 아래 약칭 + 절 번호로 표기합니다.

| 약칭 | 정식 도서명 | 역할 |
|---|---|---|
| **[필수수학AI]** | 《AI를 위한 필수 수학》 | AI 전체 수학 지도 · 생성모델 · 그래프 · 인과추론 |
| **[필수수학]** | 《개발자를 위한 필수 수학》 | 미적분 · 확률통계 · 회귀/분류 · 역전파 기초 |

## ✍️ 노트 정리 규칙 (Claude가 따르는 원칙)

> 공부하다 기록할 내용을 보내주시면 아래 원칙으로 정리합니다. 더 나은 방법이 보이면 구조를 바꾸고 알려드립니다.

1. **4층 분리 — 개념은 디스플린(수학)에, 코스는 경로+렌즈** *(2026-06 개정, 구 "단계 폴더에 개념 노트 저장" 폐기)*. 한 개념의 역할을 4층으로 나눠 *중복 없이* 둔다:
    - **① 개념 원자(SSoT):** "이게 *무엇*인가"(정의·성질·증명·기하 직관·수학 코드). **디스플린 폴더**(`수학/<관점>/`·`CS/`…), 도메인 중립, `유형: 개념원자` + `#수학/<관점>` + `관련코스`. **한 개념 = 원자 1개**(여기가 단일 주인).
    - **② 학습 경로(목차):** "*언제·어떤 순서*로 배우나". **단계 허브**의 학습 경로 표 + 진도 체크가 원자를 *AI 순서로* 링크. (원자가 어느 폴더에 있든 링크는 작동 — 목차 보장.)
    - **③ AI 적용 렌즈:** "AI에서 *어떻게 쓰이나·왜 필요*한가"(AI 필요성 사슬·어텐션/신경망/임베딩 적용). **코스 폴더**, `유형: 적용렌즈`. 개념을 *재정의하지 않고* 원자를 **트랜스클루전**(`![[원자]]`)으로 끌어와 보여준 뒤 AI 살만 더한다. (렌즈 파일명·번호는 기존 유지 → 인바운드 링크 보존.)
    - **④ 영역 다리:** "다른 분야와 *무엇이 같은가*" → [[_지식 연결 허브]](기존).
    → 효과: 수학은 수학답게 체계화(①), AI 목차·진도는 그대로(②), AI 특화 연결도 보장(③④), **개념 정의는 단 한 곳**이라 중복 불가. 선례: [[2.2 행렬 곱셈·행렬식]](원자) ↔ [[12 행렬 곱셈·행렬식 — 변환 합성·면적 배율·선형종속|12]](렌즈). *AI 고유 노트(T0X 탐구·하드웨어·구현)는 원자가 아니라 코스/CS에 그대로.*
2. **출처 태그를 붙임** — 프런트매터에 `도서: [실전선대] 1.4` 처럼 커리큘럼 절을 명시. **실제로 받은 절만 표기** — 아직 안 다룬 절을 `A.5~A.6`처럼 추정해 붙이지 말 것.
3. **개념→시각화→코드→확장 4구획**으로 구조화 — 보내주신 내용을 이 틀에 맞춰 배치하고 빈 구획은 비워둠.
4. **단계 노트 체크리스트와 연결 (책·소단원 단위로만 체크)** — 해당 항목을 `- [x]`로 체크하고 개념 노트로 링크. ⚠️ **한 체크박스가 여러 책·소단원을 묶고 있으면(예: "[실전선대] 1.1, 1.3 / [필수수학] 4.1"), 절대 묶음 전체를 체크하지 말 것.** 그 항목을 **소단원별 하위 체크박스로 쪼개고**, *실제로 끝낸 소단원 한 줄만* `[x]` 표시한다. (부모 줄은 체크박스 없는 헤더로 두어 거짓 완료 표시 방지.) 하나의 소단원만 했는데 묶음 항목 전체가 완료된 것처럼 보이면 안 된다.
5. **막힌 점은 별도 기록** — 다음에 돌아올 지점으로 표시.
6. **복습 동선 유지** — ① 개념 노트 파일명에 학습 순서 번호(`01`, `02`…) 접두, ② 노트 상단에 `이전 ｜ 허브 ｜ 다음` breadcrumb, ③ 프런트매터에 `순서`·`이해도`(🟢🟡🔴), ④ 단계 허브의 **학습 경로 표**에 한 줄로 등록. → 복습은 항상 단계 허브에서 시작.
7. **메인 / 사이드 지식 구분** — 교재 핵심(메인)은 일반 본문, 교재 밖 부가지식(사이드)은 **콜아웃 박스 + 태그**로 시각 구분. 4종: 🧭비유`[!example]` `#사이드/비유` · 🔗배경`[!info]` `#사이드/배경` · 🪂심화`[!tip]` `#사이드/심화` · 🌉타분야`[!quote]` `#연결/<도메인>`. 사이드는 [[_지식 연결 지도]]에 모임 → 도메인으로 역접근.
8. **용어 이중 관리 (둘은 항상 함께)** — 노트에 🆕 새 용어를 추가하면 **같은 작업에서 반드시** [[_용어집]]에도 한 줄(국문·영문·정의·출처 링크)을 추가한다. **노트만 갱신하고 용어집을 빠뜨리는 것은 누락(버그)** — 노트 작업을 끝낼 때 *"이번에 생긴 🆕 용어가 용어집에 다 들어갔나"* 를 마지막 검증 항목으로 확인할 것. (코드 토큰은 용어집 '영문·코드 토큰' 표에.) 헷갈릴 땐 용어집 하나만 열어 `Ctrl/Cmd+F`.
9. **예제는 목표 먼저(🎯)** — 실습·코드 섹션은 맨 위에 *"이 예제로 무엇을 확인하나"* 를 한 줄로. 목표를 알고 보면 이해가 훨씬 쉽다. (개별 숫자보다 **shape 흐름·의도**부터)
10. **3층 구조 (책 핵심 → 융합·통찰 → 참조)** — 노트는 세 층으로:
    - **📘 책 핵심(앵커):** 책이 *실제* 가르치는 것만 충실 요약(원문 복제 금지). 사실의 기준점. **형식 고정:** `> [!abstract] 📘 책 핵심 — 부록 A.X {주제}` 콜아웃 한 가지로 통일(꼬리표 "(foundation)"·"(책이 실제 가르치는 것)" 금지, 일반 `##` 헤더 금지).
    - **🧠 근본 이해·융합·통찰(헤드라인):** 왜·비유·타 분야 연결·탐구. **나의 시그니처이자 이 노트의 진짜 가치** — '제공된 지식'을 넘어 *근본 이해와 융합*을 남긴다.
    - **🎯 TL;DR · 🆕 용어 · 🔗 연결(참조):** 빠른 복습·검색용.
    → 책 핵심은 사실을 *고정*하고, 융합·통찰은 그 위에서 *확장*한다. 둘을 시각적으로(콜아웃) 구분해 "책에서 배운 것 vs 내가 확장한 것"이 한눈에 보이게.
11. **🌱 탐구는 본문 트리거 지점에** — 특정 챕터에 귀속되지 않는 재사용 조사(탐구, `T0X`)는 별도 `탐구/` 폴더에 두고, 책 척추 노트에서는 **호기심이 갈라지는 트리거 문장 바로 아래**에 요약+링크 콜아웃으로 건다: `> [!tip] 🌱 탐구 · [[T0X …]]` + 2줄 요약 + `↳ 유래 · [[_탐구 MOC]]`. 하단 '연결' 섹션에 몰지 말 것. **'파생'이라 부르지 않는다** — 부모 의존이 아니라 *유래*(어디서 갈라졌나)만 기록. 운영 원칙: [[_탐구 MOC]].
12. **그림은 재구성 + 책 원본 병기** — 책 도판(e-book 캡처)은 그대로 쓰지 말고 **색·라벨 보강해 직접 재생성**(v=파랑·w=주황·합=초록·음수=분홍, 플롯 내부 라벨은 영어). 각 재구성 그림 바로 아래 접이식 `> [!quote]- 📖 책 원본 — [그림 N-M]` 콜아웃에 원본 크롭을 병기(복습자 1:1 대조, 혼선 방지). 캡션은 일반 텍스트(인용문 `>` 금지 — 안 접힘). 명명: `<BID>그림N-M … (책원본/재구성).png`. **3D 벡터 도판은 끝점→각 축 절편 점선 가이드 + 좌표 라벨**을 넣어 좌표를 읽기 쉽게(단위구 등 끝점 아닌 3D는 제외). **하이브리드(PNG↔Plotly):** 기본은 **PNG**(플러그인·기기 무관하게 어디서나 보임 = 휴대성). *3D·정량 차트처럼 회전/탐색이 이해를 돕는 소수*만 **Plotly 코드블록을 PNG와 *병기***(Plotly 플러그인 미설치 기기에선 PNG가 폴백). **개념 다이어그램(주석·다중 패널)·책 원본 크롭은 PNG 유지** — Plotly는 데이터·3D 보강 도구지 개념 도판의 대체재가 아니다.
13. **열벡터는 명시적 2D `(N,1)`** — 수식의 열벡터는 코드에서 `np.array([[..],[..]])`(또는 `[[...]].T`/`.reshape(-1,1)`)로 `(N,1)` 표기. [필수수학]식 1D `(N,)`는 `.T` 먹통·`@` 내적 오해·`np.linalg` 에러를 부르므로, **매핑 템플릿** `[수학]→[책 1D]→[내 2D]`로 병기해 1:1 유지. (행벡터는 `(1,N)`.) 단 실무 ML의 1D·배치 2D 관용도 함께 안내. 원리: [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)]].
14. **발음 병행 표기** — **그리스 문자·기호·영어 기반 단어**는 읽는 법을 `용어({IPA}, {IPA 기반 한국어 발음}, {한국어 표기법})` 형식으로 병기. 예: `α([ˈælfə], 앨퍼, 알파)`. **본문은 노트당 첫 등장 1회만**(이후 원문만, 가독성), **용어집은 상시 등록**([[_용어집]] 🔤 발음 섹션 = 캐논). 신규 노트 기본 적용. **두문자어는 철자 읽기 일괄 금지** — 단어형(RAG 래그·LoRA 로라·CUDA 쿠다·ReLU 렐루)은 단어로, 이니셜리즘(GPU 지피유·CPU 시피유)만 철자로. **불확실하면 웹 검색 확인**.

15. **암기 3계층 (용어집과 쌍둥이 시스템)** — 외워야 할 핵심을 *용어집처럼* 별도 관리한다. (a) **본문 강조:** 반드시 암기할 사실은 본문에서 `📌`로 표시(책의 "NOTE 암기하세요" 콜아웃 포함). (b) **노트별 암기 카드:** 각 노트 끝(🆕 용어 정리 부근, `## 연결` 앞)에 `## 📌 암기 카드` 섹션 — **`| 앞면(cue) | 뒷면(외울 내용) |` 표**로, *앞면을 보고 뒷면을 떠올리는 자가 테스트*가 되게 작성(원자적·검증 가능한 정의·공식·동치·부호 규칙만; 통찰·서술은 제외). (c) **마스터 암기장:** [[_핵심 암기장]] 독립 노트가 전 노트의 암기 카드를 단계·주제별로 집약(용어집과 동일 운영). **동기화:** 노트에 암기 카드를 추가/수정하면 **같은 작업에서** 마스터 암기장도 갱신(누락=버그). 수식 안에는 발음 병기 금지(규칙 14), 안전 이모지만.

16. **개념 단위 통합 + 파일명 경량화** — 노트는 *책-절*이 아니라 **개념** 단위로 통합한다(규칙 1의 강화). (a) **한 개념 = 정식 주인 노트 1개**, 같은 개념을 가르치는 여러 책 절은 **frontmatter `도서:`에 모두 나열**(다권 통합 허용; `05`=§1.4+§1.5, `T04`=두 책 열벡터). (b) **파일명 = `NN 제목`** — 책ID·절번호는 파일명에서 빼고 `도서:`로(상세 [[_도서 ID 레지스트리]]). (c) **합병은 *해로운 중복*일 때만**(예: 입문→정식 흡수·승격); 평소엔 주인 + 1~2줄 요약 + 위임 링크(T06 패턴; 구 04 렌즈는 2026-07 통합·소멸). 새 책 내용이 기존 노트와 크게 겹치면 **주인 노트를 보강**(중복 노트 금지).

17. **전방 참조(예고)는 짧게, 그 장 핵심만 깊게** — 책이 *"N장에서 배운다 / 나중에 / 추후"* 라고 **명시적으로 미룬** 용어·응용은 **전방 참조(forward reference)** = 지금 정복 대상이 아니다. 그 문단은 *동기부여 예고편*일 뿐 → **`(🔭 예고 — N장)` 태그 + 1줄 요약(가리키는 곳만 앵커)으로 짧게** 두고 통과. 반대로 *이 장이 직접 정의·증명하는 핵심*(예: 선형독립 식 2-3·기저 정의)은 문장 단위로 정밀 정복. 판단 신호 = 본문의 "N장에서/나중에" 자체. (미해결 심화는 '막힌 점' 섹션에 deferral로.)

18. **필요성 사슬 먼저 (결핍 → 필요 → 발명)** — UHD는 *개념이 어떤 결핍·문제의 답으로 생겨났나*를 알아야 각인된다(추상 정의 나열은 암기→망각). 각 개념 노트는 책 핵심(정의) **앞에** `> [!question] 🎯 왜 이걸 배우나 — 필요성 사슬` 콜아웃을 둔다: *구체적 목표*(1단계=임베딩·의미검색·압축)에 앵커 → **각 개념을 *바로 앞 단계가 만든 결핍*의 해결책으로** 연쇄 제시 → 한 줄 사슬로 마무리. (책이 "나중에" 미룬 건 규칙 17대로 가볍게; 단 *지금 개념의 필요성*은 반드시 앞에 준다.) 예: 06 §2 = 벡터→집합→선형결합→독립→생성/부분공간→기저.

19. **외부 출처 정리 노트는 디스플린 폴더에, 코스는 링크만** — 외부 자료(블로그·교재·논문)를 충실히 옮긴 *참고 노트*는 커리큘럼 자료가 아니라 **재사용 가능한 일반 지식**이다. ⇒ **해당 디스플린 폴더**(`10_학습/수학`·`CS`·`통찰`…)에 두고 `유형: 참고-출처정리` + `#참고` + `#<분야>/<관점>` 태그, frontmatter에 `출처`·`출처URL`·`관련코스`. **코스 노트(spine)에는 본문/연결에서 링크만** 건다(선례: `수학/3D 변환 행렬`, `수학/행렬 곱셈의 두 관점`). 코스 폴더(`AI 암반 정복/`)에는 *spine(단계) + 탐구(T0X, 내 심화) + 관리노트*만 남긴다 — 외부 출처 정리는 `탐구/`에 넣지 않는다(탐구 = 내 탐구). 디스플린 `_OO MOC`에 등재.

20. **연습문제는 별도 노트 + 양방향 링크 (`PNN`, 연습/ 폴더)** — 문제 세트 하나 = 노트 하나(`PNN 주제 — 시험 개념 (N문)`). (a) **정답·풀이는 접이식 콜아웃**에 숨겨 노트 재방문 = 재시도가 되게. (b) **문제마다 "시험하는 개념" 링크 의무** — 어느 원자/렌즈의 어느 규칙을 테스트하는지. (c) **오답은 해당 학습 노트 `막힌 점`에 역기입**하고 이해도(🟢🟡🔴) 조정 근거로. (d) 검산은 코드로. (e) [[_연습 MOC]] 색인과 관련 렌즈 `연결`에 같은 작업에서 등재(용어집·암기장 동기화 원칙과 동일). 운영 상세: [[_연습 MOC]].

21. **파일명 바이트 길이 캡 — NFD 기준 255바이트** *(2026-07-18 신설 — 노트 23 파일명 손상 사고로 원인 확인)*. 이 vault를 동기화하는 iCloud 브리지는 파일명을 NFD(자모 분해형)로 처리하는데, 그 바이트열이 255바이트를 넘으면 끝부분이 잘려 파일명이 깨진다(마지막 한글 음절이 낱자모로 남음, 예 "…기하학.md"→"…기하ᄒ.md") — **재시도(rename·재생성)로 복구 불가**, Obsidian에서 직접 F2로 다시 입력해야 고쳐짐. 한글 음절은 NFC 1자=3바이트가 NFD 분해 시 자모 2~3개×3바이트로 **최대 3배 팽창**하므로 화면상 글자 수가 짧아 보여도 위험할 수 있다(실측: 262바이트=깨짐 vs 255바이트=정상). **새 파일 생성·리네임 전 확인**: `len(unicodedata.normalize('NFD', 파일명).encode('utf-8'))` ≤ 255. 넘길 것 같으면 부제를 줄이거나 공백을 압축하고 em dash " — " 대신 하이픈 "-"을 쓴다(개념 정의엔 영향 없음, 표기 차이일 뿐). 안전 어림선: 한글 비중이 높은 제목은 **NFC 기준 약 110~115바이트**(대략 55~58자) 이하로.

새 개념 노트는 [[개념 노트 템플릿 (커리큘럼용)]]에서 시작합니다.
