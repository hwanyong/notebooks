# 확률은 왜 0.0~1.0인가 — 공리·곱셈 닫힘·부동소수점 팩트체크·로그 공간

> 출발 질문(내 가설): "0~100 정수로 관리하면 직관적이고, 부동소수점(floating point, 실수를 가수×2^지수로 저장하는 방식) 오차도 없어서 더 낫지 않나?" — 결론부터: **0.0~1.0 관습의 진짜 근거는 수학(비율 정의·공리·곱셈 닫힘)이고, 컴퓨터 이전(1933 콜모고로프, 그 이전 라플라스)부터 확정된 것이다.** 부동소수점은 근거가 아니라 *부차적 궁합*이며, Gemini가 제시한 "0~1 구간이 가장 정밀하다"는 논거는 **절반이 틀렸다**(§2에서 수치로 교정).

> 🌱 **탐구 노트(재사용)** — 책 절이 아니라 독립 호기심으로 파고든 조사. 확률·손실 함수·수치 안정성을 다루는 어느 노트에서든 링크로 재사용.
> ⚙️ 원리 이웃: [[P07 특이행렬 행렬식의 수치 불안정성 — 로그 스케일 실험 (1문)|P07 로그 스케일 실험]] · [[2.8 행렬식 — 계산·선형종속성·특성다항식|원자 2.8 언더플로 경고]] ｜ 🗂️ [[_탐구 MOC]] · 🌐 [[_지식 연결 허브]]

> 🎯 **핵심 (TL;DR)**
> - **진짜 이유는 수학, 컴퓨터가 아니다.** 확률의 정의 자체가 *비율*(유리한 경우 ÷ 전체 경우 → 자연히 0~1)이고, 콜모고로프(Kolmogorov([ˌkɔːlməˈɡɔːrəf], 콜머고로프, 콜모고로프)) 공리가 $P(\Omega)=1$ 로 못 박았다(1933, 컴퓨터 이전). %는 같은 값의 *표시 형식*일 뿐.
> - **곱셈 닫힘:** 독립 사건의 동시 확률 $p \cdot q$ 는 다시 $[0,1]$ 안의 확률이다(닫혀 있음). 0~100 스케일은 $50\times50=2500$ — **곱셈에 닫혀 있지 않아** 매번 ÷100 보정이 필요.
> - **Gemini 교정:** "0~1 구간이 float로 가장 정밀, 100부터 유효숫자 손실" 주장은 **틀렸다** — IEEE 754의 *상대* 정밀도는 크기와 무관하게 $\varepsilon \approx 2.22\times10^{-16}$ 으로 동일(1.0에서 2.2e-16, 100.0에서 1.4e-16 — 검증 §2). 촘촘해지는 건 *절대* 간격뿐.
> - **진짜 수치 문제는 언더플로:** 확률을 수백 개 곱하면 0으로 붕괴(0.01⁴⁰⁰ → 선형 0.0) → **로그 공간**(곱→합, $\log 0.01^{400} = -1842.07$ 정확)으로 해결. [[P07 특이행렬 행렬식의 수치 불안정성 — 로그 스케일 실험 (1문)|P07]] 행렬식 실험과 같은 처방.

> [!note] 🗺️ 이 노트 읽는 법
> Gemini 문답을 1차 출처로 **교차검증·교정**해 정리했다. Gemini의 4개 논거 중 ①곱셈 ③정규화 ④로그는 ✅ 유지, ②부동소수점 정밀도는 ⚠️ 절반 오류라 §2에서 수치 실험으로 바로잡았다. 내 원래 가설(정수형 관리)의 반박은 §3.

> [!question] 🎯 왜 이걸 배우나 — 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 인간 직관은 "비 올 확률 70%"처럼 0~100이 편하다. 심지어 정수라면 부동소수점 오차도 없다. 그런데 왜 모든 수학·과학·코드가 0.0~1.0인가?
> 1. 확률로 하는 일의 대부분은 **결합**이다 — "A도 일어나고 B도 일어날 확률"(독립이면 곱). 0~100 스케일은 곱할 때마다 ÷100 보정이 필요하고 곱셈에 닫혀 있지 않음 → **보정 없이 곱셈이 성립하는 척도**가 필요 → 비율 그 자체인 $[0,1]$.
> 2. 확률의 근원 정의가 이미 비율이다(라플라스(Laplace([ləˈplɑːs], 러플라스, 라플라스)) 고전 확률 = 유리한 경우 수 ÷ 전체 경우 수 ≤ 1) → 콜모고로프가 $P(\Omega)=1$ 공리로 확정 → 연속 확률(밀도의 적분=1)·기댓값·여사건($1-p$)이 전부 상수 없이 맞물림.
> 3. 그런데 $[0,1]$ 실수를 컴퓨터에 넣으면 **새 결핍** — 확률 수백 개를 곱하면 표현 가능한 최솟값 아래로 떨어져 0이 됨(언더플로(underflow([ˈʌndərfloʊ], 언더플로우, 언더플로))) → **로그 공간 발명**: $\log(pq)=\log p+\log q$, 곱이 합이 되어 붕괴하지 않음.
> **한 줄:** 결합(곱셈) 필요 → 비율 정의·공리로 $[0,1]$ 확정(수학) → 곱셈 연쇄가 float에서 붕괴(결핍) → 로그 공간(발명).

## 1. 수학적 근거 — 관습이 아니라 정의다 (Gemini ✅)
- **비율 정의:** 고전 확률 = $\dfrac{\text{유리한 경우 수}}{\text{전체 경우 수}}$. 부분÷전체이므로 **정의상** $[0,1]$. 0~100은 여기에 ×100을 *표시용으로* 덧입힌 것(percent = per centum, 100마다).
- 📌 **콜모고로프 공리(1933):** ① $P(A)\ge 0$ ② $P(\Omega)=1$ ③ 서로소 사건의 가산 가법성. 기준점 "전체 = 1"이 모든 확률 이론의 뿌리 — 컴퓨터 등장 전에 확정된 수학적 선택.
- **곱셈 닫힘:** $p,q\in[0,1] \Rightarrow pq\in[0,1]$ — 독립 사건의 AND 결과가 *다시 확률*이다. 0~100 스케일은 $50\times50=2500$ 으로 척도를 이탈 → 연산마다 ÷100. n개 연쇄면 $100^{n-1}$ 로 나누는 대공사(Gemini ✅ 정확).
- **미적분 정합:** 연속 확률에서 밀도함수의 전체 적분이 1 — $\int f(x)\,dx = 1$. 기준이 100이면 모든 공식에 상수가 붙어 지저분해짐. 여사건 $1-p$, 기댓값(확률 가중 평균)도 전부 "전체=1" 위에서만 깔끔.

## 2. 부동소수점 팩트체크 — Gemini의 ②는 절반 오류 ⚠️

> [!warning] ⚠️ 정확성 — "0~1 구간이 가장 정밀, 100부터 유효숫자 손실 시작"은 **틀렸다**   `#정확성`
> IEEE([aɪ ˈtrɪpəl iː], 아이 트리플 이, 아이트리플이) 754 double의 지수 필드와 가수 필드는 **분리**되어 있다 — 큰 지수가 가수 비트를 잡아먹지 않는다. **상대 정밀도는 크기와 무관하게 머신 엡실론(machine epsilon([ˈɛpsɪlɒn], 엡실런, 엡실론)) $\varepsilon\approx2.22\times10^{-16}$ 수준으로 일정**하다(정규수 범위 내). 유효숫자 손실은 $2^{53}$ 을 넘는 정수 등 전혀 다른 지점의 얘기다. 100은 지수 6이 필요할 뿐, 15~16자리 유효숫자가 그대로다.

수치 검증(2026-07-15, `t09_probability_scale.py`):

| 주장 (Gemini) | 판정 | 실측 |
|---|---|---|
| 0에 가까울수록 표현 밀도(절대 간격)가 촘촘 | ✅ | ULP([ʌlp], 얼프, 얼프)(0.5)=1.1e-16 ≪ ULP(100.0)=1.4e-14; 양수 double의 약 절반이 $[0,1)$ 에 몰려 있음 |
| 100처럼 커지면 유효숫자(상대 정밀도) 손실 시작 | ❌ | 상대 간격: 1.0에서 2.22e-16, 100.0에서 **1.42e-16** — 사실상 동일. 손실 없음 |
| 0~100 스케일은 표현력에서 불리 | ❌(뉘앙스) | float끼리 비교하면 실질 차이 없음. 오히려 **왕복 스케일링이 유해**: `x*100/100 != x` 가 무작위 double의 **14.6%**에서 발생(100은 2의 거듭제곱이 아니라 나눗셈이 부정확) |
| (반전 — 내 가설 방향) | 🔄 | **정수 0~100은 float로 전부 정확** 표현되고, 정작 0.05 같은 십진 소수가 이진 float로 **부정확**(0.05000000000000000278)이다. "정밀도 때문에 0~1"은 방향 자체가 역 |

**정리:** 부동소수점 특성은 0~1 관습의 *근거*가 아니다(관습이 100년 먼저다). 다만 확률이 0 근처로 쏠릴 때(꼬리 확률·우도) float의 0-근처 조밀함이 *결과적으로* 잘 맞고, 그마저 한계에 닿으면 §4의 로그 공간으로 넘어간다.

## 3. "정수형 관리" 가설 반박 — 양자화가 파괴하는 것
내 원래 가설(0~100 정수, 오차 제로) 검증:
- **101단계 양자화:** 정수 0~100은 딱 101개 값. 시그모이드·softmax 출력의 미세 차이, $10^{-40}$ 같은 꼬리 확률·우도가 전부 0 또는 1%로 뭉개짐 — 오차를 없앤 게 아니라 **오차를 최소 0.5%p로 고정**한 것.
- **동적 범위:** 확률의 실전 값역은 $10^{-300}$~$1$ 에 걸친다(우도 곱·희귀 사건). 선형 정수 눈금은 이 로그적 분포를 원리적으로 못 담는다.
- **단, 완전 기각은 아님:** 고정소수점(fixed-point, 소수점 위치를 고정한 정수 표현)은 임베디드·DSP에서 실제로 쓰이고, (🔭 예고 — 4단계 [AI엔지니어링]) INT8 양자화도 모델 압축에 표준이다 — 그러나 그것은 *스케일 팩터를 동반한 압축 기법*이지 확률 척도의 재정의가 아니다.

## 4. 로그 공간 — 진짜 수치 위기(언더플로)의 처방 (Gemini ✅)
- **붕괴 실측:** $0.01$ 을 400번 곱하면 참값 $10^{-800}$ 은 double 최소 표현치(비정규수(subnormal([ˌsʌbˈnɔːrməl], 서브노멀, 서브노멀)) $\approx 4.9\times10^{-324}$)보다 작아 **선형 계산은 0.0으로 붕괴**. 로그 합은 $400\ln 0.01 = -1842.068\ldots$ 를 끝자리까지 정확히 유지.
- **원리:** $\log(pq) = \log p + \log q$ — 곱셈이 덧셈으로 바뀌어 지수가 *가수 밖의* 일반 숫자로 내려온다. 확률 $[0,1]$ ↔ 로그 확률 $(-\infty, 0]$ ↔ 음의 로그 확률 $[0,\infty)$.
- **같은 처방의 선례:** [[P07 특이행렬 행렬식의 수치 불안정성 — 로그 스케일 실험 (1문)|P07]] 이 $10^{-16}$~$10^{-1}$ 행렬식을 로그축으로 편 것과 동일 원리. (🔭 예고 — 2단계) 크로스엔트로피 손실 = **음의 로그 가능도**, "우도 곱 최대화" 대신 "로그 우도 합 최대화"가 표준인 이유가 바로 이 언더플로 회피다.

### 🧩 코드로 직접 검증
> 🎯 목표: ① 상대 정밀도가 1.0과 100.0에서 동일함(Gemini ② 반증) ② ×100 왕복 손상률 ③ 선형 곱 붕괴 vs 로그 합 생존.
> 코드: `~/LOCAL/03-00_STUDIES/AI/02_optimization/t09_probability_scale.py`

```python
import numpy as np, math, random

print(np.spacing(1.0), np.spacing(100.0))          # 2.22e-16, 1.42e-14 (절대 간격)
print(np.spacing(1.0)/1.0, np.spacing(100.0)/100.0) # 2.22e-16, 1.42e-16 (상대 — 동일!)

random.seed(0)  # ×100 왕복 손상률: 100은 2^k가 아니라 나눗셈이 부정확
print(sum((x := random.random()) != x*100/100 for _ in range(100_000)) / 100_000)  # 0.146

p = 1.0
for _ in range(400): p *= 0.01                     # 참값 1e-800
print(p)                                           # 0.0 — 언더플로 붕괴
print(sum(math.log(0.01) for _ in range(400)))     # -1842.068... — 로그 공간 생존
```
