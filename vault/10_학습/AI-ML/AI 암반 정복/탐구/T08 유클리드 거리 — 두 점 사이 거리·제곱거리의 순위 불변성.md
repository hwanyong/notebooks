# 유클리드 거리 — 두 점 사이 거리·제곱거리의 순위 불변성

> 🌱 **탐구 노트(허브형)** — 책 절 정리가 아니라, 여러 챕터에 흩어져 쓰이는 "유클리드 거리"를 한 곳에 모으는 앵커. **노름의 정의 자체는 [[1.2 벡터 크기와 단위벡터 (노름)|원자 1.2]] 소관** — 이 노트는 그걸 반복하지 않고 재인용만 한다. 이 노트 고유의 내용은 **① 두 점 일반형 ② "제곱근을 생략해도 되는 이유"(순위 불변성) ③ 여러 챕터에 걸친 쓰임 색인** 셋뿐이다.
> 🔗 발원지: [[1.2 벡터 크기와 단위벡터 (노름)|원자 1.2 §1.3]] · [[1.11 k-평균 클러스터링 — 브로드캐스팅·argmin·거리 척도|원자 1.11 §3.3]] ｜ 🗂️ [[_탐구 MOC]] · 🌐 [[_지식 연결 허브]]

> 🎯 **핵심 (TL;DR)**
> - **정의(원자 1.2 재인용, 재유도 안 함):** $\delta(p,q)=\lVert p-q\rVert=\sqrt{\textstyle\sum_i (p_i-q_i)^2}$ — 노름(원점→점)을 임의의 두 점으로 일반화한 것. [[1.2 벡터 크기와 단위벡터 (노름)|1.2]]가 이미 "$d(x,y)=\lVert x-y\rVert$"로 정의해 둔 바로 그 식.
> - **이 노트의 고유 내용 — 순위 불변성:** $\sqrt{\ }$ 는 $[0,\infty)$ 에서 **순증가함수** → $a<b \Leftrightarrow \sqrt a<\sqrt b$. 그래서 "누가 더 가까운가"만 알면 되는 알고리즘(k-평균·k-최근접 이웃 등)은 **제곱근을 생략한 제곱거리**로도 정확히 같은 순위·같은 argmin을 얻는다.
> - **실전 함의:** 제곱근은 계산 비용만 들 뿐 순위를 바꾸지 않으므로, **비교·정렬·argmin이 목적이면 코드에서 제곱근을 생략하는 것이 표준.**

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* [[1.2 벡터 크기와 단위벡터 (노름)|1.2]]는 "벡터 하나가 원점에서 얼마나 먼가"(노름)만 답했다. 그런데 실전 문제는 대부분 "이 점이 **저 점**에서 얼마나 먼가"(둘 다 원점이 아님)를 묻는다.
> 1. 원점 기준 거리(노름)로는 임의의 두 점 사이를 못 잼 → 두 점을 잇는 척도가 필요 → **일반 거리 $\delta(p,q)=\lVert p-q\rVert$**(1.2가 이미 정의: 차벡터의 노름).
> 2. k-평균 같은 알고리즘은 **150개 점 × k개 중심**처럼 이 거리를 수백~수천 번 반복 계산해야 함 → 매번 제곱근(비싼 연산)까지 구하는 게 낭비가 아닌가 하는 의문 → **제곱근을 생략해도 되는가?**
> 3. 목적을 다시 보면 "정확한 거리값"이 아니라 "**어느 중심이 가장 가까운가**"(순위·argmin)만 필요 → 순서를 보존하는 변환이면 무엇을 써도 결과가 같다 → **제곱은 순증가함수이므로 순위 보존** → 제곱근 생략이 정당화된다.
> **한 줄:** 노름(원점→점, 1.2) → 두 점 사이로 일반화(거리) → 알고리즘에서 반복 비교가 필요해짐 → "정확한 값"이 아니라 "순위"만 필요함을 깨달음 → 제곱근 생략(모노토닉 논증).

> [!abstract] 📘 책 핵심 — [실전선대] §3.3 k-평균 클러스터링 중 거리 공식 부분
> - 두 점 사이 유클리드 거리: $\delta_{i,j}=\sqrt{(\mathrm d_i^x-\mathrm c_j^x)^2+(\mathrm d_i^y-\mathrm c_j^y)^2}$ (데이터 관측치 $i$ ↔ 중심 $j$).
> - 책의 코드(`np.sum((data-centroids[ci,:])**2, axis=1)`)에는 **제곱근이 빠져 있다** — 책은 이를 짚으며 묻는다: "코드가 잘못된 것일까요?"
> - 몇 페이지 뒤 책 스스로 답한다: **"거리와 거리 제곱은 단조로운 관계이므로 두 메트릭의 결과는 같습니다. 따라서 제곱근 연산을 추가하면 결과에 아무런 영향을 미치지 않으면서 코드 복잡성과 계산 시간만 증가시키므로, 코드에서는 되도록 생략하는 것이 좋습니다."**
> - 즉 이 절의 진짜 요점은 "k-평균 알고리즘"이 아니라 **"거리 비교엔 제곱근이 불필요하다"는 일반 원리**다 — k-평균은 그 원리를 쓰는 첫 사례일 뿐.

## 🧠 근본 이해 · 융합 · 통찰

### 🔍 순위 불변성의 직접 증명
**주장:** $a,b\ge0$ 일 때 $a<b \iff \sqrt a<\sqrt b$.

**증명:** $\sqrt{\ }$ 는 $[0,\infty)$ 에서 정의된 순증가함수다 — $f(x)=\sqrt x$ 의 도함수 $f'(x)=\dfrac{1}{2\sqrt x}>0$ ($x>0$에서), 그리고 $f$ 는 $[0,\infty)$ 에서 연속·단사(injective)이므로 순서를 보존한다. 즉 임의의 음이 아닌 실수 집합에서, **제곱을 취하기 전 순서와 취한 후 순서가 정확히 같다.** $\blacksquare$

> [!tip] 🪂 왜 이게 "당연한 사실"이 아니라 강조할 가치가 있나 — 모든 변환이 순위를 보존하진 않는다   `#사이드/심화`
> 순증가함수(sqrt·제곱·로그 등, 정의역 한정 시)는 순위를 보존하지만, **모든 함수가 그런 건 아니다.** 예: $f(x)=\sin x$ 는 순위를 보존하지 않는다($\sin 10°<\sin 170°$ 이지만 $10<170$은 맞아도 $\sin$ 자체는 감소·증가를 오간다 — 국소적으로만 단조). 그래서 "argmin이 목적이면 아무 변환이나 생략해도 된다"가 아니라, **정확히 순증가(또는 순감소)함수일 때만 이 논증이 성립한다.** sqrt·제곱(음이 아닌 정의역)·log는 이 조건을 만족하지만 임의의 변환이 다 그런 건 아니라는 게 이 논증의 숨은 전제.

### 🧩 코드로 직접 검증
> 🎯 목표: "제곱거리로 뽑은 argmin"과 "실제 거리(sqrt 포함)로 뽑은 argmin"이 정말 100% 일치하는지 실증.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la3_3_distance_monotonic.py`

```python
import numpy as np

rng = np.random.default_rng(0)
data = rng.normal(0, 5, (150, 2))       # 150개 관측치, 2특징
centroids = rng.normal(0, 5, (3, 2))    # 임의의 중심 3개

# 제곱 거리(책의 코드 그대로 — 제곱근 생략)
sq_dists = np.zeros((150, 3))
for ci in range(3):
    sq_dists[:, ci] = np.sum((data - centroids[ci, :])**2, axis=1)

# 실제 거리(제곱근 포함)
real_dists = np.sqrt(sq_dists)

# 두 방식의 argmin이 전부 일치하는가?
g_squared = np.argmin(sq_dists, axis=1)
g_real   = np.argmin(real_dists, axis=1)
print("150개 점 전부 같은 클러스터에 배정?", np.array_equal(g_squared, g_real))  # True
```

관찰 포인트:
- 150개 관측치 × 3개 중심, 총 450개 거리 쌍 전부에서 **제곱거리 순위 = 실제거리 순위**가 정확히 일치.
- 이건 우연이 아니라 위 증명의 필연적 결과 — **임의의 데이터·임의의 중심 개수에서도 항상 성립**한다.
