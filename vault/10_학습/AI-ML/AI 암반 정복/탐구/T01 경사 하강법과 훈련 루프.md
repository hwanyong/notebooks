# 경사 하강법과 훈련 루프

> `backward()`로 얻은 그레이디언트(나사 조절량)를 받아, 가중치를 **조금씩 반복 갱신**해 손실을 최소화하는 과정 = "학습"의 실체.

> 🌱 **탐구 노트** — 특정 챕터에 속하지 않는 **재사용 지식**. 어느 책 절에서든 링크해 다중 연결.
> 🔗 관련 지점: [[03 계산 그래프]] (역전파에서 파생) · [[06 가중치 — 상태·초기화·학습 실패]] (손실 지형·지역 최솟값) ｜ 🗂️ [[_탐구 MOC]]

> 🎯 **핵심 (TL;DR([tiː ɛl diː ɑːr], 티엘디알, 티엘디알))**
> - 학습 = `grad` 방향으로 가중치를 **조금씩 반복 갱신**해 손실 최소화 = **경사 하강법**
> - 훈련 루프: `forward → loss → backward → optimizer.step → zero_grad` 반복
> - 함정: 지역 최솟값·오버슈팅 → **관성·적응적 학습률(Adam([ˈædəm], 애덤, 아담))** 로 보완

> [!note] 🗺️ 이 노트 읽는 법
> 이 노트는 **책의 절이 아니라, 내가 직접 조사한 파생 사고**다. 책 **A.3~A.4에서 '역전파(backprop([ˈbækprɒp], 백프롭, 백프롭))'가 처음 등장했을 때**, *"그럼 이 `grad`로 어떻게 학습하지?"* 를 Gemini([ˈdʒɛmɪnaɪ], 제미나이, 제미니)로 파고든 내용을 정리한 것. → 출처 가지: [[03 계산 그래프]]. 그래서 📘 책 핵심이 따로 없고, 본문 전체가 내 사고 흐름이다.

## 🌱 파생 출처 — A.3~A.4 역전파에서 갈라져 나온 내 조사
> 책의 독립 절이 아님. 역전파가 처음 나온 지점([[03 계산 그래프]])에서 *내가 직접 조사·정리*한 경사 하강법·훈련 루프 핵심.
- **경사 하강법:** 옵티마이저가 `grad`(기울기) **반대 방향**으로 가중치를 학습률(`lr`)만큼 한 걸음씩 갱신 → 손실 최소화.
- **훈련 루프(A.7) 표준형:**
  ```python
  for epoch in range(N):
      pred = model(X)            # forward
      loss = loss_fn(pred, y)    # 오차
      loss.backward()            # grad 계산
      optimizer.step()           # 가중치 갱신 (w ← w - lr·grad)
      optimizer.zero_grad()      # grad 초기화
  ```
- 이 루프를 반복하며 모델을 훈련. (옵티마이저: SGD([ɛs dʒiː diː], 에스지디, 에스지디), Adam 등)

> 💭 **여기서 내 사고가 분기됨 ↓** (복습 시 같은 흐름 재현)

## 🧭 받아들이기 — 내 이해 / 질문
- 의문: *"역전파는 모든 경우의 수를 다 해보고 오차 없는 케이스를 뽑는 건가? 아니면 비효율적이니 수학으로 한 번에 나오나?"*
- 답: **후자(수학의 마법).** 모든 경우를 다 대입(brute-force([bruːt fɔːrs], 브루트 포스, 브루트 포스))하는 건 불가능. 미분으로 **방향만 한 번에** 알아낸다.
- 그 다음 의문: *"모든 경우는 안 보고, 한 번 내려간 결과를 평가해서 더 나은 방향이 나올 때까지 반복하는 건가?"* → **정확히 그것.** 이게 훈련 루프.

## 1. 왜 '완전 탐색(brute-force)'은 불가능한가
- 나사(파라미터)가 $w_1, b$ 딱 2개여도, 각각 100눈금이면 조합은 $100 \times 100 = 10{,}000$.
- 실제 최신 모델은 파라미터가 **수천억 개**. 가능한 조합 수가 우주의 원자 수보다 많다.
- 슈퍼컴퓨터로 우주의 나이만큼 계산해도 다 못 본다 → **모든 경우를 보는 방식은 폐기.**

## 2. 미분의 마법 — "지름길 방향만 딱 본다" (산 내려가기 비유)
안개 낀 산꼭대기에서 골짜기(=오차가 가장 적은 곳)로 내려가는 상황.
- **모든 좌표 착륙해 보기 (불가능):** 헬기로 산의 모든 지점에 내려 가장 낮은 곳 찾기 = brute-force.
- **발바닥 기울기 느끼기 (실제 방식):** 지금 서 있는 자리의 **경사(기울기)** 만 느낀다. 발밑이 앞쪽으로 기울면 "앞으로 가면 낮아지겠구나"를 즉시 안다.
- 역전파는 이 **발바닥 기울기를 수식(미분)으로 단번에 계산**해 주는 매직. 모든 경우를 찔러보지 않는다.

## 3. 한 방에 안 끝난다 — "한 걸음씩" (학습률)
- 방향은 확실히 알지만 **골짜기까지 정확한 거리는 모른다.**
- 그래서 그 내리막 방향으로 **딱 한 걸음만 조심히** 내려간다(= 나사를 아주 살짝 돌림).
- 새 자리에서 **다시 forward 계산 → 오차 확인 → backward로 기울기 재확인** → 또 한 걸음.
- 이 과정을 수천~수만 번 반복하며 골짜기 밑바닥으로 뚜벅뚜벅 걸어가는 알고리즘 = **경사 하강법.**

> 핵심 뉘앙스: 다음 방향을 *"이쪽 맞나?" 찍어보고 평가하는 게 아님.* 미분이 "이쪽이 내리막"이라고 **이미 확실히 알려줘서** 방향엔 확신이 있다. 다만 거리(보폭)를 모르니 조심히 한 걸음씩 갈 뿐.

## 4. 훈련 루프(Training Loop([ˈtreɪnɪŋ luːp], 트레이닝 루프, 트레이닝 루프)) 5단계
내가 정리한 흐름과 실제 코드가 1:1로 맞는다.

| 단계 | 직관 | 실제 과정 |
|---|---|---|
| 1 | 일단 쏴 보고 점수 매기기 | **forward**: 현재 가중치로 예측 → 오차(loss) 계산 |
| 2 | 거꾸로 역산해 힌트 얻기 | **backward(역전파)**: loss로 `grad`(조절 방향) 얻기 |
| 3 | 나사 살짝 조절 | **update**: `grad` 방향으로 가중치 한 걸음 갱신 (학습률만큼) |
| 4 | 새 자세로 다시 평가 | **다시 forward**: 오차가 줄었는지 확인 |
| 5 | 만족할 때까지 반복 | 오차가 ≈0 이거나 더 안 줄 때까지 1~4 **무한 반복** |

의사코드:
```python
for epoch in range(N):          # 5. 반복
    a = model(x)                # 1. forward
    loss = loss_fn(a, y)        # 1. 오차
    loss.backward()             # 2. 역전파 → .grad 채움
    optimizer.step()            # 3. 가중치 갱신 (w ← w - lr·grad)
    optimizer.zero_grad()       # grad 초기화(다음 스텝 위해)
```

## 5. 경사 하강법은 만능이 아니다 — 한계와 극복
> 내 의문: *"바로 직후 단계 기준에서만 가장 효율적인 걸 고르니, 전체로 보면 비효율적이거나 도달 불가능한 상황도 생길 것 같다."* → **정확.** 경사 하강법은 **지금 발밑 경사만 보는 근시안적(Greedy([ˈɡriːdi], 그리디, 그리디))** 방식이라 다음 함정들이 있다.

![[국소·전역 최적점.jpeg|440]]
> 골짜기·봉우리가 여러 개 — 경사 하강법은 가까운 **지역 최솟값(Local minimum([ˈloʊkəl ˈmɪnɪməm], 로컬 미니멈, 로컬 미니멈))** 에 갇힐 수 있고, 진짜 바닥(**전역**)은 더 아래에 있다. (최대/최소는 부호만 다른 같은 개념)

**한계 (전체 지도를 못 보기 때문)**
- **지역 최솟값(Local Minima([ˈmɪnɪmə], 미니머, 미니마)):** 산 중턱 작은 웅덩이를 "여기가 제일 낮네!"로 착각하고 멈춤. 진짜 바닥(전역 최솟값)은 더 아래에 있는데도.
- **안장점(Saddle Point([ˈsædl pɔɪnt], 새들 포인트, 새들 포인트)):** 한 방향은 내리막, 다른 방향은 오르막인 지점. 중심부 경사가 0이라 멈춰버림.
- **평탄 구역(Plateau([plæˈtoʊ], 플래토우, 플래토)):** 끝없이 평평한 사막. 경사가 거의 0이라 조절 힌트(grad)가 안 나와 전진 불가.
- **오버슈팅(Overshooting([ˌoʊvərˈʃuːtɪŋ], 오버슈팅, 오버슈팅)):** 보폭(학습률)이 너무 크면 골짜기 바닥을 지나쳐 반대편으로 점프 → 좌우로 튕기며 바닥에 영영 못 닿는 **진동(무한 루프)**. (= 내가 말한 "무한 루프 발생")

**극복 — "과거의 기억"을 더한다**
- **관성(Momentum([moʊˈmɛntəm], 모멘텀, 모멘텀)):** 굴러오던 속도를 실어, 작은 웅덩이(지역 최솟값)는 **타고 넘어** 진짜 바닥을 찾게 함.
- **적응적 학습률(Adaptive([əˈdæptɪv], 어댑티브, 어댑티브) LR([ɛl ɑːr], 엘알, 엘알)):** 처음엔 성큼, 바닥에 가까울수록 보폭을 줄여 오버슈팅 방지.
- → 이 둘을 합친 가장 대중적인 옵티마이저가 **Adam**. (책 뒤에서 단골로 등장)

> [!quote] 🌉 융합 — 국소 최적점은 딥러닝만의 것이 아니다   `#연결/최적화`
> 손실 지형의 '지역 최솟값에 안주'는 **조직이 엉망 코드에 안주(국소 최적점)·진화가 땜질에 머무름**과 *같은 수학 구조*. 전역 최적점으로 가려면 멀쩡한 시스템을 부수고 '혼돈의 계곡'을 지나야 하는 것도 동일. → [[엔트로피와 질서 — 최소 에너지 법칙과 엔지니어링|엔트로피와 질서]] · 🌐 [[_지식 연결 허브]]

## 🌉 〔사이드 · 타 분야 연결〕 네트워크 라우팅 — 같은 '위상', 다른 알고리즘
> [!quote] 🌉 사이드 지식 — 네트워크 ↔ 딥러닝   `#연결/네트워크`
> 교재 밖 연결. 허브: [[_지식 연결 지도]] · 정밀 정리: [[두 전파 — 역전파 vs 라우팅 (그래프는 같고 메시지는 다르다)|역전파 vs 라우팅 통찰]]
> 내 통찰: *"라우터가 전체 지도 없이 국소 정보로 경로를 찾는 게 똑같아 보인다 — 그래프 구조가 유사해서?"*
> → **그래프 위상은 같지만, 같은 알고리즘은 아니다(과거 '같은 패러다임' 단정을 교정).** 공통은 *노드-에지 그래프 위 메시지 전달*이라는 **계산 골격**뿐. 역전파는 그래디언트를 연쇄법칙으로 역방향 전파(=역방향 자동미분), 거리벡터 라우팅은 거리 추정을 Bellman-Ford([ˈbɛlmən fɔːrd], 벨먼 포드, 벨먼-포드)로 전파 — *대상·수학·목적이 다르다.* 자세히 → [[두 전파 — 역전파 vs 라우팅 (그래프는 같고 메시지는 다르다)|두 전파(통찰)]].

아래 표는 *동일성*이 아니라 **수렴 동역학의 비유** — 그것도 역전파가 아니라 **경사 하강(최적화)** 쪽과의 비유다.

| 경사 하강 최적화 동역학 | 라우팅 수렴 동역학 *(비유)* |
|---|---|
| 오버슈팅(진동) | 라우트 플래핑(metric([ˈmɛtrɪk], 메트릭, 메트릭) 미세 변동에 경로가 요동) |
| 지역 최솟값 | 서브옵티멀 라우팅(이웃 hop([hɒp], 홉, 홉) 정보만 믿어 우회, 예: RIP([ɑːr aɪ piː], 알아이피, 알아이피)) |
| 안장점·평탄 → 멈춤 | 라우팅 블랙홀(다음 hop 못 정해 패킷 드롭) |
| 관성·적응적 LR로 둔감화 | 홀드다운 타이머·라우트 댐프닝으로 급변 둔감화 |

**ML([ɛm ɛl], 엠엘, 엠엘)을 네트워크에 *적용*한 사례 (역전파=라우팅이라서가 아니라, 토폴로지가 그래프라 ML을 얹을 수 있어서):**
- **TCP([tiː siː piː], 티씨피, 티씨피) 혼잡 제어(ECN([iː siː ɛn], 이씨엔, 이씨엔))·Google([ˈɡuːɡəl], 구글, 구글) BBR([biː biː ɑːr], 비비알, 비비알):** 라우터의 혼잡 신호를 피드백받아 전송률 조절. BBR은 망 상태 변화율(≈미분)로 전송률 제어 → 학습률 조절과 *유사한 제어*.
- **SDN([ɛs diː ɛn], 에스디엔, 에스디엔) 닫힌 루프 제어:** 컨트롤러가 텔레메트리(결과)→오차→플로 규칙 갱신. 피드백 제어 루프.
- **RouteNet([ˈruːtnɛt], 루트넷, 라우트넷) (GNN([dʒiː ɛn ɛn], 지엔엔, 지엔엔)):** 네트워크 토폴로지를 신경망으로 보고 지연·손실 오차로 **실제 역전파**를 돌려 파라미터 최적화 — *네트워크 위에 딥러닝을 얹은* 것(라우팅 자체가 역전파인 게 아님).

## 🆕 새 용어 정리 (글로서리)

| 용어 | 한 줄 정의 | 비유 |
|---|---|---|
| **경사 하강법** (Gradient([ˈɡreɪdiənt], 그레이디언트, 그레이디언트) Descent([dɪˈsɛnt], 디센트, 디센트)) | 기울기 반대 방향으로 조금씩 내려가 손실을 최소화하는 알고리즘 | 안개 산에서 골짜기로 한 걸음씩 |
| **기울기 / 경사** (Gradient) | 현재 위치에서 손실이 가장 가파르게 **커지는** 방향+크기 (그 반대로 가면 내리막) | 발바닥이 느끼는 비탈 |
| **학습률** (Learning Rate([ˈlɜːrnɪŋ reɪt], 러닝 레이트, 러닝 레이트), `lr`) | 한 걸음의 **보폭**. 크면 빠르지만 골짜기를 건너뛰어 발산, 작으면 안정적이나 느림 | 발을 얼마나 크게 내딛나 |
| **훈련 루프** (Training Loop) | `forward → loss → backward → update` 를 반복하는 학습 사이클 | 쏘고-고치고-다시쏘기 반복 |
| **옵티마이저** (Optimizer([ˈɒptɪmaɪzər], 옵티마이저, 옵티마이저)) | `grad`를 받아 실제로 가중치를 갱신하는 규칙. 예: `w ← w − lr·grad` (SGD), Adam 등 | 나사 돌리는 작업반장 |
| **에폭 / 스텝** (Epoch([ˈɛpək], 에퍽, 에포크) / Step([stɛp], 스텝, 스텝)) | 스텝=가중치 1회 갱신, 에폭=전체 데이터 1회 통과 | 한 걸음 / 한 바퀴 |
| **지역·전역 최솟값** (Local/Global([ˈɡloʊbəl], 글로벌, 글로벌) Minima) | 골짜기가 여러 개 — 동네 골짜기(지역) vs 가장 깊은 골짜기(전역) | 산속 작은 웅덩이 vs 진짜 바닥 |
| **안장점** (Saddle Point) | 한 방향 내리막·다른 방향 오르막, 중심 경사 0이라 멈춤 | 말안장 한가운데 |
| **평탄 구역** (Plateau) | 경사 ≈0 인 넓은 평지, grad가 안 나와 정체 | 끝없는 사막 |
| **오버슈팅** (Overshooting) | 보폭이 너무 커 바닥을 지나쳐 좌우로 진동(무한 루프) | 골짜기 건너뛰며 튕김 |
| **관성** (Momentum) | 이전 이동 속도를 실어 작은 웅덩이를 타고 넘음 | 굴러오던 공의 탄력 |
| **적응적 학습률** (Adaptive LR) | 바닥에 가까울수록 보폭을 자동으로 줄임 | 도착 직전 속도 줄이기 |
| **Adam** | 관성 + 적응적 학습률을 합친 대표 옵티마이저 | 똑똑한 작업반장 |
