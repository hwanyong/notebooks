# 가중치 — 상태·초기화·학습 실패

> **가중치(Weight([weɪt], 웨이트, 웨이트))** = 학습으로 데이터의 방향성을 새긴 모델의 핵심 상태(State([steɪt], 스테이트, 스테이트)). 같은 구조라도 어떤 데이터로 가중치를 조였느냐가 모델의 정체성을 결정한다.

> 🗺️ [[_0단계 - 닻 올리기|0단계 허브]] ｜ ⬅️ 이전: [[05 퍼셉트론·편향·은닉층|퍼셉트론·편향·은닉층]] ｜ ➡️ 다음: [[07 데이터 로더 (Dataset · DataLoader)|데이터 로더]]

> 🎯 **핵심 (TL;DR)**
> - 가중치 = 학습으로 새겨진 모델 상태(=모델의 영혼); `nn.Linear`가 자동 보유
> - `requires_grad`는 가중치 *유무*가 아니라 "학습할지(True)/고정할지(False)" **모드**
> - 같은 값 초기화 → 대칭성 안 깨져 학습 실패 → **작은 난수 + `manual_seed`** 로 초기화

> [!note] 🗺️ 이 노트 읽는 법
> 책 A.5(가중치 초기화)를 *척추*로, 그 한 주제에서 내가 파고든 의문·비유·확장을 **그 아래 가지(💭)로** 붙였다. 복습 땐 책 핵심 → 내 분기 순으로.

---

> [!question] 🎯 왜 이걸 배우나 — 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제(구체적 목표):* 같은 신경망 구조인데도 어떤 모델은 학습이 되고 어떤 모델은 아무리 돌려도 멍청한 채 멈춘다. 그 차이를 만드는 정체를 알아야 한다.
> 1. 입력을 변형할 '연결 강도'가 어딘가 있어야 함 → 학습으로 채워지는 상태가 필요 → **가중치(`nn.Linear`가 자동 보유)**
> 2. 학습이 끝난 모델은 더 안 돌려야 하고, 일부는 고정하고 싶음 → 가중치마다 "조절할까/고정할까" 스위치 필요 → **`requires_grad`(학습 모드)**
> 3. 가중치를 다 같은 값으로 시작하면 모든 뉴런이 똑같이 굴러 학습 실패 → 대칭을 깰 무작위성 필요 → **작은 난수 초기화**
> 4. 난수면 매번 결과가 달라져 디버깅·비교 불가 → 같은 출발점 재현 장치 필요 → **`torch.manual_seed`**
> **한 줄:** 모델의 정체=가중치 → 켜고 끌 모드(`requires_grad`)가 필요 → 대칭을 깨려 난수 초기화 → 재현하려 시드 고정.

> [!abstract] 📘 책 핵심 — 부록 A.5 가중치 초기화
> - `nn.Linear`의 가중치 행렬(`model.layers[0].weight`)은 생성 시 **작은 난수로 자동 초기화**되고 `requires_grad=True`(훈련 가능)가 기본.
> - **대칭성을 깨기 위해** 작은 난수로 초기화하는 게 좋음 — 같은 값이면 모든 뉴런이 같은 연산·업데이트를 반복해 학습이 안 됨.
> - `torch.manual_seed(123)` 로 난수 생성기 시드를 고정하면 초기화를 **재현 가능**.

> 💭 **여기서 내 사고가 분기됨 ↓** (복습 시 같은 흐름 재현)

## 🧭 받아들이기 — 내 이해 / 질문
- 오해 바로잡음: *"가중치 없이 훈련 가능"* 이 아니라 → **"훈련(역전파) 없이 사용(추론)만 한다"** 였음. 학습 끝난 모델을 예측만 할 땐 역전파가 필요 없어 끄는 것(→ `torch.no_grad()`), 가중치 자체가 없는 게 아님.
- 그래도 던진 질문: *"가중치 때문에 훈련이 안 되는 모델이 실제로 있나?"* → **있다.** 잘못 초기화하거나 연산 중 값이 깨지면 모델이 통째로 먹통이 된다(아래 §2).

## 1. 가중치란? — 모델의 영속 상태(persistent memory)
- 입력 행렬을 특정 방향으로 **회전·신축하는 공간 변형 필터**. `nn.Linear` 안의 `W`·`b`가 그것.
- 데이터가 층을 지날 때 신호를 **얼마나 증폭/감쇠할지 결정하는 '연결 강도'**.
- 없으면? **기억상실에 걸린 프로그램** — 입력이 들어와도 연산할 게 없고, 오차로 업데이트할 대상(나사)도 없어 훈련 자체가 불가.

## 1-1. 🔑 `requires_grad=True`의 진짜 의미 — 잠금 vs 잠금 해제
> 또 다른 오해: *"나사를 조정할 수 있게 했으니 = 가중치를 나중에 적용 = 가중치 없이도 작업 가능?"* → **아니다, 반대다.**

- `nn.Linear(50, 30)` 을 실행하는 **그 순간** 50×30 = **1,500개 가중치가 랜덤(작은 난수)으로 즉시 박힌다.** 가중치가 없는 공백 상태는 한 순간도 없다.
- `requires_grad` 는 가중치의 **유무**가 아니라 **조절 모드**를 정한다:
  - **`True`(기본값, 잠금 해제):** 연산에 참여 + 오차 나면 드라이버로 돌려 조절 허용(**훈련 가능**).
  - **`False`(잠금/프리징):** 연산엔 참여하지만 용접해 고정 → 오차가 커도 **절대 안 돌림(훈련 불가)**.
- **왜 "가중치 없이"는 불가능?** 입력×가중치 **행렬 곱**이 쉴 새 없이 일어나는데, 가중치 행렬이 비면 곱 자체를 못 해 런타임 에러로 멈춘다. 가중치는 옵션이 아니라 **구동 필수 원자재**.

**한 줄:** 가중치(나사)는 처음부터 꽉 차 있고, `requires_grad`는 그 나사를 **"학습하며 돌릴까(True)"** vs **"고정해 둘까(False)"** 의 차이일 뿐.

## 2. 가중치 '때문에' 훈련이 안 되는 치명적 상황 3가지
**① 대칭성(Symmetry([ˈsɪmɪtri], 시머트리, 시메트리))이 안 깨짐 — 모두 같은 값(0)으로 초기화**
모든 가중치를 똑같이 0(또는 같은 값)으로 두면, 모든 뉴런이 정·역방향에서 **완전히 같은 신호**를 내보내 서로 구분되게 학습되지 못함. 아무리 돌려도 똑똑해지지 않고 멈춤.
→ 그래서 **작은 난수로 초기화**(`torch.manual_seed`로 재현성 확보). "가중치 초기화"가 중요한 이유.

**② 그레이디언트 소실 / 폭주 — 값이 잠기거나 터짐**
- **소실(Vanishing):** 역전파 중 grad가 소수점 아래로 너무 작아져 사실상 0 → 나사가 잠겨 먹통.
- **폭주(Exploding):** 값이 너무 커져 `NaN`/`Inf`로 터짐(오버플로) → 구동 자체가 다운.

**③ 프리징(Freezing([ˈfriːzɪŋ], 프리징, 프리징)) — `requires_grad=False` 로 의도적 잠금**
오픈소스 거대 모델(GPT([dʒiː piː tiː], 지피티, 지피티) 등)을 가져와 **일부만 튜닝**할 때, 특정 레이어의 `requires_grad`를 `False`로 바꿔 그 가중치는 **학습 안 되게 락(Lock([lɒk], 락, 록))**. (파인튜닝/전이학습의 기본 기법)

> [!info] 🔗 사이드 · 배경 — "가중치 없이 사용 가능"의 진짜 뜻 = `torch.no_grad()`   `#사이드/배경`
> 책 본문의 그 문장은 *훈련이 끝난 뒤 **추론(예측)만 할 때 역전파 연산을 꺼두라*** 는 뜻. 추론 땐 grad가 필요 없으니 `with torch.no_grad():` 로 그래프 기록을 꺼 메모리·속도를 아낀다. (가중치를 없애는 게 아니라 *미분 추적*을 끄는 것)

## 2-1. 가중치 초기화 깊이 — 왜 매번 다르고, 왜 시드를 고정하나

**① 난수가 매번 달라지는 이유 — 의사난수(PRNG([piː ɑːr ɛn dʒiː], 피알엔지, 피알엔지))**
컴퓨터 난수는 진짜 무작위가 아니라 공식으로 만드는 **의사난수(Pseudo-Random([ˌsuːdoʊ ˈrændəm], 슈도 랜덤, 슈도 랜덤))**. 파이토치 내부에 난수 상태 머신이 돌고, `nn.Linear`를 호출할 때마다 상태가 한 단계 전진하며 새 숫자를 뽑는다. 실행 시각·시스템 엔트로피 등으로 **시작 상태가 매번 달라** 결과가 조금씩 바뀜.
→ `torch.manual_seed(123)`로 시작 상태(시드)를 고정하면 몇 번 돌려도 **같은 가중치**가 재현됨(재현성).

**② 초기 난수가 다르면 최종 성능도 달라지나? — YES**
손실 지형(Loss([lɒs], 로스, 로스) Landscape([ˈlæn(d)skeɪp], 랜드스케이프, 랜드스케이프))은 수천억 차원의 산악 지형(골짜기 여럿). 초기 난수 차이 = **출발 좌표 차이**. 출발점이 다르면 발밑 경사도 달라, 같은 데이터·같은 경사 하강법이어도 **서로 다른 골짜기(종착지)** 에 도달한다. 그래서 같은 데이터로도 성능이 95% vs 70%로 갈릴 수 있음.
→ 그래서 실무는 **시드 고정 + 여러 시드 평균**.

> [!tip] 🌱 탐구 · [[T01 경사 하강법과 훈련 루프|경사 하강법과 훈련 루프]]
> 출발 좌표(초기 난수)가 종착지를 가르는 건, 가중치를 **손실 지형에서 경사 하강법으로 한 걸음씩 내려가며** 학습하기 때문. **지역 최솟값**에 갇히는 한계(§5)가 탐구의 핵심.
> _↳ 손실 지형·출발점 얘기에서 갈라진 챕터-독립 탐구 · 전체_ → [[_탐구 MOC]]

**③ "대칭성을 깨뜨린다"의 논리 — 같은 값 초기화의 붕괴 4단계**
가중치를 모두 같은 값(0, 0.5 등)으로 두면:
1. **정방향 획일화:** 은닉층 30개 뉴런이 입력에 대해 소수점까지 똑같은 값을 계산.
2. **역전파 복제:** 상태가 같으니 30개 뉴런이 받는 grad도 완전히 동일.
3. **업데이트 동기화:** 같은 방향·같은 양만큼 나사가 돌아감.
4. **결론(차원의 함정):** 30개 뉴런이 영원히 **하나의 뉴런처럼** 동기화 → 30개를 써도 1개 쓰는 것과 다름없음.

→ 그래서 초기에 **무작위성(혼돈)** 을 주입해야 뉴런마다 역할이 **분화(Specialization([ˌspɛʃələˈzeɪʃən], 스페셜라이제이션, 스페셜라이제이션))** 되고(어떤 뉴런은 가로선, 어떤 뉴런은 세로선 담당) 대칭이 깨진다.

**그럼 우연히 같은 난수가 뽑혀 획일화될 수도?** 이론상 가능하지만 실질적으로 0.
- `Float32` 한 값의 경우의 수 ≈ **42억**. 한 퍼셉트론의 나사 수십 개가 두 뉴런에서 **전부 똑같이** 뽑혀야 하므로 확률 ≈ $(1/42\text{억})^{n}$ → **차원의 저주**로 사실상 불가능.
- 결론: 난수만 뿌려주면 무조건 서로 다른 길로 찢어진다 → 안심하고 써도 됨.

> [!quote] 🌉 사이드 · 타 분야 연결 — 분산 시스템   `#연결/분산시스템`
> 모든 노드가 완벽히 대칭이면 데드락(deadlock([ˈdɛdlɒk], 데드락, 데드록))에 빠지듯, 분산 알고리즘은 **난수 오프셋**(예: 재시도 백오프 지터)으로 대칭을 깨 충돌을 피한다. 가중치 난수 초기화도 같은 원리. 허브: [[_지식 연결 지도]]

> [!tip] 🪂 사이드 · 심화 — 실전 초기화 기법(Xavier·He)   `#사이드/심화`
> 그냥 작은 난수가 아니라, 층 크기에 맞춰 분산을 조절하는 **Xavier([ˈzeɪvieɪ], 제이비어, 자비에)(Glorot([ɡloʊˈroʊ], 글로로, 글로로))·He([heɪ], 헤이, 허) 초기화**가 표준. 그레이디언트 소실·폭주를 줄인다. → 2단계 [필수수학AI] 4장에서 수학적으로.

## 3. 🪂 〔심화·철학〕 가중치 = 데이터가 새긴 '관성의 길'
> [!tip] 🪂 사이드 · 심화 — 표현 학습(Representation Learning([ˌrɛprɪzɛnˈteɪʃən ˈlɜːrnɪŋ], 레프리젠테이션 러닝, 리프리젠테이션 러닝)) / 매니폴드   `#사이드/심화`
> - 가중치 = 입력 데이터라는 행렬을 특정 방향으로 흐르게 만드는 **관성(중력 골짜기)**. 한 번 고정되면 새 데이터도 그 길을 따라 흘러 정답을 찾는다.
> - 데이터가 누적·역전파될수록 가중치는 노이즈를 걸러내고 **진짜 신호의 방향만 뾰족하게** 깎아낸다 = 특징 추출(Feature Extraction([ˈfiːtʃər ɪkˈstrækʃən], 피처 익스트랙션, 피처 익스트랙션)).
> - 가중치가 없으면 "특색 없는 답"이 아니라 **완전한 파괴(랜덤 노이즈)**. LLM([ɛl ɛl ɛm], 엘엘엠, 엘엘엠) 가중치를 지우면 "안녕"에 의미 없는 문자열을 뱉음.
> - 같은 구조(`nn.Module`)라도 **무엇을 먹였나(데이터)에 따라 완전히 다른 존재**: 의료 데이터 → 진단 AI, 게임 코드 → 게임 물리 AI. **구조(뼈대)는 같아도 가중치 숫자가 모델의 영혼.**

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **가중치** (Weight) | 입력을 변형하는 연결 강도이자 학습으로 새겨진 모델 상태 |
| **가중치 초기화** | 학습 시작 전 가중치를 작은 난수로 설정. 대칭성 깨기 위해 필수 |
| **대칭성 문제** (Symmetry) | 같은 값 초기화 시 모든 뉴런이 똑같이 굴러 학습 불가 |
| **그레이디언트 소실/폭주** | grad가 0으로 잠김 / NaN·Inf로 터짐 → 훈련 마비 |
| **프리징** (Freezing, `requires_grad=False`) | 특정 레이어 가중치를 학습 못 하게 잠금(파인튜닝) |
| **`torch.no_grad()`** | 추론 시 역전파 추적을 꺼 메모리·속도 절약 |
| **표현 학습 / 특징 추출** | 데이터의 핵심 방향을 가중치에 새기는 과정 |
| **의사난수 / 시드** (PRNG / `manual_seed`) | 공식으로 만든 가짜 난수. 시드 고정 시 같은 초기값 재현 |
| **손실 지형** (Loss Landscape) | 가중치 공간 위 손실의 산악 지형(골짜기 여럿). 출발점이 종착지를 좌우 |
| **대칭성 깨기 / 분화** (Specialization) | 난수 초기화로 뉴런마다 역할을 다르게 만드는 것 |
| **Xavier · He 초기화** | 층 크기에 맞춰 분산을 조절하는 실전 초기화 기법 |
