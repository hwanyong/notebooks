# 텐서 이해하기

> 텐서 = 벡터·행렬을 임의 차원으로 일반화한 **다차원 데이터 컨테이너**. PyTorch([ˈpaɪtɔːrtʃ], 파이토치, 파이토치)의 모든 데이터·가중치가 텐서.

> 🗺️ [[_0단계 - 닻 올리기|0단계 허브]] ｜ ⬅️ 이전: [[01 PyTorch 설치 트러블슈팅 (Apple Silicon)|PyTorch 설치]] ｜ ➡️ 다음: [[03 계산 그래프|계산 그래프]]

> 🎯 **핵심 (TL;DR)**
> - 텐서 = 차수(rank([ræŋk], 랭크, 랭크))로 정의: 스칼라 0D · 벡터 1D · 행렬 2D · 그 이상 nD
> - dtype 기본값: 정수 `int64`, 실수 `float32`(정밀도·속도 균형, GPU 최적)
> - 모양 다루기: `.shape` 확인 / `.reshape`·`.view` 변형 / `.T` 전치 / `.matmul`·`@` 행렬곱

---

> [!question] 🎯 왜 이걸 배우나 — 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제(구체적 목표):* 신경망의 입력·가중치·출력을 담고, GPU에서 빠르게 곱하고, 학습 때 자동으로 미분까지 되는 "그릇"이 필요하다.
> 1. 숫자 데이터를 담을 게 필요 → 임의 차원(스칼라·벡터·행렬·nD) 컨테이너 → **다차원 배열(NumPy 같은)**
> 2. 그런데 일반 배열은 GPU 가속도, 자동 미분도 못 한다 → 학습엔 둘 다 필수 → **PyTorch 텐서 (배열 + GPU + autograd)**
> 3. 메모리·속도를 챙겨야 → 64비트는 과하다 → **float32 기본 dtype (GPU 최적)**
> 4. 층마다 모양이 달라 곱이 어긋난다 → 모양을 보고·바꾸고·곱해야 → **`.shape`·`.reshape`/`.view`·`.T`·`matmul`/`@`**
> **한 줄:** 다차원 그릇이 필요 → 그릇에 GPU·자동미분을 박은 게 텐서 → dtype·모양 연산으로 그 그릇을 다룬다.

> [!abstract] 📘 책 핵심 — 부록 A.2 텐서
> - **텐서란:** 차원 개수 = **차수(order([ˈɔːrdər], 오더, 오더) / rank)**. 스칼라=0D(rank 0), 벡터=1D, 행렬=2D, 그 이상=nD 텐서. 계산 관점에선 **다차원 데이터 컨테이너**.
>
> ![[그림 A-6 텐서 차수.png|480]]
>
> - **PyTorch 텐서 = NumPy([ˈnʌmpaɪ], 넘파이, 넘파이) 배열 + α([ˈælfə], 앨퍼, 알파):** ① autograd([ˈɔːtoʊɡræd], 오토그래드, 오토그래드)(자동 미분) ② GPU([dʒiː piː juː], 지피유, 지피유) 가속. (NumPy API·문법을 많이 차용)
> - **생성(코드 A-1):** `torch.tensor(1)`→0D, `torch.tensor([1,2,3])`→1D, 중첩 리스트→2D/3D.
> - **A.2.2 데이터 타입(dtype):**
>   - 정수 기본 `int64`, 파이썬 실수로 만들면 기본 `float32`. `.dtype`로 확인.
>   - **왜 float32 기본?** 64비트보다 메모리 적고 빠르며 딥러닝엔 충분한 정밀도. **GPU가 32비트에 최적화**돼 속도↑.
>   - 변환: `tensor.to(torch.float32)`.
> - **A.2.3 자주 쓰는 연산:**
>   - `.shape` — 크기 확인 (예: `torch.Size([2,3])`)
>   - `.reshape(3,2)` / `.view(3,2)` — 크기 변경. **차이:** `.view`는 메모리가 연속이어야 동작, `.reshape`는 무관(필요시 복사).
>   - `.T` — 전치(주대각선 기준 뒤집기). *크기 바꾸는 reshape와 다름.*
>   - `.matmul(...)` 또는 `@` — **행렬 곱**.

## 🧠 근본 이해 · 융합 · 통찰

### 🔍 왜 '배열'이 아니라 '텐서'인가 — 단어의 두 관점
> 책은 "텐서 = 다차원 배열"이라고만 한다. 그런데 *왜 Array([əˈreɪ], 어레이, 어레이)라 안 하고 Tensor([ˈtɛnsər], 텐서, 텐서)*라 부를까? 호기심에 파고들어 보니 관점이 둘로 갈렸다.

**① 엔지니어 관점 — "그냥 다차원 배열" (실무는 이걸로 100% 충분)**
- 프레임워크(PyTorch·TensorFlow)에서 텐서 = **다차원 숫자 배열**. rank(차원 수)만 다를 뿐 본질은 배열:
  - Rank 0 = 스칼라(`7`) · Rank 1 = 벡터(`[1,2,3]`) · Rank 2 = 행렬(이미지 가로×세로 픽셀) · Rank n = 고차원 배열(`[Batch, Channel, Height, Width]` 컬러 이미지 묶음).
- **그럼 왜 'Array'가 아니라 'Tensor'?** 단순 배열에 없는 **두 가지 무기**가 엔진에 내장돼 있어서 (= 📘의 "NumPy + α"):
  - **하드웨어 가속:** GPU/TPU 수천 코어로 넘겨 대규모 병렬 행렬 연산.
  - **autograd + 계산 그래프:** 연산 경로를 기억해 역전파 때 스스로 편미분 → 가중치 갱신. (→ [[03 계산 그래프|계산그래프]] · [[04 자동 미분 (autograd)|autograd]])
- → 그래서 텐서는 "숫자 담은 배열"이 아니라 **가속·자동미분이 내장된 지능형 배열**.

**② 수학·물리 관점 — "좌표계가 바뀌어도 안 변하는 본질"**
- 정통 수학에선 "텐서 = 다차원 배열"은 엄밀히 틀린 말. 텐서 = **좌표계 변환에 독립적인 선형 관계**를 가리키는 대상.
- 보는 각도(좌표계)를 바꿔도 **본질 값·물리 특성이 일정하게 유지되도록** 특정 규칙(선형 변환)으로 변하는 대수적 객체.
  - 예: 공간의 **힘(Force([fɔːrs], 포스, 포스))** 은 기준 축을 어떻게 회전해도 물리적 실체는 안 변함 — 그 좌표 불변의 기하학적 매핑 규칙이 진짜 '텐서'.

> [!quote] 🌉 사이드 · 타 분야 연결 — 물리·선대의 '불변량'   `#연결/물리`
> **임베딩 벡터**도 같은 성질: 공간을 회전·이동(선형 변환)해도 단어들 사이 **상대적 의미 거리(코사인 유사도)** 라는 본질은 유지된다 → 수리적으로 텐서의 성질. 그래서 RAG([ræɡ], 래그, 래그)·시맨틱 검색이 좌표계와 무관하게 동작. (→ 1단계 [[_1단계 - 선형대수와 임베딩|코사인 유사도]]) 허브: [[_지식 연결 지도]]

> [!tip] 🪂 사이드 · 심화 — 두 관점을 언제 쓰나   `#사이드/심화`
> **현업(모델 코드·인프라):** "GPU 가속 + 자동 미분 되는 고차원 배열"로 이해하면 100% 충분. **논문·수리 모델:** "좌표계 불변의 기하학적 본질"이라는 의미가 뒤에 숨어 있다고 보면 중심이 단단해진다. 그래서 0단계 환경 세팅·1단계 선형대수에서 텐서 **shape(차원) 맞추는 감각**이 전통적 데이터 포인터 제어만큼 중요해진다.

### 🧩 다른 노트에서 만난 부품들
> 이미 다른 노트에서 만난 것들이 여기서 "기초 부품"으로 정체를 드러냄.

- **`.matmul`/`@` = [[05 퍼셉트론·편향·은닉층|퍼셉트론]] 노트의 그 행렬 곱.** "`Linear`는 이중 for문 = 행렬곱"이라 했던 그 연산의 실제 API가 `matmul`/`@`. 텐서는 그 연산의 *재료*.
- **`.shape` = [[05 퍼셉트론·편향·은닉층|퍼셉트론]] data flow의 그 추적 도구.** `[1,50]→[1,30]→[1,3]`을 읽던 게 전부 `.shape`.
- **`float32` = [[06 가중치 — 상태·초기화·학습 실패|가중치]] 가중치의 그릇.** 가중치 행렬도 결국 float32 텐서. 메모리·속도 얘기가 여기서 연결.
- **개발자 시각:** 텐서 = 딥러닝의 *기본 자료형(primitive([ˈprɪmɪtɪv], 프리미티브, 프리미티브) type([taɪp], 타이프, 타입))*. 모든 게 텐서 위에서 돈다 — 입력·가중치·grad·출력 전부.

## 🧩 코드 실습 (A.2)
> 🎯 목표: 텐서를 만들고(rank), dtype을 확인·변환하고, 모양 연산(shape/reshape/T/matmul)을 손으로 돌려보기.
> 코드: `~/LOCAL/03-00_STUDIES/AI/00_setup/a2_tensors.py`

```python
import torch
t0 = torch.tensor(1)            # 0D 스칼라
t2 = torch.tensor([[1,2,3],[4,5,6]])   # 2D 행렬
t2.shape                        # torch.Size([2, 3])
t2.reshape(3,2); t2.view(3,2)   # 크기 변경
t2.T                            # 전치 → [3,2]
t2.matmul(t2.T)                 # 행렬곱 → [2,2]  (== t2 @ t2.T)
torch.tensor([1.0,2.0]).dtype   # torch.float32
```

## 🆕 새 용어
| 용어 | 한 줄 정의 |
|---|---|
| **텐서** (Tensor) | 임의 차원의 다차원 데이터 컨테이너. (엔지니어 관점) 가속·autograd 내장 배열 / (수학 관점) 좌표계 불변의 선형 관계 |
| **차수 / 랭크** (order / rank) | 차원의 개수. 스칼라0·벡터1·행렬2·… |
| **dtype** | 텐서 원소 자료형. 기본 정수 `int64`, 실수 `float32` |
| **`.reshape` vs `.view`** | 둘 다 크기 변경; `view`는 연속 메모리 필요, `reshape`는 무관 |
| **전치** (`.T`, Transpose([trænsˈpoʊz], 트랜스포우즈, 트랜스포즈)) | 행↔열 뒤집기(크기 변경 아님) |
| **`matmul` / `@`** | 행렬 곱 연산자 |
