# 모델 저장과 로드 (state_dict)

> 훈련으로 채워진 가중치를 디스크에 **저장**했다가 다시 **로드**해 재사용한다. 책 추천 방식 = '모델 전체'가 아니라 학습된 상태(가중치·편향)만 담은 **`state_dict`** 저장.

> 🗺️ [[_0단계 - 닻 올리기|0단계 허브]] ｜ ⬅️ 이전: [[08 훈련 루프 (Training Loop)|훈련 루프]] ｜ ➡️ 다음: [[10 GPU 가속과 분산 훈련 (DDP)|GPU 가속과 분산 훈련]]

> 🎯 **핵심 (TL;DR)**
> - **저장:** `torch.save(model.state_dict(), "model.pth")` — *상태(가중치)만* 직렬화.
> - **`state_dict`** = 층 ↔ 훈련 가능 파라미터(가중치·편향)를 매핑한 파이썬 딕셔너리.
> - **로드 2단계:** ① `model = NeuralNetwork(2, 2)` (구조=코드 먼저) → ② `model.load_state_dict(torch.load("model.pth"))` (상태 주입).
> - **구조 일치 필수:** 로드할 모델 구조가 저장된 원본과 **정확히 같아야** 한다(키 매핑이라).
> - 확장자 `.pth`/`.pt` 관행(자유 지정 가능).

> [!note] 🗺️ 이 노트 읽는 법
> 책 A.8을 *척추*로 한다. [[08 훈련 루프 (Training Loop)|A.7]]에서 가중치를 *채웠고*, 여기선 그 **채워진 상태를 디스크에 영속화(persist([pərˈsɪst], 퍼시스트, 퍼시스트))** 한다. 짧은 장이지만, "구조(코드) vs 상태(데이터)의 분리"라는 큰 원리가 들어 있다.

> [!question] 🎯 왜 이걸 배우나 — 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제(구체적 목표):* 08에서 어렵게 가중치를 학습시켰는데, 프로그램을 끄면 메모리의 그 값이 전부 날아간다. 매번 처음부터 다시 학습하는 건 엄청난 낭비다.
> 1. 끄면 사라지는 메모리 위 가중치 → 디스크에 영속화할 방법 필요 → **`torch.save`(직렬화)**
> 2. 모델 객체 통째 저장하면 클래스 코드 경로에 묶여 깨지기 쉬움 → 코드와 분리된 순수 상태만 저장 필요 → **`state_dict`(텐서 딕셔너리)**
> 3. 저장한 상태를 다시 쓰려면 담을 그릇이 있어야 함 → 같은 구조의 빈 모델을 코드로 먼저 만들 필요 → **`NeuralNetwork(2,2)` 재생성 후 `load_state_dict`**
> **한 줄:** 끄면 날아가는 가중치 → 디스크에 저장(`save`) → 코드와 분리해 상태만(`state_dict`) → 빈 구조에 다시 주입(`load_state_dict`).

> [!abstract] 📘 책 핵심 — 부록 A.8 모델 저장과 로드
> **① 저장 (추천 방식)**
> ```python
> torch.save(model.state_dict(), "model.pth")
> ```
> - **`state_dict`** = 모델의 각 층과 *훈련 가능한 파라미터(가중치·편향)* 를 매핑한 **파이썬 딕셔너리** 객체.
> - `"model.pth"` = 디스크 저장용 파일 이름(자유 지정). **`.pth`·`.pt`** 가 가장 널리 쓰이는 확장자.
>
> **② 로드**
> ```python
> model = NeuralNetwork(2, 2)
> model.load_state_dict(torch.load("model.pth"))
> ```
> - **`torch.load("model.pth")`** = 파일을 읽어 파라미터를 담은 딕셔너리 객체를 *재구성*.
> - **`model.load_state_dict(...)`** = 그 파라미터를 모델에 *적용*해 저장했던 **학습된 상태를 복원**.
> - 같은 세션이라면 `model = NeuralNetwork(2, 2)` 재생성은 불필요. 단 **파라미터를 적용할 모델 객체는 필요**하므로 보여 주기 위해 넣음.
> - ⚠️ **`NeuralNetwork(2, 2)` 구조가 저장된 원본 모델과 정확히 일치해야** 한다.

> 💭 **여기서 내 사고가 분기됨 ↓**

## 🧠 근본 이해 · 융합 · 통찰

### 🧩 구조(코드) vs 상태(데이터)의 분리 — 이게 핵심
`state_dict`만 저장한다는 건, **모델을 '구조'와 '상태' 두 조각으로 쪼개 본다**는 뜻이다.

| 조각 | 정체 | 어디에 사는가 | 저장 대상? |
|---|---|---|---|
| **구조(Architecture([ˈɑːrkɪtɛktʃər], 아키텍처, 아키텍처))** | `class NeuralNetwork` 코드 — 층 배치·연결 | `.py` 소스 코드 | ✕ (코드로 이미 존재) |
| **상태(State([steɪt], 스테이트, 스테이트))** | 학습으로 채워진 752개 가중치·편향 숫자 | 메모리(훈련) → `.pth`(저장) | ✅ `state_dict` |

→ 로드 = **빈 구조(코드로 재생성)에 + 상태(`.pth`)를 주입**. [[06 가중치 — 상태·초기화·학습 실패|"가중치=학습된 상태"]]라 했던 그 *상태*가 바로 여기서 직렬화되는 대상. [[08 훈련 루프 (Training Loop)|훈련 루프]]가 수백 번 덮어쓴 가중치 원본 메모리 = 이 `state_dict`.

### 🔑 왜 `torch.save(model)` 전체가 아니라 `state_dict`인가
`torch.save(model)`로 모델 객체 통째 저장도 *가능은* 하지만, 책이 `state_dict`를 추천하는 데는 이유가 있다.

- **전체 저장** = 파이썬 **pickle([ˈpɪkəl], 피클, 피클)**로 객체를 통째 직렬화 → *클래스 정의(코드 경로·구조)에 의존*. 나중에 코드를 리팩토링·파일 이동하면 로드가 깨진다(이식성↓).
- **`state_dict` 저장** = 순수 **텐서 딕셔너리**(키=층 이름, 값=텐서)만 저장 → 코드와 분리돼 이식성·호환성↑. 그래서 *표준 권장*.

> [!example] 🧭 사이드 · 비유 — `state_dict` = '세이브 파일'   `#사이드/비유`
> 게임으로 치면 **구조(코드)=게임 프로그램 자체, `state_dict`=세이브 파일**. 세이브 파일엔 캐릭터 레벨·아이템(=학습된 가중치)만 들었지 게임 엔진은 안 들었다. 그래서 게임(코드)을 깔아 둔 사람만 세이브를 불러올 수 있고(구조 일치 필요), 세이브 파일은 가볍고 이식성 좋다.

> [!quote] 🌉 타 영역 연결 — 직렬화(Serialization([ˌsɪəriəlaɪˈzeɪʃən], 시리얼라이제이션, 시리얼라이제이션))   `#연결/CS`
> 저장/로드 = CS의 **직렬화 ↔ 역직렬화**(메모리 객체 ↔ 디스크 바이트열) 그 자체. `state_dict`(텐서 딕셔너리)만 저장하는 건 **데이터·코드 분리** 원칙 — JSON([ˈdʒeɪsɒn], 제이슨, 제이슨)으로 *데이터*만 주고받지 실행 코드를 통째 보내지 않는 것과 같은 발상. pickle 전체 저장의 위험(임의 코드 실행)은 `weights_only=True`로 차단(아래 🪂). 전역 다리 → [[_지식 연결 허브]]

> [!tip] 🪂 사이드 · 심화 — 키 매핑이라 '구조 일치'가 필수   `#사이드/심화`
> `state_dict` 키는 `layers.0.weight`·`layers.0.bias`·`layers.2.weight`… 처럼 **층 이름→텐서** 매핑이다(ReLU는 파라미터가 없어 인덱스 1·3은 빠지고 Linear인 0·2·4만 등장). 로드 시 이 키(key([kiː], 키, 키))가 모델 구조와 안 맞으면 `Missing/Unexpected key` 에러(Missing([ˈmɪsɪŋ], 미싱, 미싱)·Unexpected([ˌʌnɪkˈspɛktɪd], 언익스펙티드, 언익스펙티드)). 그래서 "구조가 정확히 일치해야"가 *인터페이스 계약*처럼 작동한다. (부분 로드·전이학습 땐 `strict=False`로 일부만 채우기도)

## 🧩 코드 실습 (A.8)
> 🎯 목표: A.7에서 학습한 모델을 `state_dict`로 저장 → 새 모델에 로드 → **저장 전후 예측이 같은지** 확인.
> 코드: `~/LOCAL/03-00_STUDIES/AI/00_setup/a8_save_load.py` (검증: 저장 전후 예측 `[0,0,0,1,1]` 일치, 파라미터 752개)

```python
# 저장 — 상태(가중치)만 직렬화
torch.save(model.state_dict(), "model.pth")

# 로드 — ① 동일 구조의 빈 모델 ② 상태 주입
loaded = NeuralNetwork(2, 2)                  # 구조가 원본과 정확히 일치해야
loaded.load_state_dict(torch.load("model.pth"))
```

관찰 포인트:
- **`state_dict` 키 6개** = Linear 3층 × (weight + bias). ReLU는 파라미터가 없어 빠짐.
- **저장 전후 예측 동일** = 상태(가중치)가 완벽히 복원됨을 증명.
- **구조 불일치면 로드 실패** — `NeuralNetwork(2, 2)`를 다른 크기로 만들면 키 에러.

### 🍎 Apple([ˈæpəl], 애플, 애플) Silicon([ˈsɪlɪkən], 실리컨, 실리콘)(MPS([ɛm piː ɛs], 엠피에스, 엠피에스)) 보강 — `map_location`과 `weights_only`
> 내 장비(MPS)·실전에서 꼭 챙길 두 인자. 책 예제엔 없지만 디바이스를 넘나들면 필수.

```python
state = torch.load("model.pth", map_location=device, weights_only=True)
loaded.load_state_dict(state)
loaded.to(device)
```

- **`map_location=device`** — GPU(또는 다른 머신)에서 저장한 텐서를 *현재 디바이스*(cpu/mps)로 안전하게 재배치. 안 주면 "저장 당시 디바이스"를 찾다 에러 날 수 있다. (예: NVIDIA([ɛnˈvɪdiə], 엔비디아, 엔비디아) `cuda`에서 저장 → 내 맥 `mps`/`cpu`로 로드)
- **`weights_only=True`** — `torch.load`는 pickle 기반이라 *임의 객체(코드)* 를 역직렬화할 수 있어 **신뢰 못 할 `.pth`는 보안 위험**. 순수 텐서만 로드하도록 제한(최신 PyTorch([ˈpaɪtɔːrtʃ], 파이토치, 파이토치) 권장 기본값). 원리 → [[메모리·캐시·멀티프로세스 (시스템 원리)|CS 시스템]]

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **`state_dict`** | 층 ↔ 훈련 가능 파라미터(가중치·편향)를 매핑한 파이썬 딕셔너리 |
| **`torch.save`** | 객체(보통 `state_dict`)를 디스크 파일로 직렬화 |
| **`torch.load`** | 저장 파일을 읽어 객체(딕셔너리)로 역직렬화 |
| **`load_state_dict`** | 그 파라미터를 모델에 적용해 학습 상태 복원 |
| **직렬화 / 역직렬화** | 메모리 객체 ↔ 디스크 바이트열 변환 |
| **`map_location`** | 저장된 텐서를 로드 시 현재 디바이스(cpu/mps/cuda)로 재배치 |
| **`weights_only`** | pickle 임의코드 위험 차단, 순수 텐서만 로드(권장) |
| **`.pth` / `.pt`** | PyTorch 모델 저장 파일 관행 확장자 |
