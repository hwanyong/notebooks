# 0단계. 닻 올리기 — 데이터 연산의 공용어 & 딥러닝 인프라의 본질

> **목표:** 수학 공부 전에, 전통적 개발자 관점에서 다차원 데이터를 다루는 가속 엔진의 로우레벨 구조를 파악. 중첩 for-loop 연산이 **계산 그래프 변환**을 통해 대규모 병렬 가속 연산으로 치환되는 코드 제어 감각 익히기.

> [!abstract] 📘 책 개요 — 부록 A.1 (오리엔테이션, 책이 실제 가르치는 것)
> - **파이토치 = 오픈소스 파이썬 딥러닝 라이브러리.** 3대 핵심 요소: ① **텐서 라이브러리**(NumPy 확장 + GPU 가속), ② **autograd**(자동 미분 엔진 → 역전파 자동), ③ **딥러닝 라이브러리**(모델 구성 모듈 제공).
> - **AI ⊃ 머신러닝 ⊃ 딥러닝.** ML = 명시적 프로그래밍 대신 *데이터에서 학습*. DL = 여러 **은닉층**의 심층 신경망 → 이미지·오디오·텍스트 같은 **비정형 데이터**에 강함 → LLM에 적합.

## 💭 0단계 소감 / 회고
> 0단계를 마치며 든 생각. 복습 시 "내가 왜 이걸 배웠나"를 빠르게 되살리는 닻.

- 파이토치 사용법(문법)을 익힌 것보다 더 큰 수확: **파이토치가 LLM 내부를 어떻게 관리·구조화하는지** 깨달은 것.
- 핵심 깨달음: **파이토치는 단순 수학 계산기가 아니라, 거대한 다차원 데이터 연산의 흐름과 상태(State)를 실시간으로 추적·구조화해 주는 "동적 아키텍처 인프라"** 다.
- 그래서 "왜 파이토치를 쓰는가"를 이해함 → GPT·Claude 같은 거대 모델도 결국 이 **계산 그래프 노드·화살표가 수천억 개로 확장된 것**일 뿐. 프레임워크가 이 메모리 상태·역산 흐름을 체계화해 주지 않으면 아무리 좋은 GPU가 있어도 제어 불가.
- 한 줄 결론: **"파이토치 = 동적 아키텍처 인프라."** (개념 근거 → [[03 계산 그래프]] §5 Define-by-Run)
- 다음 큰 그림: 이 계산 그래프들이 모여 만드는 **레이어(Layer)·신경망 블록 결합 구조**로 넘어감 (A.5~).
