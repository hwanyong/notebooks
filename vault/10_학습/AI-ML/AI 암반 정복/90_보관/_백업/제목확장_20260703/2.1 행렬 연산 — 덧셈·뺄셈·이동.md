# 행렬 연산 — 덧셈·뺄셈·이동

> 🎯 **핵심 (TL;DR)**
> - 행렬 수학 연산 = **직관적(원소별 처리) vs 비직관적** 두 범주. 덧셈·뺄셈은 직관 쪽.
> - **행렬 덧셈/뺄셈 = 대응 원소끼리.** **크기가 같은 두 행렬 사이에서만** 성립. 📌
> - **λ+A 는 수학적으로 불가** — 파이썬 `3+np.eye(2)`는 브로드캐스팅(편의)이지 선형대수 연산이 아님.
> - **행렬 이동(shifting([ˈʃɪftɪŋ], 시프팅, 시프팅)) = $A+\lambda I$** — 단위 행렬에 스칼라를 곱해 더함 = **대각에만 상숫값 더하기.** 정방 행렬 전용. 📌
> - **이동해도 대각 원소만 변하고 나머지는 그대로** — 그래서 정보를 보존하며 수치 안정성을 높인다(적은 양만 이동).
> - 응용 예고: **고윳값 찾기 메커니즘 · 모델 적합 시 정규화** `(🔭 예고)`.

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 행렬(데이터 테이블·변환)을 여러 개 다루면 결합하고, 조금씩 '보정'해야 한다.
> 1. 행렬 두 개를 합치고 차이를 재야 함 → 원소별 결합 연산 필요 → **덧셈/뺄셈 (같은 크기만)**
> 2. 행렬 전체에 상수를 살짝 더해 보정하고 싶음 → 그런데 **λ+A 는 수학적으로 불가**(벡터의 λ+v 불가와 같은 문제, [[1.3 벡터 연산 — 덧셈·뺄셈·스케일링·전치·브로드캐스팅|1.3 §1.2.4]])
> 3. 행렬 세계의 "1"인 단위 행렬($I$, [[2.3 역행렬·연립방정식 — 특수 행렬·AX=B 풀이|2.3]])을 지렛대로 우회 발명 → **이동 $A+\lambda I$ (대각에만 얹기)**
> 4. 얼마나 이동? 행렬 밖 임의 숫자는 기준이 없음 → **행렬 자체에서 유도한 값**(노름·고윳값 평균)의 일부로 λ 설정 `(🔭 예고)`
> **한 줄:** 원소별 결합(덧셈·뺄셈) → 스칼라 덧셈 불가의 우회(이동 $A+\lambda I$) → 대각만 살짝 바꿔 정보 보존 + 안정성 확보.

> [!abstract] 📘 책 핵심 — [실전선대] §4.2 도입 · §4.2.1 · §4.2.2
> **① §4.2 도입 — 연산의 두 범주**
> - 행렬 수학 연산은 **직관적**(원소별 처리로 표현, 이해 쉬움)인 것과 **비직관적**(설명 복잡, 훈련 필요)인 것으로 나뉜다. 직관적인 것부터.
>
> **② §4.2.1 덧셈과 뺄셈**
> - 두 행렬을 더할 때는 **대응되는 원소끼리** 더한다:
> $$ \begin{bmatrix}2&3&4\\1&2&4\end{bmatrix}+\begin{bmatrix}0&3&1\\-1&-4&2\end{bmatrix}=\begin{bmatrix}2&6&5\\0&-2&6\end{bmatrix} $$
> - 📌 행렬 덧셈은 **크기가 같은 두 행렬 사이에서만** 성립.
>
> **③ §4.2.2 행렬 '이동' (shifting a matrix)**
> - 벡터처럼 행렬도 $\lambda + \mathbf{A}$ 스칼라 덧셈은 **불가**. 파이썬 `3+np.eye(2)`는 브로드캐스팅 — *편리하지만 선형대수학 연산이 아니다.*
> - 선형대수에서 **정방 행렬**에 스칼라를 더하는 방식 = **행렬 이동**: 단위 행렬에 스칼라를 곱해 더함 (= 대각에 상숫값 더하기).
> $$ \mathbf{A}+\lambda \mathbf{I} $$
> - 수치 예제: $\begin{bmatrix}4&5&1\\0&1&11\\4&9&7\end{bmatrix}+6\begin{bmatrix}1&0&0\\0&1&0\\0&0&1\end{bmatrix}=\begin{bmatrix}10&5&1\\0&7&11\\4&9&13\end{bmatrix}$
> ```python
> A = np.array([ [4,5,1],[0,1,11],[4,9,7] ])
> s = 6
> A + s                  # 이동이 아님! (브로드캐스팅 — 전 원소에 더해짐)
> A + s*np.eye(len(A))   # 이동
> ```
> - **이동하면 대각 원소만 변경, 나머지는 그대로.** 데이터 과학에선 **수치적 안정성**을 높이면서 행렬의 정보를 최대한 보존하려고 **상대적으로 적은 양**을 이동한다(이유는 이후 장 — `(🔭 예고)`).
> - **얼마나 이동?** λ=6? λ=.001? — 행렬 안 숫자에 비해 '크거나 작음'이 문제 → 일반적으로 **노름 또는 고윳값 평균처럼 행렬 자체로 정의되는 값의 일부**로 λ 설정 (나중에 — `(🔭 예고)`).
> - **이동의 두 가지 주요(매우 중요한!) 응용:** ① **고윳값을 찾는 메커니즘** ② **모델을 데이터에 적합시킬 때 행렬 정규화** `(🔭 예고)`.

## 🧠 근본 이해 · 융합 · 통찰

### 🔑 이동 = "스칼라 덧셈 금지"의 합법적 우회 — I가 1이라서 가능
- 벡터에서 λ+v 가 불가했던 것([[1.3 벡터 연산 — 덧셈·뺄셈·스케일링·전치·브로드캐스팅|1.3]])과 **정확히 같은 문제**가 행렬에서 재등장한다. 벡터는 금지로 끝났지만, 행렬은 **우회로가 있다**: 행렬 세계의 "숫자 1" = 단위 행렬 $I$([[2.3 역행렬·연립방정식 — 특수 행렬·AX=B 풀이|2.3]])에 λ를 실어 더하면 *크기가 같은 행렬끼리의 합법적 덧셈*이 된다.
- 결과는 **대각선에만 λ를 얹는 것** — 비대각(행렬의 관계 정보)은 건드리지 않는다. *전체를 물들이는* 브로드캐스팅(`A+s`)과 근본적으로 다르다.

### 🔑 이미 배운 것의 이름 — 릿지의 $+\lambda I$ 가 곧 '이동'
- [[2.3 역행렬·연립방정식 — 특수 행렬·AX=B 풀이|2.3]]/렌즈 08에서 본 **릿지 정규화 $X^{\mathsf{T}}X+\lambda I$** (특이행렬을 가역으로 되살림)가 바로 이 '이동'이다 — 책이 예고한 응용 ②(정규화)를 우리는 이미 역방향으로 만난 셈.
- det=0(비가역) 행렬도 대각에 λ를 얹으면 가역이 된다 — "적은 양의 이동으로 수치 안정성 확보"의 대표 사례.

> [!tip] 🔭 예고 — 고윳값·이동량 λ 기준   `#안내`
> ① **고윳값 찾기**: 이동이 고윳값 계산의 메커니즘이 되는 이유(특성 다항식 $\det(A-\lambda I)=0$ — 실전선대 §5.5·이후 장). ② **λ 크기 기준**: 노름·고윳값 평균 등 *행렬 자체로 정의되는 값*의 일부로 설정 — 해당 장 수신 시 정복(규칙 17). 지금은 "이동=대각만 살짝, 기준은 행렬 스스로"까지.

## 🧩 코드 (§4.2.1~4.2.2)
> 🎯 목표: 원소별 덧셈(크기 제약) + `A+s`(브로드캐스팅) vs `A+s*I`(이동)의 차이를 눈으로 확인.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la4_matrices.py`

```python
import numpy as np
# §4.2.1 덧셈 — 대응 원소끼리, 같은 크기만
M1 = np.array([[2,3,4],[1,2,4]]); M2 = np.array([[0,3,1],[-1,-4,2]])
print(M1 + M2)                      # [[2 6 5],[0 -2 6]] (책 예제)
# §4.2.2 이동 — A+s 는 이동이 아님!
A = np.array([[4,5,1],[0,1,11],[4,9,7]])
s = 6
print(A + s)                        # 전 원소 +6 (브로드캐스팅 — 선대 연산 아님)
print(A + s*np.eye(len(A)))         # 대각만 +6 → [[10,5,1],[0,7,11],[4,9,13]]
```
