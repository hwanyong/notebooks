# 퍼셉트론 · 편향 · 은닉층 (다층 신경망)

> 신경망의 구성 블록 3종 — **퍼셉트론**(기본 세포) · **편향**(기준점 조절 나사) · **은닉층**(공간을 꺾어 복잡한 문제 풀기). 왜 이런 구조일 수밖에 없는지.

> 🗺️ [[_0단계 - 닻 올리기|0단계 허브]] ｜ ⬅️ 이전: [[04 자동 미분 (autograd)|자동 미분]] ｜ ➡️ 다음: [[06 가중치 — 상태·초기화·학습 실패|가중치]]

> 🎯 **핵심 (TL;DR)**
> - 퍼셉트론 = 입력×가중치 합 > 임계값이면 ON/OFF 스위치; **편향** = 기준점 오프셋
> - 은닉층 = 공간을 비틀어 XOR 같은 비선형 분리; 여러 겹 = 계층적 추상화
> - **ReLU(비선형) 없으면** 층을 쌓아도 직선 하나로 **선형 붕괴**
> - 무작정 깊으면 **과적합** — 문제 복잡도에 맞는 깊이가 정답
> - 코드: `nn.Module` + `Sequential(Linear+ReLU)` → `forward` → 로짓

> [!note] 🗺️ 이 노트 읽는 법
> ⚠️ 책 A.5가 *실제* 가르치는 건 **"`nn.Module`로 다층 퍼셉트론을 빌드 → `print(model)`·파라미터 수 확인 → forward로 로짓 출력 → `no_grad`·softmax"** 까지의 **코드 흐름**이다(→ 아래 📘 책 핵심·🧩 코드). 반면 퍼셉트론·편향·은닉층이 *왜 그렇게 생겼나*의 **이론은 책이 아니라 내가 파생시킨 배경지식**이라, 본문 중간 §1~§3는 그 사고 가지(💭)다.

## 📘 책 핵심 (foundation) — 부록 A.5 (다층 신경망 빌드 → 사용)
책 A.5는 **그림 A-9 같은 다층 퍼셉트론(MLP, =완전 연결 신경망)** 을 파이토치로 빌드하고, 만든 모델을 정방향으로 *사용*하는 데까지를 한 호흡으로 가르친다.

**① 빌드 — `torch.nn.Module` 상속 (코드 A-4)**
- `nn.Module`을 상속하면 `Module`이 층·연산을 캡슐화하고 **모델 파라미터를 자동 추적**해 준다.
- `__init__`에서 신경망 구조를 정의(`self.layers = nn.Sequential(nn.Linear(...), nn.ReLU(), ...)`), `forward(x)`에서 입력이 층을 통과하는 흐름을 기술 → 이 forward가 곧 **계산 그래프**로 구성된다. (역방향 `backward`는 직접 구현 불필요 → **A.7절**)
- 마지막 층 출력 = **로짓(logits)**: softmax 전의 날 점수.
- 입력 특성 수·출력 클래스 수를 변수(`num_inputs`, `num_outputs`)로 받아 **데이터셋이 달라도 재사용**.

**② 구조·파라미터 확인**
- `print(model)` → `Sequential` 안 층이 인덱스(`(0) Linear … (1) ReLU …`)로 출력. `Sequential`은 필수는 아니지만 *특정 순서로 층을 실행*할 때 편리.
- `num_params = sum(p.numel() for p in model.parameters() if p.requires_grad)` → **2213**. `requires_grad=True`인 파라미터 = 훈련 가능(훈련 중 업데이트, A.7).
- `model.layers[0].weight` → `requires_grad=True`인 파라미터 텐서, `.shape = torch.Size([30, 50])`. 편향은 `model.layers[0].bias`.
- **`nn.Linear` = 입력에 가중치 행렬을 곱하고 편향 벡터를 더하는 층** → 종종 **피드 포워드(feed forward)·완전 연결(fully connected)** 층이라 부른다.

**③ 정방향 계산(사용)**
- `X = torch.rand((1, 50))`; `out = model(X)` → `model(X)` 호출 시 정방향 계산 자동 실행. 출력 `tensor([[-0.1262, 0.1080, -0.1792]], grad_fn=<AddmmBackward0>)` — 3개 출력 노드의 점수.
- **`grad_fn=<AddmmBackward0>`** = 계산 그래프에서 마지막에 쓰인 함수. `Addmm` = 행렬곱(`mm`) 다음 덧셈(`add`) = `Wx+b`. 파이토치가 역전파 때 이 정보를 쓴다.
- **추론만 할 땐 `with torch.no_grad():`** — 그래프를 안 만들어 메모리·계산량 절약(출력에서 `grad_fn`이 사라짐).

**④ 로짓 → 확률(softmax)**
- 파이토치는 보통 **마지막 층 로짓을 활성화 함수에 안 통과시키고 그대로 출력**한다 — 손실 함수가 softmax(이진은 sigmoid)+음의 로그 가능도를 한 클래스로 합쳐 *수치적 효율·안정성*을 얻기 때문. 확률이 필요하면 **명시적으로 softmax 호출**.
- `torch.softmax(out, dim=1)` → `tensor([[0.3113, 0.3934, 0.2952]])`. 합 = 1 → 클래스 소속 확률. 아직 **학습 0회(랜덤 초기화)** 라 세 확률이 거의 비슷.

> *(아래 §1~§3 퍼셉트론·편향·은닉층의 "왜 이런 구조인가" 개념 설명은 책의 범위를 넘어, 위 코드를 이해하려고 내가 확장한 배경지식이다.)*

## 1. 퍼셉트론 (Perceptron) — 가장 단순한 의사결정 스위치
- 인간 뉴런을 본뜬 **신경망의 최소 단위(노드)**. "여러 입력을 받아 YES(1) / NO(0)를 뱉는 스마트한 스위치".
- 작동: 입력값 $x_1, x_2$ 에 각각 중요도(가중치 $w_1, w_2$)를 곱해 더하고, 그 합이 **기준치(임계값)** 를 넘으면 1, 못 넘으면 0.

```python
# 퍼셉트론 = 조건문 한 줄
if (x1*w1 + x2*w2 > threshold): return 1
else:                            return 0
```

- 이 단순 스위치를 **수백·수천 개 그물망처럼 엮은 것**이 신경망(Neural Network).

## 2. 편향 (Bias Unit) — 기준점 조절 나사
> ⚠️ **편향(偏向, bias `b`)** ≠ **편도함수(偏導, partial derivative)**. 글자만 비슷한 다른 개념 — 편도함수는 미적분(02 노트). ([[_용어집]] ⚠️표)

- 직선의 방정식 $y = wx + b$ 의 **$b$(절편)** 에 해당. 각 층 시작점에 값이 **1로 고정된 특수 노드**로 그려짐.
- 역할: 스위치가 **언제 켜질지(ON/OFF 기준점)** 를 유연하게 조절.

### 📛 "편향 XXX" 표현들은 모두 같은 bias (형태만 다름)
> 의문: *"편향 벡터·편향 함수처럼 '편향 XXX'를 만나면 모두 가중치 유무로 인식하면 되나?"* → **아니다.** 전부 **같은 편향(bias)** 을 보는 관점만 다른 것이고, 가중치와는 별개의 독립 파라미터다.

- **편향 유닛 (Bias Unit)** — *그래프 관점.* 값이 **1로 고정된 특수 노드**.
- **편향 벡터 (Bias Vector)** — *자료구조 관점.* 뉴런 30개면 편향 30개를 모은 **1차원 배열**. (가중치는 `30×50` 2D 행렬, 편향은 `[30]` 1D 벡터)
- **편향 항 / 매개변수 (Bias Term / Parameter)** — *수식·코드 관점.* `Wx + b` 의 **`b` 자체**.

→ 셋 다 **"입력과 무관하게 더해지는 기준 오프셋(baseline offset)"**. `nn.Linear` 안에 가중치와 **함께** 들어 있는, 학습되는 파라미터다.

**가중치 vs 편향 — "유무"가 아니라 역할이 다른 두 부품**

| 항목 | 가중치 (Weight) | 편향 (Bias) |
|---|---|---|
| 정체 | 입력의 **곱하기** 채널 (Multiplier) | 시스템의 **더하기** 오프셋 (Offset) |
| 의존성 | 입력 $x$ 크기에 **비례** | 입력과 **무관**, 고정값을 더함 |
| 자료구조 | 입력×출력 **2차원 행렬** | 출력 수만큼 **1차원 벡터** |
| 비유 | 센서 **감도/증폭률(Gain)** 나사 | 센서 **영점(Zero-point) 보정** 나사 |

> [!tip] 🪂 사이드 · 인지 가이드   `#사이드/심화`
> '편향 XXX'를 만나면 머릿속에서 이렇게 번역: **"입력이 0이거나 엉뚱해도 시스템이 유지할 디폴트 기준선(오프셋)을 제어하는 부품."** 한 줄: **가중치 = 관계(곱) 조절, 편향 = 기준(덧셈) 조절.**

> [!warning] ⚠️ '편향 함수(Bias Function)'는 표준 용어가 아님
> 편향은 **데이터(숫자/파라미터)** 이지 **함수(동작)** 가 아니다 — `b = -0.0817`처럼 고정된 상태값일 뿐. 어딘가에서 '편향 함수'를 봤다면 십중팔구 다음을 잘못 부른 것:
> - **활성화 함수(Activation)와 혼동(가장 흔함)** — `Linear(Wx+b)` + `Activation(ReLU/Sigmoid)`가 한 세트로 돌아가다 보니 "편향 더해주는 함수"로 뭉뚱그린 것. 공식 명칭은 **활성화 함수**.
> - **통계의 편향-분산 트레이드오프(Bias–Variance)** — 여기서 'bias'는 뉴런의 `b`가 아니라 "모델이 정답에서 얼마나 떨어졌나"의 오차 수치. 신경망 `b`와 무관한 다른 맥락.
> 🔖 인지: **가중치·편향 = 나사(파라미터/데이터), 선형·ReLU = 함수(연산).** 편향은 함수가 아니라 함수(`Linear`) **안에서 더해지는 오프셋 숫자**.

### 🔑 "절편 비유 ↔ 스위치 비유"의 괴리 해소 — 핵심은 **"0이 기준선"**
퍼셉트론은 $y = wx + b$ 를 계산만 하고 끝나는 게 아니라, 그 결과로 최종 조건문을 실행한다:
- 계산 결과 $y > 0$ → **ON(1)**, $y \le 0$ → **OFF(0)**.
- 즉 그래프에서 **$y=0$ 이 되는 지점(x축과 만나는 곳)** 이 스위치의 **전환점**.

**① 편향이 없으면 ($b=0$) — 기준점이 락(Lock)**
```python
y = w * x          # b = 0
if y > 0: return "ON"   # w>0이면 x>0 에서만 ON
```
$w$ 를 10이든 0.1이든 바꿔도 전환점은 항상 $x=0$ 에 고정 → **뻣뻣**. "x가 5를 넘을 때 켜라" 같은 요구를 만들 수 없음.

**② 편향이 있으면 ($b \ne 0$) — 기준점을 자유 이동**
```python
y = (w * x) + b
if y > 0: return "ON"
```
$w=2$ 고정, $b$ 만 바꿔보면 전환점이 이동:
- $b=-6$ → $2x-6>0$ → **$x>3$ 에서 ON**
- $b=+4$ → $2x+4>0$ → **$x>-2$ 에서 ON**

→ $b$ 가 직선을 위아래로 들어 올려/내려 **$y=0$(기준선)과 만나는 x(전환점)를 좌우로 옮긴다.** 그래서 스위치 ON/OFF **타이밍을 데이터에 맞게 자유 배치** 가능 = "유연하다".

### 왜 층마다 독립된 편향?
각 층(Layer)은 데이터에서 **서로 다른 차원의 특징을 추출하는 독립 공간**. 1층 스위치와 2층 스위치가 가져야 할 기준점이 당연히 다르므로, 모든 층에 독립된 편향이 하나씩 붙는다.

## 3. 은닉층 (Hidden Layer) — 차원의 벽을 깨려고
입력층(데이터)과 출력층(정답) 사이에 숨어 있어 **은닉층**.

- **단층의 한계:** 퍼셉트론 한 층은 직선 하나로 가르는 문제(AND, OR)만 풀 수 있음. **XOR**(대각선으로 마주 보는 데이터)처럼 조금만 복잡해도 직선 하나로는 **절대 못 나눔**.
- **은닉층의 역할:** 입력 공간을 **꼬고·비틀고·압축**해 비선형 문제를 풀 수 있게 가공. 계층적 특징 추출:
  - 은닉층1: 점·선 같은 파편적 특징
  - 은닉층2: 그 선들을 조합해 원·네모 같은 모양
  - 출력층: 모양들을 조합해 "이건 사과다" 같은 고차원 판단
- 은닉층이 많아진 구조 = **심층 신경망(Deep Learning)**.

**구조 예시** (입력 10 → 은닉 6 → 은닉 4 → 출력 3, 각 은닉층에 편향 유닛 1개):
![[신경망 구조도 10-6-4-3.png|420]]
> 엣지(연결선) = 가중치, 각 은닉층 위의 +1 노드 = 편향 유닛. 층이 깊어질수록(10→6→4→3) 차원을 압축하며 특징을 추상화.

### 왜 50→3 직접이 아니라 은닉층 여러 개? — Shallow vs Deep
> SW로 치면 *"거대한 함수(Main) 하나에 다 짤까, 서브 루틴·모듈로 단계를 쪼갤까"* 와 같은 질문.

- **Shallow (50→3 직접):** 이중 for문 한 번 + 조건문 한 번이 전부. 입력들이 *어떻게 얽혔는지* 복잡한 인과를 못 봄 → "x₁이 크면 A" 같은 1차원 선형 분리만 가능. 텍스트 문맥·이미지 인식 같은 복잡 패턴은 못 푼다.
- **Deep (은닉층 여러 개):** 데이터를 **단계별로 추상화**. 얼굴 인식 예 —
  - 입력: 순수 픽셀 값
  - 은닉1(낮은 추상화): 점을 모아 **선/엣지**
  - 은닉2(중간): 선을 모아 **눈·코 같은 부품**
  - 은닉3(높은): 부품을 모아 **얼굴 윤곽**
  - 출력: "이건 누구다" 판단
  - → 각 층이 데이터를 *"이해하기 쉬운 상위 개념"* 으로 한 단계씩 빌드업.

**SW 공학적 이점 — 재사용성·효율성:** 수만 줄 `if-else`를 한 함수에 하드코딩(Wide)하기보다, 코어 함수 몇 개를 **중첩(Nested)·재사용**(Deep)하는 게 효율적이듯, 깊은 층은 **적은 파라미터로 더 복잡한 논리**를 표현한다(1층 출력을 2층이, 2층 출력을 3층이 재사용). → 그래서 '신경망을 깊게 쌓는다' = **심층 학습(Deep Learning)**.

### 그럼 깊을수록 좋은가? — 아니다 (최적화 vs 일반화)
> 무작위로 층을 늘린다고 정확도가 오르지 않는다. 핵심은 **풀려는 문제의 복잡도에 딱 맞는 최적 추상화 레벨**(층 수·가중치 크기)을 찾는 것. = 딥러닝 설계의 영원한 숙제 **최적화 vs 일반화**.

- **① 너무 깊으면 → 과적합(Overfitting):** 데이터를 '이해'가 아니라 **통째로 암기** → 새 데이터에 약함. (SW 비유: 단순 유틸에 디자인 패턴 떡칠한 **오버엔지니어링**.)
  - 시스템 한계: 너무 깊으면 뒤에서 온 grad가 앞으로 오다 사라지는 **그레이디언트 소실**, 혹은 앞 레이어 성능이 되레 떨어지는 **열화(Degradation)** 발생. → 지름길(skip connection)을 뚫는 **ResNet** 이 이를 해결.
- **② 정형화를 너무 잘게 쪼개면 → 노이즈의 함정:** 비정형 데이터(텍스트/이미지)를 지나치게 미시적으로 쪼개면(글자 하나, 픽셀 하나) **나무만 보고 숲을 못 봄** → 연산량만 폭발하고 거시적 의미(시그널) 추론은 실패. (SW 비유: 과도한 정규화 → JOIN 지옥.)

**요약·보완:** 그래서 현대 AI 엔지니어는 무작정 깊게 쌓지 않고, **이미지=CNN, 텍스트=Transformer** 처럼 데이터 특성에 맞춰 *미리 최적화된 아키텍처*를 베이스로 삼아 튜닝한다. (→ 4단계 [AI엔지니어링])

## 3-1. 왜 층마다 비선형(ReLU)을 끼우나? — 선형 붕괴(Linear Collapse)
> 질문: 은닉층마다 `nn.Linear` 하나 쓰는데, 그 뒤에 굳이 `nn.ReLU` 같은 비선형을 또 넣는 이유?
> 답: **비선형이 없으면 층을 아무리 쌓아도 한 층짜리 직선으로 붕괴**하기 때문.

**① 수학적 이유 — 선형은 합쳐지면 또 선형**
`nn.Linear` = $W\cdot x + b$. 비선형 없이 두 층을 이으면:
$$z = W_2(W_1 x + b_1) + b_2 = (W_2 W_1)\,x + (W_2 b_1 + b_2) = W_{new}\,x + b_{new}$$
→ 30층·100층을 쌓아도 결국 **하나의 큰 행렬 곱 한 번**으로 합쳐짐(Collapse). 깊이가 무의미해진다.

**② 논리적 이유 — ReLU = 조건문(분기)**
$\text{ReLU}(x) = \max(0, x)$ — 0보다 작으면 0으로 깔고, 크면 그대로 통과. 코드로는 분기다:
```python
# nn.ReLU() 의 정체
if x > 0: return x
else:     return 0
```
선형만 있으면 공간을 **늘리거나 회전**(직선 유지)만 가능. ReLU라는 **분기(if)** 가 끼는 순간 공간을 **접고·구부리고·꺾을** 수 있는 능력(비선형성)이 생긴다 → §3의 "공간 비틀기"가 이걸로 실현.

**요약:** `Linear` = 재료(차원) 변형, `ReLU` = 분기로 **표현력** 부여. 활성화 함수가 층 사이에 징검다리처럼 끼어야 직선 경계를 넘어 복잡한 데이터를 분류한다.

> [!tip] 🪂 사이드 · 심화 — 프로그래밍 분기와의 대응   `#사이드/심화`
> 변수 선언·연산(선형)만 반복하던 코드에 **조건문·분기(비선형)** 가 들어가야 비로소 복잡한 비즈니스 로직이 되듯, 신경망도 선형층 사이에 비선형이 끼어야 단순 직선을 넘는 로직을 표현한다.

> [!example] 🧭 사이드 · 비유   `#사이드/비유`
> - **퍼셉트론 = 스마트 스위치** (입력 가중합이 기준 넘으면 ON)
> - **편향 = 기준점 조절 나사 / 약초의 감초** (모든 층에 하나씩)
> - **은닉층 = 추상화 단계** (복잡한 로직을 함수 하나에 안 짜고 여러 단계 클래스로 나누듯, 특징을 단계별 추출)

> 복잡한 로직을 함수 하나에 다 짜지 않고 추상화 레이어(Interface/Class)를 여러 단계 두어 복잡성을 다루듯, 딥러닝은 은닉층을 깊게(Deep) 쌓아 고차원 특징을 단계별로 추출한다. → "레이어를 쌓는다"가 곧 추상화 계층화.

## 🧩 코드 구현 — 2개 은닉층 MLP (A.5)
> 코드: `~/LOCAL/03-00_STUDIES/AI/00_setup/a5_neural_network.py`

`nn.Module`을 상속해 **층 정의(`__init__`) + 순전파(`forward`)** 두 메서드로 신경망을 만든다.

```python
import torch.nn as nn

class NeuralNetwork(nn.Module):
    def __init__(self, num_inputs, num_outputs):
        super().__init__()
        self.layers = nn.Sequential(
            nn.Linear(num_inputs, 30), nn.ReLU(),   # 은닉층1
            nn.Linear(30, 20),         nn.ReLU(),   # 은닉층2
            nn.Linear(20, num_outputs),             # 출력층
        )
    def forward(self, x):
        return self.layers(x)        # 마지막 층 출력 = 로짓(logits)
```

핵심 부품:
- **`nn.Module`** — 모든 신경망의 부모. `__init__`(층 정의) + `forward`(순전파) 구조.
- **`nn.Sequential`** — 층을 순서대로 쌓아 한 묶음으로 실행.
- **`nn.Linear(in, out)`** — 완전연결층 = `W·x + b`. 앞서 본 **가중치·편향이 여기 자동으로 들어 있음**.
- **`nn.ReLU()`** — 은닉층 사이 **비선형 함수**. 이게 있어야 §3의 "공간을 꺾어" XOR 같은 문제를 푼다. (없으면 Linear만 쌓아도 결국 하나의 직선)
- **`logits`** — softmax 전 마지막 층의 날 출력값.
- **`num_inputs`·`num_outputs` 변수화** — §퍼셉트론의 "재사용" 정신. 데이터셋이 달라도 그대로.

확인 출력(입력 특성 50, 클래스 3 예시):
```
학습 가능 파라미터 수: 2,213      # 50*30+30 + 30*20+20 + 20*3+3
입력 shape : (4, 50)             # 배치 4 × 특성 50
출력 shape : (4, 3)              # 배치 4 × 클래스 3
```

> [!tip] 🪂 사이드 · 심화 — 층 크기 규칙   `#사이드/심화`
> `nn.Linear`를 이어 쌓을 때 **앞 층의 출력 수 = 다음 층의 입력 수**가 맞아야 한다(30→30, 20→20). 안 맞으면 행렬 곱 차원 에러. 이 "차원 맞물림"은 1단계 선형대수(행렬 곱셈 유효성)에서 수학적으로 다시 만난다.

## 🧩 코드 구현 — 모델 확인 & forward (A.5)
> 코드: `~/LOCAL/03-00_STUDIES/AI/00_setup/a5_model_usage.py`

**🎯 이 예제의 목표:** 학습 *전* 모델에 데이터 1건을 넣어 **출력이 어떤 형태로(3개 점수) 나오고, 그게 무슨 뜻인지(로짓→확률)** 확인하기. 개별 숫자값보다 **shape의 흐름**을 보는 게 핵심.
> 💬 메타: 예제는 *목표를 먼저* 알면 훨씬 쉽다 → 이 코스 실습 노트는 맨 위에 🎯 목표를 적는다.

```python
torch.manual_seed(123)
model = NeuralNetwork(50, 3)
model.layers[0].weight.shape   # torch.Size([30, 50])  ← 가중치 = 2D 행렬(출력30×입력50)
model.layers[0].bias.shape     # torch.Size([30])      ← 편향 = 1D 벡터(§2 편향 벡터 그대로!)

X = torch.rand((1, 50))
out = model(X)                 # grad_fn=<AddmmBackward0>
torch.softmax(out, dim=1)      # 로짓 → 클래스 확률(합 1)
```

관찰 포인트:
- **`weight` [30,50] vs `bias` [30]** — §2에서 말한 *가중치=2D 행렬, 편향=1D 벡터* 가 코드로 그대로 확인됨.
- **`grad_fn=<AddmmBackward0>`** — `Addmm` = 행렬곱(`mm`) 후 덧셈(`add`) = `Wx+b`. (→ [[03 계산 그래프|02]] grad_fn)
- **`torch.no_grad()`** — 추론 시 그래프 미생성(grad_fn 사라짐) → 메모리·속도 절약. (→ [[06 가중치 — 상태·초기화·학습 실패|05]])
- **`softmax(dim=1)`** — 로짓을 합이 1인 확률로 변환. 랜덤 초기화라 세 클래스가 비슷(≈0.33씩).

### 값이 어떻게 산출되나 — 차원 흐름(Data Flow)으로 읽기
> 매개변수(가중치)와 출력값의 상관관계가 막막하면, 곱셈을 일일이 따지지 말고 **텐서 shape 변화**만 추적한다.

```
X [1,50] → Linear(50,30) → [1,30] → ReLU → Linear(30,20) → [1,20] → ReLU → Linear(20,3) → [1,3]
```
- 입력 = **50개 특성 1건**. 각 `Linear`가 `Wx+b`로 차원을 변환·압축(50→30→20→3).
- 최종 `[1,3]` = 세 클래스 각각의 **로짓(raw score)**. 예: `[-0.1262, 0.1080, -0.1792]` → 1번 점수가 최고 = "얘가 정답일 듯".
- **로짓 → 확률(softmax):** 로짓은 음수·범위 제각각이라 "몇 % 확신"으로 못 씀. softmax가 ① 0~1 양수로, ② 합=1로 정규화 → `[0.3113, 0.3934, 0.2952]`(31/39/30%).
- **왜 다 ~33%로 비슷?** 아직 **학습(역전파) 0회**인 랜덤 가중치라 정답 방향성이 없음(찍기 수준). `loss.backward()`로 학습하면 정답 클래스가 `0.99`처럼 뾰족해진다. (→ [[T01 경사 하강법과 훈련 루프|03]])

### `Linear`의 정체 = 이중 for문 (왜 가중치가 `[30,50]`?)
> `[1,50]`이 `[1,30]`으로 바뀌는 게 막막한 건 **행렬 곱(matmul)** 의 내부 동작을 안 봤기 때문. 수학 기호 다 버리고 **이중 for문**으로 번역하면 정체가 보인다.

`Linear(50, 30)` 이 하는 일 = "입력 50개를 조합해 출력 30개 만들기":
```python
for i in range(30):            # 출력 노드 30개를 하나씩
    total = 0
    for j in range(50):        # 입력 50개를 전부 돌며 가중치 곱해 더함
        total += x[j] * W[i][j]
    out[i] = total + b[i]      # 마지막에 편향 더하기
```

**그래서 왜 `W`가 `[30,50]`?**
- 입력 50개 *전부* 가 출력 30개 *전부* 와 **All-to-All(완전연결)** 로 이어져야 함 → 나사 = 30 × 50 = **1,500개**.
- 이 1,500개를 깔끔히 관리하려고 **2차원 그리드 `[30,50]`** 으로 묶은 것. (편향은 출력당 1개 = `[30]`)
- shape 계약: `[1,50]·[50,30] → [1,30]` — 가운데 같은 수(50)가 맞물려 사라지고 양끝만 남음. (내부적으로 `W[30,50]`을 전치해 `[50,30]`으로 곱함)

> [!tip] 🪂 사이드 · 심화 — 행렬 곱 = 하드웨어 가속 인터페이스   `#사이드/심화`
> 이중 for문을 순차로 돌면 느리다. 그래서 GPU/CPU의 **SIMD(한 명령으로 병렬 처리)** 를 쓰려고 그 반복을 **'행렬 곱'이라는 포맷**으로 표현하는 것. = 0단계 "중첩 for문 → 병렬 가속 연산 치환"의 실체. 수학적 원리는 1단계 [실전선대] 4장(행렬 곱셈).

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **퍼셉트론** (Perceptron) | 입력×가중치 합이 임계값을 넘으면 1/아니면 0을 내는 기본 신경 세포 |
| **임계값** (Threshold) | 스위치가 켜지는 기준치. 이항하면 편향과 동전의 양면 |
| **편향** (Bias) | 전환점($y=0$)을 좌우로 옮겨 ON/OFF 타이밍을 조절하는 오프셋 |
| **은닉층** (Hidden Layer) | 입력·출력 사이에서 공간을 비틀어 비선형 특징을 추출하는 층 |
| **선형 분리 / XOR 문제** | 직선 하나로 나눌 수 있나 여부. XOR은 단층으로 불가 → 은닉층 필요 |
| **심층 신경망** (Deep Learning) | 은닉층이 여러 겹 쌓인 신경망 |
| **Shallow vs Deep** | 50→3 직접(단층, 선형 분리만) vs 은닉층 여러 겹(계층적 추상화·재사용) |
| **과적합** (Overfitting) | 너무 깊은/큰 모델이 데이터를 이해 대신 통째 암기 → 새 데이터에 약함 |
| **일반화** (Generalization) | 학습 안 한 새 데이터에도 잘 맞추는 능력. 과적합의 반대 목표 |
| **열화 문제 / ResNet** (Degradation) | 너무 깊으면 앞 레이어 성능이 되레 하락 → skip connection(ResNet)으로 해결 |
| **활성화 함수 / ReLU** | 층 사이 비선형 함수. ReLU = `max(0,x)` (조건문 분기) |
| **선형 붕괴** (Linear Collapse) | 비선형 없이 선형층만 쌓으면 한 층짜리 직선으로 합쳐지는 현상 |
| **softmax** | 로짓(날 점수)을 합이 1인 클래스 확률로 변환 |
| **Addmm** | 행렬곱(mm) 후 덧셈(add) = `Wx+b`. `nn.Linear`의 grad_fn |
| **행렬 곱 (matmul)** | 이중 for문(입력×가중치 합)을 병렬 처리용으로 묶은 연산. `[1,50]·[50,30]→[1,30]` |
| **완전연결 (All-to-All)** | 입력 전부가 출력 전부와 연결 → 나사 수 = 입력×출력 (그래서 `W`가 2D 행렬) |
