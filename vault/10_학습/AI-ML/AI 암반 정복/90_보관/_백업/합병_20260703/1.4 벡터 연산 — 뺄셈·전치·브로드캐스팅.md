# 벡터 연산 — 뺄셈·전치·브로드캐스팅

> 📐 **수학 개념원자(SSoT)** — 선형대수. 정의·성질·기하 직관·코드. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[04 벡터 연산 — 뺄셈·전치·브로드캐스팅|04(AI)]]

> 🎯 **핵심 (TL;DR([tiː ɛl diː ɑːr], 티엘디알, 티엘디알))**
> - **덧셈/뺄셈** = 원소별, **같은 차원에서만**. 뺄셈 기하 = *두 머리를 잇는 선*(둘째 머리→첫째 머리).
> - ⚠️ **방향이 다르면 행렬이 된다:** 열벡터+행벡터 → **브로드캐스팅**으로 (3,3) 행렬. "방향이 결과 차원을 바꾼다".
> - **전치(transpose)** = 행↔열 뒤집기. `vᵀᵀ=v`. 데이터행렬·자기전치 = **공분산 행렬**(PCA의 뿌리).
> - **스칼라-벡터 덧셈** = 수학적으론 불가(별도 객체), 파이썬은 **브로드캐스팅**으로 허용.
> - **벡터 평균** = 다 더하고 `1/N` 곱(스케일링). "데이터 조작 = 벡터 조작".
> - **영벡터·자명한 해**: 모든 원소 0; 선대 문제는 보통 "0 아닌 해(비자명)"를 찾는다.

> [!note] 🗺️ 이 노트 읽는 법
> 03(필수수학)의 덧셈·스케일링·스팬을 봤으니, [실전선대] §1.2는 *전치·브로드캐스팅·뺄셈 기하* 같은 **코드·데이터 실전 연산**을 더한다. 특히 **브로드캐스팅**은 현대 컴퓨터 선대만의 연산(전통 교과서엔 없음).

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제(구체적 목표):* 데이터(벡터)를 다루다 보면 "두 데이터의 차이"를 재고, 모양(shape)이 안 맞는 배열을 곱하고, 평균을 내야 한다.
> 1. 두 데이터(벡터)가 얼마나·어느 쪽으로 다른지 기하로 보고 싶음 → 차이 연산 필요 → **뺄셈(두 머리를 잇는 선)**
> 2. 내적·노름($v^\mathsf{T}v$)을 행렬곱으로 쓰려는데 행/열이 안 맞음 → 축을 눕히는 연산 필요 → **전치(transpose)**
> 3. 모양 다른 배열을 매번 루프로 맞추기 번거로움 → 자동 확장 연산 필요 → **브로드캐스팅**(단 방향 틀리면 행렬이 튀어나옴)
> 4. N개 데이터의 중심을 구해야 함 → 합산 + 1/N 스케일 필요 → **벡터 평균(=데이터 평균)**
> **한 줄:** 차이를 보는 뺄셈 → 축을 맞추는 전치 → 모양을 자동 확장하는 브로드캐스팅 → 중심을 구하는 평균으로, "데이터 조작 = 벡터 조작".

> [!abstract] 📘 책 핵심 — [실전선대] §1.2 벡터 연산
> **① §1.2.1 덧셈/뺄셈** — 원소별, *같은 차원에서만* (ℝ([ɑːr], 알, 알)³+ℝ⁵ 불가).
> $$ \begin{bmatrix}4\\5\\6\end{bmatrix}+\begin{bmatrix}10\\20\\30\end{bmatrix}=\begin{bmatrix}14\\25\\36\end{bmatrix},\quad \begin{bmatrix}4\\5\\6\end{bmatrix}-\begin{bmatrix}10\\20\\30\end{bmatrix}=\begin{bmatrix}-6\\-15\\-24\end{bmatrix} $$
> - **방향이 중요(식 1-4):** 열벡터 + 행벡터는? → **브로드캐스팅**으로 *행렬*이 나온다. "차원·방향이 같을 때만 (벡터로) 더해진다."
>
> **② §1.2.2 덧셈/뺄셈의 기하** — 덧셈=머리-꼬리(그림 1-2 A). **뺄셈=두 머리를 잇는 선**(둘째 머리→첫째 머리, 그림 1-2 B). *뺄셈은 직교벡터 분해·선형 최소제곱(회귀)의 기초.*
>
> **③ §1.2.3 스칼라-벡터 곱** — 각 원소에 스칼라 λ([ˈlæmdə], 램더, 람다) 곱. 그리스 소문자(α([ˈælfə], 앨퍼, 알파), λ). $\lambda=4,\ \lambda w=[36,16,4]$. 방향 안 바뀌고 크기만(음수=180° 반전, 단 같은 선상 → 그림 1-3).
>

> 🔄 재구성 [그림 1-3] — 같은 벡터(검정)에 스칼라 σ([ˈsɪɡmə], 시그머, 시그마)를 곱함. σ>1 늘림·0<σ<1 줄임·σ<0 방향 반전(같은 선상). 크기만 변하고 방향(선)은 유지.

> > ![[LA1-3 스칼라-벡터 곱 (책원본).png|620]]
> > [!quote]- 📐 영벡터(zero([ˈzɪroʊ], 지로우, 제로) vector([ˈvɛktər], 벡터, 벡터)) — 책 보충
> > 모든 원소 0(굵은 **0**). 영벡터로 푸는 건 **자명한 해(trivial([ˈtrɪviəl], 트리비얼, 트리비얼))** — 제외. 선대는 보통 "0 아닌(비자명) 해를 찾아라".
>
> **④ §1.2.4 스칼라-벡터 덧셈** — *수학적으론 불가*(벡터와 스칼라는 별도 객체). 파이썬은 **브로드캐스팅**으로 각 원소에 더함: `2 + [3,6] = [5,8]`.
> - **벡터 평균** = 덧셈 + 스칼라곱(1/N): N개 벡터 다 더하고 `1/N` 곱.
>
> **⑤ §1.2.5 전치(transpose)** — 행↔열 인덱스 맞바꿈 $m^\mathsf{T}_{i,j}=m_{j,i}$. 열벡터↔행벡터. **`vᵀᵀ=v`**(두 번 전치=원래). *데이터 행렬에 자기 전치를 곱하면 **대칭 공분산 행렬*** → PCA([piː siː eɪ], 피시에이, 피시에이)가 데이터 공간의 직교 회전인 이유.
>
> **⑥ §1.2.6 브로드캐스팅** — 현대 컴퓨터 선대만의 연산(전통 교과서엔 없음). 한 벡터를 다른 벡터의 각 원소로 반복 연산. 행벡터+열벡터 → 행렬(외적 비슷). 효율·간결(예: k-평균).

### 🔑 전치 vᵀ — 노름(vᵀv)·공분산·PCA로 이어지는 핵심
- [[02 벡터 크기와 단위벡터 (노름)|노름]]에서 본 **vᵀv**(제곱합)가 곧 이 전치. 전치로 가로 눕혀 행렬곱 = 제곱합.
- **데이터행렬 XᵀX = 공분산 행렬**(대칭) → 주성분분석(PCA)이 *데이터 공간의 직교 회전*인 근거. 전치가 단순 뒤집기가 아니라 *통계 구조*를 만든다.
- `vᵀᵀ=v`(왕복) = 가역 연산.
- **왜 전치하나(차원 정합):** 내적을 행렬곱으로 만들려 앞 벡터를 행으로 눕힘 $(1{\times}N)(N{\times}1){=}(1{\times}1)$; $a^\mathsf{T}b{=}b^\mathsf{T}a$ (≠ $ab^\mathsf{T}$, 이건 외적 행렬) → [[05 벡터 내적 — 점곱·기하·직교 + 아다마르곱·외적|왜 전치하나·OOM 함정]].

> [!tip] 🌱 탐구 · [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|전치가 1D에서 '먹통'인 이유]]
> `.T`는 **2D `(N,1)`** 에서만 `(1,N)`로 눕는다. **1D `(N,)`** 는 `.T`해도 그대로 `(N,)`(뒤집을 축 없음) — `vᵀ` 수식이 코드에서 안 먹힘. 그래서 열벡터는 `(N,1)` 명시. NumPy 내부(1D 블록+strides=C 포인터 산술)까지 → [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]. ↳ [[_탐구 MOC]]

### 🧩 스칼라-벡터 덧셈 '불가'인데 파이썬은 됨 — 또 수학 vs 코드
수학적으론 벡터+스칼라 불가(별도 객체)인데, 파이썬은 **브로드캐스팅**으로 허용. [[03 벡터 연산 — 덧셈·스케일링·스팬·선형종속|03의 '스칼라 2 vs 1×1 행렬']]·[[01 벡터 — 정의·차원·방향 (NumPy 생성·시각화)|01의 '차원 충돌']]과 같은 **칠판 수학 ≠ 코드 수학** 테마. 브로드캐스팅은 *편의*지 수학 정의가 아님 → 의도치 않은 shape 변화 주의.

> [!quote] 🌉 타 영역 연결 — 브로드캐스팅 = '여럿에 같은 연산 살포'   `#연결/CS`
> 브로드캐스팅(작은 배열을 큰 배열 전체에 퍼뜨려 연산)은 [[GPU 하드웨어 구조 — SIMT·SM·텐서코어 (ARM 관점)|GPU SIMT(하나의 명령을 수천 코어에)]]와 같은 발상의 소프트웨어판 — *루프 없이 벡터화*. 그래서 빠르다. 전역 다리 → [[_지식 연결 허브]]

> [!tip] 🪂 사이드 · 심화 — 벡터 평균 = 데이터 평균 (스케일링의 응용)   `#사이드/심화`
> N개 벡터의 평균 = 다 더하고 `1/N` 곱. "동네 주택들의 평균 가격·면적" = 주택 벡터들을 더해 `1/N` 스케일. **데이터 평균이 곧 벡터 연산** — 통계의 평균·중심화(centering([ˈsɛntərɪŋ], 센터링, 센터링))가 선대 위에 있다. ([[03 벡터 연산 — 덧셈·스케일링·스팬·선형종속|필수수학 §4.1]]의 "데이터 조작=벡터 조작" 재확인)

## 🧩 코드 실습 (§1.2)
> 🎯 목표: 덧셈/뺄셈(차원 제약)·열+행=행렬(브로드캐스팅)·전치(vᵀᵀ=v)·스칼라곱·평균을 코드로.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/la1_2_ops.py` (검증: 열+행 shape (3,3), vᵀᵀ=v True)

```python
import numpy as np
v, w = np.array([4,5,6]), np.array([10,20,30])
v + w; v - w                                  # [14 25 36], [-6 -15 -24]
# 방향 다르면 행렬! (브로드캐스팅)
col = np.array([[4,5,6]]).T; row = np.array([[10,20,30]])
(col + row).shape                             # (3,3)
# 전치
row.T.T == row                                # vᵀᵀ=v
# 스칼라-벡터 덧셈(파이썬 broadcasting) / 평균
2 + np.array([3,6])                           # [5 8]
(np.array([2,8]) + np.array([4,2])) * (1/2)   # [3. 5.] 평균
```

관찰 포인트:
- **같은 차원만 덧셈** — 다르면 `ValueError`(또는 방향 다르면 브로드캐스팅 행렬).
- **열+행 = (3,3) 행렬** — 방향이 결과를 바꾸는 결정적 예.
- **`vᵀᵀ=v`** — 전치 왕복. 데이터행렬엔 `XᵀX`로 공분산을 만든다.
