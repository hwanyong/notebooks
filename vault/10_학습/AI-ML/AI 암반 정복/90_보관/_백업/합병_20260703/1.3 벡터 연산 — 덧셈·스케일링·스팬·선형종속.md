# 벡터 연산 — 덧셈·스케일링·스팬·선형종속

> 📐 **수학 개념원자(SSoT)** — 선형대수. 정의·성질·증명·기하 직관. AI 적용·연결은 코스 렌즈가 끌어다 쓴다.
> 🔗 [[_수학 MOC]] ｜ 🌐 [[_지식 연결 허브]] ｜ 🎓 AI 렌즈: [[03 벡터 연산 — 덧셈·스케일링·스팬·선형종속|03(AI)]]

> 🎯 **핵심 (TL;DR)**
> - **덧셈** = 두 이동을 *머리-꼬리(head-to-tail([hɛd tə teɪl], 헤드 투 테일, 헤드 투 테일))* 로 연결. 수치상 원소별 합. **교환법칙** 성립(`v+w=w+v`).
> - **스케일링** = 스칼라 곱으로 길이 늘이기/줄이기. **음수 스칼라 = 방향 반전**(같은 선상이라 '방향이 진짜 바뀐 건 아님' → 선형 종속의 씨앗).
> - **스팬(Span([spæn], 스팬, 스팬))** = 두 벡터를 *스케일링+덧셈*으로 도달 가능한 **모든 공간**.
> - **선형 독립** = 방향이 달라 스팬이 *평면 전체*(무한) / **선형 종속** = 같은 선상이라 스팬이 *직선 하나*로 갇힘.
> - 선형 종속 → 나중에 **행렬식=0**, 연립방정식 못 풂. 선형 독립이라야 해를 자유로이 구성.

> [!question] 🎯 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* 벡터들이 "공간을 얼마나 덮나(정보가 겹치나·독립인가)"를 알려면, 벡터를 *조합하는* 연산부터 있어야 한다.
> 1. 벡터 하나로는 여러 데이터를 합치거나 키울 수 없음 → 결합·길이조절 연산 필요 → **덧셈 + 스케일링**
> 2. 그 두 연산을 무한히 조합하면 "어디까지 갈 수 있나"가 궁금 → 도달 가능 영역 개념 필요 → **스팬(Span)**
> 3. 어떤 벡터는 보태도 새 방향이 안 생김(같은 선에 갇힘) → 정보가 겹치나 판별 필요 → **선형 독립 / 종속**
> 4. 종속이면 차원이 무너지고 연립방정식이 안 풀림 → 붕괴를 한 숫자로 진단할 도구 필요 → **행렬식(예고)·rank**
> **한 줄:** 벡터 조합(덧셈·스케일링) → 도달 영역(스팬) → 정보 겹침 판별(독립/종속) → 차원 붕괴 진단(행렬식)으로 이어짐.

> [!abstract] 📘 책 핵심 — [필수수학] §4.1 벡터 연산
> **① 벡터 생성 (예제 4-1~4-4)** — 복습
> - 리스트 `[3,2]` / `np.array([3,2])` / 3D `[4,1,2]` / 5D `[6,1,5,8,3]`. (정의·차원·방향은 [[01 벡터 — 정의·차원·방향 (NumPy 생성·시각화)|01 노트]])
> - 벡터 용도: 물리(방향+크기) · 수학(xy 평면 방향+스케일) · CS(숫자 배열). 데이터 과학에선 *데이터의 한 조각*.
>
> **② §4.1.1 덧셈과 결합 (예제 4-5)**
> - 두 벡터 $\vec v,\vec w$ 의 이동을 하나로 결합 = **머리-꼬리 연결**. 끝점이 결과 벡터.
> $$ \vec v+\vec w=\begin{bmatrix}3\\2\end{bmatrix}+\begin{bmatrix}2\\-1\end{bmatrix}=\begin{bmatrix}5\\1\end{bmatrix} $$
> - **교환 법칙(commutative([kəˈmjuːtətɪv], 커뮤터티브, 커뮤터티브)):** $\vec v+\vec w=\vec w+\vec v$ — 순서 무관(그림 4-6).
> - 실제 의미: 데이터 합산(주택 가격+면적 등 여러 벡터를 하나로).
> ```python
> v, w = array([3,2]), array([2,-1])
> v + w   # [5 1]
> ```
>
> **③ §4.1.2 스케일링 (예제 4-6)**
> - **스칼라**(하나의 값)를 곱해 길이를 늘이거나 줄임. 각 원소에 곱.
> $$ 2\vec v=2\begin{bmatrix}3\\1\end{bmatrix}=\begin{bmatrix}6\\2\end{bmatrix},\quad 0.5\vec v=\begin{bmatrix}1.5\\0.5\end{bmatrix} $$
> - **크기만 변하고 방향은 그대로.** 단 **음수 스칼라는 방향 반전**(그림 4-9). 하지만 여전히 *같은 선상* → 선형 종속으로 이어짐.
> - 데이터 의미: **평균** = 다 더한 뒤 `1/N` 곱(스케일링). "데이터 조작 = 벡터 조작".
>
> **④ §4.1.3 스팬과 선형 종속**
> - **스팬(Span):** 두 벡터를 *스케일링+덧셈*으로 만들 수 있는 **모든 벡터의 공간**.
> - **선형 독립(linearly independent([ˈlɪniərli ˌɪndɪˈpɛndənt], 리니얼리 인디펜던트, 리니어리 인디펜던트)):** 두 벡터 방향이 다름 → 스팬이 **평면 전체(무한)**.
> - **선형 종속(linearly dependent([ˈlɪniərli dɪˈpɛndənt], 리니얼리 디펜던트, 리니어리 디펜던트)):** 두 벡터가 같은 방향/같은 선상 → 조합해도 **그 선 위에만** → 스팬이 직선으로 제한(그림 4-11). 3D면 평면에 갇힘(그림 4-12).
> - 왜 중요? 선형 종속이면 **변수가 사라져 연립방정식 못 풂**. 선형 독립이라야 필요한 벡터를 자유로이 구성 → 해 존재. 판별 도구 = **행렬식(determinant([dɪˈtɜːrmɪnənt], 디터미넌트, 디터미넌트))**(나중).

> [!tip] 🪂 심화 — 스칼라 `2` vs 1×1 행렬 `[2]`: 결과 같고 정체 다름   `#사이드/심화`
> 벡터에 *숫자 2*를 곱하는 것과 *1×1 행렬 `[[2]]`*를 곱하는 것은 **결과는 `[6,2]`로 똑같지만 타입·메커니즘이 완전히 다르다.**
>
> | | 스칼라 `2` | 1×1 행렬 `[[2]]` |
> |---|---|---|
> | 정체 | 원시 타입(순수 크기/배율) | 2차원 배열 **객체**(공간 변환기) |
> | `ndim` / `shape` | `ndim=0` | `shape=(1,1)` |
> | 연산 | **브로드캐스팅**(모든 원소에 침투) | **행렬 곱**(가로·세로 규격 맞물림: 앞 가로1=뒤 세로1) |
>
> ```python
> col = np.array([[3],[1]])         # 2×1
> col * 2                           # 브로드캐스팅 → [[6],[2]]
> np.dot(col, np.array([[2]]))      # 행렬곱 (2×1)·(1×1) → [[6],[2]]  (같은 결과)
> np.ndim(2), np.array([[2]]).shape # 0 , (1,1)  ← 정체는 다름
> ```
> - **왜 수학은 둘을 엄격히 구분?** *행렬은 공간을 비틀·변환하는 **함수(블랙박스)***. `[2]`는 "1차원 공간을 2배로 스케일하는 변환기", 스칼라 `2`는 공간 의미 없는 순수 숫자. → 행렬=선형변환 ([[3D 변환 행렬 — 동차좌표·회전·투영 (선형대수 응용)|3D 변환]]). (= [[01 벡터 — 정의·차원·방향 (NumPy 생성·시각화)|01 노트]]의 '수학 차원 ≠ `ndim`' 충돌의 연장 — 같은 `2`도 *원시값*이냐 *구조 가진 행렬*이냐로 갈린다.)

> [!tip] 🌱 탐구 · [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|NumPy 벡터 내부 — (N,1) vs (N,)]]
> [필수수학]이 벡터를 1D `[1,2,3]`(shape `(3,)`)로 쓰는 게 *왜* 문제이고, [실전선대]가 `[[1],[2],[3]]`((3,1))로 쓰는 *이유* — `.T`·`@`·`np.linalg`가 1D에서 깨지는 메커니즘과 NumPy([ˈnʌmpaɪ], 넘파이, 넘파이) 내부(1D 블록+strides([straɪdz], 스트라이즈, 스트라이드) =C 포인터 산술). **그래서 이 노트의 코드는 열벡터를 명시적 `(N,1)`로 적는다.**

### 🔑 스팬·선형종속이 데이터 임베딩과 닿는 지점
> [!info] 📍 여기는 *입문* — 정식·심화는 [[06 벡터 확장 — 집합·선형결합·선형독립·부분공간·생성·기저|06 벡터 확장]]   `#안내`
> 이 절은 [필수수학] §4.1.3의 **입문 맛보기**(스팬·선형 독립/종속의 직관)다. **정식 정의(식 2-3 비자명해)·부분공간·기저·rank 판별**은 [실전선대] §2를 정리한 **[[06 벡터 확장 — 집합·선형결합·선형독립·부분공간·생성·기저|06]]이 정식 주인.** 개념이 처음이면 여기서 감을 잡고 06으로.

- **선형 독립 = 정보가 안 겹침.** 차원들이 독립이라야 각 차원이 *새 정보*를 담는다(종속이면 중복 차원 = 낭비).
- **선형 종속 = 차원 붕괴.** 3D 벡터가 2D 평면에 갇히는 것(그림 4-12) = 실질 차원(계수, rank([ræŋk], 랭크, 랭크))이 낮아짐 → PCA([piː siː eɪ], 피시에이, 피시에이)·차원축소의 뿌리.

🔄 **재구성** [그림 4-10] — **스팬**: 방향이 다른 두 벡터를 *스케일링+덧셈*하면 평면의 **어떤 점이든** 도달. 6가지 조합 = 평면 전체를 덮음 → **선형 독립**이면 스팬 = 평면 전체(무한).

> 스칼라 곱(스케일링)·음수 반전은 [[3D 변환 행렬 — 동차좌표·회전·투영 (선형대수 응용)|3D 변환 노트]]의 **크기 행렬(diag([ˈdaɪæɡ], 다이애그, 다이애그))·반사**의 1차원 사례. 벡터 덧셈·스케일링이 모여 만드는 *선형 결합*이 곧 신경망 `Wx+b`([[05 퍼셉트론·편향·은닉층|선형층]])의 본질 — "입력 벡터들을 가중(스케일링)해 더하기(덧셈)". 같은 두 연산이 그래픽스·AI를 관통. 전역 다리 → [[_지식 연결 허브]]

> [!tip] 🪂 사이드 · 심화 — 음수 스칼라는 '방향을 바꾼' 게 아니다   `#사이드/심화`
> 음수 곱(−v)은 화살표가 반대로 가지만 **여전히 같은 직선 위**다. 그래서 v와 −v는 *선형 종속*(한 선에 갇힘). "방향이 진짜 다르다"는 건 *다른 직선*일 때만 → 그때라야 선형 독립·스팬 확장. (책 §4.1.3의 "음수 반전은 같은 선상" 강조점)

> [!tip] 🌱 탐구 · [[1.5 벡터 공리 — 덧셈·음수·역원|왜 '+'가 방향을 만드나 — 덧셈 역원]]
> "더했는데 왜 방향이?" "음수가 왜 뒤집나"의 근본 답 — **방향은 성분 비율의 *파생값***(저장값 아님), **음수=덧셈 역원**(0으로 되돌리는 짝; (-1)v=-v는 공리적 강제). 스칼라=0차원도 정리. ↳ [[_탐구 MOC]]

## 🧩 코드 실습 (§4.1)
> 🎯 목표: 예제 4-1~4-6으로 생성·덧셈(교환법칙)·스케일링(음수 반전)을 확인.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/md4_vector_ops.py` (검증: `v+w=[5 1]`, 교환법칙 True)

> [!tip] 🧭 매핑 템플릿 — [수학] → [책 1D] → [내 2D]   `#규칙`
> 열벡터 $v=\begin{bmatrix}3\\2\end{bmatrix}$ 를:
> - **책(1D shortcut):** `np.array([3,2])` → `(2,)` *(.T 먹통)*
> - **내 표기(2D 명시):** `np.array([[3],[2]])` → `(2,1)` *(수식 1:1)* — 규칙 출처 [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]

```python
import numpy as np
# 생성: 열벡터를 명시적 (N,1)로 (책의 1D는 대조용)
v = np.array([[3], [2]]); w = np.array([[2], [-1]])   # (2,1) 열벡터
v5 = np.array([[6],[1],[5],[8],[3]])                  # (5,1)
# 덧셈 (예제 4-5) — 교환법칙
print((v + w).ravel(), np.array_equal(v+w, w+v))      # [5 1] True
# 스케일링 (예제 4-6) — 음수=방향반전
a = np.array([[3], [1]])
print((2.0*a).ravel(), (0.5*a).ravel(), (-1*a).ravel())  # [6. 2.] [1.5 0.5] [-3 -1]
# 책 1D는 전치가 먹통 → 그래서 (N,1) (→ T04)
print(np.array([3,2]).T.shape, v.T.shape)             # (2,)  (1, 2)
```

관찰 포인트:
- **덧셈 = 원소별 합**, 시각적으론 머리-꼬리. `v+w == w+v`(교환법칙).
- **스칼라 곱 = 각 원소에 곱**. 음수면 방향 반전(같은 선상).
- 이 둘(덧셈+스케일링)의 모든 조합 = **스팬**.
- **열벡터는 `(N,1)` 명시** — 1D `(2,)`는 `.T`가 먹통이라 수식과 안 맞음([[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]]).

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **벡터 덧셈** | 두 벡터를 머리-꼬리로 결합(원소별 합); 교환법칙 성립 |
| **교환 법칙** | `v+w = w+v` — 더하는 순서 무관 |
| **스케일링 / 스칼라** | 스칼라(한 값) 곱으로 길이 조절; 음수=방향 반전 |
| **스팬** (Span) | 벡터들을 스케일링+덧셈으로 도달 가능한 전체 공간 |
| **선형 독립** | 방향이 달라 스팬이 차원을 꽉 채움(정보 안 겹침) |
| **선형 종속** | 같은 선/평면에 갇혀 스팬이 줄어듦(차원 붕괴) |
| **행렬식** (예고) | 선형 종속 여부 판별 도구(=0이면 종속) |
