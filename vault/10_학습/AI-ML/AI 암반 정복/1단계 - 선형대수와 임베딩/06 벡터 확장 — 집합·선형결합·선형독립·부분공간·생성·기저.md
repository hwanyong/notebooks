> [!question] 🎯 왜 AI에서 — 필요성 사슬 (AI 앵커)
> *1단계 목표(임베딩·차원축소·PCA):* 단어·문서를 벡터로 바꿔(임베딩) 수천 개를 다루고, 의미 비슷한 걸 찾고 핵심만 압축하고 싶다.
> 1. 임베딩 = 데이터를 **수천 개 벡터**로 → 떼로 다루고 *새 특징*을 만들려면 **선형 가중 결합**(회귀·신경망 층·평균의 공통 부품).
> 2. 임베딩 차원이 **중복(종속)**이면 낭비·noise → "진짜 정보 차원이 몇 개?" → **선형 독립/rank** = 차원축소의 전제.
> 3. 같은 데이터를 설명하는 좌표계가 무한 → *최소·유일*한 축이 필요 → **기저**, 그리고 데이터에 맞는 *최적 기저 찾기* = **PCA·차원축소·특징추출·임베딩**의 본질.
> **한 줄:** 임베딩(벡터 떼) → 선형결합(신경망 층·특징 합성) → 독립성(차원축소의 전제) → 최적 기저 찾기(PCA·특징추출·표현학습). 수학 본체 ↓

## 🧠 AI 특화 연결 (이 노트 고유)

> [!quote] 🌉 선형 가중 결합 = 신경망 선형층 $Wx+b$   `#연결/AI`
> 선형 가중 결합 $\sum\lambda_i v_i$ 이 곧 신경망 선형층 $Wx{+}b$([[05 퍼셉트론·편향·은닉층|퍼셉트론]])의 본질 — *입력을 가중(λ)해 더함*. "가중치를 다르게 준다 = 변조가 아니라 비중 조절"이라는 원자의 통찰이 곧 신경망 가중치·회귀 계수의 의미이고, **신경망이 '학습'한다 = 각 입력에 줄 가중치를 데이터에서 알아낸다**는 뜻. 전역 다리 → [[_지식 연결 허브]]

> [!quote] 🌉 최적 기저 찾기 = 표현 학습 · 차원축소 (PCA·ICA·SVD·임베딩)   `#연결/AI`
> 원자에서 본 "데이터의 최적 기저 찾기"는 AI에서 **표현 학습(representation learning)** 그 자체 — 신경망이 가중치에 *좋은 기저(특징축)* 를 새기는 일. 겉보기엔 다른 ML 기법(차원축소·특징추출·**PCA·ICA·요인분석·SVD·LDA**·이미지 근사·데이터 압축·**임베딩**)이 본질은 전부 *데이터를 가장 잘 설명하는 축(기저) 고르기* 한 문제로 환원된다. → [[왜 선형대수인가 — 데이터·알고리즘의 표준 언어|왜 선형대수인가]]. 전역 다리 → [[_지식 연결 허브]]

> [!quote] 🌉 선형 독립 = '정보 비중복' = 차원축소·임베딩의 동기   `#연결/AI`
> 임베딩 차원이 **독립이라야** 각 차원이 *서로 다른 의미*를 담는다. 종속 차원 = 낭비·중복 → 실질 차원(rank)이 낮아짐 = **차원축소·PCA의 직접 동기**. 종속(공선성)은 회귀·역행렬을 깨는 원인이기도([[12 행렬 곱셈·행렬식 — 변환 합성·면적 배율·선형종속|12 det=0]]). 전역 다리 → [[_지식 연결 허브]]

> [!tip] 🌱 탐구 · 기저 = 행렬의 열   `#안내`
> 기저를 *바꾸는* 행위가 곧 행렬(연산자) — 행렬 $A$ 의 각 열 = 표준기저가 도착하는 새 위치. §2.5 "기저 = 자"와 [[T06 벡터=상태 vs 행렬=연산자 (기저 재정의·SIMD·GEMM)|T06]]의 "행렬 = 기저 재정의"가 한 쌍. (수학 본체는 원자에.)
