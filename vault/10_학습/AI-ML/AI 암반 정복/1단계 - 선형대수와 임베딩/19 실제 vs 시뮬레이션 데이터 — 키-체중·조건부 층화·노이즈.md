# 실제 vs 시뮬레이션 데이터 — 키-체중·조건부 층화·노이즈

> 📊 데이터의 두 출신: **관찰로 수집**(실제) vs **아는 함수/분포에서 생성**(시뮬레이션). 같은 키-체중 문제를 양쪽에서 실연 — 그리고 "패턴 없음"이 조건부(층화) 한 번에 선형으로 풀리는 실전 반전.

> 🗺️ [[_1단계 - 선형대수와 임베딩|1단계 허브]] ｜ ⬅️ 이전: [[18 인공지능 유형과 데이터 — 인지 AI·복잡성 4종·정형 비정형|18 AI 유형과 데이터]] ｜ ➡️ 다음: [[20 수학 모델 — 선형·비선형·시뮬레이션과 AI 3중 관계|20 수학 모델]]

> 🎯 **핵심 (TL;DR)**
> - **실제 데이터** = 실제 관찰(센서·설문·웹·주식·통제 실험)로 수집 📌 — 측정 방법·기기의 부정확성 탓에 **불완전·노이즈 포함**. 생성 함수·확률 분포를 정확히 알 수 없고 **가설만 세울 수 있다.**
> - **시뮬레이션 데이터** = **이미 아는 함수**에서 생성되거나 알려진 확률 분포에서 샘플링 📌 — 편미분 방정식 해(난류·단백질 폴딩·열 확산·행성 운동…), 영화 물리 효과(〈모아나〉 물, 〈겨울왕국〉 엘사 머리카락).
> - 실전 예시(키-체중, 캐글): 첫 데이터셋은 **패턴 없음**(그림 2-1) → 알고 보니 비만·고도 비만 과다 포함 → **Index 피처로 층화(비슷한 점수끼리)하면 선형 관계가 드러남**(그림 2-3) 📌 — "데이터 탐색이라는 이름하에 속임수 없는 공정한 게임". 둘째 데이터셋은 바로 선형(그림 2-2) + 히스토그램 쌍봉 = 남/녀 **가우스 혼합** `(🔭 예고 — 정규 분포)`.
> - 시뮬 실연: $weight = -314.5 + 7.07 \times height$ 로 5,000쌍 생성 — 노이즈 없으면 **완벽한 직선**(그림 2-4), 현실감을 원하면 **정규 분포에서 키를 샘플링 + 노이즈 항 추가**(그림 2-5) → 실제 여성 5,000명 데이터(그림 2-6)와 유사해짐. **5분 만에 실데이터 흉내** — 단 노이즈의 확률 분포도 선택해야(정규 vs 균등 `🔭`).
> - **시뮬레이션 데이터의 특징 = 데이터 자체로 함수의 기능을 수행** 📌 — 생성 함수(모델)를 이해하고 있으므로 예상 밖 결과가 거의 없다.
> - 검산: $height{=}60 \Rightarrow weight = -314.5 + 7.07{\times}60 = 109.7$ ✓.

> [!question] 🎯 왜 이걸 배우나 — 필요성 사슬 (결핍 → 필요 → 발명)
> *출발 문제:* [[18 인공지능 유형과 데이터 — 인지 AI·복잡성 4종·정형 비정형|18]]에서 "관찰 후 가상의 함수를 추정한다" 했다 — 그런데 함수를 **아는** 데이터와 **모르는** 데이터는 게임이 완전히 다르다.
> 1. 관찰 데이터는 생성 함수를 모르고 노이즈투성이 → 가설·테스트가 필요 → **실제 데이터**의 규칙.
> 2. 데이터가 없거나 비싸거나(실험실·통제 측정) 위험하면? → 아는 함수+숫자 몇 개로 **생성** → **시뮬레이션 데이터** — 함수의 숫자만 바꿔 다른 시나리오 테스트.
> 3. 실데이터에서 "패턴 없음"을 만나면? → 교란 피처(Index)를 **조건으로 층화** → 숨은 선형성 발견 — 탐색은 속임수가 아니라 정당한 절차.
> 4. 시뮬을 현실처럼 만들려면? → 입력(키)을 현실적 분포에서 샘플링 + **노이즈 항** 추가 — 노이즈의 분포 선택이라는 새 결정.
> **한 줄:** 함수를 모름(실제, 가설 필요) ↔ 함수를 앎(시뮬, 생성 가능) — 실제에선 층화로 패턴을 캐고, 시뮬에선 노이즈로 현실감을 입힌다.

> [!abstract] 📘 책 핵심 — [필수수학AI] §2.2 실제 데이터와 시뮬레이션 데이터
> - 두 유형의 차이를 아는 것이 매우 중요 — 둘 다 발견과 발전에 유용.
> - **실제 데이터**: 실제 관찰로 수집 — 측정 장치·센서, 설문/의료 진단 같은 구조화된 양식, 망원경·영상 장치, 웹 사이트, 주식 시장, 통제된 실험. 측정 방법·기기의 부정확성과 오류로 **불완전·노이즈 많음**. 수학적으로 **생성 함수·확률 분포를 정확히 알 수 없음** — 모델·이론·시뮬레이션으로 **가설**을 세우고 테스트·예측에 사용.
> - **시뮬레이션 데이터**: **이미 알고 있는 함수**에서 생성된 데이터 또는 알려진 확률 분포에서 무작위 샘플링 — 알려진 수학 함수/모델에 숫자 몇 개를 가미해 데이터 포인트 생성. 예: 자연 현상 모델링 편미분 방정식의 해(난류 예측·단백질 폴딩·열 확산·화학 반응·행성 운동·파쇄 물질 운동·교통량), 〈모아나〉 물 움직임, 〈겨울왕국〉 엘사 머리카락.
> - 이어서 키-체중으로 두 예시: ① 실제 키·체중 데이터셋 2개 탐색 ② 체중이 키에 **선형 종속**이라는 가설 기반 시뮬레이션(직선/평면 패턴 예상).

> [!abstract] 📘 책 핵심 — [필수수학AI] §2.4 실제 데이터 예시 (그림 2-1~2-3)
> - 파이썬 코드는 GitHub(옮긴이 저장소: <https://github.com/EmjayAhn/essential-mathematics-for-ai>)·캐글(Kaggle) 데이터셋 2개. [NOTE] 키·체중·성별 데이터셋은 **정형 데이터**(행=사람, 열=피처) — 오디오·게시물·이미지·동영상은 비정형.
> - 목표: **키에 따라 체중이 어떻게 달라지나** = $weight = f(height)$ 로 표현 — 새 사람의 키로 체중 예측. (성별·식습관·운동·유전도 영향을 주지만 **가진 데이터엔 키·체중·성별뿐** — 더 다양한 피처를 원하면 새 데이터셋을 찾거나 수집해야.)
> - **첫 데이터셋**(oreil.ly/pxgwe): 키-체중 산점도에서 **눈에 띄는 패턴 없음** [그림 2-1 — 상단·우측은 히스토그램·경험적 누적 분포].
> - **둘째 데이터셋**(oreil.ly/8bE36): **명백한 선형 관계** [그림 2-2] — 포인트가 직선 중심으로 모임. **왜 첫째는 관계가 없어 보였나?** → 실제 데이터 작업의 흔한 어려움: 생성 함수를 모르고, 왜 그렇게 보이는지 모름. 데이터를 조사→인사이트→가설 함수 제안→테스트→성능 측정→배포→새 데이터와 안 맞으면 가설 갱신 — **모델이 비즈니스에 쓰이는 한 계속되는 피드백 루프**.
> - [그림 2-2 캡션] 키·체중 히스토그램이 **쌍봉(bimodal)** = **혼합 분포(mixture distribution)** 존재 — 실제로 **가우스 혼합**(남성/여성 두 **정규 분포**)로 모델링 가능; 남녀 하위 집합만 그리면 종 모양(bell-shape) `(🔭 예고 — 정규 분포·[그림 2-6])`.
> - **수수께끼 해결**: 첫 데이터셋엔 비만·고도 비만이 과도 포함(옮긴이 18: Index 4=비만, 5=고도 비만) → **Index 피처 점수로 분할해 비슷한 점수끼리 그리면 선형 관계가 분명히 드러남** [그림 2-3, Index=3]. 개인의 Index 값을 조건으로 쓰는 게 속임수 같지만 — **데이터 탐색이라는 이름하에 속임수 없는 공정한 게임**.
> - 이제 가설 수립: $weight = w_0 + w_1 \times height$ — 적절한 $w_0, w_1$ 찾기는 **Chapter 3** `(🔭 예고)`. **ML·딥러닝 활동의 대부분은 데이터에서 $w$를 학습하는 것** — 여기선 피처 1개(키)·관찰된 선형 패턴 가정이라 $w$ 2개뿐이지만, 수백만 개 $w$의 딥러닝도 **수학적 구조는 동일**하다. 📌

> [!abstract] 📘 책 핵심 — [필수수학AI] §2.5 시뮬레이션 데이터 예시 (그림 2-4~2-6)
> - 자체 데이터로 시뮬레이션하면 데이터 검색·실험실 구축 없이 통제된 측정값을 얻는 번거로움 회피 — 데이터를 구할 수 없거나 비쌀 때 유용, 함수의 숫자만 바꿔 다른 시나리오 테스트. 필요한 것 = 수학 함수 + 무작위성(randomness)/노이즈를 포함하는 확률 분포 + 컴퓨터.
> - 선형 종속성 가정: $weight = w_0 + w_1 \times height$. 시뮬레이션하려면 $w_0, w_1$ 값을 **가정**해야 — 실데이터 인사이트가 없으면 맥락·다양한 값으로 실험할 수밖에(여기선 실데이터가 있어 $w$ 학습 가능, 방법은 Chapter 3 `🔭`).
> - $w_0 = -314.5,\ w_1 = 7.07$ 설정: $weight = -314.5 + 7.07 \times height$ — $height{=}60$ 이면 $weight = 109.7$(키 60인치 사람 체중 109.7파운드 예측, 좌표 (60, 109.7)).
> - [그림 2-4] 54~79인치 키 5,000개 대입 → 5,000쌍 생성: **노이즈·변동이 없어 완벽한 직선**.
> - **시뮬레이션 데이터의 특징 = 데이터 자체로 함수의 기능을 수행** — 만든 함수(모델)를 이해하고 있고 큰 계산 오류·이상한 숫자가 없다면 원하는 방식으로 사용 가능, 예상 밖 결과가 드묾.
> - 더 현실적으로: **종 모양 정규 분포에서 키를 샘플링**(우리가 샘플링하는 분포 ≠ 실데이터 분포!) → 선형 모델에 대입 → **약간의 노이즈 추가**. 노이즈도 무작위성이 있으므로 **노이즈를 샘플링할 확률 분포 선택** 필요 — 정규 분포를 썼지만 균일한 랜덤 노이즈면 **균등 분포(uniform distribution)** 도 가능: $weight = -314.5 + 7.07 \times height + noise$ [그림 2-5 — 키·체중 히스토그램 모두 정규].
> - [그림 2-5] vs [그림 2-6](실제 여성 5,000명, oreil.ly/rZNBS): 실데이터 수집과 달리 **5분(코드 작성 시간)** 만에 생성한 것치곤 나쁘지 않음 — $w$·노이즈 파라미터(평균·표준 편차)를 조정하면 더 비슷해지겠지만, 곧 가설 모델에 적합한 파라미터 찾기(Chapter 3)를 설명하므로 여기서 종료.

## 🧠 근본 이해 · 융합 · 통찰

### 🔑 그림 2-1→2-3의 반전 = 조건부 확률의 산점도 버전 (MD 1.3 교차 접속)
- "전체를 그리면 무패턴, **Index=3으로 조건을 걸면** 선형" — 이건 [[1.3 조건부 확률·베이즈 정리 — 방향·뒤집기·조건부 결합·합|MD 1.3]]의 조건부($P(\cdot \mid \text{조건})$)가 시각화로 나타난 것: $f(height)$ 전체가 아니라 $f(height \mid Index{=}3)$ 을 본 것이다. **교란 피처(비만도)가 관계를 가리면, 조건화(층화)가 그것을 벗긴다.**
- 🪂 심화 `#사이드/심화`: 통계학에서 이 기법의 이름이 **층화(stratification)** — 집계 데이터와 층별 데이터의 경향이 달라지는 현상의 극단이 **심슨의 역설**(집계에선 방향이 *반전*)이다. 여기는 반전까지는 아니고 "가림→드러남"이지만 같은 계열. 책이 "속임수 같지만 공정한 게임"이라 못 박은 이유 = 조건화는 데이터 왜곡이 아니라 **모집단의 정밀화**([[1.3 조건부 확률·베이즈 정리 — 방향·뒤집기·조건부 결합·합|1.3]] "분모 교체" 통찰과 동일 구조).

### 🔑 실제 vs 시뮬레이션 = "함수를 모른다 vs 안다" — MD 1.1 확률/통계 구분의 데이터판
- [[1.1 확률 — 정의·가능도·오즈|MD 1.1]] §2.1.1: 확률(이론, 데이터 불요) vs 통계(데이터 필수). 이 절이 그 쌍의 거울상이다: **시뮬레이션 = 함수(이론)를 알고 데이터를 만들고** / **실제 = 데이터에서 함수(가설)를 캔다.** 두 책이 같은 이분법을 각자의 언어로 반복 — 학습(fit)이란 결국 "실제 데이터를 시뮬레이션할 수 있는 함수 찾기"라는 재정의가 가능해진다.
- 노이즈 추가의 확률 분포 선택(정규 vs 균등)은 [[1.4 이항 분포·베타 분포 — pmf·CDF·확률의 확률|MD 1.4]]에서 시작한 "분포" 어휘의 실전 첫 사용 — 정규·균등 분포의 정식 정의는 `(🔭 예고 — 이 장 잔여·MD 3장)`.

### 🔑 "5분 시뮬"의 진짜 교훈 — 생성 모델의 원형
- 그림 2-5가 그림 2-6(실데이터)을 흉내 낸 과정 = **입력 분포 샘플링 → 결정적 함수 → 노이즈** — 이 3단 구조가 생성 모델(generative model)의 원형이다 `(🔭 예고 — [필수수학AI] 8장 생성모델 계열)`. "우리가 샘플링하는 분포 ≠ 실데이터 분포"라는 경고는 시뮬-실제 간극(sim-to-real gap)의 씨앗.

## 🧩 코드 (§2.5)
> 🎯 목표: ① $height{=}60 \to 109.7$ 검산 ② 그림 2-4(무노이즈 직선)·2-5(정규 키+정규 노이즈) 재현 ③ 노이즈 유무가 산점도에 주는 효과 확인.
> 코드: `~/LOCAL/03-00_STUDIES/AI/01_linear_algebra/ma2_simulation.py` (그림 재구성 동일 로직)

```python
import numpy as np
rng = np.random.default_rng(42)
w0, w1 = -314.5, 7.07

assert abs((w0 + w1 * 60) - 109.7) < 1e-9          # (60, 109.7) 검산

h = rng.uniform(54, 79, 5000)                      # 그림 2-4: 완벽한 직선
w_clean = w0 + w1 * h                              # 노이즈 없음 → 함수 그 자체

h2 = rng.normal(63.5, 3, 5000)                     # 그림 2-5: 종 모양 키 샘플링
noise = rng.normal(0, 4, 5000)                     # 노이즈의 분포도 '선택'
w_noisy = w0 + w1 * h2 + noise                     # 5분짜리 유사-실데이터
```

## 🆕 용어
| 용어 | 한 줄 정의 |
|---|---|
| **실제 데이터** (real data) | 실제 관찰(센서·설문·웹·주식·통제 실험)로 수집 — 불완전·노이즈; 생성 함수는 가설로만 |
| **시뮬레이션 데이터** (simulated data) | 아는 함수/알려진 분포에서 생성·샘플링 — 데이터 자체가 함수의 기능을 수행 |
| **노이즈** (noise) | 신호를 가리는 무작위 변동 — 측정 오류(실제) 또는 의도적 추가(시뮬, 분포 선택 필요) |
| **층화/조건화 탐색** (Index 분할) | 교란 피처 값이 비슷한 부분집합끼리 그려 숨은 관계를 드러냄 — "속임수 없는 공정한 게임" |
| **쌍봉·혼합 분포** (bimodal·mixture, 🔭) | 봉우리 2개 히스토그램 = 여러 분포(남/녀 정규)의 혼합 신호 — 가우스 혼합 |
| **균등 분포** (uniform distribution, 🔭) | 구간 내 모든 값이 동일 확률 — 균일 랜덤 노이즈용 선택지(↔ 정규) |
| **경험적 누적 분포** (empirical CDF) | 데이터로 그린 누적 분포([[1.4 이항 분포·베타 분포 — pmf·CDF·확률의 확률|MD 1.4]] CDF의 표본판) — 그림 2-1 여백 그래프 |
