> [!quote]- ✅ P1(3-1) 정답·방법 — 상관계수+코사인 동시 반환 함수
> ```python
> import numpy as np
>
> def corr_and_cosine(x, y):
>     """상관계수·코사인 유사도를 모두 반환 (np.corrcoef, spatial.distance.cosine 미사용)."""
>     xbar, ybar = x.mean(), y.mean()
>     xt, yt = x - xbar, y - ybar
>     rho = (xt @ yt) / (np.linalg.norm(xt) * np.linalg.norm(yt))
>     cos = (x @ y) / (np.linalg.norm(x) * np.linalg.norm(y))
>     return rho, cos
>
> rng = np.random.default_rng(1)
> x = rng.normal(10, 2, 20); y = rng.normal(5, 3, 20)   # 평균이 0이 아닌 원본
>
> rho_raw, cos_raw = corr_and_cosine(x, y)
> print(rho_raw, cos_raw)     # -0.3380, 0.7829 — 다름
>
> xc, yc = x - x.mean(), y - y.mean()                    # 미리 평균중심화
> rho_c, cos_c = corr_and_cosine(xc, yc)
> print(rho_c, cos_c)         # -0.3380, -0.3380 — 동일
> assert not np.isclose(rho_raw, cos_raw)
> assert np.isclose(rho_c, cos_c)
> ```
> **원리:** 함수 내부에서 항상 `x - xbar`로 평균을 뺀다. 입력이 이미 평균 0이면 이 연산이 아무 효과가 없어 `xt==x`가 되고, 결국 ρ와 cos가 **같은 벡터에 같은 공식**을 적용하는 셈이라 값이 일치한다. 입력 평균이 0이 아니면 `xt≠x`라서 ρ와 cos는 서로 다른 벡터에 대한 계산이 되어 갈라진다 — 이것이 원자 1.9의 "상관계수=평균중심화된 벡터의 코사인 유사도"를 함수 동작으로 실증한 것.

> [!quote]- ✅ P3(3-3) 정답·방법 — scipy `pearsonr` 소스 대조
> ```python
> import inspect
> from scipy import stats

> 소스에서 발췌한 핵심 라인:
> ```python
> xmean = xp.mean(x, axis=axis, keepdims=True)
> ymean = xp.mean(y, axis=axis, keepdims=True)
> xm = x - xmean
> ym = y - ymean
> ```
> **책 공식과의 대응:** `xmean`/`ymean` 빼기가 정확히 평균중심화(식 3-2의 x̃=x−x̄) 단계이고, 이후 `xm`·`ym`의 내적을 노름 곱으로 나누는 구조가 이어진다 — **원자 1.9가 유도한 식 3-2와 논리적으로 동일**하다. 차이는 실전 라이브러리다운 안전장치: scipy는 `xm`의 절댓값 최댓값(`xmax`)으로 먼저 나눈 뒤 노름을 계산해 **오버플로를 방지**한다(책의 단순 공식엔 없는 수치 안정성 보강). 즉 "무엇을 계산하는가"는 같고 "어떻게 안전하게 계산하는가"만 다르다.

> [!quote]- ✅ P4(3-4) 정답·방법 — 속도 벤치마크
> ```python
> import time
>
> def my_corrcoef_only(x, y):
>     xt, yt = x - x.mean(), y - y.mean()
>     return (xt @ yt) / (np.linalg.norm(xt) * np.linalg.norm(yt))
>
> n_iter = 1000
> rng = np.random.default_rng(0)
> pairs = [(rng.standard_normal(500), rng.standard_normal(500)) for _ in range(n_iter)]
>
> t0 = time.perf_counter()
> for a, b in pairs: my_corrcoef_only(a, b)
> t_mine = time.perf_counter() - t0
>
> t0 = time.perf_counter()
> for a, b in pairs: np.corrcoef(a, b)[0, 1]
> t_builtin = time.perf_counter() - t0
> ```
> **결과(이 환경 실측):** 직접구현 13.52ms/1000회(13.5μs/회) vs `np.corrcoef` 28.18ms/1000회(28.2μs/회) — **직접구현이 약 2.08배(108%) 빠름**. 책 본문은 저자 환경에서 "약 33% 빠름"이라 보고하는데, 정확한 배율은 하드웨어·numpy 버전에 따라 달라지지만 **방향(직접구현이 항상 더 빠름)은 일치**한다. 이유는 책이 명시한 그대로 — `np.corrcoef`는 입력 검사·형변환·일반적인 N차원 처리를 위한 범용 코드 경로를 거치는 반면, 직접구현은 "정확히 두 1D 벡터"라는 좁은 가정 위에서 필요한 연산만 한다. 다만 책의 경고대로 입력 검사가 없다는 건 **오류를 놓칠 위험**과 맞바꾼 속도라는 점, 벡터가 훨씬 커지면 이 속도 이점이 사라질 수 있다는 점은 그대로 유효하다.

> [!quote]- ✅ P6(3-6) 정답·방법 — 가우스 평활화(그림 3-6 재현)
> ```python
> kernel_raw = np.array([0, .1, .3, .8, 1, .8, .3, .1, 0])
> kernel = kernel_raw / kernel_raw.sum()           # 합=1로 정규화

>
> rng = np.random.default_rng(7)
> noisy = rng.normal(0, 1, 100)

> assert smoothed.std() < noisy.std()              # 저주파 필터 = 변동성 감소
> ```
> **결과:** 원신호 표준편차 0.875 → 평활화 후 0.413(약 53% 감소). 커널 합이 1이면 필터링 결과는 원신호의 **국소 가중평균**이 되므로, 값이 급격히 튀는(고주파) 성분은 이웃 값들과 섞이며 상쇄되고 완만한(저주파) 추세만 남는다 — 이게 "저주파 통과(low-pass)" 필터라 불리는 이유.
>
> ![[P09 그림3-6 재현 (연습3-6).png|760]]
> > [!quote]- 📖 책 원본 [그림 3-6]
> > *(책 §3.2 연습문제 3-6 그림 3-6 — (A)가우스형 커널 (B)잡음 신호 (C)평활화 결과 3패널. 캡처 시 attachments에 추가)*

> [!quote]- ✅ P7(3-7) 정답·방법 — 커널 반전+영평균화 → 고주파(선명화) 필터
> ```python
> kernel_sharp_raw = kernel_raw.astype(float).copy()
> kernel_sharp_raw[4] = -1.0                        # 중앙의 1을 -1로
> kernel_sharp = kernel_sharp_raw - kernel_sharp_raw.mean()   # 영-평균화 -> 합 0
> assert np.isclose(kernel_sharp.sum(), 0.0)
>
> klen = len(kernel_sharp)
> sharpened = np.array([np.dot(noisy[i:i+klen], kernel_sharp) for i in range(100 - klen + 1)])
> ```
> **결과:** 원신호 표준편차 0.875 → 선명화 후 1.303(오히려 **증가**, 3-6의 저주파와 정반대). 커널 합이 0이면 필터는 "이웃과 얼마나 다른가"만 반응한다 — 신호가 평평한 구간에서는 중심의 큰 음수와 주변의 양수가 서로 상쇄돼 출력이 0에 가깝지만, 값이 요동치는 구간에서는 상쇄가 깨지며 큰 값이 튀어나온다. 그래서 부드러운(저주파) 성분은 감쇄되고 빠르게 변화하는(고주파) 성분만 강조되는 **고주파 통과(high-pass)** 필터가 된다 — 책이 말한 "더욱 날카로운 신호"의 정체.
>
> ![[P09 결과 (연습3-7).png|760]]

> [!quote]- ✅ P8(3-8) 정답·방법 — k=3 반복 실행의 안정성
> ```python
> def kmeans(data, k, n_iter, rng):
>     ridx = rng.choice(range(len(data)), k, replace=False)
>     centroids = data[ridx, :].copy()
>     for _ in range(n_iter):
>         dists = np.zeros((data.shape[0], k))
>         for ci in range(k):
>             dists[:, ci] = np.sum((data - centroids[ci, :])**2, axis=1)
>         groupidx = np.argmin(dists, axis=1)
>         for ki in range(k):
>             pts = data[groupidx == ki]
>             if len(pts) > 0:
>                 centroids[ki, :] = pts.mean(axis=0)
>     return centroids, groupidx
>
> # 원자 1.11과 동일한 3-무리 합성 데이터 재사용, seed만 바꿔 5회 반복
> for seed in [100, 101, 102, 103, 104]:
>     rng = np.random.default_rng(seed)
>     centroids, groupidx = kmeans(data, 3, 5, rng)
>     ...
> ```
> **결과(x좌표 기준 정렬, 5회 중):**
>
> | seed | 중심 | 크기 |
> |---|---|---|
> | 100~103 | [1.8,7.0]·[2.0,2.1]·[5.9,6.0] (진짜 중심 근처) | 50/50/50 |
> | **104** | [1.8,1.7]·[2.2,2.6]·[3.9,6.5] (**진짜 중심과 다름!**) | 22/100/28 |
>
> **정직한 답:** "일반적으로 유사한가?" — **대체로 그렇지만 항상은 아니다.** 5회 중 4회(seed 100~103)는 무작위 초기화가 달라도 정확히 같은 3개의 진짜 중심으로 수렴했지만, seed=104는 [2,2] 무리와 [2,7] 무리의 경계에서 초기 중심이 애매하게 잡혀 **두 무리를 섞어버리고 세 번째 중심이 [6,6] 무리 일부까지 침범**하는 다른(더 나쁜) 국소해로 수렴했다. 이것이 k-평균의 잘 알려진 한계 — **경사하강처럼 지역 최적해에 갇힐 수 있다**(원자 1.11 ⚠️ 박스가 예고한 "k 결정이 어렵다"는 문제와 같은 계열: 초기화도 데이터가 알려주지 않는 선택이다).
>
> ![[P09 결과 (연습3-8·3-9).png|1000]]
> (좌측 2패널이 3-8 — 좌1: 성공 사례(중심이 정확히 3무리와 일치) / 좌2: seed=104 실패 사례(파랑·초록이 [2,2]/[2,7] 경계에서 뒤섞임))

> [!quote]- ✅ P9(3-9) 정답·방법 — k=2, k=4 결과
> **k=2 (3회 반복, 전부 동일 패턴):** 항상 **기하적으로 가장 가까운 두 무리가 병합**된다. [2,7]과 [6,6] 사이 거리(≈4.12)가 [2,2]와 [2,7] 사이 거리(≈5.0)보다 가까워서, [2,7]+[6,6]이 크기 100짜리 한 클러스터로 묶이고 [2,2](크기 50)만 홀로 남는다.
> ```
> centroids: [3.86, 6.53](100)  [1.99, 2.11](50)
> ```
> **k=4 (3회 반복):** 세 무리 중 하나가 **둘로 쪼개진다.** 어느 무리가 쪼개지는지는 초기화에 따라 달라지지만(관측: [2,7] 또는 [2,2] 무리가 주로 분할), 나머지 두 무리는 온전히 유지된다.
> ```
> 예: centroids: [1.68,6.64](32)  [1.99,2.11](50)  [5.92,6.02](50)  [1.98,7.77](18)
>     -> [2,7] 무리가 [1.68,6.64]+[1.98,7.77] 두 조각으로 분할, 나머지 둘은 온전
> ```
> **해석:** k-평균은 "정답 무리 수"라는 개념을 모른다 — 그저 **k개로 나누라는 명령을 문자 그대로 수행**할 뿐이다. k가 작으면 서로 다른 진짜 무리를 억지로 합치고(과소분할), k가 크면 하나의 진짜 무리를 억지로 쪼갠다(과대분할). 이것이 원자 1.11이 경고한 "$k$는 데이터가 알려주지 않는다"의 실물 증거 — elbow method·실루엣 계수 같은 정량적 $k$ 선택 기법이 왜 필요한지(이 책·이 원자의 범위 밖) 보여주는 대목이다.
>
> ![[P09 결과 (연습3-8·3-9).png|1000]]
> (우측 2패널이 3-9 — 우1: k=2로 [2,7]+[6,6] 병합 / 우2: k=4로 [2,7] 무리가 파랑+보라로 분할)

## 코드 검산 (전체 통합)
```python
import numpy as np, time, inspect
from scipy import stats

# ── A. 상관관계 (3-1~3-4) ──
def corr_and_cosine(x, y):
    xbar, ybar = x.mean(), y.mean()
    xt, yt = x - xbar, y - ybar
    rho = (xt @ yt) / (np.linalg.norm(xt) * np.linalg.norm(yt))
    cos = (x @ y) / (np.linalg.norm(x) * np.linalg.norm(y))
    return rho, cos

rng = np.random.default_rng(1)
x, y = rng.normal(10, 2, 20), rng.normal(5, 3, 20)
rho_raw, cos_raw = corr_and_cosine(x, y)
xc, yc = x - x.mean(), y - y.mean()
rho_c, cos_c = corr_and_cosine(xc, yc)
assert not np.isclose(rho_raw, cos_raw) and np.isclose(rho_c, cos_c)   # 3-1

x2 = np.array([0., 1., 2., 3.])
rhos = [corr_and_cosine(x2, x2 + off)[0] for off in range(-50, 51)]
assert np.std(rhos) < 1e-10   # 3-2: rho는 오프셋 무관하게 일정

src = inspect.getsource(stats.pearsonr)
assert "xmean" in src and "ymean" in src   # 3-3: 평균중심화(식 3-2의 x̃=x-x̄) 단계가 소스에 실제로 존재

def my_corrcoef_only(x, y):
    xt, yt = x - x.mean(), y - y.mean()
    return (xt @ yt) / (np.linalg.norm(xt) * np.linalg.norm(yt))
pairs = [(rng.standard_normal(500), rng.standard_normal(500)) for _ in range(200)]
t0 = time.perf_counter(); [my_corrcoef_only(a, b) for a, b in pairs]; t_mine = time.perf_counter() - t0
t0 = time.perf_counter(); [np.corrcoef(a, b)[0,1] for a, b in pairs]; t_builtin = time.perf_counter() - t0
assert t_mine < t_builtin   # 3-4: 직접구현이 더 빠름

# ── B. 필터링 (3-5~3-7) ──
n = 30
signal = np.zeros(n); signal[10:20] = 1.0
edge = np.array([np.dot(signal[i:i+2], [-1.,1.]) for i in range(n-1)])
assert np.isclose(edge[9], 1.0) and np.isclose(edge[19], -1.0)   # 3-5

kernel_raw = np.array([0,.1,.3,.8,1,.8,.3,.1,0])
kernel = kernel_raw / kernel_raw.sum()
noisy = rng.normal(0, 1, 100)
smoothed = np.array([np.dot(noisy[i:i+9], kernel) for i in range(92)])
assert smoothed.std() < noisy.std()   # 3-6: 저주파=변동성 감소

kernel_sharp_raw = kernel_raw.astype(float).copy(); kernel_sharp_raw[4] = -1.0
kernel_sharp = kernel_sharp_raw - kernel_sharp_raw.mean()
assert np.isclose(kernel_sharp.sum(), 0.0)   # 3-7: 영-평균화 커널

# ── C. k-평균 (3-8~3-9) ──
def kmeans(data, k, n_iter, rng):
    ridx = rng.choice(range(len(data)), k, replace=False)
    centroids = data[ridx, :].copy()
    for _ in range(n_iter):
        dists = np.zeros((data.shape[0], k))
        for ci in range(k):
            dists[:, ci] = np.sum((data - centroids[ci,:])**2, axis=1)
        groupidx = np.argmin(dists, axis=1)
        for ki in range(k):
            pts = data[groupidx==ki]
            if len(pts): centroids[ki,:] = pts.mean(axis=0)
    return centroids, groupidx

base_rng = np.random.default_rng(0)
data = np.vstack([base_rng.normal([2,2],0.6,(50,2)), base_rng.normal([6,6],0.6,(50,2)), base_rng.normal([2,7],0.6,(50,2))])
base_rng.shuffle(data)
c3, g3 = kmeans(data, 3, 5, np.random.default_rng(100))
assert sorted([np.sum(g3==k) for k in range(3)]) == [50,50,50]   # 3-8: 좋은 초기화는 정확히 수렴

print("3장 연습문제 9문항 전체 검증 통과")
```
