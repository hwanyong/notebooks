# ✏️ 연습 MOC — 문제 풀이 관리

> **연습문제 노트의 색인 + 운영 규칙.** 탐구(`T0X`)가 *호기심의 갈래*라면 연습(`P0X`)은 *손으로 푸는 검증* — 개념 노트가 "안다"를 만들고, 연습 노트가 "할 수 있다"를 확인한다.

🔙 [[_커리큘럼 MOC]] ｜ 🧠 [[_핵심 암기장]] ｜ 📖 [[_용어집]]

## 운영 규칙
1. **한 세트 = 노트 1개**, 파일명 `PNN 주제 — 시험 개념 (N문)`. `연습/` 폴더에 보관.
2. **frontmatter:** `유형: 연습` · `관련노트:`(시험하는 렌즈/원자 링크) · `출처:` · `풀이일:` · `성적:`(맞음/전체) · `재시도:`(🔁 필요 여부).
3. **정답·풀이는 접이식 콜아웃**(`> [!quote]- ✅ 정답·풀이`)에 숨긴다 — 노트를 다시 열면 곧바로 재시도(자가 테스트)가 되도록. 문제 본문에는 답을 노출하지 않는다.
4. **문제마다 "시험하는 개념" 링크 의무** — 어느 원자/렌즈의 어느 규칙을 테스트하는지 한 줄로. 이것이 학습 노트와의 양방향 연결 고리.
5. **오답·헤맨 문제는 해당 학습 노트의 `막힌 점`에 역기입** — "P0X-n번에서 틀림 → 원인 개념" 형식. 이해도(🟢🟡🔴) 조정의 근거로 쓴다.
6. **검산은 코드로** — 풀이 스니펫을 노트에 남기고, 세트가 크면 코드 저장소(`~/LOCAL/03-00_STUDIES/AI/`)에 `pNN_*.py`로.
7. 색인(아래 표)과 관련 렌즈의 `연결` 섹션에 **같은 작업에서** 등재(용어집·암기장과 같은 동기화 원칙).
