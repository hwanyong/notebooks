> [!quote]- ✅ P2(1-2) 정답·방법 — 노름을 식 1-7로 직접 구현
> ```python
> def my_norm(v):
>     return np.sqrt(np.sum(v**2))   # 식 1-7: ‖v‖ = sqrt(Σ vᵢ²)
>
> for dim in [2,3,5,7]:
>     v = np.random.randn(dim) * 10
>     assert np.isclose(my_norm(v), np.linalg.norm(v))
> ```
> 모든 차원·방향에서 `my_norm(v) == np.linalg.norm(v)`. 실전에서는 `np.linalg.norm()`이 훨씬 간단(및 수치적으로 더 안정)하므로 직접 구현은 원리 확인용.

> [!quote]- ✅ P3(1-3) 정답·방법 — 단위벡터 함수 + 영벡터 처리
> ```python
> def unit_vector(v):
>     n = np.linalg.norm(v)
>     if n == 0:
>         raise ValueError("영벡터는 방향이 없어 단위벡터를 만들 수 없습니다 (0으로 나눔)")
>     return v / n
> ```
> **영벡터 입력 시:** `ZeroDivisionError`류의 상황(0으로 나눔) — **정의 자체가 불가능**하다. 영벡터는 크기 0·방향 없음이므로 "방향만 남긴 단위벡터"라는 개념이 성립하지 않는다([[02 벡터 크기와 단위벡터 (노름)|02 원자]]의 "영벡터 예외"와 동일한 사실). 실무에서는 `if n==0` 체크 후 명시적 예외를 던지는 게 조용히 `NaN`을 반환하는 것보다 안전하다.

> [!quote]- ✅ P4(1-4) 정답·방법 — 임의 크기의 벡터 생성
> ```python
> def vector_with_magnitude(v, mag):
>     return unit_vector(v) * mag        # 방향(단위벡터) × 원하는 크기
>
> w = vector_with_magnitude(np.array([3.0,4.0]), 10.0)   # -> [6., 8.]  (‖w‖==10)
> ```
> 원리: $v = \lVert v\rVert\,\hat v$ (크기×방향 분해, [[02 벡터 크기와 단위벡터 (노름)|02 원자]])를 뒤집은 것 — 먼저 방향만 뽑아내고(단위벡터), 원하는 크기를 다시 곱한다.

> [!quote]- ✅ P5(1-5) 정답·방법 — for 루프 전치
> ```python
> def transpose_loop(row_vec):
>     row_vec = np.atleast_2d(row_vec)      # (1,N) 가정
>     n = row_vec.shape[1]
>     col = np.zeros((n,1))
>     for i in range(n):
>         col[i,0] = row_vec[0,i]
>     return col
>
> rv = np.array([[7,2,9,4]])
> assert np.array_equal(transpose_loop(rv), rv.T)
> ```
> 핵심은 **인덱스를 바꿔 다시 쓰는 것**뿐 — `row_vec[0,i]` (1행 i열)의 값을 `col[i,0]` (i행 1열)에 옮긴다. `.T`가 내부적으로 하는 일(strides 재해석, [[T04 NumPy 벡터 내부 — shape·strides·열벡터 (N,1)|T04]])을 데이터 복사로 명시적으로 흉내낸 것.

> [!quote]- ✅ P6(1-6) 정답·방법 — ‖v‖² = v·v
> ```python
> v = np.random.randn(6)
> assert np.isclose(my_norm(v)**2, v @ v)
> ```
> [식 1-7]을 제곱하면 $\lVert v\rVert^2=\sum v_i^2$ 인데, 내적의 정의($v^\mathsf{T}v=\sum v_i v_i=\sum v_i^2$)와 **글자 그대로 같은 식**이다 — 별개의 공식이 아니라 노름이 "자기 자신과의 내적의 제곱근"이라는 원자([[02 벡터 크기와 단위벡터 (노름)|02]]) `vᵀv (제곱합 행렬곱)` 카드의 재확인.

> [!quote]- ✅ P7(1-7) 정답·방법 — 내적 교환법칙
> ```python
> a = np.random.randn(5); b = np.random.randn(5)
> assert np.isclose(a @ b, b @ a)
> ```
> **왜 교환 가능한가([식 1-9] 관점):** $a^\mathsf{T}b=\sum_i a_i b_i$ 는 **원소별 스칼라 곱을 모두 더한 것**이다. 실수의 곱셈 자체가 교환법칙을 만족하므로($a_ib_i=b_ia_i$, 각 항 단위에서 이미 성립), 그 합도 순서와 무관하다 — 행렬곱 일반(AB≠BA)과 달리 내적이 교환 가능한 이유는 **"1×1 스칼라라서"**가 아니라 **"원소별 곱셈이 스칼라 곱셈이라서"**다.

> [!quote]- ✅ P9(1-9) 정답·방법 — 직교벡터 분해 구현 + 그림 1-8 재현
> ```python
> def orth_decompose(t, r):
>     t_par  = r * (t @ r) / (r @ r)   # 식 1-13
>     t_perp = t - t_par
>     return t_par, t_perp
>
> t = np.random.randn(2); r = np.random.randn(2)
> t_par, t_perp = orth_decompose(t, r)
> assert np.allclose(t_par + t_perp, t)          # 검증 ①: 합 = t
> assert np.isclose(t_perp @ r, 0, atol=1e-10)   # 검증 ②: 직교
> ```

> 난수라 매 실행마다 벡터 모양은 달라지지만, **두 검증(합이 t·직교)은 항상 성립**해야 한다 — 이 두 조건이 "직교벡터 분해"의 정의 그 자체([[1.7 직교벡터 분해 — 투영·수직·평행 성분|1.7 원자]] "분해 검증 2가지").

> [!quote]- ✅ P10(1-10) 정답·방법 — 버그 시뮬레이션과 온전성 검사
> ```python
> def orth_decompose_buggy(t, r):
>     t_par_bug  = r * (t @ r) / (t @ t)   # 버그: 분모가 r@r 아닌 t@t
>     t_perp_bug = t - t_par_bug
>     return t_par_bug, t_perp_bug
>
> t_par_bug, t_perp_bug = orth_decompose_buggy(t, r)
> print(t_perp @ r)        # 정상: ≈ 0
> print(t_perp_bug @ r)    # 버그: 0이 아님 (예: -0.457)
> ```
> **차이:** 정상 코드는 $\beta r = r\,(t^\mathsf{T}r/r^\mathsf{T}r)$ — $r$ **자신의 크기**로 정규화한 배율만큼 $r$을 스케일한다. 버그 코드는 분모가 $t^\mathsf{T}t$이므로 **엉뚱한(t의) 크기**로 나눈 벡터가 나와 방향·길이 모두 달라진다.
> **결과가 올바른지 판단하는 법(온전성 검사):** 이 문제가 정확히 [[1.7 직교벡터 분해 — 투영·수직·평행 성분|1.7 원자]]가 이미 세워둔 **"분해 검증 2가지"**다 — ① $t_{\parallel r}+t_{\perp r}\stackrel{?}{=}t$, ② $t_{\perp r}\cdot r\stackrel{?}{=}0$. 버그 코드는 ①은 항상 통과하지만(대수적으로 $t=(t-t_{\parallel r,\text{bug}})+t_{\parallel r,\text{bug}}$ 는 항등식이라 자동 성립) **②에서 즉시 걸린다** — $t_{\perp r}\cdot r$이 0에서 크게 벗어난다. → **결론: "결과가 그럴듯해 보이는가"가 아니라 "수학적으로 반드시 성립해야 할 관계식이 실제로 성립하는가"를 코드로 assert 해야 버그를 잡는다** — 이게 이 장 전체가 반복해서 보여준 "분해는 항상 두 가지로 검증한다"는 원칙의 실전 사용례다.

## 코드 검산 (전체 통합)
```python
import numpy as np
np.random.seed(0)

def my_norm(v):
    return np.sqrt(np.sum(v**2))

def unit_vector(v):
    n = np.linalg.norm(v)
    if n == 0:
        raise ValueError("영벡터는 단위벡터를 만들 수 없습니다")
    return v / n

def vector_with_magnitude(v, mag):
    return unit_vector(v) * mag

def transpose_loop(row_vec):
    row_vec = np.atleast_2d(row_vec)
    col = np.zeros((row_vec.shape[1], 1))
    for i in range(row_vec.shape[1]):
        col[i, 0] = row_vec[0, i]
    return col

def orth_decompose(t, r):
    t_par = r * (t @ r) / (r @ r)
    return t_par, t - t_par

def orth_decompose_buggy(t, r):
    t_par_bug = r * (t @ r) / (t @ t)   # 버그: 분모가 t@t
    return t_par_bug, t - t_par_bug

# 1-2
for dim in [2,3,5,7]:
    v = np.random.randn(dim)*10
    assert np.isclose(my_norm(v), np.linalg.norm(v))
# 1-3
assert np.isclose(np.linalg.norm(unit_vector(np.array([3.,4.]))), 1.0)
try:
    unit_vector(np.zeros(3)); assert False
except ValueError:
    pass
# 1-4
w = vector_with_magnitude(np.array([3.,4.]), 10.0)
assert np.isclose(np.linalg.norm(w), 10.0)
# 1-5
rv = np.array([[7,2,9,4]])
assert np.array_equal(transpose_loop(rv), rv.T)
# 1-6
v = np.random.randn(6)
assert np.isclose(my_norm(v)**2, v @ v)
# 1-7
a, b = np.random.randn(5), np.random.randn(5)
assert np.isclose(a @ b, b @ a)
# 1-9
t, r = np.random.randn(2), np.random.randn(2)
t_par, t_perp = orth_decompose(t, r)
assert np.allclose(t_par + t_perp, t)
assert np.isclose(t_perp @ r, 0, atol=1e-10)
# 1-10 (버그가 직교성 검사에서 잡히는지)
t_par_bug, t_perp_bug = orth_decompose_buggy(t, r)
assert not np.isclose(t_perp_bug @ r, 0, atol=1e-6)

print("전 10문항 검증 통과")
```
