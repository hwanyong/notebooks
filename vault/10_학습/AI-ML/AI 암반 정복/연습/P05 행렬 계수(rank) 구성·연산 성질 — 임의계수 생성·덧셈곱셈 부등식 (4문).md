# P05 행렬 계수(rank) 구성·연산 성질 — 임의계수 생성·덧셈곱셈 부등식 (4문)

> ✏️ **연습 세트 (책 공식 [연습 문제 5-5]~[5-8]).** [[P04 행렬 계수(rank) 추정 — 열벡터 독립성 판별 (6문)|P04]]가 "주어진 행렬의 rank를 **판별**"했다면, 이 세트는 "원하는 rank의 행렬을 **구성**하고 연산 법칙을 검증"한다 — 반대 방향의 기술. 정답은 접이식. 운영 규칙: [[_연습 MOC]]
> 🎯 **이 세트가 시험하는 것:** ① 외적의 일반화(M×r @ r×N)로 임의 계수 행렬 생성 ② r이 행렬 크기를 넘어설 때의 자동 상한 ③ 계수 덧셈 부등식의 등호·부등호 경계를 직접 만들기 ④ 곱셈 부등식의 **더 엄격한 상한**(rank-1×rank-1은 항상 rank≤1 — 도달 불가능한 조합 존재) ⑤ rank(A)=rank(Aᵀ)=rank(AᵀA)=rank(AAᵀ) 항등식과 그 이유.

## 문제

**P1 (책 [연습 5-5]).** 계수 $r$ 인 $M\times N$ 행렬을 만들려면 난수 $M\times r$ 행렬에 $r\times N$ 행렬을 곱한다. 이를 구현하고 계수가 실제로 $r$ 인지 확인하라. $r>\min\{M,N\}$ 을 설정하면 어떻게 되고 그 이유는?
- 시험 개념: 외적(rank-1, [[1.6 벡터 내적 — 점곱·기하·직교·아다마르곱·외적|1.6]])의 일반화 — $r$개 외적의 합은 최대 rank $r$; [[2.7 행렬 계수(rank) — 정보 차원·최대계수·이동·확장|2.7]] $r\le\min\{M,N\}$

**P2 (책 [연습 5-6]).** 계수-1 행렬 세 쌍을 만들어 더한 결과가 각각 계수-0, 계수-1, 계수-2가 되게 하라 — 덧셈 법칙 $r(A+B)\le r(A)+r(B)$ 를 구현하는 것. 그런 다음 덧셈 대신 **곱셈**에 대해서도 같은 문제를 풀어보라.
- 시험 개념: 덧셈 상한 [[2.7 행렬 계수(rank) — 정보 차원·최대계수·이동·확장|2.7]] · 곱셈은 **더 빡빡한 상한** $r(AB)\le\min\{r(A),r(B)\}$ — 병목 규칙

**P3 (책 [연습 5-7]).** P1의 코드를 함수로 만들어 $M{=}20$, 계수 $2\sim14$ 조합마다 $A{+}B$ 와 $A@B$ 의 계수를 계산해 히트맵 2장으로 그려라.
- 시험 개념: P2의 두 부등식을 **전 구간에서** 시각적으로 재확인 — 덧셈은 대각선 경계로 포화, 곱셈은 min(rA,rB) 계단

**P4 (책 [연습 5-8]).** 행렬 $A$, $A^{\mathsf{T}}$, $A^{\mathsf{T}}A$, $A A^{\mathsf{T}}$ 의 계수는 모두 같다. 다양한 크기·모양(정방·높은·넓은)과 계수의 난수 행렬로 이를 확인하라.
- 시험 개념: [[2.3 행렬 전치 — 내적·외적 표기법·LIVE EVIL·대칭 행렬|2.3]] $A^{\mathsf{T}}A$ 곱셈 기법 · [[2.6 행렬 공간 — 열공간·행공간·영공간|2.6]] 영공간

---

> [!quote]- ✅ P1(5-5) 정답·풀이 — 임의 계수 생성
> **구현:** `M_r = rng.standard_normal((M,r)) @ rng.standard_normal((r,N))`. 이유: 왼쪽 $M{\times}r$ 을 열벡터 $r$개(각 $\mathbb{R}^M$), 오른쪽 $r{\times}N$ 을 그 열들의 결합 계수로 보면 — 결과는 **그 $r$개 열의 선형결합만으로 만들어진 열공간**([[2.6 행렬 공간 — 열공간·행공간·영공간|2.6]])이라 최대 $r$차원. 난수라 $r$개가 (거의 확실히) 서로 독립이므로 정확히 $r$.
> **$r>\min\{M,N\}$ 이면:** 요청한 $r$ 과 무관하게 결과 행렬 자체가 $M{\times}N$ 이므로 열이든 행이든 **애초에 $\min\{M,N\}$개보다 많은 독립 방향을 담을 그릇이 없다** — 자동으로 $\min\{M,N\}$ 에서 잘린다. $r$을 아무리 키워도 "그릇 크기"를 못 넘는다는 게 [[2.7 행렬 계수(rank) — 정보 차원·최대계수·이동·확장|2.7]] 의 $r\le\min\{M,N\}$ 그 자체.
> 재현 검증(2026-07-14): $(M,N,r){=}(6,8,3)\to3$, $(5,5,5)\to5$, $(6,8,10)\to6{=}\min(6,8)$(요청 10 무시), $(6,8,6)\to6$ — 전부 일치.

> [!quote]- ✅ P2(5-6) 정답·풀이 — 덧셈은 0/1/2 전부 가능, 곱셈은 2가 원천 불가능
> **덧셈 (전부 구성 가능):**
> - **계수 0:** $A=uv^{\mathsf{T}}$, $B=-uv^{\mathsf{T}}$(완전히 반대) → $A+B=0$.
> - **계수 1:** $A=uv_1^{\mathsf{T}}$, $B=uv_2^{\mathsf{T}}$(**같은** $u$, 다른 $v$) → $A+B=u(v_1{+}v_2)^{\mathsf{T}}$, 여전히 외적 하나 = rank 1.
> - **계수 2:** $A=u_1v_1^{\mathsf{T}}$, $B=u_2v_2^{\mathsf{T}}$($u$도 $v$도 전부 독립) → 공유하는 축이 없어 rank 2.
> 재현: 세 경우 각각 sum rank = 0, 1, 2 — 정확히 일치.
>
> **곱셈 (계수-2는 원천 불가능 — 이게 진짜 답):** $A{=}uv^{\mathsf{T}}$, $B{=}wx^{\mathsf{T}}$ 이면 $AB=u(v^{\mathsf{T}}w)x^{\mathsf{T}}$ — $v^{\mathsf{T}}w$ 는 **스칼라 하나**이므로 결과는 항상 (스칼라)$\times$($u$와 $x$의 외적), 즉 **rank 0(스칼라가 0일 때) 아니면 rank 1**이다. rank-1 두 개를 곱해 rank-2를 얻는 조합은 **존재하지 않는다** — "만들어보라"는 요청에 대한 정답은 구성법이 아니라 **불가능성의 증명**. [[2.7 행렬 계수(rank) — 정보 차원·최대계수·이동·확장|2.7]] 의 $r(AB)\le\min\{r(A),r(B)\}=\min\{1,1\}=1$ 이 바로 이 상한.
> **⚠️ 수치 함정(2026-07-14 발견):** `np.linalg.matrix_rank()`로 직접 확인하면 이 사실이 가끔 깨져 *보인다*. $v^{\mathsf{T}}w\approx0$(거의 직교)이 되도록 일부러 구성하면, 특잇값이 $[\sigma_1{\approx}0,\ \sigma_2,\sigma_3,\sigma_4{\sim}10^{-16}]$ 형태가 되는데 — 여기서 $\sigma_1$(진짜 신호)이 이미 부동소수점 잡음 수준까지 작아져 있으면, 행렬곱 자체가 만들어내는 반올림 오차($\sigma_2,\sigma_3,\sigma_4$)가 `matrix_rank`의 기본 상대허용오차를 넘어서 **계수를 3이나 4로 오판**한다(2000회 무작위 시행 중 24회≈1.2%). 특잇값을 직접 찍어보면 $\sigma_1$이 나머지보다 항상 압도적으로 크다는 게 드러나 수학적 계수는 언제나 1(또는 0) 임을 재확인할 수 있다 — [[2.8 행렬식 — 계산·선형종속성·특성다항식|2.8]] · [[2.7 행렬 계수(rank) — 정보 차원·최대계수·이동·확장|2.7 tol 추정]]과 같은 "이론과 실제" 계열의 함정.
> ```python
> import numpy as np
> rng = np.random.default_rng(2)
> u,v = rng.standard_normal((4,1)), rng.standard_normal((1,4))
> A = u@v
> assert np.linalg.matrix_rank(A + (-A)) == 0                       # 계수 0
> v2 = rng.standard_normal((1,4)); assert np.linalg.matrix_rank(A + u@v2) == 1   # 계수 1
> u2,v3 = rng.standard_normal((4,1)), rng.standard_normal((1,4))
> assert np.linalg.matrix_rank(A + u2@v3) == 2                      # 계수 2
> # 곱셈: rank-2 조합은 2000회 무작위 시행에서도 등장한 적 없음(이론상 불가능)
> ```

> [!quote]- ✅ P3(5-7) 정답·풀이 — 히트맵으로 본 두 부등식
> $M{=}20$, $r_A,r_B\in\{2,\dots,14\}$ 조합마다 $A{+}B$·$A@B$ 계수를 계산:
> - **덧셈 히트맵:** $r_A{=}r_B{=}2$ 지점 4(합 그대로), $r_A{=}r_B{=}14$ 지점에서 **20으로 포화**(행렬 크기 상한 $\min(M,M)=20$ 에 걸림 — 부등식이 등호가 아니라 실제로 "이하"임을 보여줌).
> - **곱셈 히트맵:** 거의 모든 지점에서 정확히 $\min(r_A,r_B)$ — 계단(staircase) 패턴, 난수라서 병목이 곧 등호로 실현된다.
> 전 격자(169개 조합)에서 $r(A{+}B)\le r_A{+}r_B$ 와 $r(A@B)\le\min(r_A,r_B)$ 둘 다 **예외 없이 성립** 확인.
> → [[LA5-9 rank 덧셈곱셈 히트맵 (재구성).png|LA5-9]] (책 그림 5-9 재현, 2패널).

> [!quote]- ✅ P4(5-8) 정답·풀이 — 네 계수가 같은 이유
> 재현(정방 5×5·8×3·3×8·6×6, 각 계수 1/중간/최대): **전부 예외 없이 $r(A)=r(A^{\mathsf{T}})=r(A^{\mathsf{T}}A)=r(AA^{\mathsf{T}})$.**
> **왜 그런가(핵심은 $r(A)=r(A^{\mathsf{T}}A)$):** [[2.6 행렬 공간 — 열공간·행공간·영공간|2.6]] 의 영공간 언어로 증명된다 — **$N(A)=N(A^{\mathsf{T}}A)$**(두 영공간이 *같은 집합*):
> - $Ax=0 \Rightarrow A^{\mathsf{T}}Ax=A^{\mathsf{T}}0=0$ (자명한 방향).
> - $A^{\mathsf{T}}Ax=0 \Rightarrow x^{\mathsf{T}}A^{\mathsf{T}}Ax=0 \Rightarrow \lVert Ax\rVert^2=0 \Rightarrow Ax=0$([[2.5 행렬 노름·대각합 — 프로베니우스·ℓ2·tr(AᵀA)|2.5]] 의 $v^{\mathsf{T}}v$=제곱합 재사용 — 노름이 0이면 벡터는 반드시 영벡터).
> 두 영공간이 같은 집합이면 차원도 같고, [[2.6 행렬 공간 — 열공간·행공간·영공간|2.6]] 이 예고했던 **계수-영공간 차원(nullity) 정리**(🔭, "$r$ + 영공간 차원 = 전체 차원" 형태)에 의해 두 계수도 같아진다 — 이 문제가 그 정리의 **특수 사례를 직접 증명**한 것이다(일반형은 여전히 🔭). $r(A)=r(A^{\mathsf{T}})$ 는 [[2.6 행렬 공간 — 열공간·행공간·영공간|2.6]] 의 "$\dim C(A)=\dim R(A)$" 정리 그대로. 넷을 조합하면 넷 다 같다.
> 비자명(계수-2인 6×4) 케이스로 재확인: $\dim N(A)=\dim N(A^{\mathsf{T}}A)=2$, 그리고 $A^{\mathsf{T}}A$ 의 영공간 벡터가 실제로 $Av=0$ 도 만족함을 수치로 확인(단순 차원 일치가 아니라 **집합 자체**가 같음을 검증).
> ```python
> import numpy as np
> rng = np.random.default_rng(7)
> A = rng.standard_normal((6,2)) @ rng.standard_normal((2,4))   # 계수-2, 6x4
> from scipy.linalg import null_space
> nsA, nsAtA = null_space(A), null_space(A.T@A)
> assert nsA.shape[1] == nsAtA.shape[1] == 2
> assert np.allclose(A @ nsAtA, 0, atol=1e-8)   # AᵀA의 영공간이 A의 영공간과 실제로 같음
> assert np.linalg.matrix_rank(A) == np.linalg.matrix_rank(A.T) == np.linalg.matrix_rank(A.T@A) == np.linalg.matrix_rank(A@A.T) == 2
> ```
