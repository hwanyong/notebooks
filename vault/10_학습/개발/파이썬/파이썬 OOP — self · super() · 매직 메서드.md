# 파이썬 OOP([oʊ oʊ piː], 오오피, 오오피) — self · super() · 매직 메서드

> 클래스를 상속해 객체를 만들 때 매번 등장하는 `self`·`super()`·던더(`__init__`/`__getitem__`/`__len__`)의 정체. 전통 언어(Java([ˈdʒɑːvə], 자바, 자바)/JS([dʒeɪ ɛs], 제이에스, 제이에스))의 `this`와 무엇이 다른가.

> 🐍 **파이썬 노트(범용 도구 지식)** — 특정 공부에 종속되지 않는 **재사용 문법**. AI·웹·데이터 어디서 클래스를 상속하든 다시 만난다.
> 🔗 영역 간 연결: 🌐 [[_지식 연결 허브]] ｜ 📖 [[_개발 MOC]]
> 🧪 지금 이 문법을 만난 곳: [[07 데이터 로더 (Dataset · DataLoader)|AI/Dataset 상속]] · [[05 퍼셉트론·편향·은닉층|AI/nn.Module 상속]]

> 🎯 **핵심 (TL;DR)**
> - **`self` = 상속과 무관.** 인스턴스 메서드의 첫 파라미터는 *무조건* `self` = "생성될 인스턴스 자기 자신".
> - **부모 접근은 `self`가 아니라 `super()`.** `super().__init__()` = 부모 생성자 호출.
> - `self`가 도는 원리 = **자동 주입**: `obj.method(x)` → 내부적으로 `Class.method(obj, x)`. 점 앞 객체가 첫 인자로.
> - **매직 메서드(던더)** = 파이썬이 약속된 순간 자동 호출. `len(obj)`→`__len__`, `obj[i]`→`__getitem__`.

> [!note] 🗺️ 이 노트의 출처
> **AI 공부(A.6 Custom Dataset) 코드를 보다 든 의문**(*"`self`는 상속받은 클래스라 들어가는 건가? 부모 접근용?"*)을 직접 조사한 파이썬 기본 문법. AI에 종속된 게 아니라 **범용 파이썬 지식**이라 이 영역(개발/파이썬)에 둔다. 현재 공부와의 다리는 → 🌐 [[_지식 연결 허브]].

## 1. `self`는 상속과 아무 상관이 없다
- `self`는 상속 여부와 무관하게, **파이썬 인스턴스 메서드(클래스 내부 함수)라면 무조건 첫 파라미터**로 들어간다.
- **`self`의 진짜 의미:** "나중에 이 클래스로 실제로 만들어질 **인스턴스(객체) 자기 자신**".
- 설계도(클래스)를 그릴 땐 나중에 생길 구체적 이름(예: `train_ds`)을 미리 알 수 없으니, 임시로 *"내 창고"·"내 레이블"*을 가리키려고 `self.features`·`self.labels`처럼 `self.`를 붙여 둔다.

## 2. 부모 접근은 `self`가 아니라 `super()`
부모 클래스의 자원(메서드·변수)에 접근할 땐 `self`가 아니라 **`super()`** 라는 별도 키워드를 쓴다.

```python
class Parent:
    def __init__(self):
        self.money = 1000          # 부모의 자산

class Child(Parent):               # Parent 상속
    def __init__(self):
        super().__init__()         # ① 부모 __init__ 실행 → self.money 세팅
        self.toy = "로봇"           # ② 나(Child)만의 새 변수

    def show_info(self):
        # self는 '나 자신' → 부모한테 물려받은 것도, 내 것도 모두 self로 접근
        print(f"물려받은 돈: {self.money}")
        print(f"내 장난감: {self.toy}")
```

- **`super().__init__()`** — 부모 생성자를 강제로 호출해 부모가 준비하는 상태(`self.money`)를 세팅.
- 한번 세팅되면 부모 것이든 내 것이든 **전부 `self.`로 접근**(상속받은 `self.money`도 `self`로).

> [!tip] 🪂 심화 — `super()`가 보일 때 vs 안 보일 때
> - **부모 기능을 켜야 하면 호출:** `class NeuralNetwork(nn.Module): def __init__(...): super().__init__()` → 부모(`nn.Module`)가 파라미터 추적 장치를 여기서 켠다.
> - **부모 기능을 안 쓰면 생략 가능:** `class ToyDataset(Dataset): def __init__(self, X, y):` 처럼 `super().__init__()`이 *없어도* 동작 — 부모의 초기화 기능을 강제로 쓸 일이 없기 때문. (그래서 "`super()`가 왜 안 보이지?"의 답)

## 3. `self`가 도는 원리 — '자동 주입'(문법적 설탕)
> 의문의 핵심: *"선언은 `(self, index)` 2개인데 호출은 `train_ds[0]`처럼 1개로 보인다. 개수가 왜 안 맞지?"*

파이썬은 메서드 호출을 내부적으로 이렇게 변환한다:

```python
train_ds.__getitem__(0)             # ① 우리가 쓰는 코드 (인수 1개처럼 보임)
ToyDataset.__getitem__(train_ds, 0) # ② 파이썬이 실제로 실행하는 연산
```

- 점(`.`) 앞에 있던 **인스턴스 자기 자신(`train_ds`)이 첫 번째 인자로 자동 주입**된다.
- 그래서 선언부 `(self, index)` 2개 = 변환된 호출부 `(train_ds, 0)` 2개로 **개수·순서가 정확히 일치.** 단지 점 문법을 쓸 때 첫 인자를 생략하게 해 주는 **문법적 설탕(Syntactic Sugar([sɪnˈtæktɪk ˈʃʊɡər], 신택틱 슈거, 신택틱 슈거))** 일 뿐.
- `self`라는 **이름은 강제가 아니다.** `def __getitem__(me, index):`로 써도 동작 — 파이썬은 첫 매개변수를 무조건 인스턴스로 취급. (다만 관례상 `self`로 통일)

## 4. 다른 언어(Java/JS)와의 결정적 차이 — `this`
> [!quote] 🌉 타 영역 연결 — 언어 설계 철학   `#연결/언어설계`
> Java·JS·C++는 `this`를 **엔진이 암묵적으로 숨겨** 처리한다(선언 안 해도 `this.x` 사용 가능). 파이썬엔 `this` 같은 예약어가 **없고**, 같은 것을 매개변수 자리에 **명시적으로** 드러낸다.
> ```javascript
> // JavaScript: this가 예약어로 숨겨져 있음
> class ToyDataset {
>   get_item(index) { return this.features[index]; }  // 선언 안 해도 this 사용
> }
> ```
> 파이썬 철학 **"명시적인 것이 암묵적인 것보다 낫다(Explicit is better than implicit([ɪkˈsplɪsɪt ɪz ˈbɛtər ðæn ɪmˈplɪsɪt], 익스플리싯 이즈 베터 댄 임플리싯, 익스플리싯 이즈 베터 댄 임플리싯))"** — 클래스 메서드도 결국 *인스턴스를 첫 인자로 받는 평범한 함수*임을 드러내려고 `self`를 매개변수로 끌어올린 것. 전역 다리: 🌐 [[_지식 연결 허브]]

## 5. 매직 메서드(던더) — 파이썬이 자동으로 부른다
`__init__`·`__len__`·`__getitem__`처럼 **언더스코어 2개로 감싼**(double-underscore = 던더) 메서드. 내가 직접 부르지 않고 **약속된 상황에서 파이썬이 자동 호출**한다.

| 매직 메서드 | 발동 트리거 | 의미 |
|---|---|---|
| `__init__(self, …)` | 객체 생성 `Cls(...)` | 초기화(속성 세팅) |
| `__len__(self)` | `len(obj)` | 길이 반환 |
| `__getitem__(self, i)` | 인덱싱 `obj[i]` | 한 항목 꺼내기 |

- 핵심: 두 던더(`__len__`·`__getitem__`)만 구현하면, `len()`·`[]` 문법을 쓰는 **어떤 코드든 내 객체를 표준 시퀀스처럼** 다룰 수 있다 = **덕 타이핑(duck typing([dʌk ˈtaɪpɪŋ], 덕 타이핑, 덕 타이핑))·프로토콜([ˈproʊtəkɒl], 프로토콜, 프로토콜)**.

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **`self`** | 인스턴스 메서드의 첫 파라미터. "생성될 객체 자기 자신". 상속과 무관 |
| **`super()`** | 부모 클래스 접근 키워드. `super().__init__()` = 부모 생성자 호출 |
| **자동 주입 / 문법적 설탕** | `obj.m(x)` → `Class.m(obj, x)`. 점 앞 객체가 첫 인자로 들어가는 변환 |
| **매직 메서드 / 던더** | `__이름__` 형태. 약속된 순간 파이썬이 자동 호출(`len()`·`[]` 등) |
| **덕 타이핑 / 프로토콜** | 약속된 메서드만 있으면 타입 무관하게 표준처럼 쓰이는 것 |
| **`this` (타 언어)** | Java/JS의 숨겨진 자기참조 예약어. 파이썬 `self`의 암묵적 버전 |
