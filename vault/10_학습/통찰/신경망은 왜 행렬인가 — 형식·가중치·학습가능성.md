# 신경망은 왜 행렬인가 — 형식·가중치·학습가능성

> "행렬이 신경망에 *쓰인다*"가 아니라 **"신경망의 *몸*이 곧 행렬이다."** 성적표 가중합 예제에서 출발해 — 신경망의 *형식(그림)* ↔ *행렬* ↔ *가중치* 를 한 줄로 꿰면, 행렬이 도구가 아니라 모델 자체임이 드러난다. 그리고 *왜 하필 행렬 형식인가* 의 가장 깊은 답은 **"학습 가능성이 행렬을 강제한다"** 이다.

> 💡 **통찰 노트(cross-domain)** — 수학(선형대수)·AI(신경망)·CS(Software 2.0)를 관통.
> 🌐 [[_지식 연결 허브]] ｜ 📖 [[_통찰 MOC]] ｜ 짝 통찰: [[왜 선형대수인가 — 데이터·알고리즘의 표준 언어|왜 선형대수인가]]

> 🎯 **한 문장**
> **신경망 = 데이터로 빚어지는, 미분 가능한 행렬 더미.** 그림의 엣지가 행렬 원소이고(형식↔행렬), 그 원소가 곧 학습되는 가중치이며(행렬↔가중치), *표현·합성·미분*이 한꺼번에 닫히는 유일한 구조라 행렬은 *선택이 아니라 필연*이다.

## 🧠 5단 전개 (구체 → 심층)

### 1. 신경망 *그림* 자체가 행렬이다
신경망 도식: 입력 노드 → 엣지 → 출력 노드. 여기서 **노드 = 벡터 성분, 엣지 = 행렬 원소.** 입력 $n$개·출력 $m$개를 모두 잇는 완전연결(fully-connected)이면 엣지가 $m\times n$개 = $m\times n$ 가중치 행렬 $W$. $W_{ij}$ = "입력 $j$ → 출력 $i$" 엣지의 굵기. ⇒ **신경망 그림을 그리는 것 = 행렬을 그리는 것.**

![[신경망 그림=가중치 행렬 대응 (재구성).png|860]]

🔄 **재구성** — (왼쪽) 신경망 한 층: 빨강 엣지 x1→y1 = $W_{11}$, 파랑 엣지 x3→y2 = $W_{23}$. (오른쪽) 같은 것이 $2\times3$ 가중치 행렬. 한 층 = $y=Wx$.

### 2. 가중치 = 행렬 원소 = 모델의 지식 그 자체
"학습한다" = **행렬 $W$의 원소(숫자)를 데이터로 조정**하는 것. 모델이 아는 모든 것 = 행렬 안의 숫자들. GPT의 "1.8조 파라미터" = 행렬 원소 개수. ⇒ 신경망 = **행렬 더미**, 가중치는 그 살. 행렬은 *연산 도구*이자 *모델 본체*. (가중치 한 뿌리: 선형결합 계수=가중평균=회귀계수=$Wx$ → [[2.2 행렬 곱셈·행렬식|행렬곱 원자]])

### 3. 왜 하필 *행렬* 형식? — 세 연산을 적용해도 결과가 다시 행렬 안에 머무는(closure, 닫힘) 유일한 구조 ⭐
학습 가능한 모델이 되려면 한 형식이 셋을 *동시에* 만족해야 한다:
- **표현** — 입력을 가중합으로 섞기 = 행렬곱 한 칸(내적).
- **합성** — 단순 블록을 쌓아 임의 복잡도 = 행렬곱 연쇄([[2.2 행렬 곱셈·행렬식|합성]]).
- **미분** — 경사하강이 $\partial \text{loss}/\partial W$ 를 요구 → **행렬곱을 미분하면 또 행렬(전치 $W^{\mathsf{T}}$)** 이 나와, **역전파 전체가 행렬 연산 안에서 돌아간다**(다른 도구로 새지 않음).

이렇게 표현·합성·미분 *어느 것을 적용해도 결과가 다시 행렬 안에 머무는*(연산에 대해 닫혀 있음 — closure, 닫힘) 유일한 구조가 행렬이다. ⇒ **"왜 행렬?"의 가장 깊은 답: 학습 가능성이 행렬 형식을 *강제*한다.** (같은 "닫힘"을 06 부분공간에서 이미 봄 — 덧셈·스칼라배에 닫힌 집합.)

### 4. 행렬의 *약점*이 신경망 깊이의 의미를 만든다 (반전 통찰)
순수 행렬곱만 쌓으면 **한 행렬로 붕괴**한다: $W_2(W_1 x)=(W_2 W_1)x$. 선형을 아무리 깊게 쌓아도 결국 선형 하나 → 깊이가 무의미. *그래서* 층 사이에 비선형(ReLU 등)을 끼워 붕괴를 막는다. ⇒ **행렬(선형)의 한계가 비선형의 필요성을 낳고, "선형(행렬) ↔ 비선형"의 교대가 표현력을 만든다.** 행렬을 이해하면 *왜 활성화 함수가 필수인지*까지 따라온다.

### 5. 행렬의 *모양(shape)* 이 곧 아키텍처 설계
$W$가 $m\times n$이면 $n$차원 입력 → $m$차원 출력. $m<n$=압축(병목·차원축소), $m>n$=확장. 신경망 설계 = **행렬 shape 사슬 설계**: 오토인코더=좁아졌다 넓어지는 모양, 분류기=마지막에 클래스 수로 좁힘. **모양이 곧 기능.**

> [!quote] 🌉 Software 2.0 — 코드를 숫자로 쓴다   `#연결/CS`
> 전통 프로그램 = 사람이 `if`/`for`를 짠다. 신경망 = 행렬 원소(가중치)가 데이터로 정해진 **연속적·미분 가능한 프로그램**. 행렬은 *코드를 숫자로 바꾼 것* — 그래서 경사하강으로 "프로그래밍"할 수 있다. (Karpathy "Software 2.0".) 전역 다리 → [[_지식 연결 허브]]

## 🔭 이 통찰이 자라는 곳 (예고)
- **4단(선형 붕괴 → 비선형 필요)** → 2단계 활성화 함수(ReLU·시그모이드)의 *존재 이유*.
- **3단(미분 가능성 = 역전파)** → 2단계 역전파([[04 자동 미분 (autograd)|autograd]])가 왜 전치·행렬 연산의 연쇄인지.
- **2단(행렬=모델)** → 3단계 LoRA(가중치 행렬을 저랭크로 근사·미세조정), 양자화(원소를 적은 비트로).

## 📌 핵심 한 줄 (암기)
- 신경망 그림 = 행렬(엣지=원소), 가중치 = 원소 = 모델의 지식.
- 행렬인 *이유* = 표현·합성·**미분**이 동시에 닫힘(학습가능성이 강제).
- 선형 합성은 한 행렬로 붕괴 → 그래서 비선형이 필수.
