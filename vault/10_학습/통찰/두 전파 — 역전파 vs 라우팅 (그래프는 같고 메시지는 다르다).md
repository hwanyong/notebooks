# 두 전파 — 역전파 vs 라우팅 (그래프는 같고 메시지는 다르다)

> "0단계에서 배운 **역전파(backprop([ˈbækprɒp], 백프롭, 백프롭))**가 **네트워크 라우터 전파**와 같은 개념 아닌가?" — 결론: **아니다.** 둘은 *노드-에지 그래프 위에서 정보가 전파된다*는 **위상(topology([təˈpɒlədʒi], 터팔러지, 토폴로지))만** 같을 뿐, 전파하는 **대상·수학·목적**이 다른 별개 알고리즘이다. 다만 더 깊이 보면 *진짜* 공통 뼈대가 있다 — **둘 다 "그래프 위 메시지 전달(message passing([ˈmɛsɪdʒ ˈpæsɪŋ], 메시지 패싱, 메시지 패싱))"**. (이 노트는 [[T01 경사 하강법과 훈련 루프|T01]]의 "같은 패러다임" 단정을 **정밀 교정**한다.)

> 💡 **통찰 노트(cross-domain)** — AI ↔ CS/네트워크 ↔ 그래프 이론.
> 🌐 [[_지식 연결 허브]] ｜ 📖 [[_통찰 MOC]]
> 🛠️ 관련: [[03 계산 그래프|계산 그래프(역전파 토대)]] · [[04 자동 미분 (autograd)|autograd]] · [[T01 경사 하강법과 훈련 루프|T01(교정 대상)]] · 그래프 알고리즘 [[왜 선형대수인가 — 데이터·알고리즘의 표준 언어|왜 선형대수인가]]

> 🎯 **핵심 (TL;DR)**
> - **같은 것 = 위상뿐.** 둘 다 *노드+에지 그래프* 위에서 값이 이웃으로 흐른다.
> - **다른 것 (별개 알고리즘):** ① 전파 대상 — **그래디언트(오차)** vs **경로 정보**. ② 수학 — **연쇄법칙·자동미분** vs **최단경로(Bellman-Ford([ˈbɛlmən fɔːrd], 벨먼 포드, 벨만-포드)·Dijkstra([ˈdaɪkstrə], 다이크스트러, 다익스트라))**. ③ 목적 — **가중치 학습** vs **패킷 전달**.
> - **진짜 공통 뼈대 = 그래프 위 메시지 전달(DP([diː piː], 디피, 디피)):** 역전파 = *역방향 자동미분*(메시지=그래디언트), 거리벡터 라우팅 = *Bellman-Ford*(메시지=거리 추정). 둘 다 "이웃 메시지를 받아 내 값 갱신". → GNN([dʒiː ɛn ɛn], 지엔엔, 지엔엔)이 이 둘을 일반화.
> - **교정:** "역전파 = 라우팅"은 과잉 단정. 정확히는 *위상+메시지전달 양식*이 같을 뿐.
> - **학습엔 역전파 필수:** 라우터식 *이산 선택*은 미분 불가라 그래디언트가 끊긴다. 현대 LLM([ɛl ɛl ɛm], 엘엘엠, 엘엘엠)의 **MoE([ɛm oʊ iː], 엠오이, 엠오이) '라우터'조차 역전파로 학습되는 *학습된 게이팅*** 일 뿐(네트워크 프로토콜 전파가 아님). → §5

## 1. 역전파 (Backpropagation([ˌbækprɒpəˈɡeɪʃən], 백프로퍼게이션, 백프로퍼게이션)) — 그래디언트를 역방향으로
**계산 그래프**(유향 그래프) 위에서, 출력 손실 $L$ 의 그래디언트를 **연쇄법칙**으로 *출력→입력 역방향*으로 흘려 각 가중치의 갱신 방향(`grad`)을 구한다. → 토대: [[03 계산 그래프|계산 그래프]], 엔진: [[04 자동 미분 (autograd)|autograd]].

> [!warning] ⚠️ 정확성 — 연쇄법칙은 "가중치를 가중치로 미분"하지 않는다   `#정확성`
> 흔한 오기: $\dfrac{\partial L}{\partial w_{in}} = \dfrac{\partial L}{\partial w_{out}}\cdot\dfrac{\partial w_{out}}{\partial w_{hidden}}\cdots$ — **틀림.** 가중치들은 서로 **독립 파라미터**라 $\partial w_{out}/\partial w_{hidden}$ 같은 항은 의미가 없다(0). 그래디언트는 **층의 활성/사전활성($a, z$)을 타고** 흐른다:
> $$ \frac{\partial L}{\partial w^{(l)}} = \frac{\partial L}{\partial a^{(L)}}\cdot\frac{\partial a^{(L)}}{\partial z^{(L)}}\cdots\frac{\partial z^{(l)}}{\partial w^{(l)}} $$
> 즉 "오차 신호가 *뉴런 값들*을 경유해 거슬러 오른다"가 정확. (가중치는 그 경로의 *끝단* 에서 한 번 곱해질 뿐.)

## 2. 라우팅 전파 (Routing Propagation([ˈruːtɪŋ ˌprɒpəˈɡeɪʃən], 루팅 프로퍼게이션, 라우팅 프로퍼게이션)) — 경로 정보를 교환
라우터들이 **경로 정보를 이웃과 교환**해 각자의 *라우팅 테이블*을 갱신, 패킷을 목적지로 포워딩한다. 두 방식:
- **거리벡터(Distance-Vector([ˈdɪstəns ˈvɛktər], 디스턴스 벡터, 디스턴스 벡터), 예 RIP([ɑːr aɪ piː], 알아이피, 알아이피)):** *이웃의 거리 추정*만 받아 "이웃까지 비용 + 이웃의 추정"으로 내 테이블 갱신 = **Bellman-Ford** 반복. 전체 지도 없이 국소 정보로 수렴.
- **링크상태(Link-State([lɪŋk steɪt], 링크 스테이트, 링크 스테이트), 예 OSPF([oʊ ɛs piː ɛf], 오에스피에프, 오에스피에프)):** 링크 상태를 망 전체에 *flooding([ˈflʌdɪŋ], 플러딩, 플러딩)* → 각자 전체 지도로 **Dijkstra** 최단경로 계산.

목적은 *학습*이 아니라 **데이터 패킷을 빠르고 안전하게 전달**.

## 3. 진짜 공통 뼈대 — '그래프 위 메시지 전달(message passing)'
표면의 "전파"라는 단어가 아니라, **계산 구조**를 보면 둘은 같은 *틀*을 공유한다 — **각 노드가 이웃이 보낸 메시지를 모아 자기 값을 갱신**하는 메시지 전달.

| | 역전파 | 거리벡터 라우팅 |
|---|---|---|
| 그래프 | 계산 그래프(DAG([diː eɪ dʒiː], 디에이지, 디에이지)) | 네트워크 토폴로지(순환 가능) |
| 메시지 | **그래디언트**(자식→부모) | **거리 추정**(이웃→나) |
| 갱신 규칙 | 연쇄법칙(곱 누적) | Bellman-Ford(min+합) |
| 실행 | DAG 1회 **역방향 패스**(정확) | 반복 **수렴**(근사, 플래핑 가능) |
| 일반화 | \-- | 둘 다 **GNN**의 특수형(메시지 패싱 신경망) |

- 이 "메시지 전달 DP"는 [[왜 선형대수인가 — 데이터·알고리즘의 표준 언어|왜 선형대수인가]]의 **그래프=인접행렬·Bellman-Ford** 와 직결. *반정환(semiring([ˈsɛmɪrɪŋ], 세미링, 세미링))만 바꾸면* 같은 골격(역전파=곱·합, 최단경로=합·min).
- 차이의 핵심: 역전파는 **비순환(DAG) 1패스**라 정확·종료 보장, 거리벡터는 **순환 그래프 반복**이라 수렴·진동(플래핑) 이슈.

## 4. 그래서 T01의 "같은 패러다임"은 — 정밀 교정
[[T01 경사 하강법과 훈련 루프|T01]] §🌉는 "둘은 같은 수학적 패러다임 (맞다)"라 단정했지만, 정확히는:
- **같은 알고리즘/패러다임이 아니다.** 같은 건 *위상(그래프)+메시지전달 양식*.
- **유효한 비유**는 "*경사하강 최적화의 수렴 동역학* ↔ *라우팅 수렴 동역학*"(둘 다 국소 정보 반복 → 진동·국소최적·댐핑). 이건 **역전파(그래디언트 계산)가 아니라 경사하강(최적화)** 쪽 비유다. T01이 둘을 한 칸에 묶은 걸 분리.
- **RouteNet([ˈruːtnɛt], 루트넷, 라우트넷)·BBR([biː biː ɑːr], 비비알, 비비알)·SDN([ɛs diː ɛn], 에스디엔, 에스디엔)**은 "라우팅 = 역전파"의 증거가 아니라 **네트워크에 ML([ɛm ɛl], 엠엘, 엠엘)/그래디언트 기법을 *적용*한 사례**(반대 방향 응용).

## 5. 왜 역전파를 라우팅으로 *대체* 못 하나 — 미분 장벽 + MoE 절충
**핵심 축 = 학습 vs 추론 환경 분리.** 두 과정은 가중치를 *바꾸느냐 / 고정하느냐*가 갈린다.

| | 학습(Training) | 추론(Inference) |
|---|---|---|
| 가중치 | *바꾸는* 중(미완성) | **고정(Frozen([ˈfroʊzən], 프로즌, 프로즌), read-only([riːd ˈoʊnli], 리드 온리, 리드온리))** |
| 필수 | **역전파**(연속 미분으로 가중치 갱신) | 가중치 갱신 없음 → **라우팅/조건부 연산**이 위력 |
| 라우팅 | 이산 라우팅은 미분 끊김 → 직접 못 씀 | 입력별로 *필요한 노드만* 골라 타 비용↓·지연↓ |

- **미분 장벽(핵심):** 라우터식 *이산 선택*("이 패킷은 3번 포트로")을 노드마다 넣으면 그 지점에서 **미분 불가(non-differentiable([nɒn ˌdɪfəˈrɛnʃəbəl], 논 디퍼렌셔블, 논 디퍼렌셔블))** → 그래디언트가 끊긴다. 역전파의 본질은 "가중치를 0.00001 만큼 흔들면 손실이 얼마나 변하나"는 *연속 기울기*인데, 이산 선택은 그 미세 변화를 정의할 수 없다.
  - ⚠️ *정확성:* Gemini([ˈdʒɛmɪnaɪ], 제미나이, 제미나이)는 이를 "조합 최적화 NP-hard([ɛn piː hɑːrd], 엔피 하드, 엔피 하드)"라 했지만 과한 표현 — 정확히는 **이산 결정이 경사 흐름을 끊어 경사기반(gradient([ˈɡreɪdiənt], 그레이디언트, 그레이디언트)) 학습이 불가**한 것이 핵심.
- **노드 목적 차이:** 라우터=원본 패킷을 *훼손 없이 전달(forwarding([ˈfɔːrwərdɪŋ], 포워딩, 포워딩))*. LLM 노드=입력을 *비틀고 압축해 의미 추출(transformation([ˌtrænsfərˈmeɪʃən], 트랜스퍼메이션, 트랜스포메이션); `Wx+b`+비선형)*. → 단순 전파만으로는 이 '연산 공장'의 가중치를 깎을 수 없다. ([[05 퍼셉트론·편향·은닉층|Wx+b]])
- **반전 — MoE(Mixture of Experts([ˈmɪkstʃər əv ˈɛkspərts], 믹스처 오브 엑스퍼츠, 믹스처 오브 엑스퍼츠)):** 현대 초거대 LLM은 실제로 '라우터'를 넣는다(Mixtral([ˈmɪkstrəl], 믹스트럴, 믹스트랄)·DeepSeek-V3([diːpsiːk viː θriː], 딥시크 브이쓰리, 딥시크 V3)·GPT-4([dʒiː piː tiː fɔːr], 지피티 포, 지피티-4) 추정). 내부에 여러 **전문가(Experts([ˈɛkspərts], 엑스퍼츠, 엑스퍼츠))** 망 + 게이팅 '라우터'가 토큰을 *일부 전문가에게만* 보냄 = **희소 활성(sparse activation([spɑːrs ˌæktɪˈveɪʃən], 스파스 액티베이션, 스파스 액티베이션))**. 1조 파라미터라도 토큰당 일부만 켜져 연산 비용↓.
  > [!warning] ⚠️ 정확성 — MoE는 역전파를 *대체하지 않는다*   `#정확성`
  > - MoE도 **역전파로 학습**된다. 게이팅이 "1번 80%·2번 20%" 식 **연속 확률값**을 주므로 *미분 가능* → 라우터 가중치도 모델과 함께 학습된다(= **미분 가능 라우팅, Differentiable Routing([ˌdɪfəˈrɛnʃəbəl ˈruːtɪŋ], 디퍼렌셔블 루팅, 디퍼렌셔블 라우팅)**). top-k([tɒp keɪ], 탑 케이, 탑케이) 이산 선택의 미분 불가 부분은 **보조 로드밸런싱 손실** + *선택된 전문가로의 그래디언트*로 우회. 즉 "라우팅을 *학습 가능*하게 만든 것".
  > - MoE의 '라우팅'은 **학습된 게이팅 네트워크**이지 BGP([biː dʒiː piː], 비지피, 비지피)/OSPF 같은 *네트워크 라우팅 프로토콜 전파*가 아니다(이름만 빌린 비유). 학습엔 여전히 연속 미분(역전파)이 필수이고, 라우터는 *어느 전문가를 쓸지* 만 고른다.
- **정리:** 학습=연속 미분(역전파) 절대 필수 / 추론·거대모델 통제=라우터식 *희소 라우팅*(MoE)이 핵심. 둘은 *대체*가 아니라 **역할 분담**(라우터도 역전파로 학습됨).

> [!info] 🔭 커리큘럼 전방 연결 — 추론 효율의 두 갈래   `#배경`
> "추론은 *고정된 가중치*로 효율을 짜내는 단계"라는 이 관점은 곧 만날 단계로 이어진다:
> - **저차원 분해(SVD([ɛs viː diː], 에스브이디, 에스브이디)·LoRA([ˈlɔːrɑː], 로라, 로라)):** 고정 가중치 행렬의 차원을 깎아 추론 효율↑ → [[_3단계 - 트랜스포머와 LoRA|3단계 트랜스포머·LoRA]] (LoRA=저차원 적응).
> - **추론 인프라(latency([ˈleɪtənsi], 레이턴시, 레이턴시)·throughput([ˈθruːpʊt], 쓰루풋, 스루풋)·라우터/캐시 오케스트레이션):** → [[_4단계 - 커리어 분기|4단계 커리어 분기(AI 엔지니어 루트)]].

## 🧠 통찰 — 한 줄
**"전파"라는 같은 단어에 속지 말 것.** 역전파와 라우팅은 *그래프 위에서 정보가 흐른다*는 위상과 *메시지 전달*이라는 계산 골격을 공유하지만, **무엇을(그래디언트 vs 경로)·어떤 수학으로(연쇄법칙 vs Bellman-Ford)·왜(학습 vs 전달)** 흘리는지가 다른 별개 알고리즘이다. 구조(위상)의 유사성과 알고리즘의 동일성을 구분하는 게 융합적 이해의 핵심.

> [!quote] 🌉 관통 — 그래프 위 메시지 전달이라는 한 골격   `#연결/CS` `#연결/AI`
> 역전파(역방향 자동미분)·거리벡터 라우팅(Bellman-Ford)·GNN은 모두 *노드가 이웃 메시지로 자기 값을 갱신*하는 메시지 전달. 반정환만 갈아끼우면 같은 뼈대 → [[왜 선형대수인가 — 데이터·알고리즘의 표준 언어|그래프 알고리즘=행렬화]]. 전역 다리 → [[_지식 연결 허브]]

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **역전파** (Backpropagation) | 계산 그래프 위 손실 그래디언트를 연쇄법칙으로 역방향 전파(=역방향 자동미분) |
| **라우팅 전파** (Routing Propagation) | 라우터가 경로 정보를 이웃과 교환해 테이블 갱신·패킷 포워딩 |
| **거리벡터 / Bellman-Ford** | 이웃 거리 추정만으로 최단경로를 반복 수렴(RIP) |
| **링크상태 / Dijkstra** | 링크상태 flooding 후 전체 지도로 최단경로(OSPF) |
| **메시지 전달** (Message Passing) | 노드가 이웃 메시지를 모아 자기 값 갱신(역전파·라우팅·GNN의 공통 골격) |
| **위상 유사성** (Topological similarity([ˌtɒpəˈlɒdʒɪkəl ˌsɪmɪˈlærɪti], 토폴로지컬 시밀래러티, 토폴로지컬 시밀래리티)) | 알고리즘은 달라도 같은 노드-에지 그래프 구조를 공유함 |
