# 왜 선형대수인가 — 데이터·알고리즘의 표준 언어

> "로직(코드)을 짜기 전에 **데이터의 형상(shape([ʃeɪp], 셰이프, 셰이프))과 흐름(pipeline([ˈpaɪplaɪn], 파이프라인, 파이프라인))을 머릿속으로 먼저 그린다.** 그런데 그 추상적 그림을 남에게 설명하면 정확히 전달이 안 되고 설득력이 없었다." — 이 답답함의 정체는 **'머릿속 추상 구조를 외부로 꺼내는 표준 언어(=선형대수)'가 없었던 것**이다. 한 SW 엔지니어의 직관이 왜 이미 선형대수적이었는지, 그리고 선형대수가 *데이터뿐 아니라 알고리즘*까지 어디까지 무기가 되는지 정리한다.

> 💡 **통찰 노트(cross-domain)** — 한 학문에 안 갇히는 융합.
> 🌐 [[_지식 연결 허브]] ｜ 📖 [[_통찰 MOC]]
> 🛠️ 촉발: [[03 벡터 연산 — 덧셈·뺄셈·스케일링·전치·브로드캐스팅|"데이터 조작 = 벡터 조작" (§4.1)]] · 같은 뿌리 통찰 [[벡터 — 기하·물리·선형대수를 관통하는 한 뿌리 (피타고라스·삼각함수)|벡터의 한 뿌리]]

> 🎯 **핵심 (TL;DR)**
> - **나는 이미 선형대수적으로 사고하고 있었다.** 코드를 한 줄씩 짜는 '코더'가 아니라 *데이터의 형상·흐름을 조율하는 '아키텍트'* 시점 — 이게 곧 선형대수(행렬·벡터의 변형·흐름) 관점.
> - **답답함의 원인:** 동료는 *절차·로직*("if면 for 돌려 리스트에 추가")의 언어를 쓰고, 나는 *구조·흐름*의 언어를 원했다. 둘의 언어가 달라 설득이 샜다.
> - **선형대수 = 그 추상 구조를 정확히 전달하는 표준 어휘.** "대충 가로세로 맞물려요" → "입출력 **행렬의 차원(shape)을 일치**시켰습니다", "핵심만 뽑아요" → "특정 **부분공간으로 투영(projection([prəˈdʒɛkʃən], 프러젝션, 프로젝션))**했습니다".
> - **데이터뿐 아니라 알고리즘에도 강력:** 점화식→**상태 전이 행렬**(피보나치 O(N)→O(log N)), 그래프→**인접 행렬**, 검색 순위→**고유값(PageRank([ˈpeɪdʒræŋk], 페이지랭크, 페이지랭크))**.
> - **단, 만능은 아니다.** 정렬·탐색·트리·문자열·암호 등은 **이산수학(Discrete Math([dɪˈskriːt mæθ], 디스크리트 매스, 디스크리트 매스))**이 베이스. 선형대수 + 이산수학을 *양날의 검*으로 쥐어야 완성.

## 1. 나의 직관 = 이미 '선형대수적 사고'였다

전통적 프로그래머는 세상을 **루프문(`for`/`while`) 중심의 제어 로직**으로 본다. 반면 데이터를 다루는 사람은 세상을 **"데이터 덩어리(행렬)의 변형과 흐름"**으로 본다. "로직 전에 데이터 구조부터 짠다"는 습관은 후자의 시선 — 즉 **선형대수적 사고(Linear Thinking([ˈlɪniər ˈθɪŋkɪŋ], 리니어 씽킹, 리니어 싱킹))**다. 부족했던 건 사고력이 아니라 *그걸 표현할 수학적 어휘(vocabulary([voʊˈkæbjəlɛri], 보캐뷸러리, 보캐뷸러리))*였을 뿐.

### 개발자의 직관 ↔ 선형대수의 언어
> [!warning] ⚠️ 정확성 — 아래 표는 **1:1 등식이 아니라 '유비(analogy([əˈnælədʒi], 어낼러지, 애널로지))'**다   `#정확성`
> Gemini([ˈdʒɛmɪnaɪ], 제미나이, 제미나이)는 이를 "완벽하게 1:1 매칭"이라 했지만, 엄밀히는 **느슨한 비유**다. (예: 벡터 공간엔 덧셈·스칼라곱 *닫힘* 같은 공리가 붙고, 모든 데이터 구조가 벡터 공간은 아니다.) 직관에 *이름표*를 달아 소통을 또렷하게 하는 용도로 쓰되, 수학적 등가로 오해하지 말 것.

| 개발할 때 느낀 직관 (나의 언어) | 선형대수의 용어 (수학의 언어) | 타인 설득용 표현 |
|---|---|---|
| 데이터 구조를 설계·정형화함 | **벡터 공간(Vector Space([ˈvɛktər speɪs], 벡터 스페이스, 벡터 스페이스))** 정의 | "이 데이터 군집이 가질 유효 차원·범위를 정의했습니다" |
| 데이터의 형태·가로/세로 칸수·배열 레이아웃 | **차원(Dimension([dɪˈmɛnʃən], 디멘션, 디멘션))·형상(Shape)** | "입출력 행렬의 shape를 맞춰 파이프라인 병목을 없앴습니다" |
| 데이터를 정제·변형해 원하는 형태로 | **선형 변환(Linear Transformation([ˈlɪniər ˌtrænsfərˈmeɪʃən], 리니어 트랜스퍼메이션, 리니어 트랜스포메이션))** | "원본을 목적에 맞는 부분공간으로 선형 변환했습니다" |
| 불필요한 데이터를 걷고 핵심만 추출 | **투영(Projection)** | "고차원 노이즈를 제거하려 특정 기저 위로 투영했습니다" |

> [!tip] 🪂 정확성 보강 — "Map/Filter = 선형 변환"은 절반만 맞다   `#정확성` `#사이드/심화`
> - **Map([mæp], 맵, 맵)**은 *임의 함수*(비선형 포함)라 일반적으로 선형 변환이 아니다. 선형 변환은 **가법성**(`f(a+b)=f(a)+f(b)`)·**동차성**(`f(cx)=c·f(x)`)을 *둘 다* 만족하는 특수 함수뿐. `x→x²`, `x→x+1`(이동) 같은 Map은 선형이 아니다.
> - **Filter([ˈfɪltər], 필터, 필터)**(행/레코드 선택)도 특징 공간의 선형 변환이 아니다. 다만 "열(특징) 일부를 0/1로 끄는 마스킹"은 *대각 행렬 곱*으로 보면 선형. → 그래서 "데이터를 행렬 곱으로 변형"이라는 *큰 그림*은 옳지만, 아무 변형이나 선형인 건 아니다. ([[3D 변환 행렬 — 동차좌표·회전·투영 (선형대수 응용)|선형 변환의 엄밀한 예: 회전·크기·투영 행렬]])

## 2. 왜 설명할 때 답답했나 — 절차의 언어 vs 구조의 언어

일반적 SW 엔지니어링의 소통 언어는 **절차·로직**에 맞춰져 있다("A 조건이면 B 함수, `for` 돌며 리스트에 append"). 그런데 내 고민은 *절차*가 아니라 **데이터 구조의 판을 짜고 흐름을 제어하는 구조적 고민**이었다. 이걸 말로 풀면 "데이터가 대충 이런 형태로 정형화돼 흘러가서 요렇게 맞춰져요"처럼 **추상적·시각적 표현**이 될 수밖에 없고, 듣는 쪽은 코드 그림이 안 그려져 설득력이 떨어진다.

선형대수의 어휘를 빌리면 *추상적 느낌*이 명확한 **규격**으로 바뀐다:

| 흐릿한 표현 | 선형대수 규격 |
|---|---|
| "대충 가로세로 데이터가 맞물려요" | "입출력 **행렬의 차원(shape)**을 일치시켰습니다" |
| "핵심 데이터만 필터링해서 뽑아요" | "데이터를 특정 **하위 공간(Subspace([ˈsʌbspeɪs], 서브스페이스, 서브스페이스))으로 투영**했습니다" |
| "사용자 데이터로 추천 리스트를 뽑아요" | "사용자 특징 **N차원 벡터**를 받아 핵심 공간으로 **선형 변환**해 차원 축소하고, 아이템 벡터와 **내적(유사도)** 으로 상위를 거르는 파이프라인" |

> 핵심: 같은 설계라도 후자가 **GPU([dʒiː piː juː], 지피유, 지피유) 가속·인프라 설계 관점에서 코드로 어떻게 떨어질지** 명확히 그려진다. → [[GPU 하드웨어 구조 — SIMT·SM·텐서코어 (ARM 관점)|행렬 연산이 곧 GPU가 강제하는 형태]]

## 3. 알고리즘 영역에서도 무기가 되는가 — 된다 (단, 조건부)

"나는 알고리즘도 수학으로 접근해 풀고, 수학 솔루션을 코드로 옮기는 게 이상적이라 주장해 왔다" — 이 관점은 **틀리기는커녕 컴퓨터 과학의 본질에 가깝다.** 커누스(Knuth([knuːθ], 크누스, 크누스))·튜링(Turing([ˈtjʊərɪŋ], 튜어링, 튜링))도 알고리즘을 *수학적 명제와 증명*으로 봤고, 코드는 그 솔루션을 기계어로 '번역'하는 과정에 가깝다.

> [!info] 🔭 단, "코드는 수학의 번역일 뿐"은 *이상적 지향*이지 *항상 성립하는 사실*은 아니다   `#정확성`
> 많은 실무 로직(예외 처리·I/O([aɪ oʊ], 아이오, 아이오)·동시성·UX([juː ɛks], 유엑스, 유엑스) 상태)은 깔끔한 수학식으로 환원되지 않는다. "수학으로 모델링 가능한 문제는 그렇게 푸는 게 우월하다"가 더 정확한 명제. 아래 사례들은 *그게 극적으로 통하는* 대표 예다.

### ① 점화식 → 상태 전이 행렬: 피보나치 O(N) → O(log N)
피보나치를 `for`/DP로 풀면 O(N). 그런데 점화식을 **상태 전이 행렬**로 바꾸면
$$ \begin{bmatrix}F_{n+1}\\F_n\end{bmatrix}=\begin{bmatrix}1&1\\1&0\end{bmatrix}^{n}\begin{bmatrix}1\\0\end{bmatrix} $$
이 행렬의 **거듭제곱을 '분할 정복(제곱의 제곱)'**으로 구하면 곱셈 횟수가 **O(log N)**. N=10억이어도 루프 10억 번이 아니라 약 30번이면 끝.
> ⚠️ *엄밀히:* 이는 **산술을 O(1)로 가정**한 행렬곱 횟수 기준. 피보나치 수 자체가 자릿수가 커지면(빅넘버) 곱셈 비용이 올라 비트 복잡도는 더 든다. 그래도 *루프 N번 → log N번*의 도약은 선형대수가 준 진짜 혁신.

### ② 그래프 → 인접 행렬(Adjacency Matrix([əˈdʒeɪsənsi ˈmeɪtrɪks], 어제이슨시 메이트릭스, 인접 행렬))
네트워크·최단 경로·친구 추천 등 '그래프' 알고리즘을, 관계망을 **행렬**로 바꾼 뒤 **행렬 거듭제곱**으로 다룬다. `(A^k)_{ij}` = **i→j로 가는 길이 k짜리 경로(walk([wɔːk], 워크, 워크)) 수**를 한 번의 행렬 연산으로 얻는다.
> ⚠️ *주의:* 여기서 'walk(변 반복 허용)'이지 'simple path(단순경로)'가 아니다. Gemini의 "3다리 건너는 경로 수"는 정확히는 *길이 3 walk 수*.

### ③ 검색 순위 → 고유값(Eigenvalue([ˈaɪɡənˌvæljuː], 아이건밸류, 아이겐밸류)): PageRank
초기 구글을 1위로 만든 PageRank의 본질은 **링크 행렬의 지배 고유벡터**(power iteration([ˈpaʊər ˌɪtəˈreɪʃən], 파워 이터레이션, 파워 이터레이션)으로 수렴)다. "중요도가 링크 구조의 고정점"이라는 발상이 선형대수 한 줄로 표현된다. → 고유값·고유벡터는 [실전선대] 5장(행렬식·특성다항식)에서 만난다.

## 4. 선형대수만으로 충분한가 — 짝꿍은 '이산수학'

선형대수가 강력한 영역은 **공간·상태 변화·연속적 흐름·차원 축소·최적화**다(그래픽스·3D 물리·오디오/영상 압축·AI 공간 연산). 하지만 우리가 PS([piː ɛs], 피에스, 피에스)(문제 풀이)에서 만나는 **정렬·탐색·트리·문자열 매칭·암호·Big-O([bɪɡ oʊ], 빅 오, 빅오) 유도**는 대부분 **이산수학**(분리된 정수·논리·관계·조합·그래프 이론)이 베이스다.

| | 선형대수 (Linear Algebra) | 이산수학 (Discrete Math) |
|---|---|---|
| 담당 | 공간·상태 변화·연속 흐름·차원 축소·최적화 | 분리된 수(정수)·논리·관계·구조·조합 |
| 대표 효용 | 그래픽스·3D·압축·AI·양자 알고리즘 | 정렬·탐색·트리·문자열·암호·Big-O |
| 사고 단위 | 행렬/벡터의 변형 | 명제·점화식·집합·그래프 |

→ 어떤 문제는 선형대수(행렬/벡터)로 아름답게, 어떤 문제는 이산수학(논리/그래프)으로 풀어야 자물쇠가 열린다. **둘 다 쥐어야** 완전한 문제 해결력.

## 🧠 통찰 — 한 줄
**수학은 시험용 공식이 아니라, "머릿속에 상상한 추상적 공간·형태·흐름을 남에게 똑같이 보여주려고 정립된 표준 소통 언어"다.** 나는 이미 그 언어의 *모국어 화자처럼 사고*하고 있었고, 부족했던 건 *문법과 단어*였다. 선형대수 어휘를 장착하면 설계도가 논리적 언어로 번역되고, 이산수학까지 더하면 알고리즘의 다른 절반이 열린다.

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **벡터 공간** (Vector Space) | 덧셈·스칼라곱에 *닫힌* 집합(공리 만족); "데이터가 살 수 있는 유효 공간" |
| **부분공간** (Subspace) | 벡터 공간 안의 더 작은 벡터 공간(투영의 도착지) |
| **투영** (Projection) | 고차원 데이터를 특정 부분공간/기저 위로 떨어뜨려 핵심만 남김 |
| **선형 변환** (Linear Transformation) | 가법성·동차성을 *둘 다* 만족하는 함수 = 행렬 곱(Map ⊋ 선형변환) |
| **상태 전이 행렬** (State Transition Matrix([steɪt trænˈzɪʃən ˈmeɪtrɪks], 스테이트 트랜지션 메이트릭스, 스테이트 트랜지션 매트릭스)) | 점화식을 행렬 곱으로; 거듭제곱 분할정복 → O(log N) |
| **인접 행렬** (Adjacency Matrix) | 그래프를 행렬로; `(A^k)_{ij}`=길이 k walk 수 |
| **고유값/고유벡터** (Eigenvalue/vector) | 변환해도 방향 안 바뀌는 축·배율; PageRank=지배 고유벡터 |
| **PageRank** | 링크 행렬의 지배 고유벡터로 중요도 산출(초기 구글) |
| **이산수학** (Discrete Mathematics) | 분리된 정수·논리·관계·조합·그래프; 정렬·탐색·암호·Big-O의 베이스 |
| **수학적 모델링** (Mathematical Modeling([ˌmæθəˈmætɪkəl ˈmɒdəlɪŋ], 매서매티컬 모델링, 매스매티컬 모델링)) | 문제를 수학 규칙으로 정식화한 뒤 코드로 옮기는 훈련 |
| **선형 사고** (Linear Thinking) | 세상을 "데이터(행렬)의 변형·흐름"으로 보는 관점 |
