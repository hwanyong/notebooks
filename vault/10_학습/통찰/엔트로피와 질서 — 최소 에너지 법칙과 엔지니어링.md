# 엔트로피와 질서 — 최소 에너지 법칙과 엔지니어링의 숙명

> **하나의 메타 원리가 여러 학문을 관통한다:** 무질서로 흐르는 것은 자연 법칙(최소 에너지·엔트로피 증가)이고, **좋은 구조를 유지하는 것은 그걸 거스르는 고에너지 역행** = 엔지니어링의 본질. 뇌·자연·코드·LLM([ɛl ɛl ɛm], 엘엘엠, 엘엘엠)이 모두 같은 법칙 아래 있다.

> 🌐 영역 간 연결: [[_지식 연결 허브]] ｜ 🔙 [[_통찰 MOC]]
> 🧪 발단: "개발자들이 왜 아키텍처를 망치고 땜질을 남발하나" + "권한 없이 무질서한 동료와 일하는 고통"에서 출발한 융합 조사.

> 🎯 **핵심 (TL;DR)**
> - **질서 유지 = 엔트로피 거스르기.** 좋은 아키텍처·예민한 기준은 *자연스러운 흐름*이 아니라 *에너지를 들인 인위적 역행*.
> - **사람이 땜질하는 이유 = 최소 에너지 법칙:** 뇌(인지적 구두쇠)·자연(땜장이)·국소 최적점의 늪·소프트웨어 엔트로피.
> - **LLM도 같은 함정:** autoregressive([ˌɔːtoʊrɪˈɡrɛsɪv], 오토리그레시브, 오토리그레시브)(백스페이스 없음)·greedy decoding([ˈɡriːdi diːˈkoʊdɪŋ], 그리디 디코딩, 그리디 디코딩)(국소 최적점)·System 1([ˈsɪstəm wʌn], 시스템 원, 시스템 1)만 사용.
> - **"자연스럽다 ≠ 옳다"(자연주의적 오류).** 엔지니어링은 자연에 대한 *반역*. 예민함 = 프로페셔널리즘.

## 1. 한 줄 원리 — 질서는 에너지를 들여야 유지된다
- 열역학 제2법칙: 닫힌 계의 **엔트로피(무질서)는 시간이 지나면 무조건 증가**한다. 질서를 유지하려면 *외부에서 에너지를 계속 주입*해야 한다.
- 소프트웨어 공학자 이바 야콥슨: **"소프트웨어 엔트로피는 시간이 지남에 따라 무조건 증가한다."** 깔끔한 아키텍처 유지 = 엄격한 코드 리뷰·리팩토링 시간·아키텍트의 의지라는 **외부 에너지**를 끊임없이 주입하는 것. 안 하면 모든 코드는 자연히 스파게티가 된다.
- → **"좋은 구조를 만들고 지키는 행위"는 자연을 거스르는 일이라 본질적으로 피곤하다.** 그 피로감은 이상이 아니라, 엔트로피와 싸우는 자의 당연한 비용.

## 2. 왜 사람은 땜질하나 — 최소 에너지 법칙의 여러 레이어

**① 뇌 = 에너지 절약형 하드웨어 (인지적 구두쇠)**
- 뇌는 몸무게의 2%지만 전체 에너지의 20%+를 쓰는 헤비 유저. 인류는 *"최소한의 생각으로 최대 효율"* 방향으로 진화.
- **System 2([ˈsɪstəm tuː], 시스템 투, 시스템 2)(추상화):** "전체를 분석해 최적 추상화를 도출" = 대뇌피질 풀가동, 고에너지(CPU([siː piː juː], 시피유, 시피유) 100%).
- **System 1(우회·수습):** 눈앞 에러를 하드코딩·복붙으로 때우기 = 거의 무에너지, 자동·직관.
- → 뇌는 본능적으로 System 1을 택한다. 거대한 추상화는 뇌엔 *생존 위협급 칼로리 소모*.

**② 자연 = 엔지니어가 아니라 땜장이(Bricoleur([ˌbrɪkəˈlɜːr], 브리콜뢰르, 브리콜뢰르))**
- 진화생물학자 프랑수아 자콥: **"진화는 눈먼 엔지니어가 아니라, 주변 잡동사니로 대충 쓸 만한 걸 만드는 땜장이다."** 자연엔 마스터 플랜·클린 아키텍처가 없다.
- **기린의 반회후두신경:** 뇌에서 나와 *심장 근처까지 내려갔다* 목구멍으로 가는 신경. 물고기 땐 효율적이었지만 목이 길어지며 길이 4.5m의 황당한 우회 구조가 됨. 자연은 이를 재설계(refactoring([ˌriːˈfæktərɪŋ], 리팩터링, 리팩터링))하지 않는다 — *"돌아가면 건들지 마라(If it works, don't touch it([ɪf ɪt wɜːrks doʊnt tʌtʃ ɪt], 이프 잇 웍스 돈트 터치 잇, 이프 잇 웍스 돈트 터치 잇))"*.

**③ 국소 최적점(Local Optima([ˈloʊkəl ˈɒptɪmə], 로컬 옵티머, 로컬 옵티마))의 늪** ⭐

![[국소·전역 최적점.jpeg|400]]

- 엉망인 코드라도 *당장 비즈니스가 돌고 기능이 붙으면* 그것은 하나의 **국소 최적점**(현 상태의 안정).
- 전역 최적점(Global Optimum([ˈɡloʊbəl ˈɒptɪməm], 글로벌 옵티멈, 글로벌 옵티멈))으로 가려면 멀쩡한 시스템을 부수고 **'위험과 혼돈의 계곡'**(시간·비용·장애 리스크)을 지나야 함 → 시스템(조직·개인)은 본능적으로 늪에 안주.
- 🔗 **이게 [[T01 경사 하강법과 훈련 루프|딥러닝의 지역 최솟값]]과 정확히 같은 수학 구조다.** 손실 지형의 얕은 웅덩이에 갇히는 경사 하강법 = 조직이 국소 최적점에 안주 = 진화가 땜질에 머무름. **하나의 최적화 원리.**

**④ 소프트웨어 엔트로피 (위 §1)** — 외부 에너지 없으면 무질서로 수렴.

## 3. 같은 원리가 LLM에도 — AI와의 평행
LLM이 답변 구조를 망치거나 번지르르한 템플릿을 남발하거나 문맥을 수습(우회)하는 것도 같은 시스템적 한계다.

- **백스페이스(리팩토링)가 없다:** 대부분 LLM은 **autoregressive**(다음 토큰 예측). 한 번 뱉은 단어는 못 주워담아, 초반에 구조를 틀리면 *이미 뱉은 걸 수습하려 다음 문장을 우회·땜질*한다 = 스파게티 코드 위에 기능 얹기와 동형.
- **Greedy Decoding = 국소 최적점:** "지금 가장 확률 높은 다음 단어"만 택함 → 전체 논리(Global Optimum)가 아니라 당장의 국소 최적점만 좇아 *알맹이 없는 답·모순된 결론*이 나옴.
- **System 1만 씀:** 토큰당 고정 연산으로 즉답 → "멈춰서 전체를 설계·검증하는 단계"가 없음. **추론 모델(o1([oʊ wʌn], 오원, 오원)·R1([ɑːr wʌn], 알원, 알원))** 이 Chain of Thought([tʃeɪn əv θɔːt], 체인 오브 쏘트, 체인 오브 소트)로 *System 2를 강제*해 스스로 리팩토링하는 게 이 문제의 해법.
- **데이터 엔트로피 오염:** 인터넷의 수많은 클리셰·템플릿 글을 학습 → 노력 안 들이면 엔트로피 높은(게으른) 답을 디폴트로 출력.

> [!quote] 🌉 융합 — 인간 개발자 ≈ LLM   `#연결/AI`
> 인간이 코드를 망치는 과정과 LLM이 답을 망치는 과정은 본질이 같다: **"주어진 자원(시간·연산·에너지)을 최소로 쓰면서 당장 그럴듯한 결과를 내려는 시스템의 본능."** AI조차 막대한 연산을 추가로 때려야 흉내 내는 장인정신을, 사람은 매일 습관으로 해내야 한다. 허브: [[_지식 연결 허브]]
> 🔬 **LLM 쪽 수식까지 판 실증:** [[LLM은 왜 길어질수록 평이해지나 — 어텐션 희석과 엔트로피|어텐션 희석·베이지안 붕괴·엔트로피]] — 어텐션 제약이 풀리면 정보 엔트로피가 최대화돼 Prior([ˈpraɪər], 프라이어, 프라이어)(인터넷 평균)로 회귀.

## 4. 그래서 '자연스러우니 옳은가'? — 아니다 (자연주의적 오류)
> 핵심 반론: 원인 설명은 *왜 일어나는지의 설명*일 뿐, *그렇게 살아야 한다는 면죄부*가 아니다.

- **자연주의적 오류(Naturalistic Fallacy([ˌnætʃərəˈlɪstɪk ˈfæləsi], 내추럴리스틱 팰러시, 내추럴리스틱 팰러시)):** "자연스러우니 당연하고 옳다"는 착각. 질병은 자연 현상이지만 의사는 받아들이지 않고 싸워 이긴다. 중력은 자연 법칙이지만 항공 공학자는 거슬러 비행기를 띄운다.
- **엔지니어링 = 자연에 대한 반역:** 본질이 *무질서(엔트로피)를 거슬러 인위적 질서·규칙을 부여하는 행위*. 본능대로 편하게 짜는 걸 "자연스럽다"고 정당화하면, 그건 엔지니어가 아니라 *흐르는 물에 몸을 맡긴 방관자*.
- **깨진 유리창·타협의 누적:** 한 번의 사소한 우회·땜질이 전체를 부패시킨다. 당장의 에너지를 아끼려 미래의 거대한 재앙을 대출받는 것.
- → **예민함은 프로페셔널리즘의 다른 이름.** 무질서를 당연시하지 못하는 그 거부감이 질서를 창조·유지하는 엔지니어 정체성의 증거.

## 5. 권한 없이 무질서한 동료와 일할 때 — 현실적 방어 전략
> 전체 시스템을 혼자 구원하려는 책임감은 자신을 태우는 지름길. 통제 가능한 것에 집중하는 냉정함이 필요.

**① 인간이 아니라 '시스템'이 악당 되게**
- 동료에게 "코드 구조가 이상하다"고 직접 말하면 *논리적 지적*이 아니라 *개인 공격·피곤한 참견*으로 받아들여지기 쉽다(권한 없을 때 특히).
- **자동 검사기 도입:** Linter([ˈlɪntər], 린터, 린터)·정적 분석(SonarQube([ˈsɒnɑːrkjuːb], 소나큐브, 소나큐브))·테스트 커버리지 제한·`pre-commit` 훅을 빌드 파이프라인에 심는다.
- **규칙의 주체를 바꾼다:** "이건 나중에 문제 됩니다" 대신 *"빌드가 통과를 안 하네요", "도구가 에러를 뱉네요"*. 인간은 동료의 지적엔 반발해도 **컴파일러·CI([siː aɪ], 시아이, 시아이) 봇의 거절엔 순응**한다.

**② 아키텍처적 격리 (Blast Radius([blæst ˈreɪdiəs], 블래스트 레이디어스, 블라스트 레이디어스) 최소화)**
- 동료의 예측 불가능한 모듈을 내 코드에 직접 결합하지 말 것. **인터페이스·어댑터 레이어**로 추상화해, 그들 코드가 망가져도 내 영역은 격리되게.
- 레포 안에서 내 작업 본진과 그들 영역이 안 엉키게 **모듈화**를 극한으로. (→ [[07 데이터 로더 (Dataset · DataLoader)|관심사 분리]]와 같은 원리)

**③ 감정적 디커플링 — '내 포트폴리오'에 집중**
- 통제 못 하는 타인의 습관·전체 붕괴는 마음에서 내려놓고, **"내가 맡은 파트만은 완벽한 구조·생산성으로 증명한다"**에 에너지를 집중.
- 아키텍처 붕괴의 부채·리스크는 결국 *관리하지 못한 조직과 본인들*이 감당할 몫.

> [!note] 💭 나의 태도 (개인 성찰)
> 나는 타협점을 찾기보다 **상시 예민하게 반응해 최선의 구조·생산성을 내는 것을 습관으로** 삼고 있다. 다만 그렇지 못한 사람들과, *강제할 권한 없이* 일할 때의 고통이 크다. → 위 §5는 그 상황의 방어 전략. 핵심: 통제 가능한 것(내 파트·시스템 도구)에 집중하고, 통제 불가한 것(타인 습관)은 시스템에 위임하거나 내려놓기.

## 🆕 새 용어 정리
| 용어 | 한 줄 정의 |
|---|---|
| **엔트로피 / 열역학 제2법칙** | 닫힌 계의 무질서는 시간이 지나면 증가. 질서엔 외부 에너지 필요 |
| **소프트웨어 엔트로피** | 시간이 지나면 코드는 자연히 무질서(스파게티)로 — 야콥슨 |
| **인지적 구두쇠 / System 1·2** | 뇌는 에너지 절약 위해 직관(S1)을 택함; 추상화(S2)는 고에너지 |
| **땜장이 (Bricoleur)** | 마스터 플랜 없이 주변 것으로 대충 만드는 자 — 자콥의 진화 비유 |
| **국소·전역 최적점** (Local/Global Optima) | 당장의 안정(늪) vs 진짜 최적; 재설계엔 '혼돈의 계곡' 통과 필요 |
| **autoregressive / greedy decoding** | 다음 토큰만 예측·선택 → 백스페이스 없음, 국소 최적점 함정 |
| **추론 모델 (Reasoning([ˈriːzənɪŋ], 리즈닝, 리즈닝), o1·R1)** | CoT([siː oʊ tiː], 시오티, 시오티)로 System 2를 강제해 스스로 검증·리팩토링하는 LLM |
| **자연주의적 오류** | "자연스러우니 옳다"는 착각 |
| **깨진 유리창 법칙** | 사소한 무질서 방치가 전체 부패를 부름 |
| **Blast Radius** | 한 변경·장애가 영향을 미치는 범위; 격리로 최소화 |
