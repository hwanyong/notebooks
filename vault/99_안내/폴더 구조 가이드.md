# 📁 폴더 구조 가이드

> 이 볼트가 시간이 지나도 **혼선 없이** 유지되도록 만든 규칙 문서입니다.
> 새 학습을 시작하거나 "이건 어디에 둬야 하지?" 고민될 때 이 문서를 먼저 보세요.

---

## 1. 한눈에 보는 구조

```
notebooks/
├── 00_Home.md            ← 전체 진입점(대시보드)
├── 10_학습/              ← 학습 기록 (분야 → 주제 누적)
│   ├── AI-ML/
│   ├── CS/
│   ├── 개발/
│   └── 기타/
├── 30_프로젝트/          ← 실제로 만드는 것 (Code agent, desktop pet …)
├── 80_템플릿/            ← 새 노트 찍어낼 양식
├── 90_보관/              ← 더 안 보는 옛 노트 (삭제 대신 여기로)
├── 99_안내/             ← 이 가이드 등 메타 문서
└── 고민상담/             ← 학습과 무관한 개인 메모
```

번호 접두사(10·20·30…)는 옵시디언 사이드바 **정렬을 고정**하기 위한 것입니다. 사이에 새 영역이 필요하면 비어 있는 번호(예: 40, 50)를 쓰면 됩니다.

---

## 2. 학습은 '주제별로 누적'한다

한 주제 = 한 노트. 날짜가 아니라 **주제로 묶어** 계속 다듬고 업데이트한다(위키/백과사전식). 같은 내용을 두 번 쓰지 말고, 한 곳에 쌓아 깊게 만든다.

---

## 3. 학습 노트 두는 법 (가장 중요)

### 3-1. 분야 고르기
새 주제는 먼저 4개 분야 중 하나에 넣습니다.

- **AI-ML** — 모델, 에이전트, 논문, LangGraph, 프롬프트 등
- **CS** — 알고리즘, 자료구조, DB 이론, 네트워크, OS 등
- **개발** — 언어/프레임워크/인프라/도구 (Docker, Prisma, React …)
- **기타** — 위 셋에 안 맞는 학습 (영어, 글쓰기, 도메인 지식 등)

> 애매하면 "이걸 나중에 어느 서랍에서 찾을까?"를 기준으로 고르세요. 완벽하지 않아도 됩니다. 나중에 폴더째 옮기면 그만입니다.

### 3-2. 주제 = 폴더 1개
하나의 주제는 **폴더 하나**로 만듭니다. 폴더 안에는 그 주제의 **대표 노트(인덱스)** 를 같은 이름으로 둡니다.

```
10_학습/개발/Docker/
├── Docker.md              ← 대표 노트(개념 요약 + 하위 노트 목차)
├── 01. 네트워크.md         ← 세부 노트
└── 02. 볼륨과 마운트.md
```

- 세부 노트가 많아지면 `01.`, `02.` 번호를 붙여 순서를 잡습니다.
- 명령어 한두 줄짜리 짧은 메모면 굳이 폴더 안 만들고 대표 노트 하나로 충분합니다. **커지면 그때 쪼개세요.**

### 3-3. 새 주제 시작하는 가장 빠른 방법
1. `80_템플릿/주제 노트 템플릿.md` 복사
2. `10_학습/<분야>/<주제>/` 안에 `<주제>.md` 로 저장
3. 해당 분야 MOC(`_AI-ML MOC` 등)에 링크 한 줄 추가

---

## 3-4. 복습 동선 (순서·링크·이해도)
노트가 쌓여도 "어떤 순서로 다시 볼지" 헷갈리지 않도록 4가지를 지킵니다.

1. **순서 번호 파일명** — 한 주제(코스) 안 노트는 공부한 순서대로 `01`, `02`, `03` 접두를 붙임 → 사이드바가 학습 순서로 정렬.
2. **상단 breadcrumb** — 각 노트 맨 위에 `🗺️ 허브 ｜ ⬅️ 이전 ｜ ➡️ 다음` 한 줄. 끊김 없이 따라 읽기.
3. **이해도 신호등** — 프런트매터에 `이해도: 🟢/🟡/🔴` (🟢 충분 · 🟡 더 봐야 · 🔴 다시). 복습 우선순위가 됨.
4. **허브의 학습 경로 표** — 묶음(코스·단계) 대표 노트에 `# · 노트 · 이해도 · 한 줄` 표를 둠. **복습은 항상 이 표에서 시작.**

> 예: `10_학습/AI-ML/AI 암반 정복/0단계 .../_0단계 ...` 의 "🗺️ 학습 경로" 표 참고.

## 3-5. 메인 / 사이드 지식 구분
공부하면 교재 핵심(메인)에 더해 배경지식·비유·심화 같은 부가지식(사이드)이 따라옵니다. 둘을 섞지 않고 **시각적으로 구분**합니다.

- **메인** = 교재가 직접 가르치는 핵심. 일반 본문·헤딩.
- **사이드** = 교재 밖 부가지식. **콜아웃 박스**로 감싸 구분(접기 가능) + 태그로 도메인 묶기. 4종:

| 종류 | 콜아웃 | 태그 |
|---|---|---|
| 🧭 비유 | `> [!example]` | `#사이드/비유` |
| 🔗 배경·연계 | `> [!info]` | `#사이드/배경` |
| 🪂 심화 | `> [!tip]` | `#사이드/심화` |
| 🌉 타 분야 연결 | `> [!quote]` | `#연결/<도메인>` |

사이드 지식은 묶음별 **연결 지도** 노트(예: `AI 암반 정복/_지식 연결 지도`)에 모아, 나중에 도메인(네트워크 등)으로 역추적합니다. → 복습 시 메인은 흐름대로, 사이드는 관심 도메인에서 바로 접근.

> [!note] 📘 책 핵심 · 🌱 탐구 — 커리큘럼 코스 전용 표준 (정본: [[_커리큘럼 MOC]] 정리 규칙 10·11)
> - **📘 책 핵심** = `> [!abstract] 📘 책 핵심 — 부록 A.X {주제}` 콜아웃으로 *책이 실제 가르친 것만* 고정 = 메인의 사실 앵커. (꼬리표·일반 헤더 금지)
> - **🌱 탐구** = 챕터에 안 묶이는 재사용 조사(`T0X`, `탐구/` 폴더). 책 노트의 **트리거 문장 바로 아래**에 `> [!tip] 🌱 탐구 · [[…]]` 요약+링크로 삽입(하단 '연결'에 몰지 않음). '파생' 아님 — 유래만 기록.

## 3-6. 용어집 (중앙 색인)
용어 설명은 **두 곳에 동시 관리**합니다(중복이 아니라 색인).

- **노트 안 인라인 표** = 맥락 속에서 배우는 용도(기존 룰 유지).
- **중앙 용어집** (묶음별 `_용어집` 노트) = 헷갈릴 때 한 곳에서 `Ctrl/Cmd+F`로 찾는 용도. 한 줄 정의 + 출처 노트 링크.

새 용어가 생기면 ①인라인 표 + ②용어집 한 줄을 같이 추가. 예: `AI 암반 정복/_용어집`.

## 3-7. 이미지 인박스 (`__temp`)
노트에 넣을 캡처·이미지를 다루는 약속.

- 사용자가 이미지를 볼트 루트의 **`__temp/`** 폴더에 넣어 둔다.
- Claude가 이미지 내용을 보고 **알맞은 노트를 판단** → 그 노트의 `attachments/` 폴더로 **이동·리네임**하고 노트에 `![[ ]]`로 **임베드**한다.
- 처리 후 `__temp/`는 비운다(폴더 자체는 유지). 이미지 최종 보관은 각 노트 옆 `attachments/`.

## 3-8. 두 학습 축 — 책 코스(선형 진도) × 지식 영역(다관점 융합)
볼트는 두 축이 **격자처럼 공존**한다. 충돌하지 않고 서로를 보완한다.

- **세로축 · 진도 (책 코스):** 책으로 공부하는 코스(예: `AI 암반 정복`)는 **책 절 순서**로 노트를 쌓고(`NN-BID-챕터`), 학습 경로 표·breadcrumb·진도 체크리스트로 **선형 복습 동선**을 유지(→ 3-4). *책이 정한 순서가 곧 척추.* 이 규칙은 그대로 유지된다.
- **가로축 · 융합 (지식 영역):** 책 진도와 무관한 **범용 원리**(CS·개발/언어·수학 등)는 영역으로 분리해, **폴더에 가두지 않고 다관점 태그(`<영역>/<관점>`, 예 `CS/하드웨어`·`CS/운영체제`) + 융합 맵**으로 관리한다. 한 노트가 여러 관점에 동시 소속되고 영역을 넘어 연결된다(예: [[_CS MOC]]).
- **잇는 다리:** 책을 읽다 범용 지식이 갈라지면 — 그 코스에서 재사용되면 🌱**탐구**로, 책과 무관한 독립 지식이면 🔗**타 영역**으로 빼고, [[_지식 연결 허브]]가 영역을 가로질러 잇는다.

→ 책은 책대로 순서대로 나아가고(세로), 거기서 나온 원리는 영역에서 융합된다(가로). **한 절에서 코스 노트·탐구·여러 영역 노트가 갈라져 나오는 게 정상**이며, 둘은 같은 지식의 두 입구다.

## 4. 이름 규칙 (간단 버전)
- 폴더/영역: `번호_이름` (예: `10_학습`)
- MOC(목차) 노트: 앞에 `_` → 폴더 맨 위에 정렬됨 (예: `_AI-ML MOC.md`)
- 세부 노트 순서: `01.`, `02.` 접두
- 날짜: 항상 `YYYY-MM-DD`
- 태그: `분야/주제` 형태 권장 (예: `#개발/docker`, `#cs/db`)
